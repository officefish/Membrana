# Membrana + Anthropic (локальный CLI)

## Когда применять

- Нужно **проверить**, что `ANTHROPIC_API_KEY` в корневом `.env` рабочий.
- Нужен **короткий разбор** `docs/ARCHITECTURE.md`, `docs/DESIGN.md` или другого файла с опорой на текст (как у роли **Teamlead** / ревью архитектуры).
- Пользователь явно просит «запусти anthropic-скрипт» / «спроси API по этому файлу».

## Запрещено

- Читать или вставлять в ответ **значение** `ANTHROPIC_API_KEY`, полные `sk-ant-…`, содержимое `.env`.
- Коммитить `.env` или предлагать хранить ключ в репозитории.

## Прокси и 403

Если пользователь видит **HTTP 403** «Request not allowed» из терминала при рабочем VPN в браузере: трафик Node/Yarn часто идёт **мимо** VPN. Предложите добавить в `.env` строку `HTTPS_PROXY=http://127.0.0.1:ПОРТ` (HTTP-порт прокси-клиента) и снова `yarn anthropic:smoke`, либо системный/TUN VPN.

## Команды (из корня монорепо)

| Задача | Команда |
|--------|---------|
| Проверка ключа и сети | `yarn anthropic:smoke` |
| Разбор файла по умолчанию (`docs/ARCHITECTURE.md`, вопрос про правила PR) | `yarn anthropic:task` |
| Другой файл | `yarn anthropic:task docs/DESIGN.md` |
| Свой вопрос | `yarn anthropic:task docs/ARCHITECTURE.md "Твой вопрос по тексту"` |

Скрипты: `scripts/anthropic-smoke.mjs`, `scripts/anthropic-task.mjs`, общий загрузчик `scripts/_anthropic-env.mjs`.

Опционально в `.env`: `ANTHROPIC_MODEL=` (иначе используется Haiku 4.5 по умолчанию в скриптах).

## После запуска

- Кратко перескажи пользователю **суть ответа модели** (без дословного копирования длинных блоков, если не нужно).
- При ошибке HTTP — сообщи код и типичные причины (неверный ключ, лимиты, регион/политика), **без** вывода тела ответа, если там может быть чувствительная информация.
