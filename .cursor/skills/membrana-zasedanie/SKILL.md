# Membrana Заседание

Канон: [`docs/ZASEDANIE_REGULATION.md`](../../../docs/ZASEDANIE_REGULATION.md) —
**черновик, подлежит ратификации консилиумом** (прецедент: формат Cowork утверждён
консилиумом 14.07).

## Суть

Заседание **≠ серия консилиумов**. Серия — N прогонов. Заседание — контейнер с тремя
несущими элементами:

- **общее объединяющее задание** — одно на все вопросы;
- **свой протокол** — `ZASEDANIE_ACTIVE.md`, отдельный от протоколов консилиумов;
- **свой аудитор** — отдельный агент, read-only, следит за процедурой.

Без этих трёх это просто несколько консилиумов подряд — и они разойдутся так же, как один.

## Почему формат вообще есть

Три ступени эскалации, каждая от провала предыдущей:

1. **Проза** (урок 11.07 в `membrana-consilium`): «пронумеруй точки, требуй эхо с вердиктом».
   17.07 её **исполнили** — и консилиум всё равно уронил центральный вопрос.
2. **Детекция** (rt-6 `findUncoveredAgendaItems`): ловит, но уже по сожжённому прогону.
3. **Структура** (этот формат): ронять нечего — вопрос один.

Срывов задокументировано шесть: 2× 11.07 (#355→ADR #356), 3× 16.07, 1× 17.07.

## Фазы

```
З0 — все вопросы в одной комнате; единственный вердикт: ПОРЯДОК (DAG)
   → ратификация порядка ВЛАДЕЛЬЦЕМ (не председателем)
     → З1..Зn — по одному вопросу в этом порядке, каждое → свой вердикт
       → сборка активных вердиктов в эпик
```

З0 — единственная общая комната, и она нужна: увидеть DAG через замочную скважину нельзя.
Изоляция защищает от **потери** вопроса, но не от **неверного ранга**.

**З0 не спецслучай:** его повестка тоже несёт один ID-вопрос («какой порядок?»), а
кандидаты лежат в теле **простым текстом без жирных ID** — тогда правило единообразно.

## Hard rules

- Один вопрос — одно заседание (**S-Z1**, машинный отказ до прогона).
- Вердикта нет → заседание **не состоялось**; «частично» не существует (**S-Z2**).
- Задание после З0 меняется только через `ZASEDANIE_AMENDMENT.md` + LGTM владельца (**S-Z4**).
- **Аудитор ≠ председатель** (**S-Z5**). 17.07 обе роли играл один агент: сам написал
  повестку, сам запустил, сам оценил покрытие — поймал гейт, а не он. В Competition жюри
  не реализует код команд; изоляция здесь проходит между тем, кто ставит вопрос, и тем,
  кто проверяет процедуру.

## Run

```bash
yarn consilium --zasedanie <id> --topic-file <повестка.md> --save-as <id>-z<n>-<slug> "…"
```

`--zasedanie` включает S-Z1: повестка обязана нести ровно один ID-вопрос, иначе **отказ
до прогона** (exit 2), кредит не тратится. Без флага обычный консилиум работает как
прежде — многовопросность там законна.

## Шов

Форму вердикта задаёт эпик `truth-graph-contour`, не этот формат. Кандидат из обсуждения
17.07: вердикт = токен, зависимость = `parents`, отзыв каскадом (S-Z3) — тогда «вёрстка в
эпик» перестаёт быть ручной. **Не проектировать здесь** — иначе две несовместимые схемы.

## Честная цена

DAG — последовательность, не параллель: `Зn` ждёт `Зn-1`. Изоляция убивает сшивки (I4
родился оттого, что Музыкант связал эхо-камеру с графом) — З0 остаётся ровно для этого.
Осознанный размен эмерджентности на покрытие.
