# Membrana Шторм

Канон (закон): [`docs/STORM_REGULATION.md`](../../../docs/STORM_REGULATION.md) — **черновик v0.1**,
ратифицирован заседанием `storm-format-ratify` 2026-07-20, канон-статус после чистого аудита.
Операционная процедура (как ведущая исполняет): [`docs/virtual-team/angelina/STORM_PROCEDURE.md`](../../../docs/virtual-team/angelina/STORM_PROCEDURE.md).
Место в ряду форматов: [`docs/FORMATS.md`](../../../docs/FORMATS.md).

## Суть

Шторм — **дивергентный** формат: беседа, из которой **рождаются тезисы**. Не планирование,
не разбор готового — часто просто живой конспект будущего доклада/инсайта/входа в заседание.
Шестой в ряду форматов (Competition · Cowork · Hackathon · Night · Meeting · **Storm**).

**Шторм ≠ заседание.** Оснастка общая (живой конспект, секретарь, правило «только
подтверждённое»), но заседание **сходится** к обязательному вердикту, а шторм **расходится**:
вердикта не несёт, директивы факультативны.

## Когда

- **Да:** мысль ещё не сформулирована, её надо породить; предмет расходится, а не сходится.
- **Нет:** сходимся к вердикту → заседание (`membrana-meeting`); готовая развилка, один
  вопрос → `yarn consilium`; исполнимая задача → `membrana-task-lifecycle`.

## Три голоса (кросс-агентский расклад)

| Голос | Кто | Право |
|---|---|---|
| **Секретарь — Ангелина** | процесс | приглашает участников, ведёт конспект, следит чтоб не врали, возвращает на доработку; **единственный гейт каскада** |
| **Участник** (5 персон) | доменная экспертиза | вносит реплику в своём предмете; вес — только через две подписи |
| **Робот-питомец** | сам агент | встревает без приглашения, **судит и критикует** владельца и участников; `origin: pet`, **голос, не гейт**, в основания гейта не входит |

## Жизненный цикл

- **Дом — директория** `docs/storm/<id>/` (обычно `storm-<slug>-<YYYY-MM-DD>/`) с двумя
  носителями: живой конспект (`THESES.md`) и сводный `REPORT.md`. Дом ≠ пространство `current_task`.
- **Открытие:** владелец **называет предмет** → секретарь заводит дом, статус OPEN.
  Пустой/безымянный дом — **не открыт** (S-S1).
- **Закрытие — пол и потолок:**
  - **Пол — «пора» владельца:** закрывает в любой момент, суверенно, раньше потолка.
  - **Потолок — семь вдохов:** неотменяемый, закрывает **даже против владельца** (S-S5).
- **Выход обязателен:** конспект + **развилка**, названная владельцем явно (включая
  «никуда» — тоже его слово). Молчаливое закрытие без развилки — **не закрыт** (S-S2).
- **Стенка контейнера:** наружу — конспект и развилка (+ директивы, если есть, с двумя
  подписями); реплики остаются внутри.

## Вес: две подписи

Реплика веса не несёт. `реплика ──(участник ЯВНО: «прошу утвердить директивой»)──▶ заявка
──(владелец ЯВНО подтверждает)──▶ директива`. Директива без **обоих следов** — ложь о
статусе, Ангелина возвращает (S-S3). Опровержение владельцем оставляет **след согласия**
(контр-довод+ответ или явная уступка); пустое падение — подавление, возврат (S-S4).

## Тайминг: вдох (детерминированный)

Вдох — единица прогресса, меряется хешем близости (SimHash64), **без часов и случайности**.
Существенное событие (тезис добавлен/изменён/снят, `d ≥ 1`) при мандате `K_min = 1` = вдох;
косметика (`d = 0`) вдоха не даёт. ≤ 7 существенных событий гарантированно жгут потолок.

## Hard rules

1. В запись только подтверждённое; директива — две подписи (участник + владелец).
2. Развилка на выходе обязательна, включая явное «никуда». Директива — нет.
3. Пора — суверенный пол, семь вдохов — неотменяемый потолок.
4. Гейт каскада один — Ангелина; питомец — голос, не входит в основания.
5. Опровержение оставляет след согласия.
6. Детерминизм тайминга: без сети, без `Date.now`, без `Math.random`.

## Как вести (процедура Ангелины)

1. **Диалог держит Ангелина** — вопросы владельцу, кандидаты-тезисы, развилки. Темп у
   диалога, не у документов.
2. **Делегирование по предикату** (`scripts/lib/angelina-delegate.mjs`), не рефлексом:
   долгоживущий `THESES.md` → **писарь**; фактура из репозитория параллельно беседе →
   **аналитик**; влезающее в один контекст — сама (штат не самоцель).
3. **Каждый возврат субагента — через `isValid` до внесения.** Невалидный — назван
   владельцу и перезапущен; молча выбросить или использовать нельзя.
4. **Подтверждение владельца — единица истины:** в `THESES.md` только подтверждённое,
   неподтверждённое живёт как кандидат с явной пометкой.
5. **Выход:** `REPORT.md` — каждый тезис подтверждён, развилки «куда едет» и «осталось
   нерешённым» явными таблицами; хвосты — в `JOURNAL.md` с провенансом «день + источник».

Модули контракта (worktree `feat/storm-format-impl`, #698): `scripts/lib/storm-lifecycle.mjs`
(предикаты), `storm-codex.mjs` (QC1-5), `storm-breath.mjs` (SimHash64, счётчик 0..7).

## Швы наружу

- **С заседанием:** ветвь развилки = заголовок будущего заседания (`handoff: конспект
  шторма → вход заседания`). Дивергенция → схождение.
- **С движком задач:** факультативная директива (две подписи) → GitHub-issue. Норма-
  называние, не файловый провод — здесь не проектируется.

> Открытый долг канона: именование живого документа не выровнено (`THESES.md` в процедуре ↔
> `конспект`/`CONSPECTUS.md` в регламенте) — свести отдельной правкой, не молча.
