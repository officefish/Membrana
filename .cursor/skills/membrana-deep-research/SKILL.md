# Membrana deep research

Три запроса к Perplexity по теме спринта. Схема **Landscape / Fit / Risk** — что известно
снаружи / как ложится на наш канон / чем аукнется.

> Имя с префиксом `membrana-` намеренно: встроенный скилл `deep-research` (веб-поиск
> Claude) — другой инструмент. Здесь — Perplexity + привязка к карточке спринта.

## When to use

Есть **внешняя** неизвестность: чужие подходы и форматы, практики 2025–2026, лицензии,
известные грабли. Признак: на вопрос **нельзя ответить чтением нашего репозитория**.

## When NOT to use

- **Ответ внутри репозитория** — канон, код, история PR, ADR. Тулинг-спринты 2026-07-15
  (#485/#492/#497/#504/#510) ресёрчем не выиграли бы ничего: всё было в самом репо.
- **Инсайт** → `yarn insight research <id>` (та же машинерия, свой контур).
- **Вместо консилиума/ADR** — ресёрч это **вход** для решения, а не решение.
- **«Расскажи про X вообще»** — три рана на общий вопрос это не ресёрч, а шум.

## Поток

```bash
yarn task:register --id <slug> --title "…" --size M --issue N --research
# → в промпт добавлена секция «Вопросы для research (Q1–Q3)» с заготовкой

# 1. Агент формулирует 3 вопроса ИЗ КОНТЕКСТА спринта (только он знает, что неизвестно)
# 2. Владелец подтверждает формулировки
yarn research <task-id> --dry-run    # показать, что уйдёт в Perplexity
yarn research <task-id>              # 3 запроса → docs/tasks/research/<task-id>.md
```

Без ключа `PERPLEXITY_API_KEY` — печатает запросы для MCP/ручного прогона (каскад
API → MCP → руками, как у инсайтов).

## Как формулировать вопрос

Вопрос **конкретный и самодостаточный**: Perplexity ответит на что угодно — включая
обрывок. Живой урок (#402, 2026-07-12): вопрос ушёл обрезанным на слове «giving», и
пришёл добросовестный ответ **про благотворительные пожертвования в США** вместо памяти
персон. Сбой был **тихий**: скрипт напечатал «обновлён», ран потрачен.

| Плохо | Хорошо |
|-------|--------|
| «Landscape: как делают X» | «Landscape: какие практики сложились в 2025–2026 для X — на каком шаге зовут, куда кладут результат, как связывают с трекером?» |
| «Fit: подходит ли нам Y» | «Fit (Membrana): для маленькой команды с агентами и жёстким каноном — по каким признакам решают, что Y нужен, а когда превращается в шум?» |
| «Risk: какие риски» | «Risk: чем рискует команда, принимая выжимку как вход для архитектурного решения — устаревание, галлюцинации, ложная уверенность; какие приёмы верификации применяют?» |

## Гарды (валят прогон ДО траты рана)

- **Плейсхолдеры** `<…>` в вопросах — секция не заполнена.
- **Оборванный вопрос** — незакрытая скобка, хвостовая запятая, длина < 25 символов.

Оба — прямое следствие #402: обрезанный запрос неотличим от нормального, поэтому ошибка
должна быть **до** вызова, а не сюрпризом в выжимке.

## Что делать с выжимкой

**Выжимка — вход, а не решение.** Проверяй утверждения по нашему коду: поисковая LLM
уверенно говорит о том, чего у нас нет. Ссылки `[1][2]` в ответе — повод открыть источник,
а не аргумент сам по себе.

Дальше: выжимка → консилиум/ADR → решение → код. Не «в выжимке написано, значит делаем».

## Связанное

- `membrana-insight` — свой ресёрч-контур (`yarn insight research <id>`), та же машинерия.
- `membrana-consilium` / `membrana-adr` — где принимается решение.
- `membrana-task-lifecycle` — старт спринта, куда встроен флаг.
