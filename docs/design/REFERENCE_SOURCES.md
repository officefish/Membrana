# Источники референсов для дизайна

> Где команда собирает референсы для лендингов, презентационных материалов и
> интерактивных нарративов. Собрано из трёх запросов к Perplexity (модель `sonar`),
> 2026-07-17. **Выжимка — вход для решения, а не решение:** названия и оценки
> («самый большой архив») сверять глазами при первом заходе.

## Главный принцип: две разные библиотеки

Референсы делятся на два несмешиваемых слоя — путать их вредно:

1. **Продуктовые лендинги** — то, что реально конвертит (структура, оффер, паттерны).
2. **Иммерсивный интерактив-нарратив** — полёт мысли, WebGL, карты-истории.
   Эталон команды — [Bear 71](https://en.wikipedia.org/wiki/Bear_71) (NFB, интерактивный
   документальный нарратив на карте: телеметрия медведицы → сюжет). Живёт этот жанр
   **не в дизайн-галереях, а в архивах интерактивной документалистики** (см. §2).

Связь с нашим визуальным направлением: целевой облик research-tree — «живой мозг с
нейросетевыми связями» (см. память `research-tree-visual-direction`), и §2 — основной
питомник идей под него.

## 1. Лендинги и презентационные сайты

| Слой | Куда идти | За чем |
|---|---|---|
| **Вау / визуальный край** | Awwwards, Godly, FWA, CSS Design Awards | Типографика, моушн, «эффект». Оговорка: много «красиво, но конвертит плохо» — не тащить вслепую в прод. |
| **Работающие лендинги** | Lapa Ninja (7300+, крупнейший архив), Land-book (упор на конверсию), Landing Folio (лучшие фильтры — по индустрии/элементам), SaaS Landing Page, One Page Love (одностраничники) | Структура, оффер, конверсионные паттерны. |
| **Реальные UI-паттерны** | Mobbin (400k+ экранов, потоки), Refero (SaaS-компоненты, апдейт еженедельно), Pageflows (видео реальных user-flow) | Как это устроено, а не как выглядит на дриббле. |
| **Идеи / направление** | Dribbble, Behance, Are.na, Savee | Ранняя визуальная разведка. Оговорка: Dribbble — часто нереализованные концепты. |
| **Ниши (2025–26)** | Footer.Design, Dark Mode Design, Unsection (секции лендинга), Brutalist Websites, Httpster | Точечные референсы компонентов/стиля. |

**Три и не распыляться** (прямая рекомендация Perplexity): **Mobbin** (реальный UI) +
**Land-book** (структура лендинга) + **siteInspire** (чистое брендовое направление).
Awwwards/Godly — только когда нужна смелость, и всегда в паре с конверсионной галереей.

## 2. Интерактив-нарратив «как Bear 71»

Ключевое: финишные работы жанра лежат **не в дизайн-галереях, а в архивах
интерактивной документалистики**.

- **[i-docs](http://i-docs.org/)** — профильный каталог жанра; Bear 71 там каталогизирован
  ([карточка](http://i-docs.org/bear-71/), проверено 2026-07-17). Нелинейный / партиципаторный сторителлинг.
- **[Docubase (MIT Open Documentary Lab)](https://docubase.mit.edu/)** — исследовательский
  архив документальных инноваций, трансмедиа, веб-нарративов
  ([карточка Bear 71](https://docubase.mit.edu/project/bear-71/): карта-надзор, migrating dots
  телеметрии, зритель сам трекается через вебкамеру — проверено 2026-07-17).
- **[Experiments with Google](https://experiments.withgoogle.com/bear71)** — живая реставрация
  самого Bear 71 (оригинал был на Flash); удобно смотреть механику вживую.
- **[Pudding.cool](https://pudding.cool/)** — data-driven визуальный сторителлинг: карты,
  графики, где данные *ведут* сюжет (логика телеметрии Bear 71).
- **Godly / Awwwards / FWA / Hoverstat.es / Codrops** — за технической стороной: WebGL, 3D,
  canvas, нелинейная навигация.

**Карта-как-интерфейс** (суть Bear 71):

- **[ArcGIS StoryMaps Gallery](https://storymaps.arcgis.com/)** — тысячи интерактивных карт-историй.
- **[Mapbox Storytelling](https://www.mapbox.com/showcase)** — демо scroll-driven карт + медиа.
- **[Knight Lab](https://knightlab.northwestern.edu/projects/)** (Northwestern) — карты, таймлайны, scrollytelling.

## 3. Чем собирать и хранить (мудборды команды)

Гибрид, а не один инструмент:

- **Клиппинг откуда угодно:** **Savee** (лучший для дизайнеров — тянет PNG/SVG с любого
  сайта), альтернатива — **Cosmos** (спокойный анти-Pinterest).
- **Личная библиотека-хранилище:** **Eagle** — офлайн, данные у тебя, глубокие теги/цвета/
  AI-поиск, держит десятки тысяч ассетов без лагов. (Библиотека референсов — тот случай,
  где локальное владение оправдано, в отличие от общего server-first курса.)
- **Совместная сборка и показ:** **Milanote** (красивые борды, презентационный PDF) или
  **Miro** (энтерпрайз-коллаборация).
- **Связная ресёрч-сеть:** **Are.na** — каналы вместо плоских папок, для концептуального копания.
- **Не для команды:** Pinterest (шумно, водяные знаки, слабый экспорт).

**Рекомендованный контур:** ловишь в **Savee/Cosmos** → складываешь надолго в **Eagle**
(с тегами) → финальный мудборд и показ в **Milanote/Figma**.
