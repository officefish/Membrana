## Аргументы ролей

- **Teamlead:** решение обосновано строго через §5; развилка read-доступа растворена (агент читает,
  партнёр в git не входит ни в одной топологии). Инвариант «сток, не исток» = запрет исходящих
  рёбер, не запрет чтения.
- **Структурщик:** `apps/comms-studio` — leaf-ноль; `check:boundaries` даёт позитивный + негативный
  тест инварианта; чтение через `fs-read`, не `import` (иначе ребро).
- **Математик:** `S_A = 0`, `M_A ≈ 0`; B даёт `S=0` лишь при авто-зеркале с `M>0` → «дешёвая» не
  выполнена → оговорка §5 не активируется; гибрид — максимум `M`.
- **Музыкант:** психологический контекст партнёра — Drive/монитор, а не IDE; это верно для A так же,
  как для B, поэтому «партнёр в IDE» не аргумент за B.
- **Верстальщик:** живое чтение закрывает реальную боль (устаревшие снимки описаний компонентов);
  выход — каталог `out/**`, экспорт в Drive, обратный импорт в git запрещён.
