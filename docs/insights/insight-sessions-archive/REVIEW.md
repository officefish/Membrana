# Review: Архивация сессий AI-агентов

> Virtual team · 2026-06-28

---

[Teamlead / Vesnin]: Инсайт закрывает реальный пробел: git даёт «что попало», но не «что рассматривалось и почему». Адаптерная схема + скруб на границе нормализации — правильная архитектура. Два открытых вопроса (домен backend, JSONL-поля claude-code) должны быть сняты **перед** кодом, не в процессе. Рекомендую: **adopted**, но реализацию не начинать до фиксации домена. Горизонт quarter обоснован — это инфраструктура, не продукт.

[Структурщик / Ozhegov]: Адаптерный контракт `SessionAdapter.discover/read/normalize` — чистая граница. Scrub внутри `normalize()` — правильное место (данные очищаются до хранения, не после). `background-sessions` как новый сервис соответствует single-responsibility; `background-office` stateless by design нарушать нельзя. Замечание: `files_touched` из claude-code JSONL — убедиться, что поле существует или деривируется из диффов ходов, а не фантазируется.

[Математик / Dynin]: Per-turn hash — корректный механизм дедупа для append-only сессий. Content hash сессии как идемпотентный ключ — правильно. Важно: регулярки скруба должны покрываться детерминированными тестами с fixture-токенами до деплоя в ритуал. Поле `diff` в Turn — опционально, но ценно для восстановления нарратива; деривировать из `git diff HEAD~1` на момент архивации, если claude-code не пишет его сам.

[Музыкант / Kuryokhin]: Прямой связи с аудио-доменом нет, но архив сессий захватит контекст отладки аудио-сессий (mic latency, detector runs) — это ценно для post-mortem анализа. Поддерживаю. Замечание: `linked_issue` из `MAIN_DAY_ISSUE.md` — хрупкая связь, если issue не обновлялся в день. Документировать как «приближение, не точная связь».

[Верстальщик / Rodchenko]: UI-аспект отсутствует в первой версии. В перспективе: session browser в cabinet (timeline сессий, поиск по content_hash) — интересная возможность для отладки и демонстрации прогресса. Минимальное мнение по текущей задаче: поддерживаю, никаких frontend-блокеров нет.
