## Проблема / наблюдение

Нужен процесс, собирающий, нормализующий и архивирующий сессии работы с AI-агентами
(Cursor / Claude Code / Codex). Целевая функция: **первичен архив без потерь** —
верный воспроизводитель хода мысли разработчика и агента. Конечная задача — возможность
восстановить историю разработки не только по репозиторию, но и по сессиям.

## Гипотеза

Адаптерная схема (один адаптер на окружение → общая нормализованная `Session`) с
per-turn дедупом и скрубом секретов на границе нормализации даёт append-only канал
без потерь, который встраивается в вечерний ритуал и позднее выносится в ночной фон.

## Scope

- **In scope:** JSONL-сессии claude-code (первый адаптер), скруб `sk-ant-`, `ghp_`, `lin_api_`, high-entropy хвостов после `=`; дедуп по per-turn hash; корреляты `branch` / `HEAD` / `linked_issue`.
- **Out of scope:** векторный индекс (Pinecone), адаптеры Cursor и Codex (пока структура не подтверждена с машины).

## Открытые вопросы

1. **Домен backend** — `background-sessions` (новый) vs `background-office` vs `media`. Решение не принято.
2. **Точные пути и поля JSONL** claude-code — снять с рабочей машины.
