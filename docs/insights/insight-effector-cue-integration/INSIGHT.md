## Проблема / наблюдение

Партнёр Денис предложил ВИЗОР-МПВС — counter-UAS **эффектор** (прожектор подсветки
4–14 млн Кд, дальность 4–7.5 км, луч 2.5°, опц. лазер ослепления). Вопрос: наш ли
это слой и можно ли интегрировать.

Разбор вскрыл **общий класс, шире одного устройства**: Membrana — слой
**обнаружения**, а C-UAS по стандарту 2024–2026 (detect-then-cue / slew-to-cue)
это многослойная цепь **сенсор → C2 → эффектор**. Membrana сегодня занимает
только первое звено, и притом не полностью: даёт присутствие + класс, но **не
пеленг** (моно-микрофон, DOA нет). У эффектора-кандидата — обратный дефект:
**ручное наведение, нет цифрового интерфейса**.

**Ядро наблюдения — двойной разрыв.** Между «Membrana обнаружила» и «эффектор
навёлся» два пробела: у нас нет выхода-азимута, у ВИЗОР нет входа-целеуказания.
Ни один не закрывается покупкой устройства — оба лежат в архитектуре.

## Гипотеза

Ценность не в конкретном ВИЗОР, а в **контракте целеуказания (cue)** как будущей
границе Membrana с любым эффектором. Если (позже, не сейчас) Membrana научится
давать **азимут с известной погрешностью**, появляется общая точка сопряжения с
любым C-UAS-эффектором через слой C2 — ВИЗОР лишь один из них.

Обратное тоже верно: **без пеленга эффектор бесполезен** (research: акустика без
DOA — только триггер «начать поиск», а луч 2.5° при ошибке в пару градусов на
1 км уходит мимо). Значит предпосылка интеграции — **пеленгация у нас**, а не
интерфейс у вендора.

## Вердикт консилиума (5 ролей) — вход, принят

| Вопрос | Решение |
| --- | --- |
| Программная интеграция (наш код правит ВИЗОР) | **Нет** — управлять нечем (ручное, нет интерфейса) |
| Системная сопряжённость (соседнее звено) | **Отложено с условием** — после stage-gate 1→2 и пеленга |
| Актуальность сейчас | **Неактуально** — приоритет Single-Node Detection First |
| Порог возврата к теме | σ(азимут) ≤ **0.6°** на рабочей дальности (иначе луч 2.5° не накрыть) |
| Пакетный граф | **Не трогаем** — вендор в граф не тянем; `effector-cue-service` — YAGNI до пеленга |
| Характеристики | Железо добротное для класса (7.5 км, 14 кКд, азимут 360°), но **чужой слой** |

## Scope (черновик)

- **In scope (когда дозреем):** контракт целеуказания `cue { azimuth, ±σ, class,
  confidence, ts }` как граница Membrana↔эффектор; операторская карточка «дрон,
  грубый сектор» с бейджом «не для точного наведения».
- **Out of scope (сознательно, сейчас):** любой код под ВИЗОР; управление
  устройством; `effector-cue-service`; закупка; пеленгация (отдельный крупный
  предшественник — микрофонная решётка/DOA, свой эпик).

## Условие возврата (что должно стать правдой)

1. Взят stage-gate 1→2 детекторов (P≥85 / R≥90) — база честна.
2. Появился пеленг с σ(азимут) ≤ 0.6° на рабочей дальности.
3. Есть эффектор с цифровым входом целеуказания **или** оператор в контуре как
   явный проектный сценарий.

До выполнения 1–2 тема сопряжения не открывается — иначе строим верхний этаж без
фундамента (эшелон 0 не закрыт).

## Ответ Денису (формула команды)

«Мы — уши, не прожектор. Устройство добротное и решает соседнюю задачу того же
контура, но точки соединения с Membrana в коде сегодня нет: у ВИЗОР ручное
наведение без цифрового интерфейса, а мы пока даём детекцию, а не точный пеленг.
Вернёмся к сопряжению, когда дойдём до пеленгации и сможем дать азимут с
точностью, достаточной для луча 2.5°.»
