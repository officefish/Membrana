## Дельта Phase 2 (что уточнилось при постройке)

- Форма артефакта закрытия расширена **опциональной** `acceptanceHistory:
  [{acceptedBy, headRev}, ...]` — по ней наблюдаются переприсвоение приёмки
  (для `escalationRate`) и отзыв/регресс (для риторической `reworkRate`).
  Без истории обе деградируют честно: событие просто не наблюдается, ошибкой
  не является. Кто пишет историю — вопрос шва (closure lifecycle, интеграция).
- Атрибуция `throughput`: по `leadPersona` карточки снимка (join по `taskId`
  артефакта), при отсутствии карточки — явная корзина `(без персоны)`, не
  домысливание по `acceptedBy`.
- Окно батча по умолчанию выводится из `capturedAt` обоих снимков — владелец
  времени подтверждён: снимки, не мои часы; вырожденное окно (снимки перепутаны)
  — жёсткая ошибка.
- Снимок без honest-провенанса `computeTeamMetrics` **отвергает** (исключение) —
  зеркально принципу «снимок без шапки холод не принимает».

## Пробелы (фиксируются, не блокеры)

- **Проверка по живому Linear API НЕ состоялась (Phase 2, 19.07) — канал недоступен,
  пробел фиксируется по правилу brief (constraint 3).** Три факта попытки:
  1. office `GET v1/linear/issue/DRU-93` → **401** с локальным `OFFICE_API_TOKEN`
     (известная ловушка: локальный токен ≠ office-токен VDS);
  2. прямой Linear GraphQL (интроспекция типа `Issue`) → **403** — похоже на
     гео/IP-блок с текущей сети (прецедент 403-как-гео у OpenRouter);
  3. **структурная находка**: даже при живом токене названный канал не ответит
     на вопрос двухслойности — `LinearIssueView` в
     `packages/background-office/src/modules/linear/linear.service.ts` запрашивает
     только `{id, identifier, title, description, state, labels, comments, url}`;
     полей `assignee`/`delegatedAgent`/под-задач в GraphQL-запросе офиса **нет**.
     Расширение сервиса — вне моей зоны (пакет office). Вопрос на Interface
     Consilium: кто дополняет issue-view полями двухслойности.
  Факты «двухслойность `assignee`/`delegatedAgent`» и «под-задача — полноценная
  сущность со своим статусом» остаются со статусом «по ресёрчу» — пометка в
  `docs/tasks/UNITS_DICTIONARY.md` §6 не снята. Контур снимко-центричный,
  на DoD блока не влияет.
- Форма снимка соседа — допущение (см. первую таблицу); сведение — Interface
  Consilium, шов №2 и №3 из brief.
- Код миграционного режима гейта снимка (три режима) — зона соседа; мой отказ-II
  оперирует только своими режимами soft/hard, переключение — параметром `mode`.
