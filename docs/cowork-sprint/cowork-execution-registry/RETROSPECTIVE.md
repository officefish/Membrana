## Обязательная строка: адаптировали vs переписали

**Переписано блоков: 0. Адаптеров: 6.** Octopus-мёрж трёх веток — ни одного
конфликта. Дефектов резки, потребовавших правки блоков, нет. Метрика формата
повторяет первый коворк (0/3) при втрое большем объёме: 52 файла, ~5000 строк
блоков + ~1600 строк интеграции, 137 тестов блоков + 16 smoke.

## Что резка угадала

- **Работы 2+4 в одном блоке** (предупреждение хендоффа о пересечении) — шов
  «носители следа ↔ единица счёта» не породил ни одного адаптера: он растворён
  внутри блока `units-trace-measure`.
- **Шов приёмки сошёлся сам**: блок Ангелины в изоляции угадал форму `acceptedBy`
  дословно той, что независимо построил блок учёта. Ранняя договорённость была
  бы потрачена впустую.
- Три блока независимо пришли к одним и тем же вопросам на консилиум (таблица
  exit-кодов, владелец времени) — швы brief оказались настоящими.

## Самый толстый шов (заметка интегратора)

Двойное именование канона записи: производитель отдаёт `state/stateType/assignee/
completedAt`, контракт §1.2 требует `status/leadPersona/closedAt` — погашено
проекцией (адаптер а), но это самая широкая трансляция резки. Поле `archivedAt`
контракт назвал обязательным минимумом, а производитель его не имеет вовсе —
деградация постоянная, не случайная; кандидат на правку контракта следующего цикла.

## Interface Consilium — что дал формат

Напряжение вердиктов двух ЗАСЕДАНИЙ (M2 «мягкий код ненулевой различимый» ↔ M4
«мягкий режим exit 0») вскрылось только при сведении EXPECTATIONS и снято
различением «вердикт-код / exit процесса» — оба вердикта исполнены буквально.
Код `11 LEGALITY_MIGRATING`, который комната M4 не назвала, назван блоком с
честной пометкой и ратифицирован консилиумом. S-C2 (несводимые EXPECTATIONS)
не понадобился.

## Гейт заработал в день рождения

Вшитый отказ-I немедленно поймал 5 живых active-карточек без `leadPersona` и
остановил штатный push (код 22). Реестр починен координаторским назначением
(`ownerlessRate = 0`), гейт не ослаблялся. Закон «в task:archive отказ-I hard»
распространён и на `--dry-run` — иначе показ врал бы о проходимости.

## Живая проверка Linear (закрыла 3 факта ресёрча)

- Поле делегата называется **`delegate`**, не `delegatedAgent` — ресёрч ошибся
  в имени; осторожность блока данных (не запрашивать неверифицированное имя)
  оправдалась.
- Под-задачи `parent`/`children` — полноценные объекты (подтверждено).
- **Корень «400 office→Linear» — не ключ, а geo-блок**: Linear отдаёт
  `403 RESTRICTED_COUNTRY_BLOCKED` всем RU-IP, включая мск-VDS. Ключ валиден,
  работает через прокси. Гейт владельца переформулирован: нужен egress
  (proxy-aware Linear-клиент офиса), не перевыпуск ключа.
- `DRU-93`: связка была ручной строкой без механизма. Слово владельца: текущий
  Linear-workspace — расходный, можно сносить.

## Процессные находки (в tooling-ретро)

- **Сессионный лимит** оборвал двух блок-агентов посреди Phase 2; возобновление
  сообщением с сохранённым контекстом отработало без потерь — паттерн надёжен.
- **`yarn install` в свежих worktree на Windows флакает** (EPERM symlink,
  UNKNOWN write/rmdir): 4 попытки на worktree интеграции, 2 на блоке данных;
  лечится полной зачисткой `node_modules` + одиночным повтором; два
  параллельных install в одном дереве гарантированно портят состояние.
- **Мёртвая запись `apply:lead-persona`** в `package.json` — скрипта
  `apply-lead-persona-defaults.mjs` не существует.
- commit-msg хук требует трейлер конкретной модели (`Claude Opus 4.8`) — warn
  на каждом коммите другой модели; кандидат на правку хука.
- Пути в задании интегратору устарели против репо (`.husky/` → фактически
  `.githooks/`; `task-archive.mjs` → `archive-task.mjs`) — агент разобрался сам,
  но карта хуков в доке пригодилась бы.
