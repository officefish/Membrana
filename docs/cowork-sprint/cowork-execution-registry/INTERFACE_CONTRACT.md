## 1. Сигнатуры и схемы на швах

### 1.1. Шов `lead-persona` ↔ `units-trace-measure` (субъект ↔ след)

- **Артефакт закрытия, поле приёмки — канон:** `{ acceptedBy: "<persona-id>", headRev: "<sha>" }`.
  Опционально `acceptanceHistory: [{acceptedBy, headRev}, ...]` — пишет closure
  lifecycle (адаптер е), не блоки; без неё `escalationRate`/`reworkRate` деградируют честно.
- **`angelina` — допустимое значение `leadPersona`** и `acceptedBy`: персона канона,
  нижний регистр, латиница, как пятеро советующих. Двойного счёта нет: ведущая
  ведёт процесс, исполнителя задачи не подменяет.
- **След — персона, принявшая выход.** Приёмка происходит строго ПОСЛЕ
  `isValid = true`; невалидный возврат следа не порождает. Подписи субагента не
  существует; его единственный публичный признак — род: канонические строки
  `SUBAGENT_KINDS = { analyst, scribe }` (экспорт `scripts/lib/angelina-delegate.mjs`).
- Отдаваемые чистые функции: `decideDelegation({fitsOneContext, needsParallelFactua,
  isLongLivedDocument}) → {delegate, kinds}`; `isValid(ret, kind) → {valid, missing}`.

### 1.2. Шов `snapshot-cold-migration` ↔ `units-trace-measure` (снимок ↔ измерение)

- **Канон формы снимка — `linear-snapshot@1` производителя:**
  `header{ capturedAt, sourceRevision, source, trigger, recordCount }` + `records[]`.
  Шапка потребителя — строгое подмножество шапки производителя.
- **Обязательный минимум полей записи** (под пять величин): `id`, `status`,
  `leadPersona`, `createdAt`, `closedAt`, `archivedAt`, `githubIssueRefs`.
  Поле отсутствует → величина честно не считается, не домысливается.
- **Запись снимка ≠ единица счёта.** Запись — сырая запись источника (Linear-issue);
  единица счёта — GitHub-issue через биекцию центральной под-задачи
  (`githubIssueRefs`). Величины считаются ТОЛЬКО по единицам счёта.
- **Движение:** снимок передаёт `status` источника как есть, производитель не
  фильтрует; интерпретация «что движется» — у словаря единиц (`UNITS_DICTIONARY.md`).
- Отдаваемое производителем: `fresh(snapshot, sourceCursor) → bool` (курсор
  инъецируется, O(1), вне тела гейта); офлайн-reader холода; `wellArchived`/
  `classifyDebt`/счётчик (`closureArtifact` — в канонической форме §1.1).
- Отдаваемое потребителем: `computeTeamMetrics(prev, curr, artifacts, window)`;
  `renderTeamMetricsReport`; `computeReworkRhetoric` (отдельный экспорт, только аудит дня).

### 1.3. Ограничение `delegatedAgent` (внешнее, проверяемое)

До живого ключа Linear на VDS и расширения `LinearIssueView` (оба — вне спринта,
гейты владельца) поле `delegatedAgent` в снимке ВСЕГДА `null`; ни одна величина
на нём не строится (проверено: пять величин читают `leadPersona` и статусы).
Имена `assignee`/`delegatedAgent` не склеиваются (вердикт M1).

## 2. Единая таблица вердикт-кодов

**Два пространства, не путать:**
- **вердикт-код** — значение в `{code, verdict}` чистой функции; различает
  «в порядке / мягко / жёстко» (исполняет M2: «протух» — ненулевой различимый);
- **exit процесса** — проекция адаптера: жёсткий вердикт → `exit = вердикт-коду`;
  мягкий → `exit 0` + обязательная печать замечания с кодом; чистый → `exit 0`
  (исполняет M4: мягкий режим не останавливает работу; и закон M2 эпика 1:
  отказ = `exit ≠ 0`).

| Код | Имя | Домен | Класс |
|---|---|---|---|
| 0 | OK | все | чисто |
| 10 | SNAPSHOT_STALE | снимок | мягкий |
| 11 | LEGALITY_MIGRATING | гейт законности | мягкий (назван блоком, ратифицирован консилиумом) |
| 12 | ACCEPTANCE_UNCONFIRMED | зубы, отказ-II soft | мягкий |
| 20 | SNAPSHOT_NO_INPUT | снимок | жёсткий |
| 21 | LEGALITY_HARD | гейт законности | жёсткий |
| 22 | LEAD_PERSONA_MISSING | зубы, отказ-I | жёсткий |
| 23 | ACCEPTANCE_FALSE | зубы, отказ-II hard | жёсткий |

Конвенция диапазонов: `0` чисто · `1–19` мягкие · `≥20` жёсткие. Код следует за
вердиктом (закон спринта). Новый код — только правкой контракта. Локальные коды
внутри блока (`0/1` зубов) — внутреннее дело блока, наружу транслируются
адаптером (б).

## 3. Инварианты потока данных

- **Владелец времени спринта — производитель снимка** (office): `capturedAt`
  UTC ISO-8601. Окно отчёта выводится из `capturedAt` обоих снимков (вырожденное
  окно — жёсткая ошибка); «день» журнала Ангелины — календарная производная
  `YYYY-MM-DD`, в таймлайн не претендует.
- **Провенанс единым словарём:** поля `capturedAt`/`sourceRevision` называются
  одинаково во всех трёх honest-шапках (снимка, холода, отчёта); шапка отчёта
  несёт их для ОБОИХ снимков. Общий закон: **вход без провенанса отвергается,
  выход несёт провенанс.**
- «Два снимка» батча = два материализованных pull-снимка; тело ни одного гейта
  и ни одной метрики в сеть не ходит; частота — вечерний батч, вебхук — только
  триггер (сигналы коалесцируются).
- `archive.jsonl` append-only, единственный писатель — office-батч; движение не
  дублируется; снимок без honest-шапки холод не принимает.
- Кириллические ключи возвратов субагентов (`фактура`, `источник`, `пробелы`,
  `дифф`, `что_открыто`) — канон домена Ангелины; ни один шов спринта их не
  пересекает; транслитерация на интеграции ЗАПРЕЩЕНА как лишний слой.

## 4. Глоссарий стыка (термин = граница)

| Термин | Значение |
|---|---|
| снимок | материализованный адресуемый pull-срез с провенансом; слово «кэш» запрещено (M2) |
| запись снимка | сырая запись источника (Linear-issue); НЕ единица счёта |
| единица счёта | GitHub-issue через биекцию центральной под-задачи |
| приёмка | событие «персона приняла выход»: `acceptedBy` + `headRev`, после `isValid = true` |
| род субагента | `analyst` \| `scribe`; единственный публичный признак безымянного субагента |
| вердикт-код | код в структуре возврата чистой функции (таблица §2) |
| exit процесса | бинарная проекция вердикта адаптером (жёсткий → код, мягкий/чисто → 0) |
| honest-шапка | провенанс входа/выхода; без неё вход отвергается |

## 5. Интеграционный smoke (исполняемые тесты интеграционной ветки)

1. Фикстурный источник → producer → снимок `linear-snapshot@1` с полной шапкой;
   `fresh`: свежий → 0; протухший → вердикт 10, exit 0 с печатью.
2. Адаптер (а) проецирует `records[]` → вид батча; `checkLeadPersona` на карточке
   без персоны → вердикт 22, `exit ≠ 0`; отказ-II soft → 12, exit 0.
3. `computeTeamMetrics(prev, curr, ...)` → отчёт с ревизиями ОБОИХ снимков и
   подписью «wip = очередь намерений, не производительность».
4. `classifyDebt` на смешанном наборе (work/legality/parked/archived-pre-passport)
   → счётчик читает только `work`.
5. Cold-writer: запись с honest-шапкой принята, без шапки — отвергнута.
6. `decideDelegation`(долгоживущий документ) → `scribe`; `isValid` обоих родов.
7. Все коды — по таблице §2; собственные тесты трёх блоков зелёные в собранной ветке.
