## 1. Что этот блок строит

Блок материализует вердикт M1 — контракт ведущей персоны — в пять носителей:

1. **Промпт Ангелины** — файл ведущей персоны, структурно отличный от пяти советующих.
2. **Журнал памяти** — схема из ровно трёх полей с провенансом «день + источник».
3. **Предикат делегирования** — чистая булева функция трёх наблюдаемых входов.
4. **Шаблон обезличенного субагента** — четыре обязательных пункта, два рода.
5. **Валидатор возврата `isValid`** — наблюдаемый провал вместо молчаливого.

Плюс **первая процедура** — шторм (не утро): единственная процедура, которую
Ангелина получает в этом спринте.

Ключевое различение, которое блок обязан удержать в каждом носителе:
**ведущая ≠ шестая советующая**. У советующей несущее — «Кто ты в традиции» +
зона + запреты зоны (`PROMPT_STRUCTURER.md` и четыре его собрата). У ведущей
несущее — **правила делегирования**; предметной зоны у неё нет, её «зона» — сам
поток работы. Всё существующее вокруг персон (реестр псевдонимов, формат промпта,
журналы позиций в `docs/virtual-team/memory/*.md`) рассчитано на советующих —
поэтому носители Ангелины строятся отдельной конструкцией, а не копией.

## 2. Устройство изнутри

### 2.1 Промпт ведущей персоны — `PROMPT_ANGELINA.md`

Три обязательные секции (вердикт M1, DoD-1):

| Секция | Что несёт | Почему несущая |
|---|---|---|
| **Цели диалога** | Ангелина держит рантайм-диалог с владельцем и сопровождает его ПО дню; оркеструет фоновых субагентов; фиксирует подтверждённое | функция ведущей — оркестрация диалога, форма следует функции |
| **Критерии качества** | результат собран из валидированных возвратов; подтверждения владельца отражены в живом документе; единый голос — итог принадлежит Ангелине, не субагентам; честная шапка провенанса у всего, что уходит в журнал | без критериев «done» — зелёная лампочка при треске внутри |
| **Правила делегирования** | предикат `delegate` (§2.3) + правило выбора рода + правило одного контекста | убери их — либо всё сама (нет оркестрации), либо хаос; это несущая конструкция промпта ведущей |

Явные запреты в промпте (вердикт M1, «чего быть не должно»):

- **Запрет делать работу исполнителя при наличии субагента** — дирижёр не дует в
  трубу, пока в оркестре есть трубач. Симметричный ограничитель: если все три входа
  предиката ложны — делает сама, делегирование влезающей задачи = накладной расход.
- **Нет секции предметной зоны ответственности** — иначе шестая советующая.
- **Нет конкретной задачи** — конкретная задача живёт в промпте субагента.
- **Нет права отвечать за субагента его голосом** — смешение слоёв; возвраты
  валидируются и синтезируются, а не транслируются.

Шапка файла — по образцу существующих промптов (персонаж, аватар), но **без**
секций «Кто ты в традиции» / «Зона ответственности» / «Взаимодействие с ролями»
в смысле советующих. Ссылка на журнал и на процедуру шторма.

### 2.2 Журнал памяти — схема `angelina/MEMORY_SCHEMA.md` + носитель `angelina/JOURNAL.md`

Ровно три состава-поля (вердикт M1, DoD-2) — не больше и не меньше:

| Поле | Состав записи | Пример статуса |
|---|---|---|
| **Незакрытые хвосты** | формулировка + статус | `открыт` / `ждёт владельца` / `делегирован (род)` |
| **Напоминания владельца** | дословная просьба «не забудь про X» | — |
| **Итог дня** | одна строка на день | — |

Провенанс каждой записи: **`день + источник`** — календарный день (`YYYY-MM-DD`)
и источник напоминания/факта (реплика владельца в диалоге, возврат субагента,
процедура). НЕ «позиция/голос/как рассудила жизнь» — этот вопрос для ведущей
бессмыслен, у неё нет позиций.

Носитель живёт в `docs/virtual-team/angelina/JOURNAL.md`, а **не** в
`docs/virtual-team/memory/` — каталог memory/ принадлежит советующим и их
формату (позиции, калибровка); класть туда журнал другого устройства значит
смешать леммы. Это осознанное решение блока, вынесено в EXPECTATIONS как
предложение соседям и интеграции.

Потребитель журнала — утренний диалог следующего дня: журнал обязан читаться
«без гадания» (наблюдаемость по Верстальщику), иначе утро начнётся со слепого
места.

### 2.3 Предикат делегирования — `scripts/lib/angelina-delegate.mjs`

Чистая булева функция трёх наблюдаемых входов, без сети, без состояния:

```
delegate(task) = ¬fitsOneContext(task) ∨ needsParallelFactua(task) ∨ isLongLivedDocument(task)
```

Форма реализации (Phase 2): `decideDelegation({fitsOneContext, needsParallelFactua,
isLongLivedDocument}) → {delegate: boolean, kinds: string[]}` — входы already
наблюдены и переданы булевыми, функция только выносит вердикт. Род помощника
выводится из двух род-указывающих входов:

- `isLongLivedDocument = true` → **писарь** (`scribe`);
- `needsParallelFactua = true` → **аналитик** (`analyst`);
- оба истинны → оба рода (по субагенту на род, не универсал);
- `delegate = true` при пустом `kinds` (только `¬fitsOneContext`) — общий триггер
  «переросло контекст»: род не определён входами, выбирается при декомпозиции
  по природе работы; функция честно возвращает `kinds: []`, а не выдумывает род.

Полная таблица истинности — 8 комбинаций (unit-тесты покрывают все, DoD блока):

| # | fitsOneContext | needsParallelFactua | isLongLivedDocument | delegate | kinds |
|---|---|---|---|---|---|
| 1 | true | false | false | **false** — делает сама (эталон: шторм 19.07 инлайном) | `[]` |
| 2 | true | true | false | true | `[analyst]` |
| 3 | true | false | true | true | `[scribe]` |
| 4 | true | true | true | true | `[analyst, scribe]` |
| 5 | false | false | false | true | `[]` — род при декомпозиции |
| 6 | false | true | false | true | `[analyst]` |
| 7 | false | false | true | true | `[scribe]` |
| 8 | false | true | true | true | `[analyst, scribe]` |

Единственный «делаю сама» — строка 1. Правило одного контекста защищает от обоих
рефлексов: «всегда делегирую» (пятикратный токен за иллюзию оркестра) и «никогда
не делегирую» (солист, клиппящий по контексту).

### 2.4 Шаблон обезличенного субагента — `angelina/SUBAGENT_TEMPLATE.md`

Четыре обязательных пункта (вердикт M1, DoD-4), шаблон один, род — параметр:

1. **Одна конкретная задача** одной формулировкой (не список, не «помоги с»).
2. **Прямой запрет коммуникации с владельцем** — «возвращай выводы материнской
   персоне; вся коммуникация с владельцем — на родителе».
3. **Валидируемый формат возврата** — структура с полями рода (§2.5), не
   свободный текст.
4. **Границы фактуры** — на чём стоять (пути, снимки, прежние решения), чего не
   выдумывать; всё нужное субагенту — в его стартовом промпте.

Свойства обоих родов, зашитые в шаблон: субагент **обезличен** (имени, атрибуции
и подписи не существует — не сочинять), **порождающий** (создаётся на задачу,
уничтожается после возврата), **память не персистит** (журнал — только у ведущей),
о других субагентах не знает. Топология без обратных рёбер:
ведущая → субагент → структура возврата → `isValid` → синтез в диалог.

### 2.5 Валидатор возврата — `scripts/lib/angelina-validate.mjs`

`isValid(ret, kind) → {valid: boolean, missing: string[]}` — чистая функция,
гоняется ведущей **до сборки** результата в диалог; провал наблюдаем, не молчалив.

| Род | Обязательные поля возврата |
|---|---|
| **аналитик** (`analyst`) | `фактура` (непустая) · `источник` (непустой — где взято) · `пробелы` (поле обязано присутствовать; пустой список — легален и означает «пробелов не вижу») |
| **писарь** (`scribe`) | `дифф` (что изменено в живом документе) · `что_открыто` (поле обязано присутствовать; пустой список легален) |

`valid = true ⟺ все обязательные поля присутствуют ∧ содержательные поля непусты`.
Возврат свободным текстом, возврат с отсутствующим полем, возврат чужого рода —
отвергаются с перечислением `missing`. Тесты — на оба рода, позитив и негатив
(DoD блока: «isValid отвергает возврат без обязательных полей»).

### 2.6 Первая процедура — шторм, не утро: `angelina/STORM_PROCEDURE.md`

Ратифицировано (эпик Р1): риск низкий (документы, не прод-код), процедура
обкатана вживую 19.07 дважды, задействует оба рода субагентов. Утро дороже —
завязано на тимлида с полномочием и контур 2; в этот блок не входит.

Скелет процедуры (документ Phase 2):

- **Вход:** тема шторма от владельца, одной формулировкой.
- **Ход:** Ангелина держит диалог; тезисы-кандидаты фиксируются в живом
  `THESES.md` — по предикату это `isLongLivedDocument` → **писарь**; добыча
  фактуры из репозитория/реестров/истории — `needsParallelFactua` → **аналитик**;
  влезающее в контекст Ангелина делает сама (эталон 19.07).
- **Каждый возврат** — через `isValid` до внесения в диалог.
- **Выход:** сводный `REPORT.md`, каждый тезис которого подтверждён владельцем
  в ходе беседы; итог дня + незакрытые хвосты — в журнал с провенансом.
- **Критерий качества:** единый голос — отчёт написан Ангелиной, следов
  «разных рук» субагентов в тексте нет.

## 4. Границы блока

- **Не пересматриваю вердикты** — M0/M1 применяются как есть; неисполнимость → `BLOCKER.md`.
- **Не строю** процедуру утра, полномочие Веснина, контур 3 (сервер), sunrise-flash — соседние работы и циклы.
- **Не трогаю** промпты и память пяти советующих персон — источники только для чтения.
- **Не проверяю Linear API** — brief поручает живые проверки блокам `units-trace-measure` и `snapshot-cold-migration`; у этого блока внешних неизвестных нет.
- **Не договариваюсь об интерфейсах** — всё стыковое одностороннее, в `EXPECTATIONS.md`.
