# Заседание `scripts-boundary` — M0 (установочное)

Общее задание: `docs/meeting/scripts-boundary/MEETING_BRIEF.md`.
Вход: `docs/tasks/research/scripts-boundary-container.md` (проверен по коду).

M0 держит все кандидаты в одной комнате и выносит **единственный** вердикт —
порядок зависимостей (DAG). Ранг фундамента виден только когда все кандидаты
разом; изолированный запуск защищает от потери вопроса, но не от неверного ранга.

## Единственный вопрос заседания M0

**O1 — Каков порядок зависимостей (DAG) между кандидатами защиты контура
`scripts/`: какой вопрос — фундамент, что от чего зависит, что можно вести
параллельно, и в каком порядке созывать M1..Mn?**

## Кандидаты (простым текстом, ранг назначает это заседание)

Каждый — потенциальное отдельное заседание Mn. Задача M0 не решить их, а
упорядочить.

1. Семантика отказа шага: убрать `|| true`, ввести явную классификацию
   критичный/некритичный (весь контур падает громко вне явного списка). Есть
   готовый вердикт B1 и карточка `rt-9`.

2. Персистентный журнал прогонов ритуала: кто/что/когда/результат по каждому
   прогону цепочки. Сейчас нет; ресёрч называет базой отказоустойчивости.

3. Версионирование схемы артефактов + гейт свежести на каждом чтении. Артефакты
   датированы и читаются следующим прогоном; при смене структуры парсер ломается
   молча. Гейт свежести есть только у payload дайджеста (баг B2/#586).

4. Ретрай транзиентных сбоев для внешних шагов (office, git): где, сколько
   попыток, для каких шагов. Сегодня латали руками.

5. Реестр процессов + регламент: форма записи и чем держать живым. Ресёрч и наш
   канон согласны — ручной протухнет (уже: 4 теста-сироты), реестр обязан быть
   производным от кода.

6. Класс объекта `scripts/`: делать workspace-пакетом (turbo/lint/typecheck
   накрывают штатно, у правил границ появляется корень) или сознательно держать
   снаружи — и чем тогда защищать контур.

## Что должен вернуть M0

- DAG: рёбра «X — фундамент для Y».
- Корень (фундамент заседания) — один или несколько независимых.
- Что параллелится, что строго последовательно.
- Кандидаты, которые не требуют консилиума (есть готовый вердикт/канон) — их
  исполнять, а не заседать по ним.
- Порядок созыва M1..Mn.

Вердикт называет полный список посылок (правило про лишнюю посылку — зеркало
реестра правды).
