## Работы (для `docs/STORM_REGULATION.md`, шестой формат)

### Р1 — контейнер и жизненный цикл (M1)

- `isOpen(s) = exists(dir(s)) ∧ topic(s)≠∅ ∧ topicByOwner(s) ∧ ¬isClosed(s)`.
- `isClosed(s) = (pora(s) ∨ sevenBreaths(s)) ∧ forkNamed(s)`, где
  `forkNamed ⟺ fork ∈ {directions ∪ {"никуда"}}`. Триггер без развилки — не закрыт.
- **Тотальность/взаимоисключение:** `isOpen ∧ isClosed = false`; `isOpen ∨ isClosed ∨
  isNotStarted = true`.
- **Стенка:** наружу — конспект + развилка (обязательны), директива **факультативна**;
  реплики остаются внутри.
- **Шторм vs заседание** — одна черта: `isRequired(output)` = false для директивы шторма,
  true для вердикта заседания. Оснастка общая.
- Чистые предикаты над `docs/storm/<id>/`, без сети, юнит-тест на каждый конъюнкт +
  анти-паразит-тест (зелёный статус при ложном конъюнкте обязан падать).

### Р2 — роли и гейты (M2)

- **Кодекс Ангелины** — чистый предикат `codex: Document → pass ⊕ fail(reasons⊆QC)`,
  детерминированный; несёт `codexVersion` (**semver**: major = ужесточён/удалён QC,
  minor = добавлен непрерывающий, patch = формулировка).
- **QC (замкнутое множество):** QC1 свежесть (`sourceDigest==sha256(sources)`) · QC2
  структура (обяз. секции, вкл. «Список посылок») · QC3 honest-шапка · QC4 две подписи
  T10 (`signatures≥2`) · QC5 посылки-не-выводы. `codex = QC1∧…∧QC5`.
- **Возврат** = адресный акт `returned(reasons, codexVersion)` (перечислены павшие QC);
  отличен от derived-статуса «не прошёл».
- **Питомец = ГОЛОС, не гейт.** Гейт каскада **один — Ангелина** (у выхода один владелец,
  как в `registry-relocation`). Реплика питомца логируется с origin `pet`, **никогда не
  попадает в `reasons`**.

### Р3 — тайминг и вычисление (M3, числа за Дыниным)

- **SimHash 64 бита** всего конспекта (упорядоченные нормализованные тезисы).
- **Событие** = смена состояния тезиса (добавлен/изменён по существу/снят); реплика без
  смены — не событие. **Пауза** = «ход завершён без новой смены», граница окна счёта.
- **Вдох** ⟺ `popcount(опорный XOR текущий) ≥ K_эфф`. Косметика → `d=0` (нормализация).
- **Смягчение:** `K_эфф = K0 − g(a)`, `g(a)=⌊(K0−K_min)·a/(a+c)⌋`, `K_эфф∈[K_min,K0]`,
  `K_min≥1`, `a` = число принятых артефактов. Константы `K0,c,K_min` — подписанные, не magic.
- **Счётчик** `{опорный_хеш, счётчик:0..7, K_эфф}` — derived-проекция append-only лога,
  `step(state,event)→state'`, без сети/`Date.now`/`Math.random`.
- **Завершимость — ИСПРАВЛЕНО Дыниным (поправка по финальному аудиту 20.07).**
  Ложная лемма «дрейф монотонно растёт до 64» **снята**: Хэммингово расстояние SimHash
  не монотонно. Верная теорема: **завершимость даёт ПОТОЛОК СЧЁТЧИКА (≤7 вдохов),
  безусловно; порог `K_эфф` регулирует ЧАСТОТУ вдохов, не факт остановки.** Сходимость
  хеша ни при чём.
  - **Разрыв, который потолок один не закрывает:** при `K_min>1` дрейф может застрять
    ниже порога → вдохи прекращаются → счётчик замирает `<7` → авто-закрытие не срабатывает
    (такой шторм закрывает «пора» владельца или критерий готовности, не потолок).
  - **Рекомендация Дынина в регламент — мандат `K_min=1`:** тогда любое существенное
    событие (`d≥1`) есть вдох, ≤7 существенных событий **гарантированно** жгут потолок,
    увернуться можно лишь косметикой (`d=0` = не прогресс). Только так «неотменяемый
    потолок» держится — иначе его обходят под-пороговым дрейфом.
- **Инварианты `K_эфф(a)` (контракт M3, тестируется без живого шторма):** форма —
  гипербола `K_эфф(a) = K_min + (K0−K_min)·c/(c+a)` (тождественна `K0 − g(a)`); `a=0→K0`,
  `a→∞→K_min`; монотонно не возрастает. `K0 ≥ K_min ≥ 1`, `c > 0`. **`c` — число
  артефактов «полуспада»** (при `a=c` смягчение ровно половинное), в тех же единицах, что
  `a`; калибруется внутри диапазона-заглушки, не с нуля.

### Р4 — место и подводка (M4)

- **`docs/STORM_REGULATION.md`** — шестой формат (competition · cowork · hackathon ·
  night · meeting · **storm**), помечен **дивергентный**.
- **Три провода:** подводка дикого агента → регламент · `PROMPT_ANGELINA.md` (оператор,
  не голос) → регламент · индекс форматов → регламент. Каждый — с якорем-файлом **или**
  явным открытым долгом.
- **Критерий провода (булев):** якорь-файл существует И обратная достижимость из индекса.
  **Долг ≠ провод.** В README не рисовать живую ссылку на несуществующий якорь.
- **Шов с заседанием:** выход-развилка, ветвь = заголовок заседания; контракт
  `handoff: конспект → вход заседания` (проверен на самом этом заседании).
- **Шов с движком:** факультативная директива (две подписи) → GitHub-issue по инварианту
  законности — **норма-называние**, не файловый провод; движок не проектируем.

## Порядок исполнения

`Р1 → {Р2 ⊥ Р3} → Р4`. Обратный ранг: Р3 раньше Р1 — тайминг на несуществующем потоке
событий; Р4 раньше Р1 — рама без картины.
