## Что решено капитаном (не переоткрывать — это вход, не вопрос)

1. **Аналогия:** контейнер имущества по GROUP_CONTAINERIZATION — контейнер +
   инструменты + Google Drive «под капотом» (метаданные в git, вещдоки-бинарь наружу).
2. **Родовой узор, третий домен:** версионируемый сабграф = **НАБОР** (скрипты→кит,
   процедуры→фрейм, имущество→набор; все PINNED_SUBGRAPH_VERSIONING).
3. **Набор формируется на основании ЧЕКА:** один чек (покупка) → один набор активов.
4. **Реестр — git-дом `docs/assets/`;** вещдоки (фото, сканы чеков) — Drive-ссылками.
5. **Две записи на актив:** чек (провенанс, неизменяем) + фотография (датированный
   акт подтверждения, годен только сегодняшним числом — гейт RT-9).
6. **Порядок фаз:** настройка доступа к Drive — ПРЕДПОСЛЕДНЯЯ фаза (перед приёмкой).
