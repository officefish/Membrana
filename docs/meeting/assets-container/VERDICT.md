## РАТИФИЦИРОВАНО капитаном (22.07)

Свод M0–M4 принят каноном.

**Поправка капитана — особенность контейнера: часть документов под .gitignore.**
Реестр имущества несёт чувствительное (содержимое чеков: цены, серийники, платёжное;
конфиг доступа к Drive/креды). Значит внутри git-дома `docs/assets/` — свой разрез:
- **В git (коммитим):** публичный реестр — что за актив, вид, держатель, статус
  свежести, структура набора. То, что системе нужно для аудита по сегодняшнему числу.
- **Под .gitignore (только локально):** чувствительные документы — детали чеков
  (цены/серийники/персональное), конфиг доступа к Drive. Точный список — определяет
  фаза И1 (схема), это её граница.

**Следствие безопасности:** контейнер с намеренно-игнорируемым чувствительным — ровно
тот случай, где нужен секрет-скан на pre-commit. Смыкается с долгом `gitleaks-absent`
(попугай): пока gitleaks не стоит, случайный коммит чувствительного не ловится. gitleaks
из «привычного мелкого долга» становится пререквизитом контейнера имущества.

## РАТИФИЦИРОВАНО — поправка M1: сервер-БД за API для чувствительного (капитан, 22.07)

Чувствительное имущество (детали чеков, креды/ссылки Drive) — в **сервер-БД за API**,
НЕ под gitignore (gitignore-разрез снят как фрагильный). Довод капитана: выбор хранилища
диктует **классификация данных (чувствительность), не объём** («не потому что данных
много, а потому что они чувствительные»). Три яруса:
- **git** — публичный реестр (актив, держатель, свежесть, набор); гейт RT-9;
- **сервер-БД за API** — чувствительное (доступ-контроль, аудит, шифрование на покое);
- **Google Drive** — бинарь (фото, сканы).

**Принцип (капитан, снимает возражение о нестабильности office):** «Мы ОБЯЗАНЫ
действовать с работающими серверами. Если нестабильность сервера покажет себя в
динамике — поменяем либо сервер, либо провайдера». Ответ на «сервер нестабилен» —
не «архитектурить в обход сервера», а «сервер должен работать; иначе замена».
Следствие: #933 (стабильность office) — не блокер-в-обход, а **обязательство**;
монитор сетевого окружения (#933) — инструмент, показывающий **динамику**, на которой
принимается решение «менять сервер/провайдера».

Эпик тяжелеет до server-контура (БД+API, prisma:migration). Фаза «доступ» (предпоследняя)
включает провижн БД+API + креды Drive. Готов к регистрации по слову капитана.
