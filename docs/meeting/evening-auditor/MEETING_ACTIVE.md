## Вердикт M1 — граница процесса (корень DAG закрыт)

Протокол: [`evening-auditor-m1-boundary-2026-07-18.md`](../../seanses/evening-auditor-m1-boundary-2026-07-18.md) · 18 реплик · LGTM Teamlead

**Граница = функция `audit(logs, dayPlan, truthCrystals[]) → derivedReport`.**
Область определения **замкнута тремя входами** — это и есть черта.

| Внутри | Снаружи |
|---|---|
| Сверка: лог дня ↔ план дня | `mint` нового owner-токена |
| Перепроверка актуальности кристаллов | Назначение задач и DoD |
| Разрешение конфликта источников — **приоритет кода над логом** (rt-2) | Исправление кода/документа |
| Порождение derived-отчёта как факт | Перемер метрик детекторов / переслушивание звука |
| | Выбор носителя, веток, копий отчёта (→ F) |

**Инварианты (проверяемы чистой функцией, без сети и звука):**
- **I1** — `verdict` тернарный: `ok | drift | stale`. `stale` = честное «не смог
  проверить», а не ложный `ok`. → 🔒 **КАРАНТИН: предмет H**
- **I2** — `resolvable(evidence-ref) ∨ verdict = stale`: нерезолвимое сырьё не даёт зелени.
- **I3** — `dedup(evidence по origin-hash)`: эхо не удваивает вес.

Форма элемента `{claim, verdict, evidence-ref, scope: sverka|recheck}`, блоки отчёта и
контракт-тесты в DoD → 🔒 **КАРАНТИН: предмет F**.

> **🔒 Карантин по hard rule 2 (принято владельцем 2026-07-18 — «согласимся с
> аудитором»).** Положения, помеченные выше, вынесены M1 за пределы его компетенции:
> I1 принадлежит **H** («механический признак ≠ вердикт»), форма элемента и блоки —
> **F** (сток). Они **не связывают** комнаты H и F: те созываются с чистого листа и
> вправе решить иначе. Здесь положения сохранены только как *материал предложения*,
> не как действующая норма. Действующая часть вердикта M1 — сигнатура функции, списки
> внутри/снаружи, I2 и I3.

Статус: **derived-токен** (origin: посылки владельца + консилиум 18.07). Новых
owner-токенов не введено, ацикличность сохранена.

### Находка о рекурсии (ключевая)

Аудит M1 назвал три дефекта. M1′ созывался их исправить — и **воспроизвёл все три**:
посылки в новой конфигурации, колонизацию по новым адресам (F, T вместо H, F), счётчик
вместо разбора — уже в риторике председателя, а не в скрипте.

**Один прогон переоткрытия не лечит класс.** Лечит либо гейт с зубами на проверки 3–5
(`meeting:audit` **не реализован** — это Db), либо **шаблон вердикта**, разделяющий:
посылки / выводы / DoD моей комнаты / материал для чужих комнат. Прозой класс уже не
удерживается — ровно тот вывод, на котором стоит сам формат заседания.

## Бесхозные признаки — дом назначен (рекомендация аудита №5)

Повестка M1 требовала назвать признак, по которому проведена черта: **период, источник,
детерминированность, владелец предиката**. M1 назвал только источник; переоткрытие M1′
сузило рамку так, что остаток не вошёл ни в одну повестку — «списан сужением», как
сказал аудитор. Назначаю дом явно:

| Признак | Состояние | Дом |
|---|---|---|
| **источник** | ✅ назван в M1 (три входа) | закрыт |
| **период** | ✅ назван в M1′ де-факто (день против истории) | закрыт, зафиксировать формулировкой при сборке эпика |
| **детерминированность** | ⏳ объявлена смежной, вердикт запрещён в B | **→ Da** (воспроизводимость: без сети, по коммиту) |
| **владелец предиката** | ⛔ не назван нигде (слово «владелец» — 0 вхождений в M1′) | **→ Da** (кто производит каждую из трёх дорожек и когда) |

**Обоснование дома:** оба остатка — про то, откуда берутся данные и можно ли им верить
воспроизводимо. Это предмет `Da` («поля для воспроизводимости»), а не `H` (конституция
вердикта) и не `T` (читатель). При созыве M2′ = Da повестка **обязана** нести оба:
иначе провал M1 закроется вторично сужением.
