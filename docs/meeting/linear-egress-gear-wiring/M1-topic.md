## Единственный вопрос заседания M1

**E2 — какой должна быть форма egress-пути office↔media↔Linear через media-VPS NL:
кто инициирует боевой pull к Linear, где живёт клиент GraphQL к api.linear.app, и что
именно видит / хранит office (MSK) после прохождения тракта?**

Ответ — одна согласованная топология (роли узлов + направление запросов), не меню
«можно и так и так» без выбора.

---

## Кандидаты формы (простым текстом — не вопросы этой комнаты)

1) Office инициирует батч-pull, а media-NL — только сетевой egress (прокси/туннель):
   клиент Linear остаётся в office, IP-выход — через NL. Паспорт зовёт производителя
   снимка «office-батч»; этот кандидат ближе к букве паспорта.

2) Media-NL держит клиент Linear и сам делает полный pull; office получает только
   готовый артефакт снимка (push или pull артефакта по внутренней сети). Office в
   api.linear.app не ходит вовсе.

3) Гибрид: media исполняет GraphQL по заказу office (office шлёт «сделай pull», media
   возвращает тело снимка), без долгоживущего туннеля из MSK наружу. Секреты и trust —
   следующая комната (К2); здесь только кто говорит с Linear и куда ложится результат.

## Вход (факты / нормы — не вердикты)

- Владелец: egress = media-VPS NL; агент/ноут в РФ в Linear не ходит; туннель в прод
  агент сам не поднимает.
- Паспорт §6: вход гейтов — снимок `linear-snapshot@1`; производитель назван как
  office-батч полным pull; вебхук — только триггер.
- Паспорт §9 / замер 19.07: Linear `403 RESTRICTED_COUNTRY_BLOCKED` с РФ и с MSK office.
- В коде office уже есть исходящий Linear GraphQL-клиент (`LinearService` →
  `api.linear.app/graphql`) и приём webhook; боевой pull снимка с NL ещё не проведён.
- Пилот 20.07: движение Linear — stub / deferred egress.

---

## Что комната должна выдать

- Выбранную форму пути: кто инициирует, где клиент Linear, что видит office.
- Явное отношение к формулировке паспорта «office-батч» — сохраняется, уточняется
  или переименовывается смыслом (без пересмотра аксиомы «гейты читают снимок»).
- Что **не** решается здесь и уходит в К2 (секреты) / К3 (контракт снимка DoD) —
  одной строкой каждый, без вердикта по ним.
- DoD планировочный (проверяемые признаки «форма выбрана»), без деплоя туннеля.
