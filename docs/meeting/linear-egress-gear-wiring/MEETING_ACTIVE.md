## Порядок (вердикт M0 — **НЕ РАТИФИЦИРОВАН**)

Источник: [`linear-egress-gear-wiring-m0-order-2026-07-20.md`](../../seanses/linear-egress-gear-wiring-m0-order-2026-07-20.md).

Кандидаты: **К1** архитектура egress · **К2** секреты/trust · **К3** контракт снимка/DoD pull ·
**К4** подводка процедур (расщеплён: **К4a** каркас, **К4b** гейт closure) · **К5** stub и снятие.

```
Слой 0:  К1 (форма egress-пути)     К4a (каркас процедур спринта)
              │                           (независимая компонента)
              ▼
Слой 1:  К2 (секреты / trust boundary)
              │
              ▼
Слой 2:  К3 (контракт снимка + DoD первого боевого pull)
              │
       ┌──────┴──────┐
       ▼             ▼
Слой 3: К5         К4b
     (снятие stub) (гейт closure по снимку)
```

| Ребро | Обоснование комнаты |
|---|---|
| К1 → К2 | топология задаёт, на каком узле живёт секрет |
| К2 → К3 | trust boundary входит в доказательство провенанса снимка |
| К3 → К5 | критерий снятия stub = успешный боевой снимок |
| К3 → К4b | предикат closure проверяется по снимку |

**Независимы явно:** К4a ⟂ К1 (параллельны на слое 0). Циклов нет.

**Фундамент:** К1 (форма пути; маршрут media-NL уже норма владельца).

**Нуждается в фактуре до существа (не до порядка):** К2 — inventory секретов на VPS;
К3 — живой ответ Linear с media-NL. К1 и К4a от недобытой фактуры не зависят.

**Риски обратного ранга:** К3 раньше К1 — контракт снимка под неверным pull-инициатором.
К5 раньше К3 — критерий снятия stub привяжется к фикстуре.
