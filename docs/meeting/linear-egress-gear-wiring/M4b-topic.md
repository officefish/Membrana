## Кандидаты (простым текстом — не вопросы этой комнаты)

1) Closure-гейт читает: артефакт closure `{acceptedBy, headRev}`, карточку
   (leadPersona), снимок `S` офлайн. Обязательно: `pullOk(S)` ∧ Issue закрыт на
   `headRev` ∧ `acceptedBy` непуст ∧ единица есть в снимке (linearId/githubIssueRefs).
   `contentDigest` — **ещё не обязателен** в первой волне; мягкий код если снимок
   stale (10), жёсткий если снимка нет / ¬pullOk (20) или acceptance ложна (23).
   Снятие stub — не условие closure (ортогонально К5).

2) То же, но `contentDigest` **обязателен** сразу: без digest → SNAPSHOT_NO_INPUT (20).
   Жёстче против подмены тела при том же header.

3) Closure не читает снимок вовсе — только git acceptance; снимок только для метрик.
   Отклоняется, если ломает «движение проверяется по снимку».
