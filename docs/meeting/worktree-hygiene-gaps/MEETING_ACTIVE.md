## Фактура, добытая по коду (2026-07-20, вход для M1/M2)

- Teardown-инструмент **уже есть**: `yarn repo:clean --worktrees` (#492) — сносит мёртвые
  ветки и worktree, источник истины = состояние PR (squash слепит `git branch --merged`).
- Свежесть базы **уже есть**: `yarn branch:check-base --strict` (exit 1 для хука/CI).
- `worktree-bootstrap` (#711) — только заведение, teardown-части нет.
- **Ни один инструмент git-дерево-гигиены не подключён к ритуалам и хукам.** `ritual:day`
  = morning-care (осведомлён о коллизии worktree, но не чистит) + `deps:watch`; `ritual:evening`
  = 11-шаговый манифест + `deps:watch`. В ритуалах бежит только deps-гигиена.
- Скилл `membrana-worktree`: teardown = `git worktree prune` «периодически, руками».
- Итог: подрезки нет → на 20.07 накопилось 30+ деревьев.

## Порядок из M0 (вердикт, ратифицирует владелец)

**K2 → K1.** Фундамент — **K2** (lifecycle/teardown дерева + гейт «нет хвостов»);
надстройка — **K1** (синхрон базовых веток с `main`). Обратного ребра нет: предикат
сноса дерева не содержит distance(base, main). Риск инверсии: механизм свежести,
построенный по популяции 30+ деревьев (большинство под снос), — переинжиниринг по
мёртвым, а гейт свежести шумит на хвостах.

### M2 (K1) — статус: гейт-блок при консистентном содержании

Четыре прогона (обычный, скелет посылок, посылки-до-DoD, строгий порядок секций) — во
всех комната вынесла вердикт, но **ни разу не написала секцию `## Список посылок`** →
S-M2: заседание по K1 формально НЕ состоялось. Посылки задним числом дописать нельзя
(правило 6, прецедент M2″ 18.07). Не token-truncation (16384 хватает, футер на месте) —
комната осознанно заканчивает на DoD. Специфично для M2: M0 и M1 секцию вынесли.

**Содержание вердикта K1 (воспроизведено 4× почти идентично, как черновик, НЕ протокол):**
- Событие/частота: шаг `ritual:day` (вход агента в дерево) + явный `yarn worktree:sync`; не cron, не per-commit хук.
- Предикат (чистая функция после `git fetch`): классы `fresh` (behind==0) / `ff-able` (behind>0, clean, ahead==0) / `diverged` (ahead>0) / `dirty`; метрики против `origin/main`.
- Режим: авто-действие **только** `--ff-only` на `ff-able` (громко логирует); `diverged`/`dirty` — только сигнал, решает агент дерева; авто-мёрдж/rebase запрещён.
- Область: список деревьев **читается из реестра K2** (не хардкод), `main` исключён (identity); синхроним только легитимные.
- Exit-код: находка расхождения ≠ сбой (`exit-code-semantics-audit`); отчёт честный, a11y (иконка+текст).
- Пороги (`behind≥10`, `age≥7дн`) — в конфиге, не хардкод.

### Вердикт M1 (K2, кратко)

4 класса дерева: `canon` (не трогать), `sprint-closed` (кандидат к сносу + причина «PR
#NNN merged»), `sprint-open` (активно), `unregistered` (человеку на разбор). Предикат
сноса: `sprint ∧ prClosed` (¬uncommitted ∧ ¬unpushed ∧ prState∈{merged,closed}); сеть-ошибка
`gh` → `unknown`, не сносить. Различитель канон/спринт — явное поле `kind` в `WORKTREE.md`,
каноническое множество закрыто (main/tooling/product/codex/cursor). Гейт «нет хвостов»:
`repo:clean --dry-run` шагом `ritual:day` (report), `--execute` руками; хук не трогаем.
Кандидат «по возрасту» отвергнут (вводит время в предикат, ложные срабатывания). Первый
прогон M1 отклонён гейтом правило-6 (нет секции посылок) → `rejected/`; перепрогон прошёл.

### Аудит M1 (`yarn meeting:audit`, exit 66)

Проверки [1][2][3][6] — ✓ на обоих протоколах. Проверка [4] — FAIL на M0 (W0) и M1 (W1):
**ложное срабатывание**. Механизм: `meetingVerdictProblems` метит «колонизацией» любой
ID-токен в секции DoD, а «свой» ID распознаёт только жирным (`**Wx`, `extractAgendaIds`).
Обе комнаты называют СВОЙ вопрос в DoD простым текстом («для W0», «только W1») — гейт
читает собственный вопрос как чужой. По существу колонизации нет: M0 решал лишь порядок,
DoD M1 явно исключает K1. Известный тупой греп-ярлыка (регламент § Аудитор, проверки 2–5
«меряют ярлык, не вердикт»). Протоколы под гейт не правились (запрет подделки).
