## Единственный вопрос заседания M2

**W2 — как базовые ветки долгоживущих деревьев (`main`, `tooling`, `product`, `codex`,
`cursor`) синхронизируются с `main`: с какой частотой и по какому событию, чьими руками
(агент/скрипт/хук), по какому ПРОВЕРЯЕМОМУ предикату база считается «протухшей», и в каком
режиме синхрон вмешивается (сигнал vs авто-подтягивание)?**

Ответ — одна согласованная процедура синхрона (событие + предикат протухания + исполнитель
+ режим), не меню без выбора.

---

## Кандидаты формы (простым текстом — не вопросы этой комнаты)

1) Сигнал-в-ритуале: `branch:check-base --strict` (уже есть) шагом `ritual:day` по каждому
   легитимному дереву; красный = сигнал агенту «подтяни `main`», подтягивание — руками.
   Гигиена видима, но не авто-мутирует чужое дерево.

2) Авто-подтягивание базы: ритуал сам делает fast-forward базовой ветки к `main`, если нет
   расхождения/конфликта. Риск: мутирует состояние дерева, которое, возможно, держит другая
   сессия (коллизия из ядра консилиума).

3) Гейт-перед-стартом-спринта: свежесть базы проверяется не по расписанию, а в момент
   ответвления task-ветки (branch:check-base как часть старта); протухшая база блокирует
   старт до подтягивания. Синхрон «ленивый», по требованию.

Комната вправе комбинировать (например, 1 как сигнал + 3 как гейт) и отвергнуть 2.

---

## Вход (факты / нормы — не вердикты)

- **факт:** `yarn branch:check-base` (branch-base-freshness.mjs) сравнивает HEAD с `origin/main`;
  `--strict` → exit 1 (пригоден для хука/CI).
- **факт:** `branch:check-base` НЕ вызывается ни ритуалом, ни хуком.
- **факт:** `morning-care` осведомлён о коллизии worktree (не рвётся, если ветку держит
  соседнее дерево), но базу не синхронизирует.
- **факт:** `ritual:day` уже содержит шаг `deps:watch --mode morning` — прецедент «гигиена свежести в утреннем ритуале».
- **норма (вердикт M1, предшественник):** синхронизировать имеет смысл только легитимные деревья
  (`canon`/`sprint-open`); дерево класса `sprint-closed`/`unregistered` — под снос, не под синхрон.
- **норма (ядро консилиума):** базовая ветка — держатель позиции, не место работы; спринт всегда в task-ветку.
- **норма:** нельзя молча мутировать дерево, которое держит другая сессия (`|writers|≤1`).

## Что комната должна выдать

- Событие и частота синхрона (по расписанию ритуала / по старту спринта / оба).
- Проверяемый предикат «база протухла» (чем меряется: `branch:check-base` behind-count / дни).
- Исполнитель и режим: сигнал агенту vs авто-fast-forward; как режим уживается с `|writers|≤1`.
- Ответственного за то, что N баз не разъехались (роль/шаг, не «все вообще»).
- DoD планировочный (проверяемые признаки «синхрон определён»), без правки прод-кода в этом прогоне.
