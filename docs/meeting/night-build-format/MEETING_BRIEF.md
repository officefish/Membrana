# Заседание: формат night-build v2 — доработка по трению ночи 17.07

## Общее задание

Ночь тулинга 17.07 (6 фаз NB, 649/649 тестов) прошла, но формат night-build
оказался сырым. Пять наблюдений процесса — доклад агента владельцу — плюс внешний
research (Perplexity, `docs/tasks/research/night-build-format-v2.md`, проверен по
коду) образуют многовопросную материю: доработка регламента + тулинга. Заседание
упорядочивает её и выносит вердикты по одному.

**Вопрос заседания:** как доработать формат night-build (регламент
`NIGHT_SPRINT_REGULATION.md` — черновик + тулинг `night:open/close`), чтобы
следующая ночь шла без трения 17.07.

## Почему заседание, а не консилиум

Материя не разваливается на независимые вопросы (все про один формат), но и не
решается одним консилиумом; цена ошибки в ПОРЯДКЕ высока: регламентные правки
(taxonomy, каденс, guardrails) — фундамент для тулинга (night:open/close), не
наоборот. `S-M1` разбирает по одному.

## Вход: наблюдения (доклад 17.07) + research

Пять кандидатов (research-валидация в скобках):

1. **Edit-class taxonomy**: тест-гейт различает класс правки — код (юнит
   обязателен) / конфиг (линтер) / инфра (plan) / интеграция (sandbox). Сегодня
   регламент требует тест безусловно; NB1/NB6 — не-юнит классы (Q1§4, Q2§2).
2. **Гранулярность кванта + каденс чекпоинта** по риску/сложности/зависимости,
   45–60 мин или логич. блок; NB-фазы сейчас плоские, разного веса (Q2§1).
3. **Переформулировать guardrails**: hard (scoped-профиль `always-yes` + изоляция
   worktree + ветка origin/main) — первичны; «мёрж утром» — вторичный процесс.
   Research: процесс без hard = театр безопасности (Q3).
4. **`night:open`**: в параллельном режиме советовать ветку от **origin/main**
   (локальный main залочен соседней сессией — эпизод 17.07).
5. **`night:close`**: генерировать merge-handoff (что смёржить + follow-up) из
   коммитов ветки по тегам, а не руками (Q2§3).
