## Вопрос заседания

**E6 — семантика рода категорий: что означает род (объект / отношение / ориентир)
для обращения процедуры с категорией, и чем проверяется корректность обращения?**

История долга: M2 положила поле `род` в схему статьи словаря, но правила
использования рода явно передала M3; M3 при урезании повестки (после шести
отказов гейта) долг вынес в хвост заседания. Комната закрывает ровно этот долг.

---

## Решено — посылки, не предмет спора

- M2: статья словаря несёт пять полей, включая `род` ∈ {объект, отношение,
  ориентир}; словарь полуоткрыт; проверка — чистый `check()` по маркерам `@cat:`.
- M2: ядро словаря — четыре категории: виртуальная команда, задача,
  ответственность, стратегическая цель (T10).
- M3: граница слоёв — ранги, единый файл правил, блокирующий CI-зуб.
- Контейнер процедуры без кода и тестов (T12); ритуалы — процедуры (T13).

## Что комната должна выдать — РОВНО ДВА пункта

1. Лемму каждого рода: что процедуре МОЖНО и чего НЕЛЬЗЯ делать с категорией
   этого рода (объект — владеть/создавать/архивировать? отношение —
   устанавливать/разрывать? ориентир — сверяться, но не мутировать?), и род
   каждой из четырёх категорий ядра.
2. Проверку: чем ловится обращение процедуры с категорией против её рода —
   расширение `check()` из M2, отдельный предикат или честное «машинно не
   ловится, работа аудитора» (тогда — что именно смотрит аудитор).

## Структура прогона — ЖЁСТКО

- Посылки оглашаются ВСЛУХ первой репликой Teamlead («Список посылок», до 10
  позиций, факт/норма).
- Финал — ровно два блока: секция **«Список посылок»**, затем таблица
  **«Итоговое решение»** ≤6 строк. Definition of Done не писать.

## Запреты этой комнаты

- Не пересматривать вердикты M1–M5 и ратифицированный эпик.
- Не расширять ядро словаря (это ADR + LGTM владельца, M2).
- Не проектировать механизм парсера ответственности (T9).
