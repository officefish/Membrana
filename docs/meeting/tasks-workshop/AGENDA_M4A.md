# M4A — повестка: контракт inspectElement для карточки

> Заседание `tasks-workshop` · фаза M4A · слой 3 (ранг 3). Зависит от M1–M3.
> Единственный ID-вопрос (S-M1). Задание — [`MEETING_BRIEF.md`](./MEETING_BRIEF.md).

## Вход (факты, не вопросы)

- `inspectElement: Element → Detail | recurse` — SHOULD; отсутствует у четырёх мастерских
  из шести, включая задачи. У процедурной мастерской он есть (`procedures:workshop --inspect`).
- Дом процедур двумерен (элемент сам контейнер); у реестра задач элемент — карточка,
  но карточка эпика имеет дочерние (`parentEpic`).
- Связи карточки живут снаружи: GitHub-иссью, Linear, промпт-файл, инсайт.
- Прецедент мис-филинга: #915 отдан мастерской задач, вердикт шторма — процедурной.

## **O4 — Что показывает inspectElement для одной карточки и ходит ли он в сеть?**

Ответ обязан задать минимум паспорта, решить вопрос сети (живое состояние иссью и Linear
против кэша реестра), и назвать базу рекурсии при спуске в дочерние карточки эпика.
Отдельно: остаётся ли инструмент read-only безусловно.

### Форма вердикта

- Минимальный состав паспорта элемента.
- Решение про сеть: ходит / не ходит / ходит по флагу; что при недоступности.
- База рекурсии и глубина спуска.
- Список посылок обязателен.
