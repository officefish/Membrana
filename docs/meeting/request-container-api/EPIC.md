## Что заседание НЕ переоткрывает

Семь тезисов шторма — вход, а не предмет спора: пара «модель+провайдер» выясняется у сервера · агент в процедурах без ключей · вызов есть заявка · ответственность переезжает на сервер · кабинет = зона полномочий · серверную механику держит Фаррелл · контейнер регулирует очередь и считает токен-экономику.

## Вердикт A3 (ратифицирован)

**Что подписывает `countersign`:** хеш содержимого закрываемой единицы работы. **Где живёт:** серверный переход **«выполнено → принято»**, между исполнением заявки и архивацией. Правка содержимого после приёмки роняет подпись дрейфом.

**Чем отличим второй актор:** **асимметрией ключей**, а не кабинетом. Клиент ключей не имеет вовсе (Т2) и умеет только подать заявку; акт приёмки производится там, куда он не дотягивается. Отсюда `executed.by ≠ accepted.by` по построению.

**Ограничение, без которого это театр:** асимметрии **недостаточно**. Если приёмку запускает заявка клиента — агент авторствует собственную приёмку руками Фаррелла. Приёмка обязана быть **критериальной, а не заявочной**: совершается, когда сходится предикат над фактами вне контроля заявителя (боевой прогон, статус проверок, серверное время, факты квоты). Заявка «прими меня» входом приёмки не является.

**Выдача кабинета — A3 не несёт.** Флаг ведущего к K1 подтверждён.

### Уточнение вердикта K1 (не переоткрытие)

Запись K1 «кабинет меняет владелец **через комнату A3**» **исправлена** на: **«выдаёт и меняет сервер; по чьему распоряжению — открытый вопрос, комнатой не закрыт»**. Содержательная часть K1 (что перечисляется, чем проверяется граница, что при отказе) остаётся в силе — снята только ссылка на несуществующий механизм.

### Замечание о ведении A3

Комната **не состоялась как многоголосая**: цепочка провайдеров исчерпана (основной лимитирован до 01.08, запасной не ответил). Вердикт вынесен **ведущим руками** по слову владельца; реплики персон намеренно не сочинялись — иначе несостоявшееся обсуждение выдавалось бы за состоявшееся. Канал помечен в протоколе явно.

**Побочная находка:** механизм «спросить у сервера, какая пара установлена» **уже существует** — overlay каналов забирается с office по адресу и токену. Локальная правка реестра резолвером **не читается** (проверено прогоном и откачено), поэтому смена пары — операция на сервере, а не в файлах репозитория. Это подтверждает Т1 шторма делом.

## Вердикт M2 (ратифицирован)

Мастерская контейнера запросов несёт **два глагола**: `audit(заявка)` — проверка заявки на валидность целиком; `decompose(заявка)` — разложение на шаги исполнения. Третий глагол **`inspectElement` объявлен неприменимым** на текущем доме — это осознанная граница с причиной, а не недоделка: достраивается только расширением контракта дома (поле `supportsElementInspection: false` с причиной). Фаррелл вызывает оба глагола как операции семьи `audit-family` и **логирует результат**; вызов неподдерживаемого глагола → ошибка с записью в `llm-calls`.

**Существенное:** атлас показывал `inspectElement` перечёркнутым как долг — теперь это **решение**, а не долг. Пустое состояние объявлено, а не замаскировано.

### Замечание ведущего (НЕ часть вердикта)

Вердикт ушёл в реализацию плотнее, чем спрашивала повестка (имя сервиса, класс ошибки, поле YAML). Для комнаты границ форма допустима, но существование `@membrana/llm-calls-service` именно в таком виде **ведущим не сверялось** — проверить до реализации, чтобы не повторить класс «вердикт на несуществующем».

## Вердикт K1 (ратифицирован)

Кабинет перечисляет **процедуры** (замкнутый перечень) + **лимиты** (квоты) + **счёт** (израсходованное). Выдаёт **сервер** при авторизации; клиент получает снимок и кэширует. Граница проверяется **синхронным предикатом на сервере** перед исполнением; ложь → `403` со **специфичным кодом** (`procedure_not_allowed` / `quota_exceeded` / `channel_unavailable`) и `reset_at`. Каждый отказ на границе **логируется в `llm-calls`**. Повтор без изменения конфига запрещён.

Разведены обе оси, как требовала повестка: **что** можно просить (перечень процедур) и **сколько** (лимиты и счёт). Клиентский снимок объявлен фильтром «опционально, не блокирует» — полномочие осталось полномочием, а не рекомендацией.

### Замечания ведущего (НЕ часть вердикта)

1. **Опора на несуществующее обещание.** В решении «кто меняет кабинет» комната написала «владелец через комнату A3». По ратифицированному порядку A3 — про **второго актора и подпись приёмки**, не про выдачу полномочий. При созыве A3 проверить: либо A3 действительно несёт выдачу, либо запись K1 уточняется до «выдаёт и меняет сервер; по чьему распоряжению — открытый вопрос».
2. **Фактура DoD не сверена.** `AuthResponse` и таблица `agent_cabinets` стоят в посылках фактом, но ведущим **не проверялись**. До реализации сверить существование — иначе повторится дневной класс «вердикт на несуществующем».

## Фактура на входе (замерено)

- Контейнер запросов существует: `docs/audit/llm-calls` (семья `audit-family` атласа), оснащён на **два глагола из трёх** — `inspectElement` отсутствует.
- Слой каналов частично живой: `consilium` резолвит пару через `llm-procedure-resolve`, дефолт — цепочка `anthropic → openrouter`. **Проверено делом:** при лимите Anthropic комната M0 заговорила запасным каналом; лимит перестал быть смертью процедуры.
- Дыра актора не закрыта: полномочия кабинета отличимости субъекта не дают — предмет A3.

## Замечание о ведении (не рушит вердикт)

Гейты S-M1 дважды отказали до прогона: повестка без ID-вопроса, затем повестка без требования списка посылок. Оба отказа по делу — вердикт не встал бы на фундаменте. Первые три попытки прогона шли **кодом с боковой ветки**, где `consilium` не знает слоя каналов и бьётся в лимит насмерть; комната пошла только со свежего дерева.

Гейт `rt-6` предупредил, что метка `O1` не найдена в теле протокола. Сверено глазами: вопрос отвечён по существу под собственным заголовком — «уронено молча» не произошло.
