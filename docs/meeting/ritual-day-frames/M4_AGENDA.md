# Повестка M4 — граница кит ↔ фреймы утра

> Заседание `ritual-day-frames`, фаза M4 (кандидат 4 по M0).
> Задание: [`MEETING_BRIEF.md`](./MEETING_BRIEF.md) — прочесть целиком до реплик.
> Предшественники: M0 · M1 (инвентарь) · M2 (preflight/frames/post) · M3 (holders) — посылки.

---

## Вопрос заседания

**B1 — где проходит граница между `kits/angelina-morning` и очередью фреймов
утра: что остаётся пином файла в ките, что уходит в `pin` кадра (`frames` /
`preflight` / `post`), как запретить дубль «весь файл и отрезок об одном», и
как не сломать `kitVersion` до закрытия F2/F3 спринта #900?**

Не писать DoD миграции на живой MANIFEST и timing vs #900 целиком — это M5;
здесь только правило границы и последствия для пинов.

---

## Фактура

- **Кит** `kits/angelina-morning`: 13 roots + pins целых `scripts/*.mjs` /
  `scripts/lib/*` (SHA); процедура держит `kitVersion: kits/angelina-morning`.
- **BRIEF / тезис про кит и фрейм** (не переоткрывать): кит пинит **файлы**
  scripts; фрейм — **отрезки/команды** сценария; доменные имена одного узора,
  не дубли.
- **M1**: 5 roots без отдельного фрейма — пины внутри кадров / страж
  (`morning-care`, `worktree-sync`, `repo-clean`, `deps-watch` →
  `morning-hygiene`; `morning-gate` → пин-страж). 8+ кадров несут артефакт/дверь.
- **M2**: `morning-wiring` ∈ `preflight` (три **двери-документа**, не root
  кита); `frames` / `post` — provisional id.
- **M3**: карта holder’ов; `leadPersona: angelina` не трогать.
- **#900**: F1–F3 (контракт `frames[]` / pin / auditPins) ещё не обязаны быть
  в main — фактурный барьер для M5; для B1 — не проектировать зубы заново,
  только не сломать `kitVersion` до них.

## Что комната должна выдать

- Правило: что пинится **только** в ките, что — **только** в кадре, что может
  быть в обоих слоях без дубля смысла (если вообще может).
- Для каждого класса из M1 (контейнер-кадр с несколькими roots; кадр = один
  root-артефакт; preflight-двери; post-ручной): как выглядит `pin` кадра на
  уровне нормы (без реализации segmentHash — это #900).
- Запрет дубля «весь файл в `kit.pins` + тот же смысл целиком в `frame.pin`».
- Правило миграции пинов относительно `kitVersion`: что можно менять в
  заседании до F2/F3, что ждёт зубьев #900 (без полного DoD приёмки — M5).
- Явная судьба `engines[]` процедуры (сейчас только `morning-care`) vs кадры.

## Список посылок — обязателен

Вердикт обязан нести секцию с заголовком ровно **«Список посылок»**: полный список
входов, у каждого пометка **факт** или **норма**. Только входы.
Вердикты M0–M3 — легальные посылки-предшественники.

## Запреты этой комнаты

- Не пересматривать контракт segmentHash / auditPins зубьев #900 (потреблять
  EPIC `procedure-frames`, не пересобирать).
- Не назначать/менять holder’ов — M3 закрыт.
- Не решать «когда `frames[]` появляется в MANIFEST» целиком — M5.
- Не менять `kitVersion` путь и не распиливать кит в этом вердикте без нужды
  для границы.
- Не трогать три двери m3 по составу — только как класс «документ-пин
  preflight».
