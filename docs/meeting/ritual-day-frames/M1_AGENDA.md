## Вопрос заседания

**I1 — каков полный инвентарь очереди фреймов утреннего ритуала: какие id
кадров покрывают цепочку `yarn ritual:day` и пост-артефактные шаги (ласточка,
hermes), что входит в каждый кадр (команды / двери-документы), а что остаётся
только корнем кита без отдельного фрейма?**

Ответ — таблица/очередь с id; полная декомпозиция сценария (T6), не «фрейм по
мере боли». Holder’ы и точные `pin` отрезков — не этой комнаты (M3/M4).

  `morning-care` → `worktree-sync` (`|| true`) → `repo-clean` (`|| true`) →
  `deps-watch --mode morning` (`|| true`) → `plan-week-if-monday` →
  `strategy-day` → `daily-standup` → `main-day-probe` → `main-day-issue` →
  `angelina`.

  `morning-gate.mjs` + `telegram-swallow.mjs` + `hermes-brief.mjs`. Swallow /
  hermes в yarn-цепочке **нет**: вручную после артефакта (BRIEF / CLAUDE.md).
- **Процедура** `docs/procedures/ritual-day/`: `leadPersona: angelina`,
  `kitVersion: kits/angelina-morning`, `engines[]` = только точка входа
  `morning-care.mjs`; `frames[]` ещё нет.
- **Уже решённый кадр** (не переоткрывать состав): id `morning-wiring`, holder
  ozhegov, три двери (CLAUDE.md · зеркало developer-rhythm · AGENTS.md) —
  вердикт m3 / EPIC `procedure-frames`. В этой комнате — **только факт, что
  такой id существует**; его ранг в очереди — комната M2.

## Что комната должна выдать

- Упорядоченный список id фреймов утра (черновые имена допустимы, если
  помечены как provisional).
- Для каждого id: краткий scope (какие шаги/артефакты кадра) — без holder и
  без механики pin.
- Явный список roots кита / шагов, которые **не** получают отдельный фрейм
  (остаются только пином файла в ките или внутри чужого кадра) — с причиной.
- Место `morning-wiring` в описи как **существующий id** (включён в очередь
  или помечен «слот зарезервирован»); позицию «первый / не первый» не решать —
  это M2.
- Критерий полноты T6: холодная сессия читает очередь и видит весь сценарий
  утра без раскопок в `package.json`.

## Список посылок — обязателен

Вердикт обязан нести секцию с заголовком ровно **«Список посылок»**: полный список
входов, у каждого пометка **факт** (проверяемо по коду или замеру) или **норма**
(наше решение). Только входы — выводы этой комнаты в посылки не вносить.
Вердикт M0 — легальная посылка-предшественник.
