## M0 — вердикт (председательское предложение, ждёт ратификации владельца)

**Дефект прогона (назван честно).** Консилиум гонялся дважды через openrouter/sonnet-4.6
(Anthropic лимит). **Оба прогона контаминированы**: комната систематически игнорировала
дословно заданные кандидаты K2–K4 и достраивала свои из слова «leveling» (прогон 1 —
агрегация/переход/верификация; прогон 2 — порог-левелинга/видимость/rebase-политика),
дважды ложно сославшись на «обрезанную повестку» (агенда 5 КБ < лимита 12 КБ, инжектится
полностью — обрезки не было). Класс «declared≠working»: зелёный прогон, контаминированное
содержание. Протокол оставлен специменом дефекта.

**Робастный вывод (дважды подтверждён):** **K1 `disposition` — единственный фундамент**;
без словаря состояний остальные три теряют область определения (контрактная зависимость).

**Председательское предложение порядка** (по РЕАЛЬНЫМ кандидатам, не по выдумке комнаты;
M0 ратифицирует владелец, не консилиум):

```
K1 disposition  →  K4 гейт+регистрация  →  K2 манифест-отчёт  →  K3 фрейм-рельсы
   (фундамент)      (решение на диспозиции)  (отчёт об исходе)     (сборка, терминал)
```

- **K1→K4** контрактная: гейт решает stop-on-trash / register-unfinished / pass-ready — оперирует диспозицией.
- **K4→K2** контрактная: манифест-отчёт (влито/застряло/мусор поимённо) верстается из исходов гейта.
- **{K1,K2,K4}→K3** контрактная: фрейм-рельсы (#1094) пакуют уже определённые сюжет и служебные фреймы — терминал.
- Независимых пар нет; голова-гейт (K4) раньше сантехники-отчёта (K2) — по К-Kuryokhin.
