## Объединяющее задание

**Свести подтверждённые тезисы шторма (T1–T13 + R2) в регламент зонтичной процедуры
`membrana-leveling`: детерминированное выравнивание общего рабочего пространства к
`main` — два скрипта по очереди (`main-fill` → `workspace-level`), голова-гейт «не
уснуть над мусором», разложенные на фрейм-рельсы (T12) с гранулами-предикатами
`ready`/`disposition`/`isLeveled`/`legit` — так, чтобы следующий агент вечером
детерминированно вливал готовое в `main`, убирал мусор и регистрировал незавершённое,
не пряча грязь под коммит-и-забыл.**

## Решено владельцем — переспорить нельзя (тезисы шторма 24.07)

- **T1–T3.** Предмет — процедура `membrana-leveling`: детерминированное выравнивание
  **общего** рабочего пространства (проект как целое), различия «свой/чужой» нет — ось **состояние** (R2).
- **T2/T9.** Зонтичная: два скрипта по очереди — `main-fill` (ветки→main, сериализованный
  `pr:ship`-поезд #700) → `workspace-level` (main→деревья + гигиена).
- **T4/T5/T10.** Цель — `main` актуален: всё **готовое** «не доехавшее» оказывается в `main`;
  конечное состояние — точка, удобная для продолжения. Центр тяжести — вечер, **обе половины
  обязательны** (обнаружение мусора И `main-fill` — не или-или).
- **T6/R2.** «Не доехало» = только `ready`. `live` (живая правка сейчас) пропускается;
  WIP не везётся. `trash = dirty ∧ ¬registered` **останавливает** гейт и называется поимённо.
- **T7.** Висящая работа — плохая практика: незавершённое остаётся легитимно незавершённым
  и **регистрируется**; незарегистрированное = мусор.
- **T8.** `ready` штампуют ОБА: тимлид (штамп) + чистая функция
  `ready = CI_green ∧ ¬conflicts(main) ∧ PR_approved`.
- **T11.** Мусор ≠ незавершёнка — разные диспозиции. Времянки — **убираются** (`%TEMP%`/scratchpad),
  не регистрируются; незавершённое настоящее — **регистрируется**. Коммит-и-забыл запрещён для обоих.
- **T12.** `membrana-leveling` — процедура во фреймах (не скрипт-монолит): голова-гейт и два
  скрипта — фреймы; `ready`/`disposition`/`isLeveled` — гранулы. Ложится на фрейм-рельсы.
- **T13.** Отдельный фрейм «временных файлов» — штатный фрейм каркаса (обобщён паттерном #1094).
