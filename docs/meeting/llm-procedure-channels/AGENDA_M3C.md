# M3c — повестка: поведение при отказе канала

> M3c · кандидат 5 · зависит от C1 · параллельно X1/T1 · secretary

## **F1 — Что делает процедура при отказе effective-канала?**

Fallback-цепочка vs жёсткий STOP; кто меняет цепочку (админ overlay vs только git);
связь с `source` из C1.

### Кандидаты (текст)

1. STOP сразу: ненулевой exit, явная ошибка; смена канала только overlay/defaults.
2. Цепочка в defaults/overlay: `providers: [openrouter, anthropic, …]`; пробовать по порядку; исчерпание → STOP.
3. Цепочка только для code-review; consilium — STOP (дорогой прогон).
4. Авто-fallback на ollama без записи в overlay — молча.

### Форма вердикта + **список посылок обязателен.**
Запреты: не поля телеметрии детально; не UI; не шов ключей (X1).
