## Порядок (вердикт M0 — **РАТИФИЦИРОВАН владельцем 2026-07-19**)

Структура **двухслойная**, не цепь: одна аксиома, один операционный фундамент,
две независимые ветви.

```
К4 (уровень счёта — аксиома-словарь, входящих рёбер нет)
 ├→ К1 (основание гейтов — операционный фундамент)
 │    ├→ К2 (форма холодного хранилища)
 │    └→ К3 (миграция и включение законности)
 └→ К3
```

| Ребро | Обоснование комнаты |
|---|---|
| К4 → К1 | единица счёта задаёт тип входа чистой функции гейта |
| К4 → К3 | единица счёта задаёт знаменатель законности: что значит «69% без Issue» |
| К1 → К2 | природа входа гейта (снимок-контракт против живой сети) определяет, что охраняет хранилище |
| К1 → К3 | источник истины задаёт критерий законной архивации |

**Независимы явно:** К2 ⟂ К3 (связь только через К1) · К4 ⟂ К2 (прямого ребра нет).
Циклов нет.

**Риски обратного ранга:** К2 раньше К1 — носитель спроектирован под, возможно,
отменённый контракт-снимок, работа в мусор. К3 раньше К4 — критерий законности считает
не ту единицу, миграцию 151 карточки переигрывать.

### Что из этого следует для архивации

**1. Записей без паспорта — 411 из 779, больше половины.** По инварианту «задач без
Issue не бывает» они все ретроактивно незаконны. «Правильно заархивировать» для них
означает решение, а не перенос: выписывать паспорт задним числом, объявить рубеж
законности с датой, или ввести класс «архивная запись без паспорта» с явной пометкой.
Молча перенести — значит внести в новый стек 411 записей, нарушающих его же инвариант.

**2. Активных из июня — 114.** Это половина активных, возрастом месяц и больше. Либо
работа в полёте пять недель, либо мёртвые карточки, которые никто не закрыл. Ровно та
популяция, где 29.06 из 38 кандидатов 19 требовали решения. **Перенос активных — это
не миграция, а инвентаризация.**

**3. Двухконтурность обоснована независимо от выбора хранилища.** 779 записей в одном
файле 647 КБ: горячий путь читает целиком, чтобы получить 221 активную. Разделение
горячего и холодного нужно уже сейчас, до всякого переезда.

**4. Целостность архивной части высокая** — `archivedAt` есть у всех 551. Это хорошая
новость для миграции: дата закрытия не восстанавливается, она уже записана.

**Что замером НЕ выяснено** (и потому К3 разблокирована лишь частично): сколько из 114
июньских активных — живая работа, а сколько мёртвые. Это не выводится из полей, только
разбором. Второй незакрытый замер — частота асинхронной записи для К2 — не сделан вовсе.

## Предложение владельца 19.07: «токсичные задачи» — материал для К3

**Замысел.** Пометить проблемные записи прямо в текущем реестре, закрывать кусками
постепенно, формально трактовать как технический долг. Владелец отмечает существенное:
**многие из них наверняка реализованы** — работа сделана, карточка не закрыта.

**Уточнение председателя: в куче три РАЗНЫХ долга, и смешивать их нельзя.**

| Долг | Что это | Чем закрывается | Оценка объёма |
|---|---|---|---|
| **записи** | работа сделана, карточка открыта | сверкой с кодом и историей — не работой | часть из 114 июньских активных |
| **работы** | не сделано, настоящий технический долг | собственно работой | остаток тех же 114 |
| **законности** | нет паспорта (Issue) | решением о рубеже или выпиской задним числом | 411 из 779, **ортогонален первым двум** |

**Почему различение несущее, а не педантизм.** Если все 411 переехать как «технический
долг», метрики нового стека получат сотни фиктивных незакрытых работ: доля задач без
владельца и незавершённая работа взлетят на пустом месте. Внешняя практика называет это
ложной точностью — цифра растёт, смысл не меняется. Мы сами четыре часа назад приняли
вердикт, требующий у каждой величины честного источника; импорт долга записи под видом
долга работы этот вердикт нарушит на первом же отчёте.

**Ловушка разбора.** Отличить долг записи от долга работы **нельзя по документам** —
только по коду и истории. Это ровно тот дефект, который ритуал допустил 16.07 (вердикты
строились на доках и статусах Issue, и он соврал дважды за день). Значит процедура
разбора обязана смотреть в код, а не в поля карточки.

**Что «токсичная» должна означать, чтобы быть проверяемой.** Предлагается определение:
запись, чью **законность или завершённость нельзя установить без разбора**. Пометка —
не приговор и не диагноз, а честное «неизвестно», требующее работы. Разбор снимает
пометку в один из трёх долгов выше либо просто закрывает карточку.

**Куда это ложится.** Пометка ставится **до переезда**, чтобы миграция несла уже
классифицированное. Разбор кусками — фоновая аналитическая работа с чтением кода:
естественный кандидат на субагентов-аналитиков ведущей персоны (работа Р1 первого
заседания), а не на ручной проход владельца.

**Статус: предложение, не вердикт.** Предмет фазы М4 (миграция и законность).

## Ресёрч по Linear — выполнен 19.07

[`linear-agent-identity-facts.md`](../../tasks/research/linear-agent-identity-facts.md).
Кратко, **как вход и не как решение** (проверять по живому API):

| Вопрос | Ответ выжимки |
|---|---|
| Личность агента | сущность **Agent (app user)**, OAuth `actor=app`, свой `id`/имя/аватар, флаг `isAgent`, права `app:assignable` / `app:mentionable`; в UI не входит |
| Семантика ответственности | `assignee` — человек, отвечает за результат; `delegatedAgent` — агент, отвечает за выполнение. **Агент не может быть единственным ответственным** |
| Подзадача | **полноценный объект**: свой статус и свой `assignee`, доступны через API. **Чеклист в описании — не сущность**, ни состояния, ни исполнителя |
| Блокирующие связи | `blockedBy` / `blocking` видны в GraphQL, **вебхуками не приходят** — внешний гейт требует периодического опроса, не реакции на событие |

**Совпадение, отмеченное председателем:** двухслойная ответственность Linear
(ответственный человек + делегат-исполнитель) независимо повторяет вердикт M2 первого
заседания — «след персоны, принявшей выход, а не подпись субагента-автора».
