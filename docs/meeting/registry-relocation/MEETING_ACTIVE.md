## Порядок (вердикт M0 — **РАТИФИЦИРОВАН владельцем 2026-07-19**)

Структура **двухслойная**, не цепь: одна аксиома, один операционный фундамент,
две независимые ветви.

```
К4 (уровень счёта — аксиома-словарь, входящих рёбер нет)
 ├→ К1 (основание гейтов — операционный фундамент)
 │    ├→ К2 (форма холодного хранилища)
 │    └→ К3 (миграция и включение законности)
 └→ К3
```

| Ребро | Обоснование комнаты |
|---|---|
| К4 → К1 | единица счёта задаёт тип входа чистой функции гейта |
| К4 → К3 | единица счёта задаёт знаменатель законности: что значит «69% без Issue» |
| К1 → К2 | природа входа гейта (снимок-контракт против живой сети) определяет, что охраняет хранилище |
| К1 → К3 | источник истины задаёт критерий законной архивации |

**Независимы явно:** К2 ⟂ К3 (связь только через К1) · К4 ⟂ К2 (прямого ребра нет).
Циклов нет.

**Риски обратного ранга:** К2 раньше К1 — носитель спроектирован под, возможно,
отменённый контракт-снимок, работа в мусор. К3 раньше К4 — критерий законности считает
не ту единицу, миграцию 151 карточки переигрывать.

## Блокировка замерами — назвал вердикт M0

Комната отметила: К2 и К3 заблокированы **не внешней фактурой про Linear**, а
**нашими собственными незамеренными величинами**:

- К2 — как часто пойдёт асинхронная запись; жив ли «единственный писатель»;
- К3 — реальное состояние 151 карточки без Issue.

Председатель ожидал блокировки со стороны Linear-ресёрча и ошибся: запущенный ресёрч
разблокировал не то, что мешало. Замеры остаются несделанными.

## Ресёрч по Linear — выполнен 19.07

[`linear-agent-identity-facts.md`](../../tasks/research/linear-agent-identity-facts.md).
Кратко, **как вход и не как решение** (проверять по живому API):

| Вопрос | Ответ выжимки |
|---|---|
| Личность агента | сущность **Agent (app user)**, OAuth `actor=app`, свой `id`/имя/аватар, флаг `isAgent`, права `app:assignable` / `app:mentionable`; в UI не входит |
| Семантика ответственности | `assignee` — человек, отвечает за результат; `delegatedAgent` — агент, отвечает за выполнение. **Агент не может быть единственным ответственным** |
| Подзадача | **полноценный объект**: свой статус и свой `assignee`, доступны через API. **Чеклист в описании — не сущность**, ни состояния, ни исполнителя |
| Блокирующие связи | `blockedBy` / `blocking` видны в GraphQL, **вебхуками не приходят** — внешний гейт требует периодического опроса, не реакции на событие |

**Совпадение, отмеченное председателем:** двухслойная ответственность Linear
(ответственный человек + делегат-исполнитель) независимо повторяет вердикт M2 первого
заседания — «след персоны, принявшей выход, а не подпись субагента-автора».
