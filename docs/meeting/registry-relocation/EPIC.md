## Работы

### Р1 — словарь единиц (аксиома, делается первой)

- **Единица ответственности** — центральная задача, она же GitHub-issue. На ней ровно
  один ответственный: персона, **принявшая выход**. Контейнер (Linear-issue)
  ответственности не несёт.
- **Единица счёта величин**: время до закрытия, доля задач без владельца, пропускная
  способность, доля эскалаций — на GitHub-issue. **Незавершённая работа** — на
  движущейся под-задаче (биекция с Issue).
- **«Закрыто»** = закрытие центральной задачи. Закрытие контейнера производно;
  **закрытие подчинённой под-задачи событием не является**.
- **Двухслойность Linear**: `assignee` ↔ ответственная — тождество по функции;
  `delegatedAgent` ↔ исполнитель — соответствие **через адаптер**, имена не склеивать.
- **Двойной счёт** исключён биекцией «центральная под-задача ↔ GitHub-issue»; контейнер
  и дополнительные — вне области счёта.

### Р2 — снимок-контракт и расслоение гейтов (фундамент)

- Вход гейта — **снимок**. Живой запрос дисквалифицирован: `gate(fetch()) ≠ gate(fetch())`.
- **Снимок ≠ кэш.** Снимок — контракт: адресуемый, версионированный, с провенансом;
  молчаливое устаревание = нарушение. Слово «кэш» в этой подсистеме запрещено.
- **Производитель** — office-батч полным pull-опросом. Вебхук — только триггер «пора
  снимать», телом источника не является (вебхуки теряются, есть лимиты частоты).
- **Провенанс** `(capturedAt, sourceRevision)`; предикат свежести сравнивает курсор
  источника с ревизией снимка — **один дешёвый запрос вне тела гейта**, тело офлайн.
- **Три кода возврата:** в порядке · протух (мягкий, замечание) · входа нет (жёсткий отказ).
- **Расслоение:** содержание → git (сеть не нужна), движение → снимок Linear.
  **Ни одна проверка не ходит в сеть в момент вердикта.**
- **Вердикт первого заседания о чистой функции ВЫЖИВАЕТ с уточнением входа**, не
  отзывается: «два снимка» = два материализованных pull-снимка Linear.

### Р3 — холодный контур

- **Ступень 2 достаточна:** `archive.jsonl`, append-only, читается офлайн без сети.
  647 КБ — три порядка до гигабайтного триггера.
- **Postgres и MongoDB отклонены как избыточные.** Оправдание Postgres (конкурентная
  запись) практически отсутствует; MongoDB не нужна при однородных карточках.
- **Посылка «нет второго писателя» ослаблена, не умерла**; триггер переформулирован
  проверяемо: «второй писатель, **доказанно пересекающийся во времени**».
- **Конкуренция растворена, а не обслужена:** вечерняя архивация редуцируется до
  **сигнала триггера**, своей дорожки записи не ведёт. Производитель один.
- **Назначение контура названо тремя пунктами:** независимость от поставщика ·
  офлайн-детерминизм гейтов · долговременность за горизонтом подписки.
- **Хранится:** содержание карточки + доменные поля, которых нет во внешнем источнике —
  `dependsOn` (413 записей, обязательно), `strategicWave`, `leadPersona`.
  **Движение не дублируется.**
- **Условие приёмки: honest-шапка.** Одна запись = снимок карточки с обязательной
  шапкой (дата снимка, ревизия head, источник). **Снимок без шапки холод не принимает.**
  *(Восстановлено по находке финального аудита: при первой сборке эпика требование
  было утеряно.)*
- **Критерии перехода фальсифицируемы:** на индекс — когда время скана превысит бюджет
  гейта; на Postgres — блокировки чаще раза в неделю при доказанном пересечении
  писателей, либо гигабайты, либо второй пересекающийся писатель.

### Р4 — миграция и включение законности

- **`wellArchived(r) = inCold(r) ∧ hasPassport(r) ∧ archivedAt(r) ≠ null`** — три
  проверяемых предиката на снимке, без сети.
- **411 без паспорта разделены:** 153 активных → выписка Issue задним числом
  (легализация через PR); 258 архивных → **рубеж законности с датой**, класс
  `archived-pre-passport`, поле `legalizedBy`.
- **Класс долга** `debtClass ∈ {record, work, legality, owner-knowledge}` вычисляется
  **до переноса**. Счётчик незакрытых работ читает **только** `work` — 258 архивных
  без паспорта не поднимают его ни на единицу.
- **Три режима гейта**: мягкий → миграционный → жёсткий. Комната задала коды возврата
  для **мягкого (0)** и **жёсткого (≠0)**; код миграционного режима не назван — при
  постановке работы назвать явно, а не домыслить. Переключение на жёсткий — по предикату
  «незаконных записей ноль», не по дате. Режим и счётчик видны в шапке отчёта.
- **`parked`** — состояние для долга владельческого знания: не мигрирует, не блокирует,
  из счётчиков исключено, помечено датой запроса владельцу.
- **Порядок:** мягкий гейт и снимок → классификация → легализация → перенос архивных
  office-батчем → жёсткий режим. Работа между шагами не встаёт.

## Порядок исполнения

```
Р1 (аксиома, входящих рёбер нет)
 ├→ Р2 (фундамент)
 │    ├→ Р3   ─┐ независимы между собой,

```

**Риски обратного ранга:** Р3 раньше Р2 — носитель спроектирован под, возможно,
отменённый контракт-снимок, работа в мусор. Р4 раньше Р1 — критерий законности считает
не ту единицу, миграцию 153 активных переигрывать.
