---

## Вопрос заседания

**F3 — как устроен полиморфизм и рекурсия инструментов мастерской в двумерном
(процедурном) доме: диспетчер по роду процедуры, якорный документ как база рекурсии
и конечность спуска, поведение стек-подобных операций по вложенным домам?**

---

## Фактура (вход шторма + M2)

- **Процедурный контейнер — 2D-массив** (T5, T9): список процедур × собственный список
  элементов каждой процедуры; «рассмотреть-элемент» полиморфен — глагол один, реализация
  по роду процедуры (спринт/ритуал/заседание).
- **Якорный документ** (T7): у каждой процедуры свой; база рекурсии, ниже листа нет —
  спуск конечен.
- **M2 уже развёл роли**: мастерская объявляет **что** (`inspectElement`), кит/процедура
  поставляет **как** (реализация); слабая связанность — мастерская не знает реализаций.
- **Плоский дом (ветки, задачи)**: элемент = лист; `inspectElement` не рекурсирует.

## Что комната должна выдать

- Устройство диспетчера: **свитч по роду внутри мастерской** vs **каждый род процедуры
  приносит свою реализацию** (мастерская зовёт по общему интерфейсу). M2 склонил к
  второму («что/как») — закрепить или уточнить; критерий «открыто для расширения» (новый
  род процедуры приносит свой обзор, мастерскую не трогаем).
- Где физически живёт реализация рода: у кита процедуры? у самой процедуры (её структура,
  эхо T10 procedure-frames «лицо/обзор выводится из структуры»)? — назвать носитель.
- Базу рекурсии формально: `inspectElement(element)` спускается, пока элемент — контейнер;
  упёрся в якорный документ (не контейнер) → терминал. Предикат конечности (у каждого
  листа есть якорь) и что делать, если якоря нет (дыра в полу — зуб или находка).
- Поведение стек-подобных операций по этажам: `decompose` плоского дома раскладывает
  лист; `decompose` процедурного — процедуры по роду; `decompose` спринта — его фреймы.
  Один глагол на этажах — рекурсия или только полиморфизм верхнего этажа? Предел спуска.
- Явную границу: это **форма** диспетчера и конечность, НЕ достройка процедурного дома
  по существу (T9 отложено) и НЕ механика `frames[]` (заседание #900).

## Список посылок — обязателен

Секция с заголовком ровно **«Список посылок»**: полный список входов, каждый — **факт**
или **норма**. Только входы. Вердикты M0, M1, M2 — легальные посылки-предшественники.
