## Что комната должна выдать

- Состав отрезков фрейма `frames[0]` процедуры `ritual-day`: какие именно строки
  CLAUDE.md / AGENTS.md / зеркал скиллов входят (по родам якорей из M2), что
  осознанно НЕ входит (документ шире процедуры, T8).
- `id` фрейма и позицию в очереди (подводка — первый элемент очереди T6;
  полная декомпозиция остальной очереди — работа спринта, не этой комнаты:
  назвать явно, что очередь стартует с одного фрейма и это легально по M1).
- Точку запуска аудита дрейфа (передана из M2): CI / утренний ритуал /
  pre-commit — с обоснованием от прецедента («три дня без хозяина» — какой канал
  поймал бы за день).
- Порядок адресного ремонта (T3/T5 живьём): прецедент → находка аудита →
  заказ holder'у (Ожегов) — как заказ оформляется (карточка, issue, строка
  ритуала), чтобы «закрою прямо сейчас» первой попавшейся сессии перестал быть
  маршрутом по умолчанию.
- Судьбу нерезолвящейся ссылки на прецедент 22.07 (ещё на ветке): ждать мерджа,
  ссылаться на THESES шторма, или precedents[] манифеста пополняется после.

## Список посылок — обязателен

Вердикт обязан нести секцию с заголовком ровно **«Список посылок»**: полный список
входов, у каждого пометка **факт** (проверяемо по коду или замеру) или **норма**
(наше решение). Только входы — выводы этой комнаты в посылки не вносить.
Вердикты M0, M1, M2 — легальные посылки-предшественники.

## Запреты этой комнаты

- Не пересматривать механику пина (форма якоря, хэш, ядро аудита) — посылка M2.
- Не пересматривать контракт `frames[]` — посылка M1.
- Не переписывать паттерн PINNED_SUBGRAPH_VERSIONING — комната K4.
- Не выполнять сам ремонт дверей и не писать код — заседание решает форму,
  исполнение — спринт (#900) руками holder'а.
- Не пересматривать тезисы владельца (MEETING_BRIEF, «переспорить нельзя»).
