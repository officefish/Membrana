## СТАТУС: ОТКРЫТО · фаза M0

## Задание

**Довести сущность процедуры до механики: контракт `frames[]`, пин отрезков,
первый фрейм «подводка утра», уточнение паттерна — чтобы по прецеденту фрейм и его
лицо находились без раскопок, а дрейф отрезка ловился аудитом.**

Полностью: [`MEETING_BRIEF.md`](./MEETING_BRIEF.md). Изменение только через
`BRIEF_AMENDMENT.md` + LGTM владельца (S-M4).

## Роли

| Роль | Кто | Ограничение |

| Аудитор | **отдельный** агент, read-only | не пишет повестку, не голосует, не чинит (S-M5) |
| Владелец | druid | задание, ратификация порядка M0 |

## Порядок (вердикт M0 — **РАТИФИЦИРОВАН владельцем 2026-07-22**)

Линейная цепь, независимых пар нет, топологическая сортировка единственна:

```
1 контракт frames[] → 2 механика пина отрезков → 3 первый фрейм «подводка утра» → 4 уточнение паттерна
```

Протокол M0: [`m0-order`](../../seanses/procedure-frames-m0-order-2026-07-22.md).
Фундамент — K1 (без схемы негде описать остальное); сток — K4 (оговорку снимают по
прожитому прецеденту, не теоретически). Блокер по фактуре: K1 требует замера текущей
схемы MANIFEST и живущих процедур `docs/procedures/`. Риск обратного ранга: схема в
воздухе / пины на недоопределённой механике / оговорка «вне области» возвращается живой.
