# ADR-0014 — Синхрон базовых веток рабочих деревьев с `main` (K1)

> **Статус:** ACCEPTED · 2026-07-20 (LGTM владельца)
> Решения Р1–Р5 действуют. Реализация — спринт `worktree-hygiene-order` (GitHub #717 / Linear DRU-234).

## Контекст

Развилка K1 заседания `worktree-hygiene-gaps` (надстройка над K2 по ратифицированному
владельцем порядку M0). Вопрос W2: как базовые ветки долгоживущих деревьев (`tooling`,
`product`, `codex`, `cursor`) синхронизируются с `main` — частота, исполнитель, предикат
«протухания», режим вмешательства.

Комната M2 четырежды вынесла содержательно идентичный вердикт, но ни разу не оформила
обязательную секцию «Список посылок» → по S-M2 заседание по K1 формально не состоялось,
а дописать посылки задним числом запрещает правило 6 (`docs/MEETING_REGULATION.md`).
Решение владельца (ратификация M0 + выбор 2026-07-20): зафиксировать K1 через ADR —
содержание стабильно (4×), а форма-блокер лежит в тулинге консилиума, не в существе.
K1 — ниже консилиум-гейта: новый core-контракт/пакет/узел палитры не вводится, это
рантайм-процедура над готовым каноном ветвления.

## Решение

### Р1 — Событие и частота: сигнал на входе в дерево, не cron, не per-commit хук
- Синхрон-проверка — шаг `ritual:day` (утренний вход агента в дерево) плюс явная команда
  `yarn worktree:sync`. **Границы:** не git-hook (per-commit/pre-push не трогаем — цена на
  каждый коммит несоразмерна), не cron (мутация фонового расписания вне сессии запрещена
  ядром консилиума). Прецедент шага-гигиены в утре — уже существующий `deps:watch`.

### Р2 — Предикат: чистая функция после `git fetch`, метрики против `origin/main`
- `worktree-sync-check` (read-only) — чистая функция `(refs, now) → {behind, ahead,
  mergeBase, class}`, детерминированная при фиксированном входе, без `Date.now()` внутри
  (время — параметр; юнит-тест бит-в-бит). Классы: `fresh` (behind==0) · `ff-able`
  (behind>0 ∧ clean ∧ ahead==0) · `diverged` (ahead>0) · `dirty` (рабочее дерево грязное).
  `git fetch` — отдельный явный шаг с таймаутом ВНЕ чистой функции; при неудаче предикат
  считается на локальных refs с пометкой «возможно неактуально». **Границы:** метрики
  `rev-list --count` против `origin/main`, не локального `main`.

### Р3 — Режим: авто только `--ff-only`, расхождение/грязь — только сигнал
- Автодействие разрешено **исключительно** при классе `ff-able` и выполняется `git merge
  --ff-only origin/main` (перемотка указателя не нарушает границу владения), с громкой
  строкой в лог входа (`ff <branch>: <old> → <new>, +N`). `diverged` и `dirty` — **только
  сигнал** агенту дерева, без мутации; авто-merge и авто-rebase **запрещены**. **Границы:**
  соблюдён инвариант `|writers(worktree)| ≤ 1` — молча чужое дерево не мутируем; находка
  расхождения ≠ сбой, ненулевой exit-код для «нашлось расхождение» не использовать
  (`exit-code-semantics-audit`).

### Р4 — Область синхрона читается из реестра K2, `main` исключён
- Список деревьев берётся из носителя K2 (M1 lifecycle, поле `kind`/каноническое множество),
  **не хардкодится** второй раз. Синхроним только легитимные (`canon` / `sprint-open`);
  `sprint-closed`/`unregistered` — под снос (K2), не под синхрон. `main`-дерево исключено
  как identity (`behind==0` всегда). **Границы:** при отсутствии материализованного носителя
  K2 — явный отказ с указанием зависимости, не самодельный список-синоним.

### Р5 — Пороги — в конфиге, отчёт видимый и a11y
- Пороги `behind_threshold` (дефолт 10) и `stale_age_days` (дефолт 7) — в конфиге, не
  хардкод, отзываемы без правки кода. Отчёт: шапка `origin/main` hash + время fetch; строка
  на дерево `base '<tree>' отстала: −N, предок <дата> (M дн)`, при `ahead>0` — `+K свои →
  rebase, не ff`; danger-цвет только при `stale`; иконка+текст (a11y), без «мёртвой двери».

## Список посылок (авторские — ADR, не ритуальный вердикт комнаты)

1. **факт:** `branch:check-base` (`--strict` → exit 1) существует, но не подключён к ритуалу/хуку.
2. **факт:** `ritual:day` уже несёт `deps:watch` — прецедент гигиены свежести в утреннем ритуале.
3. **факт:** `morning-care` знает о коллизии worktree, но базу не синхронизирует.
4. **норма (вердикт M1, K2):** синхронизируем только легитимные деревья; носитель списка — реестр K2.
5. **норма (ядро консилиума `tier2-git-hygiene-multi-agent`):** базовая ветка — держатель позиции; `|writers|≤1`, молча чужое дерево не мутируем.
6. **норма (вердикт M0, ратифицирован владельцем 2026-07-20):** K1 — надстройка над фундаментом K2.

## Definition of Done (для будущей реализации)
- [ ] `scripts/lib/worktree-sync-check.mjs` — чистая функция `(refs, now) → {behind, ahead, mergeBase, class}`; юнит-тесты на все 4 класса + граница порогов; без `Date.now()` внутри.
- [ ] `scripts/worktree-sync.mjs` + `yarn worktree:sync`: `git fetch` (таймаут) → предикат → `--ff-only` при `ff-able`, СТОП+отчёт при `diverged`/`dirty`.
- [ ] Область читается из носителя K2 (нет второго хардкод-списка), `main` исключён; отказ при отсутствии носителя.
- [ ] Шаг подключён в `ritual:day`; находка расхождения ≠ ненулевой exit; отчёт по формату Р5 (a11y).
- [ ] Пороги в конфиге; тесты зелёные на затронутых пакетах; границы пакетов не нарушены (`scripts/` вне графа `packages/*`).
