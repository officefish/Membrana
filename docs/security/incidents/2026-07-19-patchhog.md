# Инцидент: patchhog — холодное письмо о «уязвимости vite»

**Дата:** 2026-07-19 · **Класс:** входящее security-сообщение (cold outreach)
· **Статус:** закрыт · **Регламент:**
[INCIDENT_RESPONSE_REGULATION.md](../INCIDENT_RESPONSE_REGULATION.md)
· **Роль:** эталонная карточка (вердикт M3)

## Что произошло

Владельцу на почту пришло письмо от автора сервиса patchhog: «в вашем
membrana-core в package.json vite 8.0.12 с high-severity дырой (fs.deny bypass
на Windows), обновитесь до 8.0.16» + реклама своего сканера + гиперссылка
«See how the rest holds up». Ссылка — обёртка Яндекс.Почты, внутри base64:
`go.patchhog.dev/go/<уникальный токен>` (персональный трекинг). Страница —
тизер «We found 1 issue» с размытыми находками и «Unlock 1 finding → Sign in
with GitHub»; сервис просит OAuth + webhook + опционально write-доступ для
«auto-fix». Владелец перешёл по токен-ссылке из браузера (адрес помечен живым).

## Проверка (чек-лист регламента, секция а)

1. **Сверка с репозиторием:** vite закреплён `^5.3.0` (в lock 5.4.21), никакого
   8.0.12; репозиторий называется `officefish/Membrana`, не «membrana-core».
   Конкретика письма выдумана. Критичность «поплыла» между касаниями: в письме
   high, на странице medium — шаблонная генерация.
2. **Декодирование ссылок:** конечный URL вытащен из параметра `l=` (base64)
   без перехода; опознан персональный токен-трекинг. Репутация домена — нулевая
   (ни одного независимого упоминания), хостинг Vercel, сайт признаёт холодные
   рассылки как тактику.
3. **Локальная проверка существа:** `yarn npm audit --all --recursive` — 96
   advisories (2 critical, 34 high), в том числе реальный vite fs.deny bypass
   (GHSA-fx2h-pf6j-xcff) в установленной 5.4.21 — dev-сервер-only.

## Вердикт

Шаблонная лид-generation-рассылка с выдуманной конкретикой, НЕ целевой фишинг.
Ирония: по классу уязвимости спамер случайно оказался прав — дыра того же
класса реально была в дереве (в другой версии). Главный риск был не в письме,
а в воронке: «Sign in with GitHub» + write-доступ = поверхность supply-chain.

## Действия

Проведено заседание `security-posture` (M0–M3, все ратифицированы 19.07):

- гигиена-патчи: 96 → 84 advisories (undici, ws, multer, form-data — в ноль);
- gitleaks-гейт (pre-commit + CI), скан истории 1439 коммитов — секретов нет;
- ревизия доступов: репо-уровень чист (webhooks 0, deploy keys 0, App 0);
- Dependabot alerts включены; дозор `yarn deps:watch` в ритуалах
  (`DEPS_WATCH_REGULATION.md`);
- апгрейд-спринт (fastify 5-стек, vite 7, vitest 3, tar 7, electron) — карточки
  `sec-upgrade-*` в реестре;
- этот регламент реагирования.

## Урок

О дырах узнали от спамера — свой дозор должен опережать чужие «отчёты».
Переход по токен-ссылке подтвердил адрес: пункт «запрещено» регламента родился
из этого эпизода. Проверка заняла минуты и не потребовала чужого сканера —
чек-лист секции (а) воспроизводит ровно её.
