## (а) Протокол входящих security-сообщений

**Презумпция недостоверности**: любое внешнее сообщение о «уязвимости в вашем
репозитории» считается недостоверным, пока не сверено с фактами.

Чек-лист проверки (эталон применения — [карточка patchhog](./incidents/2026-07-19-patchhog.md),
проверка заняла минуты и не потребовала чужого сканера):

1. **Сверить заявленное с репозиторием**: версии — с `package.json` / lock-файлом,
   имя репозитория — с фактическим remote. Расхождение = шаблонная рассылка.
2. **Декодировать ссылки, не переходя**: вытащить конечный URL из обёрток
   (почтовые редиректы, base64), опознать токен-трекинг (уникальный хвост).
3. **Проверить существо локально**: `yarn npm audit` (двери зависимостей),
   gitleaks при подозрении на утечку. Свой аудит первичен, чужой отчёт — вторичен.

**ЗАПРЕЩЕНО** (запрет словом, не только акцентом):

- **запрещён** переход по токен-трекинг ссылкам из письма (подтверждает адрес);
- **запрещён** «Sign in with GitHub» / OAuth на сторонних «отчётах об уязвимости»;
- **запрещена** установка тулинга, предлагаемого в холодном письме.

**Фиксация**: каждый эпизод — карточка `docs/security/incidents/<дата>-<slug>.md`
(шаблон — [`_TEMPLATE.md`](./incidents/_TEMPLATE.md)). Пересмотр порога дозора
после инцидента — по [`DEPS_WATCH_REGULATION.md`](./DEPS_WATCH_REGULATION.md).

## (б) Правило выдачи third-party доступов

**Deny-by-default.** GitHub App, OAuth-приложение или токен с write-доступом к
репозиториям выдаёт **только владелец проекта**, явным «да» до выдачи.

## (в) Дисциплина секретов и `.env`

- `.env` не коммитится; живёт **в каждом worktree отдельно** и проверяется
  per-worktree глазами, не однострочником (память инцидента
  office-token-in-main-worktree).
- **При подозрении на утечку — ротация первой, разбор потом.** Скомпрометированный
  токен отзывается/перевыпускается до выяснения, как он утёк.
- Машинный гейт — gitleaks (pre-commit мягкий + CI, см. `.gitleaksignore`) —
  остаётся как есть; этот регламент задаёт норму поверх него.
- **Расширение gitleaks-baseline санкционирует владелец** явной записью в PR;
  молчаливое добавление отпечатка = swallow, запрещено.
