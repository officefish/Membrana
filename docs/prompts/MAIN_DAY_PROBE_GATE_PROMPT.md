### Что построить (продуктовое описание)

Read-only препроцессор, который **проверяет посылки развилки до того, как план назначит работу**. Он не чинит, не закрывает Issue, не переписывает план — только выносит вердикт и роняет генерацию при нарушенной посылке.

Правило: **ни одна развилка A/B не публикуется без {маркер в коде + связанная карточка/Issue + вердикт}**.

---

### Архитектура / контракт

| Слой | Путь | Ответственность |
| --- | --- | --- |
| Чистое ядро | `scripts/lib/main-day-probe.mjs` | `probeAssertion(claim, evidence) → { verdict, evidence }` — чистая, тотальная, без I/O |
| Обвязка | `scripts/main-day-probe.mjs` | Read-only сбор фактов (символ в дереве, `issue.state`), таблица, exit-код |
| Впайка | `package.json` → `ritual:day` | Хвост цепочки; падение блокирует генерацию магистрали |

Контракт вердикта:

```js
/** @typedef {{ claim: string, marker: {kind:'symbol'|'file'|'test', value:string}, issue?: number }} Assertion */
/** @typedef {{ verdict: 'holds'|'violated'|'unknown', evidence: string }} ProbeResult */
```

- `holds` — посылка подтверждена: маркера нет, как план и утверждал.
- `violated` — посылка нарушена: маркер есть в дереве вопреки плану; ИЛИ Issue open при существующем маркере. Расхождение код↔Issue — **находка** (реестр протух), а не шум.
- `unknown` — данных нет (`gh` недоступен, нет сети). Graceful: `unknown` **не роняет** гейт — отсутствие данных не алерт (эталон `drift-anchor-divergence.mjs`).

**Отклонение от буквы консилиума — осознанное, зафиксировать в PR.** Протокол говорит «чистая функция рядом с `evaluateProdMainDivergence` (#413)», то есть в `@membrana/core`. Кладём в `scripts/lib/`: `@membrana/core` — **продуктовые** контракты, уходящие в бандл клиента и борда, логика планирования ритуала там не место. `scripts/lib/` уже держит чистые проверяемые ядра (`task-closure-review.mjs`, `deep-research.mjs`, `repo-clean.mjs`). Дух решения — «чистое ядро + тонкая обвязка, не новый пакет» — соблюдён. Спорно → решает Ozhegov.

**Запрещено:**

- Автопочинка: закрывать Issue, править план, писать в реестр. Только вердикт — иначе находка «реестр протух» скрывается.
- `git log -S` на каждом ритуале (дорого, хрупко). Достаточно `exists(symbol)` по `packages/**` + `issue.state` — миллисекунды, детерминировано.
- Новый пакет; новые зависимости.
- `|| true` в впайке: падение проверки обязано ронять генерацию — в этом её смысл.
