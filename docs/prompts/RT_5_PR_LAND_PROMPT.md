## Что построить

Скрипт `task:pr-land <N>`: локальный `git merge origin/main` (драйвер авторешает реестр) → push → `gh pr merge`. Плюс норма: «на CONFLICTING-реестр-PR — `git merge origin/main`, не редактировать руками».

## Прецедент (живой — на нём и демонстрировать)

Union-драйвер **уже есть** и зарегистрирован (`merge.registry-union.driver`, `.gitattributes`), но серверный `gh pr merge` его не видит → реестр-PR всегда CONFLICTING. Проверено живьём 16.07 (PR #532): `git merge origin/main` разрулил реестр сам, руками править не пришлось.

## Нормы эпика (обязательны)

- **Живая демонстрация на прецеденте выше** — unit-тестов недостаточно. PG3 #533: гейт соврал при 12 зелёных тестах ядра, поймал только реальный прогон.
- **Регрессия проверяется возвратом бага** — верни поломку, тест обязан покраснеть. Иначе это декорация.
- **Сначала проверь, что инструмента нет.** За 15–16.07 подтвердилось трижды: `isBaseHeldElsewhere` жил в `pr:ship`, механика `skipped` — в `morning-care`, union-драйвер был зарегистрирован.
- Новых пакетов нет; `@membrana/core` не трогаем (ритуальной логике там не место).

## СНЯТО (TF-1, #554): «хук отклоняет merge-коммиты» — находка была ложной

Ранее здесь стояло: «хук `commit-msg` отклоняет merge-коммиты, норма упирается в хук». **Это неверно — не чинить.**

Проверено прямым прогоном [`.githooks/commit-msg`](../../.githooks/commit-msg) (строка 17 освобождает `Merge*|Revert*|fixup!*|squash!*`):

```text
Merge branch 'origin/main' into feat/x   → exit 0  (хук пропускает)
merge: origin/main в day/2026-07-16      → exit 1  (строчными; «merge» не тип)
```

Ломало **собственное** `-m "merge: ..."` строчными, а не хук. Обход `SKIP_COMMIT_MSG=1` был не нужен.

**Норма для `task:pr-land`:** мёржить **без `-m`** — дать git сгенерировать `Merge branch ...`, хук пропустит. Никакого обхода в скрипт не закладывать.
