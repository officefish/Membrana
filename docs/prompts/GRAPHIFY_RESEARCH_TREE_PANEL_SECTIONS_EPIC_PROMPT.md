# Промпт: эпик graphify-research-tree-panel-sections — два графа как разделы панели за гейтом

> **Task-промпт эпика.** Под-эпик к `panel-partner-access` (#463). Размер **L** (4 M-фазы).
> Артефакт: **4 PR** (по фазе), каждая — свой closure review.
> Реестр: эпик `graphify-research-tree-panel-sections` + фазы `grp1…grp4`.

---

## Контекст

Владелец хочет опубликовать два визуальных блока — **graphify** (граф кода) и
**research-tree** (граф знаний, `apps/demos/Research-Tree/`) — **не публично**, а за
регуляцией доступа: инвесторам/управленцам — research-tree (с git-time-travel),
техпартнёрам — graphify. Механизм доступа уже построен эпиком #463
(`ally` + `grants[]`, owner-матрица «Пользователи», промокоды).

**Консилиум-гейт пройден ДО кода:**
[`docs/seanses/graphify-research-tree-panel-sections-2026-07-16.md`](../seanses/graphify-research-tree-panel-sections-2026-07-16.md)
(25 реплик). **Спор НЕ переоткрывать.**

## Архитектурные инварианты (вердикт консилиума)

- **Физика (b): блоки — независимые SPA-артефакты.** Панель отдаёт их пре-собранную
  статику через **маршрут-мост** `/panel/section/<id>/*`. Прямых импортов
  `apps/panel` ↔ блоки НЕТ (линт границ импортов; panel зависит максимум от `@membrana/core`).
- **Три леммы топологии:** **раздел** (статья реестра секций панели), **блок-артефакт**
  (своё SPA), **маршрут-мост** (deploy-правило путь→grant).
- **Доступ — два гранта поверх #463, БЕЗ новой роли:** `grant:research-tree`,
  `grant:graphify`. Предикат тот же: `canAccess(role, section) ∨ id ∈ grants`.
- **Тир = комбинация грантов** на пользователе (owner чеканит в owner-матрице):
  инвестор → `research-tree`; техпартнёр → `graphify`; owner → оба. Кода «тиров» нет.
- **Единственный арбитр доступа — гейт панели** (реестр секций + `canAccess`).
  Reverse_proxy только **исполняет** вердикт: проверка HMAC-подписи cookie + вхождение
  требуемого `grant-id`. **Логика ролей в Caddy запрещена.**
- **git-time-travel — офлайн, детерминированно** → статический снапшот-индекс (JSON).
  Прод-git веб-контуром панели в рантайме НЕ трогается.
- **СКВОЗНОЙ ИНВАРИАНТ (владелец): визуал research-tree промежуточный и СМЕННЫЙ.**
  Реф текущего вида — древо из игры Ancestors: The Humankind Odyssey; целевой облик —
  «живой мозг с нейросетевыми связями» (будущий **дизайн-эпик** за дизайнерами, вне
  scope). Сейчас на внешний вид сил НЕ тратим; presentation держим отделённым от
  модели/данных графа, текущий `@xyflow/react`-рендер НЕ канонизируем как облик.

## Фазы (порядок жёсткий; фаза = PR = closure review)

1. **GRP1 — маршрут-мост + реестр секций** (+ ADR топологии):
   - ADR (расширение [ADR-0008](../adr/ADR-0008-root-domain-scenarios-docs-topology.md)):
     `panel.mmbrn.tech` несёт разделы `research-tree`/`graphify` за гейтом; membrana.space не трогаем.
   - Реестр секций `apps/panel`: две статьи `{id, minRole, grants:[…], render: мост}`;
     UI-карточки гейтятся `canAccess`, неразгрантованное — **не отрисовано** (нет мёртвых дверей).
   - Маршрут-мост `/panel/section/<id>/*` отдаёт статику блока **только** при валидной
     HMAC-cookie с нужным grant-id; ролей в конфиге прокси нет.
   - **Контракт-тест моста**: валидная/невалидная подпись × наличие/отсутствие гранта.
   - Линт границ: `apps/panel` не импортирует исходники блоков; граф без циклов.
2. **GRP2 — два гранта + owner-матрица** (расширение #463 PU3):
   - `grant:research-tree`, `grant:graphify` в реестре секций/`PANEL_SECTIONS`.
   - Owner-матрица «Пользователи» показывает новые разделы; промокоды чеканятся с этими грантами.
   - Empty-state «залогинен, грантов ноль» — строка-манифест, не белый экран.
   - Тесты предиката и парсинга /me; ноль новых зависимостей.
3. **GRP3 — research-tree за гейтом**:
   - Блок собирается как самостоятельный артефакт, отдаётся через мост под `grant:research-tree`.
   - git-time-travel питается **офлайн-построенным детерминированным снапшот-индексом** (JSON);
     прод-git в рантайме не читается.
   - Рамка раздела (заголовок + бейдж уровня + навигация shell) — мгновенно; тело — `skeleton`
     + `aria-busy`; cold-load до первого осмысленного кадра ≤ 3 с (иначе стрим индекса чанками);
     фокус-ловушка на входе в изолированный контейнер/iframe.
   - **Presentation как заменяемый порт** (dependency inversion) — под будущий дизайн-эпик.
4. **GRP4 — graphify за гейтом (условная фаза)**:
   - Уходит за гейт **только по закрытии #529** И выполнении трёх булевых условий:
     (1) cold-load ≤ бюджета (≤ 3 с), (2) сборка НЕ тянет исходники graphify в граф
     зависимостей `apps/panel`, (3) #529 закрыт.
   - Отдаётся тем же мостом под `grant:graphify`.

## Запрещено (вердикты консилиума)

- Прямые импорты `apps/panel` ↔ блоки; циклы в графе зависимостей.
- Новая роль вместо грантов; логика ролей в reverse_proxy.
- Чтение прод-git веб-контуром панели в рантайме.
- Канонизировать текущий рендер research-tree как окончательный облик.
- graphify за гейт до закрытия #529 и трёх условий очереди.
