### Что построить (продуктовое описание)

1. **Журнал субъектного опыта персоны** — `docs/virtual-team/memory/dynin.md`:
   записи типов «позиция» / «голос» / «прогноз» (+ исход, если уже известен из
   архива), каждая запись со ссылкой-provenance на протокол-источник и датой.
   Строгая схема записи, жёсткий токен-бюджет **<5K** на журнал целиком.
2. **Детерминированный extractor** (ретро-наполнение + инкрементальный дописок):
   читает `docs/seanses/*-PROTOCOL.md` и `docs/insights/*/REVIEW.md`, фильтрует
   реплики/голоса персоны (regex/тег по имени, фиксированный парсер таблиц
   голосований), пишет журнал. Одинаковый вход → одинаковый журнал.
3. **Инъекция за флагом**: загрузчик `yarn ask` / consilium при включённом флаге
   (env/CLI, по умолчанию **выключен**) подмешивает top-N записей журнала в промпт
   персоны в пределах токен-бюджета.
4. **Человек-гейт на запись**: extractor не коммитит журнал сам — вывод идёт как
   diff/файл под явное подтверждение человека (обычный PR-флоу).

### Архитектура / контракт

| Слой | Путь | Ответственность |
|------|------|-----------------|
| Extractor | `scripts/persona-memory-extract.mjs` (или `packages/services/persona-memory` — решить на планировании, по прецеденту hermes-brief достаточно scripts) | Парсинг протоколов/REVIEW → журнал; **не знает** про промпты персон |
| Журнал (контракт) | `docs/virtual-team/memory/<persona>.md` | Схема записи: тип + дата + provenance-ссылка + текст; единственная точка сопряжения слоёв |
| Загрузчик | точка сборки промпта `yarn ask` / consilium | Инъекция top-N за флагом; **не знает** про формат протоколов |

**Запрещено** (проголосованные ограничения review, дословно):

- LLM на этапе extraction (только детерминированный парсинг; LLM-суммаризация — фаза 3, вне задачи).
- Векторные хранилища / embeddings (Research Q2: plain Markdown в git достаточно при <5K).
- Калибровка прогноз↔факт, changed-my-mind/superseded-записи — фаза 2.
- Изменение состава команды или самих `PROMPT_*.md`-характеров.
- Автономные **действия** персоны на основе памяти; cron-рутины персон (уроки заморозки #344).
- Hermes как 6-я роль (остаётся функцией ритма).
- UI/вьюер над памятью (позиция Верстальщика: журналы читаются в редакторе/на GitHub).
- Прямые импорты в UI-слои.
- Авто-коммит журнала без человек-гейта.

### Тесты

| Область | Минимум |
|---------|---------|
| Детерминизм extractor | снапшот: фикстурный протокол + REVIEW → байт-в-байт одинаковый журнал (сортировки byCodePoint, не localeCompare — урок hermes-brief) |
| Парсер таблиц голосований | фикстуры REVIEW.md → корректные записи «голос» с provenance |
| Токен-бюджет | журнал > бюджета → top-N отсечение детерминировано |
| Инъекция | флаг выключен → промпт без изменений; включён → блок памяти в пределах бюджета |
