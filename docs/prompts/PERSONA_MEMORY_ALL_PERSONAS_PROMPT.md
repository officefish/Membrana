## Контекст

Пилот суточный (13→14.07): журнал Dynin (35 записей, <5K токенов) в консилиуме
`quality-control-contour-2026-07-14` дал обещанную непрерывность позиций. Вывод:
узкое место — не время наблюдения, а частота использования (`--with-memory`
default OFF). Решение владельца 2026-07-14: расширить на всех, включить по
умолчанию в консилиумах.

## Что сделать

1. Построить журналы: `yarn persona:memory <slug>` для `vesnin`, `ozhegov`,
   `kuryokhin`, `rodchenko` (экстрактор параметризован, `PERSONA_ROLE_LABELS`
   полон — нового кода экстракции НЕ требуется).
2. **Человек-гейт:** 4 журнала предъявить владельцу как отдельные читаемые diff
   (можно одним PR, но по-файлово); `importance.json` — ручка владельца, самому
   флаги не проставлять.
3. Загрузчик консилиума: `--with-memory` становится **умолчанием только в
   `yarn consilium`** (+ новый флаг `--no-memory`); `yarn ask` — без изменений
   (opt-in). Тесты на оба пути.
4. Замерить фактическую суммарную инъекцию 5 журналов (лог строкой в вывод
   консилиума: «memory: 5 персон, N токенов»); гард: суммарно > 25K → warn.
5. `yarn persona:memory <slug> --check` зелёный для всех 5 (идемпотентность).

## Запреты

- Не трогать фазу 2 (калибровка) и фазу 3 (LLM-суммаризация/pgvector) — отдельные гейты.
- Экстракция остаётся детерминированной (без LLM) — канон REVIEW фазы 1.
- Слои 2–3 субъектности (компьютер/бюджет персоны) не упоминать в коде.
