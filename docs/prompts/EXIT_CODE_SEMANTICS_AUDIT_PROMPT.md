## Контекст

Находка спринта `ritual-step-manifest-sf` (#613, 18.07), вскрытая **живым
прогоном** — ни тесты, ни `--dry` её не видели.

Часть скриптов — **репортёры**: ненулевой код у них значит «отработал, и мне
есть что сказать», а не «сломался».

| Скрипт | Код | Значение |
| --- | --- | --- |
| `insight-drift.mjs` | 3 | найден дрейф реестров (1 = настоящая ошибка) |
| `truth.mjs cool` | 2 | граф правды нарушен, процессы заблокированы |

Бинарная модель «exit 0 → ok, иначе провал» объявляла успешный отчёт
«пропущенным некритичным» — прямая ложь о шаге. Под `|| true` вопрос не вставал
вовсе: там любой код становился нулём. **Снятие проглатывания обнажило
семантику, которую оно скрывало** — значит, тот же класс ошибки ждёт везде, где
мы будем снимать `|| true` дальше.

**Масштаб непроверенного:** осмотрены 10 шагов вечерней цепочки. В `scripts/` —
**216 скриптов**. Утренняя цепочка, ночные, деплойные не смотрели вовсе.

## Промпт целиком

### Что построить

1. **Карта кодов возврата.** Для скриптов, входящих в ритуальные цепочки
   (`ritual:day`, `ritual:evening`, ночные), установить ПО КОДУ, какие коды они
   отдают и что каждый значит. Искать не только литералы `process.exit(1)`, но и
   вычисляемые формы — литеральный grep их пропускает:
   `process.exit(bad ? 2 : 0)`, `process.exit(drifts.length === 0 ? 0 : 3)`,
   `process.exitCode = …`, проброс `res.status` из `spawnSync`.

2. **Классификация каждого ненулевого кода** в одну из трёх корзин:
   - **сбой** — скрипт не сделал работу;
   - **находка** — сделал, и есть что предъявить (→ `findingExitCodes`);
   - **не определено** — автор не задумывался; кандидат в отдельный долг,
     решать НЕ в этом спринте.

3. **Объявить `findingExitCodes`** в манифестах там, где корзина «находка».

4. **Зафиксировать карту** как проверяемый артефакт, а не прозу: гард падает,
   если скрипт из ритуальной цепочки отдаёт незадокументированный ненулевой код.

---

### Запрещено

- **Менять коды возврата самих скриптов.** Задача — описать и объявить, а не
  унифицировать. Унификация кодов — отдельное решение, скорее всего требует
  консилиума (меняет контракт 216 скриптов).
- Автопочинка найденного (#533): нашёл «не определено» — предъяви, не назначай.
- Расширять на непроверяемые пути (деплой в прод, внешние API).
- Трогать `morning-ritual-steps.json` до архивации #605.

### Тесты

| Область | Минимум |
|---------|---------|
| Карта | Каждый скрипт ритуальных цепочек присутствует; сирот нет |
| Гард | Падает при незадокументированном ненулевом коде в цепочке |
| `findingExitCodes` | Для каждого объявленного кода — тест, что `stepStatus` даёт `ok`, а `isFinding` — `true` |
| Регрессия | Настоящая ошибка (обычно 1) у того же шага остаётся сбоем |

### Definition of Done

- [ ] Карта кодов возврата ритуальных скриптов — проверяемый артефакт.
- [ ] Каждый ненулевой код классифицирован в одну из трёх корзин.
- [ ] `findingExitCodes` объявлены везде, где корзина «находка».
- [ ] Гард на незадокументированный код написан и **проверен на живом падении**
      (не только «тест зелёный» — спринт-родитель показал, что это разные вещи).
- [ ] Корзина «не определено» вынесена списком в Issue как отдельный долг.
- [ ] `yarn test:scripts` зелёный.
- [ ] LGTM Teamlead.
