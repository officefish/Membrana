# Промпт: W3 — Mintlify-зеркало (гранулы + ловушки)

> **M** · `bw-w3-mintlify` · [#949](https://github.com/officefish/Membrana/issues/949) · lead **ozhegov** · parent `bestiary-workshop`

## Контекст

T9–T12: Mintlify-ядро = гранулы улова + код/дока ловушек; внизу git-шина;
монитор — **зеркало**, не второй источник истины. Дом не переезжает.
Прецедент: `branch-mintlify-engine` (#823) — server-first, pages + `docs.json`,
пин инструкций отдельно от hygiene-kit.

Вход W2: форматы и примеры улова/ловушек уже в `docs/audit/bestiary/`.

## Промпт целиком

1. Минимальный набор страниц в `apps/docs` (или каноническом docs-приложении репо):
   - обзор мастерской бестиария / слои (класс · ловушка · улов · specimen);
   - зеркало ≥1 гранулы улова и ≥1 карточки ловушки (контент **ссылается** на git-пути,
     не копирует engines).
2. Навигация (`docs.json` / эквивалент) + провод из README дома бестиария
   («монитор → …»).
3. Явная лемма в README дома и на overview-странице: **истина = git**; Mintlify
   может отставать — сверка по путям/SHA при необходимости (как у #823), без
   обязательного полного pin-манифеста всего репо в этом спринте.
4. Объём: **thin mirror** — не cookbook на все классы; расширение — follow-up.
5. Не трогать внешний mintlify-community без ok владельца.

## Acceptance criteria

- [ ] ≥2 страницы зеркала (overview + улов или ловушка) в docs-приложении
- [ ] Навигация + провод из `docs/audit/bestiary/README.md`
- [ ] Лемма «git = истина / Mintlify = монитор» зафиксирована
- [ ] LGTM ozhegov (owner ok)

## Out of scope

Кит и пины скриптов (W4); полный каталог всех классов; sync внешнего community-репо;
новые детекторы; перенос `docs/audit/bestiary/` в `apps/docs`.
