## Контекст

Ревизия репозитория 2026-07-15 показала: **открытых PR всего 3, и все живые** — чистить
там нечего. Мусор — в ветках: 74 локальных, 59 удалённых, 10 worktree.

**Корень проблемы.** Репозиторий мёржит PR **squash**: коммиты ветки не становятся
предками `main`, поэтому `git branch --merged` их не видит. На замере он признал влитыми
**9** удалённых веток из **42** реально мёртвых. Штатный git-инструмент здесь слеп, и
именно поэтому ветки копятся: разовая ручная чистка вернёт ту же гору через месяц.
Единственный надёжный источник истины — **состояние PR на GitHub**.

Что НЕ трогаем: персона-ветки (канон [`TASKS_MANAGEMENT.md` §7а](../TASKS_MANAGEMENT.md)),
живой cowork (#487), главное дерево (в нём работает параллельная сессия).

### Что построить

`yarn repo:clean` — **dry-run по умолчанию**, удаление только по явному `--execute`:

| Флаг | Что делает |
|------|-----------|
| (без флагов) | отчёт: что мертво, что остаётся и **почему** |
| `--execute` | удалить локальные мёртвые ветки |
| `--execute --remote` | + удалить их на `origin` |
| `--execute --worktrees` | + убрать worktree архивных спринтов |

**Мертва = PR ветки в состоянии MERGED или CLOSED.** Всё остальное остаётся.

### Архитектура / контракт

| Слой | Путь | Ответственность |
|------|------|-----------------|
| Правила | `scripts/lib/repo-clean.mjs` | чистые функции: `decideBranch`, `decideWorktree`, `latestPrByBranch`, `makeArchivedSprintPredicate` |
| I/O + отчёт | `scripts/repo-clean.mjs` | git/gh, печать решений, исполнение |

**Гарды (обязательны, каждый — с причиной в отчёте):**

1. **Персона-ветки и `main`** — канон §7а. Ветка персонажа живёт между задачами; у
   `ozhegov` последний коммит от 2026-05-14, и это норма, а не заброшенность.
2. **Ветки, занятые worktree** — включая те, что появились в параллельной сессии.
3. **Текущая ветка сессии.**
4. **Открытый PR.**
5. **Нет PR и нет на `origin` при уникальных коммитах** — единственная копия работы.
6. **Worktree:** убирать только при совпадении всего — спринт `archived` в реестре,
   дерево чистое, не `locked`, не главный checkout, не текущая сессия.

**Запрещено:**

- Классифицировать по `git branch --merged` / git-предкам — при squash это ложь.
- Удалять что-либо без явного `--execute`.
- Молчаливый отсев: всё пропущенное печатается с причиной.
- Снимать `locked` с worktree автоматически.
- Трогать данные реестра и чужие worktree.

### Тесты

| Область | Минимум |
|---------|---------|
| Удаление | MERGED → удалять; CLOSED → удалять; squash-случай (`aheadOfMain > 0` при MERGED) → удалять |
| Гарды | персона-ветка с MERGED PR → оставить; занята worktree; текущая ветка; открытый PR; незапушенная с коммитами |
| Несколько PR у ветки | решает самый свежий (старый CLOSED не хоронит ветку с новым OPEN) |
| Worktree | archived+чистый → убрать; активный спринт; грязный; `locked`; главный checkout; detached HEAD |
| CLI | dry-run по умолчанию; `--remote` без `--execute` не удаляет |

### Definition of Done

- [ ] `yarn repo:clean` — отчёт по категориям с причинами.
- [ ] Гарды покрыты тестами на фикстурах.
- [ ] Чистка выполнена; персона-ветки, cowork и главное дерево не тронуты — **проверено
      прогоном после чистки**, а не на слово.
- [ ] Норма в `AGENTS.md`/`CONTRIBUTING.md`: почему `git branch --merged` не работает.
- [ ] `yarn test:scripts` — зелёный.
