## Контекст

Идея: отдельный скилл deep research и флаг спринта. **Проверка показала, что машинерия уже
есть и сломана.**

`yarn insight research <id>` уже делает ровно этот каскад: три запроса Landscape / Fit / Risk
к Perplexity с фолбэками API → MCP → руками, ключ в корневом `.env`. Но парсер вопросов
(`parseResearchQuestions`) режет многострочный вопрос до первой строки — открытый **#402**.
Воспроизведено 2026-07-15 на живом инсайте: Q1 обрывался внутри скобки
«(Linear/Jira/GitHub Projects,». Именно так 12.07 Perplexity ответил **про благотворительные
пожертвования в США** вместо памяти персон, а скрипт напечатал «RESEARCH.md обновлён».

**Вывод:** строить флаг поверх сломанного парсера — размножить тихий мусор на каждый старт
спринта. Сначала #402, потом обобщение.

Что НЕ трогаем: контур инсайтов (свой), формат `registry.json`, вторую реализацию каскада
не заводим.

### Что построить

1. **#402 — фундамент.** Парсер собирает многострочный вопрос целиком (терминатор —
   следующий пункт или секция, а не первый перевод строки); переносы внутри вопроса
   склеиваются (вёрстка ≠ смысл). **Гард** `findTruncatedQueries`: незакрытая скобка,
   хвостовая запятая, длина < 25 → ошибка **до** траты рана.
2. **`yarn task:register --research`** — кладёт в промпт заготовку секции
   «Вопросы для research (Q1–Q3)». Опт-ин.
3. **`yarn research <task-id> [--dry-run]`** — три запроса → `docs/tasks/research/<id>.md`.
   Вопросы формулирует **агент из контекста спринта**, владелец подтверждает.
4. **Скилл `membrana-deep-research`** — с явной границей «когда НЕ звать».

### Архитектура / контракт

| Слой | Путь | Ответственность |
|------|------|-----------------|
| Парсер + гард + Perplexity | `scripts/lib/insight-ritual.mjs` | `parseResearchQuestions`, `findTruncatedQueries`, `perplexityAsk` |
| Обобщение на любой источник | `scripts/lib/deep-research.mjs` | `runDeepResearch`, заготовка секции, пути |
| CLI | `scripts/research.mjs` | чтение промпта задачи, каскад, запись выжимки |
| Флаг | `scripts/task-register.mjs` | `--research` → секция в заготовке промпта |

**Запрещено:**

- **Вторая реализация каскада.** Парсер читает секцию из ЛЮБОГО markdown → для спринтов
  берётся тот же парсер/гард/прокси, источник другой. Копия разъехалась бы (ср. `pr:land`
  vs `pr:ship`).
- **Фолбэк по заголовку для спринтов.** У инсайтов он осмыслен; для спринта молча дал бы
  три рана на «расскажи про X вообще» → только `parseResearchQuestions` (явные вопросы).
- Имя скилла `deep-research` — занято встроенным скиллом Claude; префикс `membrana-`.
- Трогать контур инсайтов и формат реестра.
- Печатать ключ Perplexity.

### Тесты

| Область | Минимум |
|---------|---------|
| #402 | многострочный вопрос целиком; следующая H2 не затекает; переносы склеены; фикстура инцидента (обрыв на скобке) |

### Definition of Done

- [ ] #402 закрыт; воспроизведение на живом инсайте больше не режет вопросы.
- [ ] Гарды валят прогон **до** траты рана.
- [ ] `yarn research <id>` — **боевой прогон** даёт выжимку по теме (проверить глазами, а
      не по коду возврата: сбой этого класса тихий).
- [ ] Скилл называет, когда НЕ звать.
- [ ] `yarn test:scripts` — зелёный.

### Out of scope

- Автозапуск ресёрча на каждом спринте — флаг опт-ин.
- Верификация выжимки автоматом (ссылки, свежесть) — отдельная задача.
- Реализация `insight:research`-подкоманд помимо существующих.

## Вопросы для research (Q1–Q3)

> Заполнить ДО `yarn research deep-research-sprint-flag`. Вопрос — конкретный и самодостаточный:
> Perplexity ответит на что угодно, включая обрывок (инцидент 2026-07-12, #402).

1. **Landscape:** какие практики сложились в 2025–2026 для того, чтобы вносить внешний
   ресёрч (поисковые LLM вроде Perplexity/Sonar) в цикл разработки с AI-агентами —
   на каком шаге его зовут, куда кладут результат и как связывают с задачей в трекере?
2. **Fit (Membrana):** для маленькой команды с агентами и жёстким каноном в репозитории —
   по каким признакам решают, что внешний ресёрч нужен, а когда он превращается в шум,
   и как отличают вопрос, на который ответ есть внутри кода, от вопроса про внешний мир?
3. **Risk:** чем рискует инженерная команда, принимая выжимку поисковой LLM как вход для
   архитектурного решения — устаревание, галлюцинации, отсутствие ссылок, ложная
   уверенность; какие приёмы верификации выжимки применяют на практике?
