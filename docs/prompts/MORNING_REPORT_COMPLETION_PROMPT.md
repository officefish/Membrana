### Что построить (продуктовое описание)

**Ф1 — доклад по задачам (гейт успеха спринта).** После чеканки магистрали
владелец получает доклад-зеркало 5-блочного плана (T1/T4/T7): структура =
`frame()` (магистраль → подкрепление → перспективные → экспериментальные →
санитарные), каждый блок — состояние задач с живыми ссылками; собирается
Тимлидом по канону дня (ядро M3-S `standup(dayIssue, engineSnapshot) → Plan`
уже в main — вшить в живую цепочку, назначения персонам + handoff-ссылки).
Шаблон — механика кодом, НЕ правило строкой в скилле (T4, находка №7).

**Ф2 — ласточка тем же зеркалом.** Черновик партнёрам = то же зеркало линзой
Ожегова, короче: центральная фразой, блоки по строчке (T8). Структурный гейт
(M4-R: `structuralHash(report)=structuralHash(plan)`, protectedTokens, чек
ссылок) не пускает воду — «вода в ласточке» = недосмотр гейта Ангелины (T10).

**Ф3 — перезапуски по критериям.** Гейт при красном не правит документ руками
и не ретраит вслепую: повторно отдаёт задание генератору с поправкой (диагнозом),
до соответствия критерию (T9/T11). Критерии per-документ: структура/свежесть —
Ангелина, содержание — Тимлид. Живой анти-пример: 6 слепых отказов гейта скелета
21.07 без диагноза обрезки.

### Архитектура / контракт

| Слой | Путь | Ответственность |
|------|------|-----------------|
| Ядро доклада | `scripts/lib/` (чистые функции) | зеркало плана: `frame()` → секции доклада; structuralHash; без fs/сети |
| Генераторы | `scripts/` | сборка доклада после main-day-issue; ласточка-черновик из доклада |
| Цепочка | `package.json` (`ritual:day`) | провод: доклад — после чеканки магистрали (касание 2 из T7) |
| Гейты | `scripts/angelina.mjs`, `scripts/morning-gate.mjs` | структурный чек доклада/ласточки; перезапуск с поправкой |
| Скилл | `.cursor/skills/membrana-morning-ritual/` | обновить сценарий: три касания, шаблоны |

**Запрещено:**

- Синтез магистрали докладом — магистраль только из owner-choice (канон M3-G).
- Свободная проза в ласточке — только зеркало блоков через линзу (T8).
- Новые файлы-перекрёстки: не раздувать `test:scripts`-строку конфликтными
  правками — новые тесты добавлять в существующие тест-файлы домена, где уместно.
- Трогать `docs/procedures/` — это дом соседнего спринта #781.

### Тесты

| Область | Минимум |
|---------|---------|
| Зеркало | structuralHash(доклад) == structuralHash(план) на фикстурах; пустой блок = явное «— пусто —» |
| Ласточка | protectedTokens сохранены; внутренний жаргон из стоп-листа не проходит |
| Провод | тест интеграции: цепочка `ritual:day` собирает доклад после канона (не «символ существует») |
| Перезапуск | красный гейт → повторная выдача с поправкой; счётчик попыток ограничен, стоп громкий |
