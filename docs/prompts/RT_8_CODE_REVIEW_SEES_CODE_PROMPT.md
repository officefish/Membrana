## Что построить

`daily`-ревью должно оценивать **код дня** (дифф слитых за сутки коммитов), а не остаток рабочего дерева. Плюс гвард свежести и тест на слепоту.

**Не трогать:** ревью по запросу (`--pr` / `--branch` / `--uncommitted` / `--staged`) — оно честное, тянет полный дифф.

## Прецедент (живой — на нём и демонстрировать)

15.07: `collectRepositoryContext` не обходит дерево исходников (самодокумент, строки 206–207); `git diff --name-status` на чистом после коммитов дереве пуст. Ревью перечислило крупные слияния дня (PC-1/PC-2/cowork FREE) и оценило их как **T0 docs-only «Runtime не затронут»** — потому что смотрело на остаток рабочего дерева (docs-снапшоты). Вердикт ушёл в MAIN_DAY_ISSUE 16.07 как «✅ Вход в Фазу 0».

## Источник

Расследование 16.07 (находки 1–6). Смежно с `rt-7` (вход из метаданных, не из состояния) и `rt-1` (замкнуть контур достоверности).

## Направление (уточнить по итогам deep research + консилиума)

- daily получает дифф слитых за сутки коммитов — `todaysCommits()` уже есть, нужен их `git diff`.
- Гвард свежести: `|| true` в вечернем ритуале глотает сбой (15.07 фолбэк без кредита отдавал 127), гварда даты вниз по цепочке нет — родня `rt-5`.
- Тест на слепоту: PG3-урок — поведение, ломающее смысл инструмента, невидимо для набора.
- Открытый вопрос для консилиума: чинить daily до «видит код» ИЛИ переименовать артефакт в «сводку дня», чтобы downstream не читал T0 как «код проверен».

## Нормы эпика (обязательны)

- **Живая демонстрация на прецеденте 15.07** — unit-тестов недостаточно.
- **Регрессия проверяется возвратом бага.**
- **Сначала проверь, что инструмента нет** — механика диффа уже есть в `collectReviewContext` (режимы pr/branch); возможно, daily надо просто на неё переключить.
- Новых пакетов нет; `@membrana/core` не трогаем.
