## Контекст

Тело `MAIN_DAY_ISSUE` генерит LLM → структура плавает от прогона к прогону (19.07 таблица
роутинга была, 20.07 исчезла), а экстрактор доклада ждёт устойчивых маркеров. Заседание M2
провело **стенку `Slot → Text`**: детерминированный слой владеет каркасом и отбором, LLM —
только текстом.

**Не трогаем:** S (стендап), R (доклад), D (сны) — их вопросы отдельны. Только каркас плана.

## Что построить

1. **`frame(): Slot[]`** — чистая функция, **тотальный константный порядок**. Ровно **5 слотов
   ВСЕГДА** присутствуют: `магистраль` · `подкрепление` · `перспективные` · `экспериментальные`
   · `санитарные`. Пустой слот — **легальное явно озвученное состояние**, не исчезновение.
2. **`buildTop3(snapshot): Task[]`** (≤3) — **детерминированный ранжировщик** (не модель):
   `top3 = take(3, rank(candidates(snapshot)))`, чистая функция от снимка с провенансом.
   При `n<3` — честно меньше, **без добивки**.
3. **Балансировка** — ключ сортировки внутри `rank` (не три из одной зоны при альтернативах),
   не отдельный этап модели.
4. **Контракт `fill(Slot, Context) → Text`** — не имеет доступа к записи `id/order/title` и к
   выбору топ-3 (только вербализация входного `Task[]`).
5. Типы `Slot`, `PlanDraft`.

## Архитектура / контракт

| Слой | Путь | Ответственность |
|------|------|-----------------|
| Чистое ядро | `scripts/lib/day-plan-frame.mjs` | `frame`, `buildTop3`, `rank`, типы — без сети/DOM/LLM |
| Тесты | `scripts/day-plan-frame.test.mjs` | инварианты DoD, без моков |

**Инварианты (M2):**
- Детерминированный слой владеет `id/order/title` пяти слотов + составом топ-3; LLM — только текст.
- `frame()` даёт стабильный список id/order/title между вызовами.
- `buildTop3` на фиксированном снимке даёт тот же список (включая `order`); `n<3` → меньше без добивки.
- Балансировочный ключ разводит зоны при наличии кандидатов.
- DAG без обратных стрелок: `fill` не влияет на `frame`/`top3` (проверка отсутствием импортов назад).
- Гейт проверяет **скелет** (id/order/title + состав top3), вариативность текста — норма.

**Запрещено:** LLM/сеть в ядре; плавающая структура; добивка топ-3 при `n<3`; обратные рёбра `fill→frame`.
