# Эпик: Membrana Device — профиль сборки клиента с единственным модулем борда

> Размер **M**. Канон решения: [ADR-0012](../adr/ADR-0012-membrana-device-build-profile.md)
> (заменяет ADR-0011 — премиса «паритета палитры» не подтвердилась кодом).
> **Не переоткрывать:** пакет `client-board-registry` не создаётся, 16 плагинов не переезжают,
> host-обвязка борда не выносится, `packages/device-board` не трогается.

## Цель

Лёгкая настольная версия «только борд» — такой сборки **никогда не делалось**.
Слово владельца (16.07): «просто урезанная версия студии без модулей и плагинов, а точнее
с одним единственным модулем борда».

Состав: **pairing** (связка с сервером) + **список юзеркейсов**, идентичный модульному +
при загрузке юзеркейса **открывается борд**. Это всё.

## Инварианты (ADR-0012)

- Device = **профиль сборки самого `apps/client`**, не отдельное приложение → обвязка борда
  достаётся даром, выносить в пакет нечего.
- **Отдельная точка входа, НЕ рантайм-флаг.** Плагины импортируются статически в шапке
  `registerClientModules.ts:2-17` → сидят в главном бандле (3.1 МБ) вместе с ~16 МБ весов
  YAMNet. Флаг бандл не облегчит; нужны `registerDeviceModules` + свой `main`, чтобы Vite
  вытряс импорты.
- **Журнал и библиотека — сервисы, не модули.** UI наружу не торчит, но мосты в `main`
  остаются: рантайм борда пишет trace и отчёты через journal/sample-playback. Уберёшь мосты —
  борд онемеет.
- `packages/device-board` без новых зависимостей; Studio продолжает собираться.

## Фазы

1. **MD1 — профиль регистрации:** `registerDeviceModules` (один модуль `device-board`, ноль
   плагинов) + профильный `main`; мосты journal/playback сохранены. Проверка: бандл профиля
   заметно легче студийного (иначе смысл потерян).
2. **MD2 — оболочка:** `apps/membrana-device` — копия студийной (`main`/`preload`/`paths`),
   пути логов по README; скрипт сборки → `device-dist`.
3. **MD3 — тракт:** упаковка + смоук (аналог MS5); pairing → список UC → борд открывается,
   trace пишется.

## Связь

сырой. Device — сужение для поля, следующий цикл.
