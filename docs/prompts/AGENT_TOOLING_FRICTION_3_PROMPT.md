## TF-1 — первым: снять ЛОЖНУЮ находку

**Утверждение «хук `commit-msg` отклоняет merge-коммиты» — неверно.** Проверено прямым прогоном хука:

```text
Merge branch 'origin/main' into feat/x   → exit 0  (хук освобождает Merge*)
merge: origin/main в day/2026-07-16      → exit 1  (моё -m строчными; «merge» не тип)
```

[`.githooks/commit-msg`](../../.githooks/commit-msg) строка 17: `Merge*|Revert*|"fixup!"*|"squash!"*|"amend!"*) exit 0`.

Я сам ломал, передавая `-m "merge: ..."`, обходил `SKIP_COMMIT_MSG=1` и записал несуществующий баг в три места.

**Сделать:**
- Снять утверждение из: комментария к #521, `RITUAL_TRUST_CONTOUR_PROMPT.md`, карточки `rt-5` в реестре.
- Норма: **мёржи без `-m`** — дать git сгенерировать `Merge branch ...`, хук пропустит.

**Почему первым:** без этого `rt-5` пойдёт чинить то, чего нет.

## TF-6/TF-7 — мета-слой: РАСКОЛ (ключевое решение)

Обязательство обновлять — норма, а норма держится на дисциплине. Инвентарь протух **не** из-за отсутствия обязательства: рукописный список из 253 команд физически не удержать. Против тихого сбоя механика сильнее нормы — урок всей сессии 16.07.

Но и генерация не покрывает всё: «`git branch --merged` здесь врёт (#492)» — это **инсайт**, из `package.json` не добывается.

Поэтому:

| Часть | Как живёт | Почему так |
| --- | --- | --- |
| Инвентарь: команды / скиллы / чистые леммы `scripts/lib` / хуки | **Генерируется** из `package.json`, `.cursor/skills/README.md`, экспортов, `.githooks/` | 253 позиции рукой не удержать — протухнет снова |
| Грабли и уроки («что врёт», «какая привычка стоит») | **Руками + обязательство TF-7** | Инсайт не генерится; здесь норма уместна |

**TF-6:** `yarn tooling:overview` — генерит обзор; секция `AGENTS.md` заменяется генерируемым блоком либо ссылкой на источник истины. Датированный снимок 08.07 **удалить** — он врёт молча.

**TF-7:** в `membrana-tooling-needs` (SKILL.md): при заведении/одобрении инструмента — обязательство обновить **грабли-раздел** (не инвентарь: он генерится).
