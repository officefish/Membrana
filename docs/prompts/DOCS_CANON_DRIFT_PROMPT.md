## Контекст

Побочный результат ревизии репозитория #492. Сверка живого канона с репозиторием.

**Диагноз — не «доки отстали».** Спорные команды **никогда не существовали**: 0 коммитов
по `git log -S` за всю историю `package.json`. Регламенты описывали систему, которую
собирались построить.

Точнее всего видно на персона-ветках: персонажи как **роли живы** (`leadPersona` /
`supportPersonas` в каждой карточке реестра, ролевые промпты ведут консилиум, code review
и closure review), мёртв только **ветковый механизм** — 1 PR из 330 (#132), ветки `vesnin`
нет ни локально, ни на GitHub (404). Реальные правки контрактов `@membrana/core` шли
обычными ветками под **ADR** (#375 → ADR 0002, #413 → ADR 0003, #419 → ADR 0004). Механизм
эволюционировал: ветка `vesnin` → **консилиум + ADR**; старый канон забыли снять.

## Промпт целиком (для вставки агенту)

### Кто ты

**Координатор виртуальной команды Membrana** под руководством **Vesnin** (Teamlead).
Lead-персона — **ozhegov** (Структурщик: точность термина), поддержка: vesnin, dynin.

### Что построить

1. **Персона-ветки → реальность.** `TASKS_MANAGEMENT` §7а, `CONTRIBUTING`, `.cursorrules`,
   `VIRTUAL_TEAM_PROMPT`: персонажи = роли; ветковый механизм — исторический; фактический
   флоу = task-ветка + squash-PR; архитектурный гейт = консилиум/ADR. Историю **сохранить
   справкой с фактами**, а не стереть.
2. **Расхождения по командам** — честная пометка, а не выдумывание несуществующего:
   - `COMPETITION_SPRINT_REGULATION`: `comp:phase/score/close` → «не реализовано (бэклог)»;
   - `HACKATHON_REGULATION`: `hackathon:close` **реализован** (доку отстала) → поправить;
   - `INSIGHT_REGULATION`: реально `yarn insight research <id>` — док писал
     `yarn insight:research` через двоеточие, и агент падал;
   - `OFFICE_VDS_..._RUNBOOK`: `_ssh-office-local-forward.mjs` не существует.
3. **Гард `yarn docs:verify-canon`** — иначе канон разъедется снова (как ветки в #492).

### Архитектура / контракт

| Слой | Путь | Ответственность |
|------|------|-----------------|
| Правила | `scripts/lib/docs-canon.mjs` | `isCanonDoc`, `isArchiveDoc`, `hasHonestMarker`, `findMissingCommands`, `findMissingScriptPaths` |
| I/O + отчёт | `scripts/verify-docs-canon.mjs` | обход доков, печать находок, exitCode |

**Запрещено:**

- **Трогать архивы** (`day-sprint/`, `seanses/`, `discussions/`, `insights/`,
  `tasks/archive/`, `reviews/`) — исторические записи; их правка = **фальсификация истории**.
- Трогать доки, которые честно говорят правду: проектный бриф `REMOTE_NIGHT_CODER`
  (показывает *предлагаемые* записи package.json), `VDR_VALIDATION_SCOPE_BRIEF` (сам пишет
  «не существуют»).
- Реализовывать недостающие команды в этой задаче (отдельные спринты) — только честная пометка.
- Оживлять ветковый механизм персонажей — это смена флоу, консилиум.
- Удалять существующие ветки `ozhegov`/`boyarskiy`/`dynin` — `repo:clean` их защищает.

### Тесты

| Область | Минимум |
|---------|---------|
| Границы | канон vs архив vs промпт задачи |
| Честные пометки | «не реализовано», «Запланировано», «optional», «опционально», «не существуют» → не ошибка |
| Ловим | несуществующая команда без пометки (с номером строки), битая ссылка на `scripts/*` |
| Не ловим | проза («(yarn download)», пересказ в брифе), глоб (`yarn cabinet:*:prod`), плейсхолдер (`synthesis-<sprint>`), встроенные yarn, версии («yarn 4»), подкоманда существующего CLI |

### Definition of Done

- [ ] §7а и таблица `VIRTUAL_TEAM_PROMPT` описывают реальность; история сохранена справкой.
- [ ] 4 расхождения устранены честной пометкой.
- [ ] `yarn docs:verify-canon` — зелёный; ложные срабатывания не трогают правдивые доки.
- [ ] `yarn test:scripts` — зелёный.
- [ ] Ссылочная целостность: ни одна ссылка не ведёт в удалённый раздел.

### Out of scope

- Реализация `comp:phase/score/close`, `hackathon:open/day-close`.
- Подключение гарда в pre-push/CI — сначала пожить руками (как `repo:clean`).
- Проверка ссылок на `.md` и якоря — расширение гарда отдельной задачей.
