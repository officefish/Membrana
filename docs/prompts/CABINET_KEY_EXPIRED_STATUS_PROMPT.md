# Task prompt: Кабинет различает «ключ устарел» и «сопряжён · offline»

> **Тип:** task-промпт (одиночная задача `cabinet-key-expired-status`)
> **Размер:** M (1 PR: background-cabinet view + cabinet UI + тесты)
> **Ожидаемый артефакт:** PR в `main`, зелёный CI, LGTM Teamlead
> **Источник:** владелец, 2026-07-07 — «ключ устройства может устареть — это нормальная
> ситуация, обеспечивающая контур безопасности; кабинет должен фиксировать статус
> особым образом именно как „ключ устарел“, а не просто „сопряжён · offline“»

---

## Контекст

Истечение ключа доступа — **штатный механизм безопасности** (`NodeAccessKey.expiresAt`,
duration при выпуске). Сейчас карточка узла в кабинете показывает только
`online` / `сопряжён · offline` / `не сопряжён` — оператор не отличает
«связи нет» от «ключ истёк, нужно перевыпустить и пересопрячь».

Данные уже есть, миграция НЕ нужна:

| Что | Где |
|-----|-----|
| Срок ключа | `NodeAccessKey.expiresAt` (+ `revokedAt`) |
| Каким ключом сопряжено устройство | `Device.pairedKeyId` (PL2: сохраняется для истории) |
| Статус сопряжения | `Device.pairingStatus` — enum `paired/revoked/unpaired` (PL2) |
| Карточка узла | `apps/cabinet/src/pages/NodesPage.tsx` (NodeCard, бейджи CX1) |
| View узлов | `fetchMembraneMe` → `apps/cabinet/src/api/membrane.ts`, серверный membrane view |

**Архитектурное решение (Структурщик):** enum `DevicePairingStatus` НЕ расширяем
(миграция + догма PL2 «ключ хранится для истории»). «Устарел» — **производный статус
на чтении**: сервер вычисляет по `pairedKeyId → NodeAccessKey.expiresAt < now()`
и отдаёт в membrane view полем устройства (например,
`pairedKeyStatus: 'active' | 'expired' | 'revoked'` + `pairedKeyExpiresAt`).

## Требование (дословно)

Кабинет фиксирует истечение ключа сопряжённого устройства особым статусом
«ключ устарел» — это не то же самое, что «сопряжён · offline».

## Скоуп

1. **Server (`packages/background-cabinet`)**: membrane view (узлы для кабинета)
   отдаёт по сопряжённому устройству производный статус ключа:
   `pairedKeyStatus: 'active' | 'expired' | 'revoked'` + `pairedKeyExpiresAt` (ISO).
   Вычисление — join по `pairedKeyId` (ключ может быть уже удалён — тогда по
   `pairingStatus`); чистая функция + unit-тест.
2. **Cabinet (`apps/cabinet`)**: NodeCard — бейдж **«ключ устарел»** (`badge-warning`
   или `badge-error` — по DESIGN.md, решает Верстальщик) с `title`
   «Ключ истёк <дата>. Выпустите новый ключ и пересопрягите устройство»;
   показывается ВМЕСТО `online/offline`, когда ключ истёк (истёкший ключ важнее
   транспортного статуса — оператор должен действовать).
   Иерархия бейджей (pure-функция + тесты): `не сопряжён` → `ключ отозван`
   (pairingStatus=revoked) → `ключ устарел` → `online` / `сопряжён · offline`.
3. Кнопка/переход «Ключи» из карточки уже есть (CX-карточка) — достаточно title-подсказки.
