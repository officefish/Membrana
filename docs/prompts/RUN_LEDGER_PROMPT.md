## Контекст

Три первых живых прогона процедуры `one-shot` (23.07: #1046 и #1047 — согласия, #1051 —
отказ тимлида на штампе) показали разрыв: **контракт кадра можно соблюсти и доказать по
артефактам, не коснувшись рельсов процедуры**. Очередь кадров объявлена в манифесте
(`frames: [first-frame]`), но исполнителя очереди нет — `procedures:workshop` умеет только
смотреть (`--inspect` / `--audit` / `--decompose`). «Прогон» сегодня = человек читает прозу
README, а машинного следа прогон не оставляет вовсе.

Шторм «прогон процедуры» ([`docs/storm/storm-procedure-run-2026-07-23/`](../storm/storm-procedure-run-2026-07-23/REPORT.md),
12 тезисов, закрыт полом владельца) назвал конструкцию: каждая вызванная процедура порождает
**честный прогон**, внутри которого живут математически защищённые подтверждения отработки
кадра **либо явного отказа**. Подпись рождается серверным вызовом — этим снято возражение
Математика («подпись не ловит ничего, если агент и процедура суть один код в одном
контексте»): сервер есть физически отделённая сторона.

**Что уже есть в репозитории и переиспользуется, а не пишется заново:**

- `node:crypto` даёт `sha256` и `ed25519` из коробки — проверено на живой машине: подпись
  64 байта, `verify: true`. **Новых зависимостей быть не должно.**
- `scripts/lib/storm-breath.mjs` — живой пример детерминизма без часов и случайности
  (SimHash64 + Хэммингово расстояние), включая `normalize()`. Смотреть как на образец стиля.
- `scripts/lib/storm-codex.mjs` — `sha256` для свежести (QC1), проверка двух подписей (QC4).
- `packages/background-office/src/linear-snapshot/` — паттерн «снаружи истина, внутри
  слепок». В этой карточке **не трогаем**, но контракт модуля должен его допускать.

**Что НЕ трогаем:** `packages/background-*` (сервер — следующая карточка), `deploy/`,
миграции, существующие процедуры и их манифесты.

### Что построить (продуктовое описание)

Чистый модуль **цепи прогонов**: append-only лог, где каждый прогон процедуры становится
листом Merkle-дерева, а голова дерева подписывается Ed25519. Контур назван владельцем
дословно:

```
канонформа → sha256-лист → Merkle-корень → Ed25519-чекпойнт  +  verify(checkpoint, proof)
```

1. **Канонформа прогона** — детерминированная сериализация записи прогона: LF (не CRLF),
   UTF-8 NFC, сортированные ключи объектов, никаких часов и случайности внутри самой формы.
2. **Лист** — `sha256` канонформы. **Дерево** — Merkle над листьями в порядке очереди.
3. **Чекпойнт** — `{ size, root, signature, keyId }`, подписанный Ed25519.
4. **Доказательства** — `inclusionProof(index)` и `consistencyProof(sizeOld, sizeNew)`.
5. **`verify(checkpoint, proof)`** — проверка **офлайн**: без сервера, без сети, из гита.

**Форма — подписанный transparency log, а не блокчейн.** Консенсус не нужен: сервер
единственный секвенсор. В коде и README писать «цепь прогонов с подписанными чекпойнтами»;
слово «блокчейн» тянет майнинг и распределённое согласие, которых у нас нет.

### Архитектура / контракт

| Слой | Путь | Ответственность |
|------|------|-----------------|
| Канонформа | `scripts/lib/run-ledger/canonical.mjs` | объект прогона → канонические байты; NFC, LF, сортировка ключей |
| Дерево | `scripts/lib/run-ledger/merkle.mjs` | листья, корень, inclusion/consistency proofs |
| Подпись | `scripts/lib/run-ledger/checkpoint.mjs` | Ed25519 sign/verify головы дерева, `keyId` |
| Фасад | `scripts/lib/run-ledger/index.mjs` | `appendRun`, `checkpointOf`, `verify(checkpoint, proof)` |
| Тесты | `scripts/run-ledger.test.mjs` | `node --test`, без сети |

**Запрещено:**

- Новые npm-зависимости. Всё есть в `node:crypto` (`createHash('sha256')`,
  `generateKeyPairSync('ed25519')`, `sign`/`verify`).
- `Date.now()` / `Math.random()` внутри канонформы и дерева — время и порядок приходят
  ВХОДОМ (их даёт секвенсор), иначе воспроизводимости нет.
- Приватный ключ в репозитории. Публичный — обязан лежать в гите, иначе проверять нечем.
- SHA-1 в новых цепях. `segmentHash` пинов имеет форму 40 hex (sha1) — реализация отложена
  (F2), выбор ещё свободен; сюда её не тянуть (SHAttered, 2017).
- Трогать `packages/background-*` и `deploy/`.

### Тесты

| Область | Минимум |
|---------|---------|
| Канонформа | один и тот же прогон с CRLF и LF даёт ОДИН хеш; порядок ключей не влияет; NFC-эквивалентные строки совпадают |
| Merkle | корень для 0/1/2/3/8 листьев; inclusion proof для каждого индекса; отрицательный — подменённый лист не проходит |
| Consistency | лог из N и N+K чекпойнтов совместим; переписанная середина — НЕ совместима |
| Подпись | verify чужим ключом падает; изменённый `root` при валидной подписи падает |
| Симметрия исходов | прогон-**отказ** ложится в цепь и доказывается наравне с успехом |
| Офлайн | весь тест-файл проходит без сети и без сервера |
