## Контекст

Шторм branch-taxonomy (21.07, PR #771) ратифицировал: ревью ведёт один из пяти,
назначенный главным (T3), с собственной памятью (T4) и бестиарием антипаттернов
(T5). Стек владельца: процедурный слой (#781) и контейнеры готовы — ревью
рефакторится третьим слоем. Бестиарий — семя `insight-weekly-antipattern-audit-bestiary`
(17.07, поручено 19.07).

## Что построено

1. **Ф1** `scripts/lib/review-lead.mjs` — каскад назначения ведущего (явное слово
   `--lead` → `leadPersona` карточки, чей id виден в ветке/диффе → скоуп путей
   диффа большинством → Teamlead + «вне конвенции»); основание всегда названо.
   Провод в `code-review.mjs`: блок ведущего (персона + основание + память T4 +
   бестиарий T5) в промпте ПОСЛЕ обрезаемого контекста (не зверь B1); вердикт
   ведущего — пропуск/блок первой строкой. Назначение — усилитель, не гейт
   запуска: без git/памяти ревью не падает.
2. **Ф2** `docs/bestiary/` — 7 зверей с живыми вещдоками, append-only, зверь без
   вещдока запрещён.
3. **Ф3** `docs/procedures/code-review/` — README + MANIFEST по контракту Р1;
   держатель vesnin, ведущий прогона per-diff; `validateProcedure` зелёный.

**Граница:** грамматика веток / `resolveHolder` — соседний #785, не тронуто.

- [x] Бестиарий в промпте ведущего; каждый зверь с вещдоком (тест).

      (артефакт: `docs/discussions/uncommitted-code-review.md`, 21.07 вечер).
