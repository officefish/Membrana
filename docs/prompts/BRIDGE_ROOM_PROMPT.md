## Семантика жизненного цикла (канон — слово капитана, две поправки)

- **Открытие — ЯВНОЕ.** По слову капитана; комната НЕ обязана открываться с утра.
- **Закрытие — НЕ ЯВНОЕ (автоматическое).** Шаг **вечернего** ритуала закрывает
  открытую комнату сам, без отдельной команды.
- **При закрытии конспект уезжает ФРЕЙМОМ в репозиторий** (не ручным коммитом —
  через контракт фрейма отправки, шторм фреймов / #900).

## Этапы (проектировал Ozhegov — структура, слабая связанность, один источник истины)

### Б1 · Ядро состояния комнаты (чистое)
`scripts/lib/bridge-room.mjs` — конечный автомат `closed → opened → closed`,
чистые функции без fs/сети: `openRoom(state, {day, cap})`, `closeRoom(state)`,
`isOpen(state)`. **Один источник истины** — файл `docs/bridge/state.json`
(читает/пишет вызывающий, не ядро). Идемпотентность: повторное `open` открытой
комнаты → тот же state (не второй дом); `close` закрытой → no-op с честной
пометкой. Тесты — на переходы и идемпотентность.

### Б2 · Дом конспекта + попугай (append-only)
Дом дня `docs/bridge/<дата>/CONSPECTUS.md`. Реестр техдолгов `docs/bridge/DEBTS.md`
— **append-only** (попугай «запомнил — не забудет»): `{id, долг, вещдок,
статус open|settled, дата}`. `bridge debt add/settle --evidence` — правка только
с вещдоком; удаление запрещено (settled помечается, не стирается). Стартовый
набор — 7 долгов 22.07 (хвост снов #746, office #933, провод #592, gitleaks,
ласточка #918, зубы #924/#925, жаргон-ресёрч).

### Б3 · Провод открытия — попугай зачитывает долги
`bridge open` печатает живые (`open`) долги из DEBTS.md первой репликой комнаты —
голосом попугая, немногословно. Пустой реестр → «долгов нет» (честный
empty-state, не молчание — урок 22.07).

### Б4 · НЕявное закрытие в вечернем ритуале + отправка фреймом
Шаг `ritual:evening` зовёт `bridge close` **мягко**: комната открыта →
финализация конспекта + постановка на отправку **фреймом** (не прямой коммит);
комната не открыта → тихий no-op с честной строкой («мостик не открыт», не
пустота). Граница: закрытие само НЕ пушит — готовит фрейм-заявку (сходится с
инсайтом роутинга: конспект едет заявкой, не спонтанным пушем).

### Б5 · Жилец процедурного слоя
`docs/procedures/bridge/` — README (определение + держатель **angelina** как
ведущая комнаты) + MANIFEST (engines: bridge-room.mjs; frames[] когда контракт
#900 готов — до тех пор очередь фреймов комнаты конспектом в README).
`validateProcedure` + `procedures:registry --check` зелёные; регистрация
жильцом реестра слоя.
