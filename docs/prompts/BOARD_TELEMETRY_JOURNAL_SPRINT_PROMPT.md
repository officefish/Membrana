## Контекст

Вкладка «Журнал» правого сайдбара device-board (#269, `device-board-journal-sidebar`)
показывает живой хвост **scenario-трейсов** (`getScenarioTraceLines` host-контракта,
`BoardJournalPanel`: follow-tail, копировать/скачать/очистить, drag-ресайз).
Владельцу в этом месте нужен **журнал телеметрии** — треки и отчёты детекции,
как в кабинете и в клиентском модуле «Журнал».

Настоящий журнал телеметрии на клиенте уже существует:
`apps/client/src/modules/telemetry-journal/` — `useLiveJournal`
(`@membrana/telemetry-journal-service`), рендереры `LiveJournalItemRow`/`LiveJournalPager`
(lp5), события `liveJournalHub`, автообновление `useLiveJournalAutoRefresh`.

| Документ / код | Зачем |
|----------------|-------|
| [`packages/device-board/src/components/board-journal-panel.tsx`](../../packages/device-board/src/components/board-journal-panel.tsx) | Текущая вкладка (trace-хвост) — UX сохранить для трейсов |
| [`packages/device-board/src/components/board-right-sidebar.tsx`](../../packages/device-board/src/components/board-right-sidebar.tsx) | Вкладки «Узлы | Журнал», ресайз |
| [`apps/client/src/modules/telemetry-journal/`](../../apps/client/src/modules/telemetry-journal/) | Готовые рендереры/хуки телеметрии — переиспользовать, НЕ копировать |
| `check:boundaries` | **device-board зависит только от core** — телеметрия в пакет не импортируется |
| [`docs/DESIGN.md`](../DESIGN.md) | UI-канон; a11y, empty/loading |

**Ключевое архитектурное ограничение:** `@membrana/device-board` не может импортировать
`telemetry-journal-service`/клиентские модули (границы пакетов). Телеметрия попадает в
сайдбар **слотом от хоста** (render-prop/ReactNode через `DeviceBoardShell`) — пакет
остаётся тонким, рендереры живут в клиенте.

## Фазы

### BTJ1 — Слот журнала в сайдбаре: «Журнал» = телеметрия хоста, трейсы → «Трейс» · M

**Пакет:** `packages/device-board`.

- `DeviceBoardShell` принимает опциональный слот журнала (`journalSlot?: React.ReactNode`
  или render-prop — решает Структурщик в PR).
- Правый сайдбар: вкладка **«Журнал»** рендерит слот (если хост его передал);
  трейсы переезжают во вкладку **«Трейс»** без потери UX #269 (follow-tail,
  копировать/скачать/очистить, JOURNAL_VISIBLE_TAIL_LINES, drag-ресайз общий).
- Без слота (кабинетная перспектива, автономные хосты) — поведение как сейчас:
  вкладки «Узлы | Трейс», журнала телеметрии нет.
- Тесты: pure-логика видимости вкладок (слот есть/нет); существующие тесты панели зелёные.

### BTJ2 — Клиент: компакт-панель телеметрии для слота · M (после BTJ1)

**Пакет:** `apps/client` (device-board module wiring + telemetry-journal reuse).

- Компонент `BoardTelemetryJournalPanel` (клиент): компактный список последних
  элементов журнала телеметрии устройства — треки/отчёты live, через `useLiveJournal`
  + `LiveJournalItemRow` (переиспользование lp5-рендереров; никакого дублирования логики).
- Автообновление как в модуле (`useLiveJournalAutoRefresh` / liveJournalHub);
  состояния loading/empty по DESIGN.md; a11y (`role="log"` или эквивалент).
- Прокинуть слот в `DeviceBoardShell` в клиентской обвязке борда.
- Компакт-режим: без тяжёлых фильтров модуля — последние N элементов + ссылка/кнопка
  «Открыть журнал» (переход в модуль телеметрии; под захватом переход заблокирован
  бордом-локом CX4 — кнопку дизейблить с подсказкой, не ломать лок).
- Тесты: pure-хелперы компакт-списка (обрезка N, маппинг); существующие тесты модуля зелёные.

### BTJ3 — Docs + smoke · S (после BTJ2)

- `docs/DEVICE_BOARD_*`/CONCEPT: вкладки сайдбара «Узлы | Журнал | Трейс», источник
  журнала (слот хоста), границы пакета.
- Ручной smoke чек-лист в PR: запуск сценария с микрофоном → в «Журнале» появляются
  треки/отчёты; «Трейс» показывает scenario-трассу; ресайз/копирование живы.

## Итог реализации (BTJ, 2026-07-07)

Раскладка правого сайдбара борда: **«Узлы | Журнал | Трейс»**.

| Вкладка | Содержимое | Источник |
|---------|-----------|----------|
| **Узлы** | инспектор нод + палитра | device-board (как было) |
| **Журнал** | телеметрия устройства (треки/отчёты детекции) | **слот от хоста** — клиент передаёт `BoardTelemetryJournalPanel` (lp5-рендереры `LiveJournalItemRow` + `useLiveJournal`); device-board телеметрию **не импортирует** |
| **Трейс** | живой хвост scenario-trace (#269) | `getScenarioTraceLines` host-контракт (без изменений) |

**Границы пакета:** `@membrana/device-board` зависит только от core; телеметрия
приходит `journalSlot?: React.ReactNode`. Без слота (кабинет, автономные хосты)
вкладки — «Узлы | Трейс». Активная вкладка нормализуется к видимой
(`normalizeRightSidebarTab`). «Открыть журнал» под захватом заблокирован (CX4 лок).

Файлы: `right-sidebar-tabs.ts` (BTJ1), `board-right-sidebar.tsx` слот (BTJ1),
`apps/client/.../BoardTelemetryJournalPanel.tsx` + `boardTelemetryCompact.ts` (BTJ2).
