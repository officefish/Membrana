## Подтверждённые тезисы (14 + R2)

- **T1.** `membrana-leveling` — детерминированное выравнивание общего рабочего пространства.
- **T2.** Зонтичная процедура: два скрипта по очереди — `main-fill` (ветки→main) → `workspace-level` (main→деревья+гигиена).
- **T3.** Объект — общее рабочее пространство (проект как целое).
- **T4.** Главная цель — `main` актуален: всё **готовое** недоехавшее оказывается в main.
- **T5.** Конечное состояние — точка, от которой удобно продолжить общую работу.
- **T6.** «Не доехало» = только `ready`; чужую/живую работу не трогаем; WIP не везётся.
- **T7.** Висящая работа — плохая практика; незаконченное **регистрируется** легитимно, незарегистрированное = мусор. Сопряжено с вечерним ритмом.
- **T8.** `ready` штампуют оба: тимлид + чистая функция `ready = CI_green ∧ ¬conflicts(main) ∧ PR_approved`.
- **T9.** `main-fill` = сериализованный `pr:ship`-поезд (#700), не батч.
- **T10.** Центр тяжести — вечер (гейт-обнаружение), НО свежая законченная работа обязана оказаться в main. Не или-или.
- **T11.** Мусор ≠ незавершёнка: временные/прикладные файлы — **в корзину** (`%TEMP%`/scratchpad), настоящее незавершённое — **регистрируется**. Коммит-и-забыл запрещён.
- **T12.** `membrana-leveling` — процедура во фреймах со своим набором гранул и фреймов (на фрейм-рельсах, как one-shot).
- **T13.** У процедуры есть отдельный фрейм про работу с её **временными файлами**.
- **T14.** Структура фреймов процедуры — **мягкий рекомендательный паттерн** (как контейнер). **Сюжет — это серия «сюжетных» фреймов** (тег «сюжетный»), не один фрейм; вокруг — служебные фреймы: «подвести провода» + временных файлов + «довести результат до сервера/соседей».
- **R2.** «Свой/чужой» — ложная ось (одно общее пространство). Реальная ось — состояние: `live` пропускается, `trash = dirty ∧ ¬registered` останавливает гейт и называется поимённо.

## Эмерджентный каркас (кандидат, не канон)

Фреймы `membrana-leveling`: **F0** времянки (рамка) · **F1** инвентаризация+диспозиция (`ship`/`register`/`bin`) · **F2** `main-fill` (поезд `pr:ship`) · **F3** `workspace-level` (main→деревья+гигиена) · **F4** манифест (3 раздела: влито / застряло почему / **мусор найден**). Гранулы — предикаты `ready` / `disposition` / `isLeveled` / `legit`.
