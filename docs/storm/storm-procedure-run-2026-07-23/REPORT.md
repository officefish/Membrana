## Конспект: двенадцать тезисов, подтверждённых владельцем

**Диагноз (T1–T3).** Предмет — что значит процедуру **прогнать**, а не соблюсти; различение
предъявлено практикой: контракт был соблюдён и доказуем, а рельсы в прогоне не участвовали.
Очередь кадров объявлена, исполнителя нет — «прогон» сегодня есть чтение прозы человеком, и
README тем самым не артефакт процедуры, а её **обход**. Машинного следа прогон не оставляет,
причём не по забывчивости: части прогона («находка», «выстрел», «прогон») не имеют
определений — реестру нечего регистрировать.

**Направление (T4–T6).** Тимлид — это тимлид, а не формально архитектор систем; право BLOCK
не решается назначением роли. Если не проверять руками — надо строить надёжные автоматические
механики контроля. Каждая вызванная процедура порождает **честный прогон**, внутри которого,
помимо отчёта, живут **математически защищённые подтверждения** отработки кадра либо явного
отказа. Отказ подтверждается наравне с отработкой — он не «отсутствие подтверждения».

**Конструкция (T7–T10).** Кадр печатает артефакт с провенансом; без файла кадр не пройден.
Подпись рождается **серверным вызовом** — этим снято возражение Математика («подпись не ловит
ничего, если агент и процедура суть один код в одном контексте»): сервер и есть физически
отделённая сторона. При этом артефакт кадра — **контент** процедуры, а прогон в виде
итогового отчёта имеет **самостоятельную ценность**. Прогон-прецедент архивируется на
сервере, в репозитории остаётся **слепок**, подтверждающий архивацию, — он и есть
окончательное основание закрывать процедуру (для one-shot — заодно основание закрывать
контейнер в Linear). Слепок несёт **и содержание**: подпись считается от контента и хеша
предыдущего прогона; гонок прогонов нет — очередью управляет сервер.

**Криптография (T11–T12).** SHA-256; форма — подписанный transparency log, не блокчейн
(консенсус не нужен, сервер единственный секвенсор). `sha256` и `ed25519` есть в
`node:crypto` из коробки — новых зависимостей ноль. Выбран **inclusion proof** как рабочее
доказательство «прогон в логе» (log₂(N) хешей). Проверки **исполняются** на сервере
(канонизация — одна функция на приёме, кодировки перестают быть проблемой), но
**криптография не зависит от жизненного цикла сервера**: чекпойнт и доказательство обязаны
проверяться без него — из гита, чужими руками, после его смерти. Промежуточную формулировку
«офлайн-проверка отпадает» владелец снял прямо перед закрытием: цену он не принимал,
принимал только удобство.

## Развилка — названа владельцем явно

| Ветвь | Решение |

| Заседание | нет (рекомендация питомца отклонена владельцем) |
| Инсайт | нет |
| **Спринт** | **ДА — «берём `run-ledger`»** |
| Никуда | нет |

**Куда едет:** спринт **`run-ledger`**, контур назван владельцем дословно:

> канонформа → sha256-лист → Merkle-корень → Ed25519-чекпойнт, плюс `verify(checkpoint, proof)`

Плюс серверный архив прогонов в `background-office`; в репозитории — слепок, подтверждающий
архивацию. **`verify` — часть контракта, а не опция:** он и есть исполнение T12, то есть
независимость криптографии от жизненного цикла сервера.

**Опора в существующем коде** (разведка питомца, вдох 5): модуля архива прогонов в office
нет, но паттерн «снаружи истина, внутри слепок» уже живёт как `src/linear-snapshot/`;
соседи по духу — `drift-anchor` и `llm-channels` (учёт вызовов по procedure-id).

## Осталось нерешённым (хвосты в спринт, не в тезисы)

1. **Цепь защищает историю, а не суждение.** Прогон может быть честно записан, подписан — и
   опираться на кривую рамку (кандидат К4: рамку для судьи готовит тот, кого судят). Граница
   силы конструкции; принять цепь за гарантию качества будет ошибкой.
2. **Что такое прогон для процедуры БЕЗ кадров** — утренний ритуал, консилиум, сам шторм?
   У шторма детерминированный след уже есть (вдохи), у ритуала — нет.
3. **Порог состоявшегося прогона** (вопрос Математика, В2): развести «прогон состоялся»
   (была попытка и обратная связь) и «прогон дал предписание» (есть вердикт). Сегодня
   исполнитель мерил зелёными тестами, то есть успехом ПРАВКИ, а не прогона.
4. **Ключ сервера — новый корень доверия:** где живёт приватный ключ, ротация, судьба старых
   подписей после ротации, публичный ключ в гите.
5. **`segmentHash` пинов — 40 hex, форма SHA-1.** Реализация отложена (F2), выбор ещё
   свободен; на новые цепи sha1 не тянуть (SHAttered, 2017).
6. **Семантическая ось.** SimHash64 из `storm-breath` рядом с криптографической подписью даёт
   машинный детектор пустого прогона (подписан, но семантическое расстояние ноль). В тезисы
   не вносилось — кандидат для спринта.
7. **Не подтверждённые кандидаты К3–К9** остаются в `THESES.md` как есть: не отвергнуты,
   просто не подтверждены на закрытии.
