## Подтверждённые тезисы (21)

**Суть и состав**
- **T1.** Нужен контейнер основных СТРАТЕГИЧЕСКИХ документов — по ним агенты сверяют курс, из них рождаются планы/стратегии/дорожные карты. README — один член, не единственный.
- **T2.** Контейнер двумерный; три списка данных: гранулы · шаблоны · релизы.
- **T3.** Гранула — мельчайший элемент документа (абзац/строка), на основе которого всё собирается.
- **T4.** Шаблон — композиция гранул, из которой генерируется документ. _(изначально «генератор» — переименовано по Ожегову.)_
- **T5.** Генератор — СКРИПТ, рендерящий релиз по шаблону; это КОД, живёт в плоском `scripts/` (контейнер без своего кода).
- **T6.** Релиз — сгенерированный документ, не пишется руками.

**Природа гранулы, версии, валидация**
- **T9.** Гранула ≠ кристалл правды: кристалл = проверяемый факт (граф правды), гранула = инструкция-несущая документа. Разные башни.
- **T10.** У гранулы есть версия.
- **T11.** Шаблон завязан на версии гранул: версия изменилась/исчезла → шаблон НЕВАЛИДЕН → требует ЯВНОГО акта валидации.
- **T12.** Генератор может сгенерировать невалидный релиз — но тогда он честно помечен экспериментальным.
- **T13.** Генератор маршрутизирует по валидности: валидный шаблон → релизы, невалидный → эксперименты.
- **T15.** Эксперименты в КАРАНТИНЕ: агенты читают только по спец-разрешениям.
- **T18.** Подписант валидации — в движке ОПЕРАТОР; по T20 акт server-mediated → агент сам себя не валидирует.
- **T20.** Валидация — серверный запрос на API **office** (server-mediated, office = авторитет).

**Операторы, движок, источник истины**
- **T7.** С контейнером работают АИ-операторы — он делается не только в чате.
- **T8.** Контейнер подкреплён движком-рендерером, как задачи подкрепляет Linear.
- **T14.** Движок выбирается оператором через ПАНЕЛЬ из трёх: Notion · Coda · Affine; после выбора — полная синхронизация.
- **T16.** Источник истины — git-контейнер; движки ПОДКРЕПЛЯЮТ, не отбирают контроль (git побеждает при конфликте).
- **T17.** Утечка пока разрешена: проект открыт, документы публичны (scoped «пока»).
- **T19.** Affine — приоритетный, самохост ДОМА на `mmbrn.tech`; чужой движок → ЗАГЛУШКА со ссылкой на актуальный движок оператора.

**Наблюдаемость**
- **T21.** Нужен БОРД состояния сети в харнес-панели (здоровье серверов); 2 сервера — начало, будет и 22.

## Осталось нерешённым (в коворк / уточнить)

- Гранулярность гранулы: абзац / строка / **утверждение** (Ожегов предложил «утверждение», не ратифицировано).
- Как шаблон ссылается на гранулы: id / порядок / секции; версия гранулы — хеш (lockfile) или семантика.
- Правило конфликта при синке: git побеждает (T16), но механика реконсайла не расписана.
- Полнота синка на гетерогенных провайдерах: Affine-first — full; Notion/Coda — возможно lossy release-only (не ратифицировано).
- Членство: предикат «документ есть релиз контейнера» (кандидат Dynin: у него есть шаблон).
- Подписант валидации для АГЕНТ-сгенерированного шаблона (T18 открыт; office-медиация задаёт механику, но не роль).
