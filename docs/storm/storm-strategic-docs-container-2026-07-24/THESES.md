## Подтверждено владельцем — тезисы (сказаны на мостике)

**T1. Нужен контейнер для основных СТРАТЕГИЧЕСКИХ документов** — тех, по которым АИ-агенты сверяют курс и **из которых рождаются планы, стратегии, дорожные карты**. Корневой README — один член, но не единственный.

**T2. Контейнер двумерный; три списка данных:** гранулы · шаблоны · релизы.

**T3. Гранула** — мельчайший элемент документа (абзац/строка), на основе которого всё собирается.

**T4. Шаблон** — особая композиция гранул, из которой генерируется итоговый фундаментальный документ. _(изначально названо «генератор» — переименовано по Ожегову: точность.)_

**T5. Генератор** — **скрипт**, который по шаблону (из гранул) рендерит релиз. Это КОД, а не список контейнера → по норме «контейнер без своего кода» живёт в плоском `scripts/`.

**T6. Релиз** — сам сгенерированный документ (README и др.), не пишется руками.

**T7. С контейнером работают АИ-операторы** — он делается не только в чате.

**T8. Контейнер подкреплён особым движком-рендерером**, подобно тому как задачи подкрепляет Linear.

**T9. Гранула ≠ кристалл правды.** Кристалл = тезис-принцип, проверяемый факт (граф правды). Гранула = **инструкция в промпте, несущая конструкция документа**. Разные атомы — контейнер документов НЕ есть граф правды (владелец, 24.07). _(снимает развилку 1: словарь не сливается с графом.)_

**T10. У гранулы есть версия.**

**T11. Шаблон завязан на версии своих гранул: если версия гранулы исчезает или меняется — весь шаблон становится НЕВАЛИДНЫМ и требует ЯВНОГО акта валидации изменений** (владелец, 24.07).

**T12. Генератор может сгенерировать невалидный релиз — но тогда он честно помечен экспериментальным** (владелец, 24.07).

**T13. Генератор маршрутизирует по валидности шаблона: валидный шаблон → в релизы, невалидный → в эксперименты** (владелец, 24.07).

**T14. Движок-рендерер выбирается оператором ЧЕРЕЗ ПАНЕЛЬ из трёх провайдеров — Notion, Coda, Affine; после выбора оператор должен иметь полную синхронизацию с выбранным** (владелец, 24.07). _(рифма с панелью LLM-каналов: провайдер на выбор; и с Linear у задач — T8.)_

**T15. Эксперименты — в КАРАНТИНЕ: агенты читают их только по специальным разрешениям** (по умолчанию недоступны) (владелец, 24.07). _(снимает открытый вопрос: эксперименты — карантинная зона, не свободный статус.)_

**T16. Источник истины — git-контейнер. Движки (Notion/Coda/Affine) лишь ПОДКРЕПЛЯЮТ, не отбирают контроль** — при конфликте побеждает git; провайдер = зеркало/поверхность (владелец, 24.07).

**T17. Утечка пока разрешена: проект открыт (opensource), документы публичные** → server-first-гейт на выбор провайдера ПОКА не применяется («пока» — scoped, при закрытии части канона пересмотреть) (владелец, 24.07).

**T18. Подписант акта валидации — в движке действует ОПЕРАТОР.** Для шаблона, сгенерированного АГЕНТОМ, подписант отдельно не назван — но по T20 акт server-mediated, значит агент сам себя не валидирует (владелец, 24.07).

**T19. Affine — приоритетный провайдер; самохостится ДОМА, на поддомене `mmbrn.tech`.** Если оператор выбрал другой движок (Notion/Coda) — на домашнем месте только ЗАГЛУШКА со ссылкой на актуальный движок оператора (владелец, 24.07). _(Affine-first гасит утечку по умолчанию; полный синк версий/пинов реален там, где схему держим сами.)_

**T20. Валидация — через СЕРВЕРНЫЙ запрос на API office.** Акт валидации server-mediated (office = авторитет), не локальный и не инлайн у агента (владелец, 24.07). _(office как control-plane: каналы LLM И валидация доков; закрывает «лису при курятнике».)_

**T21. Нужен БОРД в харнес-панели об общем состоянии сети** (здоровье серверов). Сейчас 2 сервера — только начало, со временем будет и 22 (владелец, 24.07). _(Несущее для server-first: контейнер, каналы LLM и валидация опираются на серверы — без наблюдаемости сети office-гейт слеп; сегодня office таймаутил, это и вскрыло нужду.)_

## Кандидаты (НЕ подтверждены — вклад агента, ждут слова владельца)

- **К-read-only:** релиз правится только через гранулу/шаблон + пересборку; ручная правка релиза запрещена (стена).
- **К-freshness:** свежесть/провенанс живёт на грануле; свежесть релиза = свежесть его гранул (ложится на каскад Ангелины).
- **К-2D:** два авторских списка-источника (гранулы × шаблоны), релиз — порождённый выход; генератор — движок сбоку.
- **К-lockfile:** «версия гранулы в шаблоне» = пиновка (как package-lock): шаблон пинит `гранула@версия`; дрейф → невалиден до ре-валидации.
- **К-эксперименты-4й:** «эксперименты» — это отдельная (4-я) зона контейнера рядом с релизами, или статус самого релиза? (открыто)
