## Подтверждённые тезисы (все — слова владельца, провенанс в THESES)

| # | Тезис |
|---|-------|
| T1 | Шипование — только через скилл; пробел ответственности закрывается на шип-гейте |
| T2 | Ответственный: явное слово → иначе выводится из имени ветки |
| T3 | Ревью ведёт один из пяти, назначенный главным: антипаттерны → пропуск/блок |
| T4 | На ревью работает собственная память ведущего |
| T5 | У антипаттернов свой бестиарий (инсайт 17.07) |
| T6 | Процедурный контент → ведущая персона процедуры (шторм → Ангелина); по сложным коммитам ответственность распределяет парсер на code-review |

| T8 | Ветка вне конвенции → возврат агенту на доработку |
| T9 | Проектируем не парсер, а его место жительства (маршрут расследования прецедентов) |
| T10 | Скрипты ≠ процедуры: скрипты — функции; процедуры оперируют категориями (команда, задача, ответственность, стратегическая цель) |
| T11 | Скрипты — плоский контейнер; процедуры управляют по ссылке; группировка — виртуальными наборами (киты) |
| T12 | Контейнеру процедуры не нужны собственные скрипты и тесты — код централизован, связь ссылкой |
| T13 | Ритуалы — процедуры (порождают код и артефакты) |
