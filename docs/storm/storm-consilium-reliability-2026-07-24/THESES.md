## Подтверждено владельцем — тезисы (системная ветвь)

**T1 (из C4, Vesnin) — консилиуму нужен контракт канала.** Какая модель допустима для
какого класса вопроса; обход на запасную модель — либо явный, либо запрещён для
структурных/высокоставочных прогонов. Без контракта канала это не процедура, а рулетка.

**T2 (из C5, Kuryokhin) — запасная модель это другой инструмент.** Её прогон не выдаётся за
полноценный молча. При закрытом сильном канале высокоставочный консилиум либо явно понижает
доверие к своему вердикту, либо не гонится вовсе.

**T3 (из C6, Фаррелл/pet) — у консилиума нет своего фрейма «провода».** Утром узаконили
паттерн #1094, чей смысл — служебный фрейм «подвести провода», и тут же погнали консилиум без
проводов. Системный фикс = приложить свой же свежий канон к своему инструменту.

**T4 (из C7, синтез — ХРЕБЕТ) — «фрейм проводов консилиума» = контейнер, не пункт списка.**
T3 оказывается не одним тезисом, а рамкой; T1/T2 и бывшие хвосты C1–C3 — его содержимое,
три провода одного фрейма:

| Провод фрейма | Из тезиса | Что несёт |
|---|---|---|
| **Канал** (LLM-provider) | T1 + T2 + (C3) | консилиум = LLM-процедура с пиновой цепочкой под класс вопроса; закрытый сильный канал → **явная деградация в витрину**, не молчком |
| **Вход-сигнал** | (C1) | для класса вопроса объявлено, что подводится: структурный → агенда, БЕЗ repo-контекста/RAG/памяти |
| **Контракт вход↔вердикт** | (C2) | ID повестки — провод между входом и выходом; без ID страж (rt-6) слеп, «зелёный» нефальсифицируем |

Пять разрозненных претензий стали тремя проводами одного фрейма. Бывшие C1/C2/C3 —
не хвосты, а **содержимое T4** (их предметность сохранена в таблице выше).
