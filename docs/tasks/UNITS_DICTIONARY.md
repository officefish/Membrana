## 1. Словарные статьи

| Термин | Значение | Чем НЕ является |
|---|---|---|
| **Центральная задача** | = **GitHub-issue** — удостоверение задачи: конкретная работа с конкретным ответственным. **Единица ответственности и единица счёта.** Задача без Issue незаконна (норма владельца). | Не контейнер движения и не строка чеклиста. |
| **Контейнер** | Linear-issue (parent) — единица движения: агрегирует центральную задачу и дополнительные, ползёт по колонкам. | Ответственности **не несёт**, выхода не производит, в счёт **не входит**. |
| **Центральная под-задача** | Linear-под-задача, связанная биекцией со своей GitHub-issue: одна сущность в двух носителях («удостоверение» ↔ «движение»). | Не самостоятельная вторая единица — это та же единица работы. |
| **Дополнительная под-задача** | Атрибут центральной, односторонне производная от неё. Блокирующая дополнительная держит закрытие центральной, но отдельной единицей счёта не становится. | Не самостоятельная строка счётчика. |
| **Ответственный** | Ровно **одна персона, принявшая выход** (`leadPersona` карточки; `assignee` в Linear). Инвариант: `|owns⁻¹(t)| = 1`. | Не физический производитель: субагент-исполнитель обезличен по контракту. |
| **Исполнитель** | Тот, кто произвёл работу (обезличенный субагент; в Linear — `delegatedAgent`). | Не носитель ответственности; подписи не имеет и не обязан иметь. |
| **«Закрыто»** | Событие для величин и гейтов = **закрытие центральной задачи** (= закрытие её GitHub-issue). | Закрытие контейнера — производное (наступает при закрытой центральной). Закрытие подчинённой под-задачи — **не событие**. |

## 2. Правило биекции — запрет двойного счёта

```
count-unit(work) = центральная под-задача ⟺ её GitHub-issue   (биекция)
```

- Каждая работа отображается **ровно в одну** единицу счёта.
- Контейнер (агрегатор) и дополнительные под-задачи (атрибут/производная) —
  **вне области счёта по построению**.
- Инвариант проверяем: `Σ по центральным = счёт`; `Σ по контейнерам ≠ счёт`.
- Отношение «один контейнер : N GitHub-issue» счёта не ломает: считаем по
  центральной (вердикт T14 шторма переезда).

## 3. Двухслойность Linear — тождество и адаптер

| Пара | Характер соответствия |
|---|---|
| `assignee` ↔ ответственная (принявшая выход, `leadPersona`) | **Тождество по функции**: один человек, отвечает за результат. |
| `delegatedAgent` ↔ исполнитель | **Соответствие через адаптер, имена не склеивать**: `delegatedAgent` — конкретный слот агента Linear, наш исполнитель — обезличенный субагент. Адаптер — предмет интеграции, не словаря. |

Агент не может быть единственным ответственным (факт Linear).

## 4. Величина → единица счёта

| Величина | Формула | Единица счёта |
|---|---|---|
| `leadTime` | `closedAt − createdAt` | GitHub-issue (центральная) |
| `ownerlessRate` | доля единиц с `leadPersona = ∅` | GitHub-issue (центральная) |
| `throughput(p, w)` | закрытых единиц персоны за окно | GitHub-issue (центральная) |
| `escalationRate` | доля единиц со сменой `leadPersona` / переприсвоением приёмки | GitHub-issue (центральная) |
| `wip(p)` | число движущихся единиц персоны | движущаяся Linear-под-задача — **через биекцию** с Issue, дрейфа нет |

`wip` — очередь намерений, не производительность (обязательная подпись отчёта).

## 5. Носители следа (импорт из вердикта M2)

След единицы работы живёт ровно в двух носителях: **карточка реестра**
(`leadPersona`, назначение) и **артефакт закрытия** (`{acceptedBy, headRev}`,
подтверждение приёмки). Коммит-сообщение и тело PR носителями **не являются** —
обезличенные каналы авторства, гейты их не читают.

## 6. Достоверность фактов Linear

| Факт | Статус проверки |
|---|---|
| Под-задача — полноценный объект: свой статус и свой `assignee` через API | по ресёрчу (`docs/tasks/research/linear-agent-identity-facts.md`); живым API не подтверждён |
| Двухслойность `assignee` + `delegatedAgent`; агент не может быть единственным ответственным | по ресёрчу; проверка по живому API — office `v1/linear/issue/:id` (см. EXPECTATIONS блока) |
| `blockedBy`/`blocking` видны в GraphQL, вебхуками не приходят | по ресёрчу; проверяет блок данных |

Пометка снимается только результатом живой проверки; до тех пор словарь честно
несёт «факт по ресёрчу».
