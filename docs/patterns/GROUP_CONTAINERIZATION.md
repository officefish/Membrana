# Паттерн: контейнеризация обобщённой группы

Статус: **рекомендация** (SHOULD). Родился 2026-07-21 из двух независимых реализаций,
сошедшихся к одной форме: контейнер git-аудита ([`docs/audit/git/`](../audit/git/README.md),
PR #760) и контейнер аудита реестра задач ([`docs/audit/tasks/`](../audit/tasks/README.md),
PR #764).

## Контекст

В проекте накапливается **большая обобщённая группа** однородных сущностей или
артефактов: ветки репозитория, карточки реестра задач, скрипты, документы,
инсайты, датасеты. Группа растёт стихийно, её артефакты (снимки, разборы,
кеши, промпты) расползаются по корню репо и чужим каталогам, а работать с ней
начинают несколько агентов параллельно.

## Проблема

Без выделенной формы группа порождает три хронических дефекта:

1. **Артефакты без дома** — снимки и кеши оседают в корне, `docs/archive/` «на
   глаз», temp в tracked-путях; при параллельных сессиях — гонки в общих файлах.
2. **Нет источника «текущего»** — каждый агент пересобирает состояние группы
   ad-hoc, результаты не сравнимы между сессиями, история не накапливается.
3. **Неуправляемые мутации** — массовые операции над группой (чистка веток,
   архивация карточек) делаются по механическому признаку, без свидетельств
   и слова владельца.

## Решение

Управлять группой через **контейнер** — выделенный каталог с пятью органами:

| Орган | Что это | Форма |
|-------|---------|-------|
| **Контракт** | Граница: что можно писать, что коммитить, как звать агента | `README.md` контейнера, таблица «Разрешено / Запрещено» |
| **Реестр** | Канонический снимок «текущего» состояния группы (overwrite) + опциональные dated-снимки | `registry/<GROUP>_LIST.md` с Meta (Date, SHA, Source) |
| **Кеш** | Сырые дампы и промежуточные артефакты, не источник истины | `cache/` под gitignore (+ `.gitkeep`) |
| **Инструменты** | Детерминированные команды: инвентарь, декомпозиция, аудит; пишут в контейнер сами (`--report`) | `yarn`-скрипты + скиллы |
| **Агент** | Операторский вход: роль, сценарии, safety | `AGENT_PROMPT.md` |

Принципы поверх органов:

- **Источник истины назван явно.** Контейнер хранит производные снимки; где
  живёт истина (сам git, `registry.json`, файловая система) — сказано в контракте.
- **Сценарии с жёстким входом.** Деструктивные/дорогие сценарии требуют явного
  параметра в текущем сообщении (HARD GATE) — запрет угадывать из контекста сессии.
- **Мутации — слово владельца.** Инструменты читают и предлагают свободно;
  массовые изменения группы — только по явному ok, поштучно и со свидетельствами.
- **Разборы копятся как история** — dated-файлы `analysis/` не перезаписываются.

## Подвид `manifest-only` (вердикт Р1 заседания procedural-layer, 21.07)

Контейнер **определения** (не данных): несёт канон (`README.md`), машиночитаемый
`MANIFEST.json` со ссылками наружу (`engines[]` — код в плоском `scripts/`,
`precedents[]` — прогоны в их домах) и НИЧЕГО исполняемого: **код, тесты и кэш
запрещены**. Зуб заселённости — `validateProcedure(dir)`
(`scripts/lib/validate-procedure.mjs`): `resolvable` ∧ `readmeNonEmpty` ∧
`manifestSchemaOk`, в CI. Реализация: [`docs/procedures/`](../procedures/README.md).
К чеклисту ниже подвид добавляет пункт: манифест ссылок обязателен, локальный
код — дефект.

## Чеклист соответствия

Реализация ссылается на этот паттерн и копирует чеклист в свой README, заменяя
каждый `[ ]` на ✅ либо ⚠ с причиной отступления:

1. [ ] Выделенный каталог; артефакты группы не живут вне его.
2. [ ] `README.md`-контракт с таблицей «что писать / что коммитить».
3. [ ] Канонический overwrite-реестр с Meta; dated-снимки опциональны, не дубль.
4. [ ] `cache/` под gitignore, отделён от commit-friendly markdown.
5. [ ] Инструменты пишут в контейнер сами; источник истины назван.
6. [ ] `AGENT_PROMPT.md` со сценариями; у опасных сценариев HARD GATE.
7. [ ] Массовые мутации — только по слову владельца, поштучно, со свидетельствами.
8. [ ] Провода: контейнер виден из `AGENTS.md`, `docs/audit/README.md` (или
   соответствующего каталога) и связанных скиллов.

## Границы применимости

- Группа маленькая или короткоживущая → контейнер избыточен, хватает каталога.
- Группа уже имеет собственный движок с контрактом (например, инсайты с
  lifecycle-регламентом) → не дублировать; паттерн применять только к недостающим
  органам.
- Контейнер — форма *управления* группой, не форма исполнения. Версионирование
  сабграфов (кит / фрейм) — отдельный паттерн
  [`PINNED_SUBGRAPH_VERSIONING`](./PINNED_SUBGRAPH_VERSIONING.md) (семя — issue
  #761). **Область версионирования** (список-контейнер вне / сабграф под ним в
  области; домены кит/фрейм) — единая формулировка там же (`copies=1`, m4 / #930);
  здесь не дублировать.
- Контейнер — про *пространство* группы (где живёт, кто пишет). Оснастка, которой
  над домом **работают** (осмотр, декомпозиция, рассмотрение), — отдельная ось
  «операций», паттерн [`HOME_WORKSHOP`](./HOME_WORKSHOP.md) (мастерская). Дом и его
  мастерская — разные сущности: дом хранит, мастерская работает.
