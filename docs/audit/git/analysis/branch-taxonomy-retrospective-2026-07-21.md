## Проект «как надо было»: три оси в грамматике имени

Если бы деревья и ветки проектировались заранее под мультиагентную работу и
вывод ответственного, грамматика была бы:

```text
<держатель>/<тип-или-формат>/<scope>-<slug>
```

| Сегмент | Значения | Откуда берётся | Что даёт |
|---------|----------|----------------|----------|
| 1. Держатель | персона (`ozhegov`, `dynin`, `vesnin`, `boyarskiy`, `rodchenko`, `angelina`) | явное слово владельца → иначе `leadPersona` карточки → иначе маппинг scope→персона | **ответственное лицо машинно** |
| 2. Тип/формат | `feat` `fix` `docs` `chore` \| `night` `cowork` `meeting` `truth` `research` | закрытый словарь (реестр, зубы на новый префикс) | жанр работы; снимает дубль feat/feature |
| 3. Scope-slug | scope из **производного** реестра (turbo workspaces + каталоги docs) + слаг; если работа от карточки — слаг = id карточки | реестр скоупов производный от кода, не ручной (анти-бестиарий) | привязка к домену и к задаче |

Правила вывода ответственного для шип-гейта (по убыванию приоритета):

1. Явно указано при шипе → оно.
2. Слаг = id карточки реестра → `leadPersona` карточки.
3. Сегмент 1 — персона → она.
4. Маппинг scope→персона из конфига (докс → Ожегов, scripts/CI → Структурщик,
   детекция/DSP → Математик/Музыкант, UI — Верстальщик, …).
5. Ничего не сработало → Teamlead по умолчанию + **находка «ветка вне
   конвенции»** — кричать в вердикте, не проглатывать (анти-Молчун).

Деревья: имя дерева = `Membrana-<держатель>-<миссия>` для сессий; долгоживущие
деревья-роли — закрытым реестром в конфиге `classifyWorktree` (он уже есть и
уже знает классы деревьев — провод готов).

Агент-исполнитель (claude/codex/cursor) в имени ветки НЕ нужен: исполнитель
фиксируется в коммите (`Co-Authored-By`), а ответственность несёт персона —
это разные оси, и сегодняшние `cursor/…`-ветки путают их.

  это норма с зубами (гейт в шип-скилле), по регламенту ей место на
  консилиуме/заседании до реализации.

## Рекомендации (кандидаты, решение владельца)

2. Реестр скоупов — производный (workspaces + docs-каталоги), с зубом.

4. Реестр деревьев-ролей в `classifyWorktree`; сессионные деревья — по грамматике.
