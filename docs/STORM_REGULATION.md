# Регламент: Шторм

**Статус:** черновик (ратифицирован заседанием `storm-format-ratify` 2026-07-20, с
исправлениями по финальному аудиту) · **Версия:** 0.1 · **Заведён:** 2026-07-20

Шторм — **дивергентный** формат: беседа, из которой рождаются тезисы. Не планирование,
не разбор готового — часто просто **конспектирование будущего доклада**. Шестой в ряду:
[`COMPETITION_SPRINT`](./COMPETITION_SPRINT_REGULATION.md) ·
[`COWORK_SPRINT`](./COWORK_SPRINT_REGULATION.md) · [`HACKATHON`](./HACKATHON_REGULATION.md) ·
[`NIGHT_SPRINT`](./NIGHT_SPRINT_REGULATION.md) · [`MEETING`](./MEETING_REGULATION.md) · **шторм**.

## Когда использовать

**Да:** мысль ещё не сформулирована, её надо породить; предмет расходится, а не сходится;
нужен живой конспект будущего доклада/инсайта/входа в заседание.

**Нет:** сходимся к вердикту → [`MEETING`](./MEETING_REGULATION.md) (конвергентный).
Готовая развилка → консилиум. Исполнимая задача → движок задач.

**Шторм ≠ заседание.** Оснастка общая — живой конспект, секретарь, правило «только
подтверждённое». Расходятся по **направлению мысли** и по **выходу**: заседание сходится
к обязательному вердикту (DAG, одна комната — один вопрос); шторм расходится, вердикта не
несёт, директивы факультативны.

## Три голоса

| Голос | Кто | Право | Ограничен |
|---|---|---|---|
| **Секретарь** — Ангелина | процесс | приглашает участников, ведёт конспект; страж свежести/структуры; **следит, чтоб не врали**; **возвращает на доработку**; **единственный гейт каскада** | **версионируемым кодексом правды** |
| **Участник** (5 персон) | доменная экспертиза | вносит реплику в своём предмете | своим предметом; вес — только через две подписи |
| **Робот-питомец** | сам агент | встревает без приглашения, **судит и критикует** владельца и участников; **голос, не гейт** | ничем — свободный наблюдатель без кодекса |

- Секретарь **приглашает**, содержательное предложение принадлежит **названному
  участнику** и по умолчанию есть гипотеза.
- Питомец даёт агенту легитимный прямой голос: своё суждение не протаскивается под именем
  персоны. Реплика питомца несёт `origin: pet` и **никогда не попадает в основания гейта**.
- Гейт каскада **один — Ангелина**. Два гейта = размытая ответственность.

## Кодекс правды Ангелины

Детерминированный предикат `codex(document) → pass ⊕ fail(reasons ⊆ QC)`; тот же документ
+ та же версия = тот же вердикт, без сети и скрытого состояния. Несёт `codexVersion`.

**Версия — semver:** `major` — удалён/ужесточён QC-пункт (ломает прошлые `passed`);
`minor` — добавлен непрерывающий; `patch` — уточнена формулировка.

**QC (замкнутое множество), `codex = QC1 ∧ … ∧ QC5`:**

| # | Проверка |
|---|---|
| QC1 | **свежесть** — `sourceDigest == sha256(текущие источники)` |
| QC2 | **структура** — обязательные секции присутствуют (вкл. «Список посылок») |
| QC3 | **honest-шапка** — заявленное не противоречит фактам |
| QC4 | **две подписи** — `signatures ≥ 2` (см. ниже) |
| QC5 | **посылки-не-выводы** — в списке посылок нет заключений этой же комнаты |

**Возврат** — адресный акт `returned(reasons, codexVersion)`: перечислены павшие QC.
Отличается от derived-статуса «не прошёл» (отсутствие валидного `passed`): «вернула» —
событие, «не пропустила» — состояние.

## Вес: две подписи (правило «только подтверждённое»)

Реплика веса не несёт. Автомат трёх состояний, рёбра только вперёд, по явному действию:

```
реплика ──(участник ЯВНО просит: «прошу утвердить директивой»)──▶ заявка
заявка  ──(владелец ЯВНО подтверждает)──────────────────────────▶ директива
```

Владелец может отклонить заявку → тезис остаётся репликой. Директива обязана нести
**оба следа** (заявку участника и подтверждение владельца); без одного — это не директива,
а ложь о статусе, которую Ангелина возвращает.

## Опровержение владельцем: парирование (к согласию, не к подавлению)

Если владелец явно оспорил реплику — запускается проверка парирования: у опровергнутого
проверяется содержательный контр-довод. Есть — участник парирует своим голосом, идём к
**согласию**. Нет — уступает, реплика падает **честно**. Цель — согласие, не «владелец
прав, потому что владелец». Павшая реплика несёт **след**: контр-довод и ответ на него,
либо явную уступку. Пустое падение — подавление, Ангелина возвращает.

> Зеркало двух подписей: одно правило не даёт неподтверждённому **возвыситься** под
> авторитетом владельца, другое — живому доводу **подавиться** под ним. Runaway-риск
> («девять раундов спора») не проектируется: вступать в спор — ответственность владельца,
> агенты покладисты. Граф для обид не заводим.

## Контейнер и жизненный цикл

**Дом — директория** `docs/storm/<id>/`: вмещает конспект и порождённые файлы. Разрез
**движение ↔ содержание**: пока шторм открыт — живое (на гейте 3, server-first, серверное
пространство питомца); при закрытии — оседает в git. Дом ≠ пространство `current_task`.

**Предикаты (чистые, над директорией, без сети):**

```
isOpen(s)   = exists(dir(s)) ∧ topic(s) ≠ ∅ ∧ topicByOwner(s) ∧ ¬isClosed(s)
isClosed(s) = (pora(s) ∨ sevenBreaths(s)) ∧ forkNamed(s)
              forkNamed ⟺ fork ∈ {directions ∪ {"никуда"}}
```

Тотальность и взаимоисключение: `isOpen ∧ isClosed = false`;
`isOpen ∨ isClosed ∨ isNotStarted = true`. Третьего смысла нет.

**Открытие:** владелец называет предмет → секретарь заводит дом, статус OPEN, цепляет
фонового писаря. Пустой/безымянный дом — **не открыт**.

**Закрытие — пол и потолок:**
- **Пол — «пора» владельца:** закрывает в любой момент, суверенно, раньше потолка.
- **Потолок — семь вдохов:** неотменяемый, закрывает **даже против владельца**
  (само-связывание против затягивания). Владелец волен закрыть скорее, но не длить дольше.

**Выход обязателен:** конспект + **развилка**, названная владельцем явно (включая «никуда»
— тоже его слово, не умолчание). Директива факультативна. Молчаливое закрытие без развилки
— не закрыт.

**Стенка контейнера:** наружу — конспект и развилка (обязательны) + директивы (если есть,
с двумя подписями); реплики остаются внутри.

## Тайминг: механизм вдоха (детерминированный)

Вдох — единица прогресса, меряется хешем близости, без часов и случайности.

- **Состояние:** SimHash 64 бита всего конспекта (упорядоченные нормализованные тезисы).
- **Событие:** смена состояния тезиса (добавлен / изменён по существу / снят). Реплика без
  смены — не событие. Хеш пересчитывается после каждого события до **паузы** (ход завершён
  без новой смены — граница окна счёта).
- **Вдох** ⟺ `popcount(опорный XOR текущий) ≥ K_эфф`. Косметика → `d = 0` (нормализация),
  вдоха не даёт.
- **Смягчение артефактами:** `K_эфф(a) = K_min + (K0 − K_min)·c/(c + a)`, где `a` — число
  принятых Ангелиной артефактов; `c > 0` — число артефактов «полуспада» (при `a = c`
  смягчение половинное). Больше материала → ниже порог → вдох легче (само-темп).
- **Инварианты (контракт, тестируются без живого шторма):** `K0 ≥ K_min ≥ 1`, `c > 0`,
  `K_эфф` монотонно не возрастает по `a`, `K_эфф ∈ [K_min, K0]`.
- **Мандат `K_min = 1`** (рекомендация Дынина, ратифицирована владельцем 20.07). Тогда
  любое существенное событие (`d ≥ 1`) есть вдох, и **≤ 7 существенных событий
  гарантированно жгут потолок**; увернуться можно лишь косметикой (`d = 0` = не прогресс).
  Так неотменяемый потолок держится — иначе его обходят под-пороговым дрейфом.
- **Счётчик рантайма** `{опорный_хеш, счётчик: 0..7, K_эфф}` — derived-проекция append-only
  лога событий, `step(state, event) → state'`. Без сети, без `Date.now`, без `Math.random`.
  Вещдок вдоха несёт `{опорный_хеш, счётчик, K_эфф, d, a, K0, c}` отдельными полями.
- **Завершимость:** даёт **потолок счётчика** (≤ 7 вдохов) — безусловно. Порог регулирует
  **частоту** вдохов, не факт остановки; сходимость хеша ни при чём. Один метр (тот же
  хеш) даёт и второй сигнал — череда неудачных хешей = дивергенция стихла = готовность к
  закрытию.

> `K0`, `c` — калибруются первыми живыми штормами внутри диапазона-заглушки, не с нуля.

## Анти-паразит (управляющий принцип)

Зелёная ложь процедуры — дефект её **конструкции**, не исполнителя: щель, где шаг
рапортует успехом, не сделав дела. Формат проектируется так, чтобы паразиту негде было
жить. Для каждого «зелёного» статуса — проверить все конъюнкты предиката; известные щели:
пустой дом притворяется открытым; триггер без развилки — молчаливое закрытие; косметика
накручивает вдохи-пустышки; «никуда» неотличимо от пустоты; реплика подаётся директивой.
Все закрыты предикатами выше.

## Швы наружу

- **С заседанием:** выход-развилка, одна ветвь = заголовок будущего заседания. Контракт
  `handoff: конспект шторма → вход заседания` (дивергенция → схождение).
- **С движком задач:** факультативная директива (две подписи) → GitHub-issue по инварианту
  законности. Это **норма-называние**, не файловый провод; движок здесь не проектируется.

## Hard rules

1. **В запись только подтверждённое.** Директива — две подписи (участник + владелец).
2. **Развилка на выходе обязательна**, включая явное «никуда». Директива — нет.
3. **Пора — суверенный пол, семь вдохов — неотменяемый потолок.**
4. **Гейт каскада один — Ангелина.** Питомец — голос, его реплика не входит в основания.
5. **Опровержение оставляет след согласия** (контр-довод+ответ или уступка).
6. **Детерминизм тайминга:** без сети, без `Date.now`, без `Math.random`.

## Stop rules

| ID | Условие | Действие |
|---|---|---|
| **S-S1** | Дом без предмета, названного владельцем | Не открыт |
| **S-S2** | Триггер закрытия без развилки | Не закрыт — Ангелина возвращает |
| **S-S3** | Директива без двух подписей | Не директива — ложь о статусе, возврат |
| **S-S4** | Реплика подавлена без следа парирования | Возврат (правило 5) |
| **S-S5** | Семь вдохов сгенерированы | Шторм закрыт — даже владелец не оспорит |
