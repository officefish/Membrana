# Регламент Cowork Sprint

> **Cowork Sprint** — формат разработки, в котором **одна общая разработка** режется на **три блока**, и три команды выполняют их параллельно. До точки интеграции каждая команда работает над своим блоком **изолированно** — концептуально независимо, без компромиссов ради состыковок; общие интерфейсы вырабатываются только на поздних этапах (Interface Consilium) и собираются адаптерами в единый результат. Победителя нет — **мёржатся все три блока**.
>
> Консилиум формата: [`docs/seanses/cowork-sprint-format-2026-07-14.md`](./seanses/cowork-sprint-format-2026-07-14.md) (20 реплик, единогласно).
> Донор инфраструктуры: [`COMPETITION_SPRINT_REGULATION.md`](./COMPETITION_SPRINT_REGULATION.md) v1.0.
> Связано: [`VIRTUAL_TEAM_PROMPT.md`](./VIRTUAL_TEAM_PROMPT.md), [`TASK_PROMPT_WORKFLOW.md`](./prompts/TASK_PROMPT_WORKFLOW.md), скилл `membrana-worktree`.

---

## Когда использовать

| Режим | Когда |
|-------|--------|
| **Competition Sprint** | Несколько равноправных решений ОДНОЙ задачи; выбор по фактам, победитель один |
| **Cowork Sprint** | Разработка естественно режется на ≥3 слабо связанных блока; ценна концептуальная чистота каждого блока; ранняя интерфейсная договорённость навредила бы дизайну |
| Дневной M/L | Один очевидный путь |

**Cowork Sprint уместен, если:**

- разработка режется по **естественным швам** системы (слой / пакет / домен данных) на блоки с пустым пересечением файловых зон;
- у каждого блока есть **собственный DoD**, проверяемый без соседей (на стабах);
- стоимость ранних интерфейсных компромиссов выше стоимости поздних адаптеров.

**Cowork Sprint НЕ уместен**, если блоки плотно связаны «горячим» интерфейсом (постоянный обмен по шву) — тогда резка ложная и формат выродится в переговоры. Не смешивать в один слот с Night Build / Hackathon / Competition.

## Участники

### Три команды блоков (исполнители)

| Команда | Ветка | Изоляция |
|---------|-------|----------|
| **Block A** `<slug-a>` | `cowork/<sprint-id>/<slug-a>` | своя файловая зона, свой CONCEPT.md, свой EXPECTATIONS.md, свои стабы |
| **Block B** `<slug-b>` | `cowork/<sprint-id>/<slug-b>` | то же |
| **Block C** `<slug-c>` | `cowork/<sprint-id>/<slug-c>` | то же |

Имена блоков — человекочитаемые слаги из brief (не alpha/beta/gamma: это язык соперничества, здесь его нет). Каждая команда — в **своём worktree** (скилл `membrana-worktree`; прецедент comp-detection-alarm: формат 3-агентов-в-worktree устойчив).

### Координатор (не команда)

Ведёт ACTIVE-файл и фазовые гейты, объявляет точку интеграции по предикату, ведёт Interface Consilium и **интеграционную ветку** (команды блоков интеграцию не ведут — конфликт интересов «мой блок прав»). Жюри и голосования в формате НЕТ.

---

## Hard rules изоляции

1. До Interface Consilium **запрещены**: merge/rebase/cherry-pick между ветками блоков; чтение чужих веток и чужих `EXPECTATIONS.md`; любые интерфейсные договорённости между командами (в чате, в доках, через «случайные» правки общих файлов).
2. **Файловые зоны блоков не пересекаются** (зафиксированы в brief). Общие корневые файлы (`package.json`, `docs/tasks/registry.json`, `app.module.ts`-подобные точки сборки) в изолированной фазе **не трогает никто** — нужные строки вносятся на интеграции.
3. Думать об интерфейсах — можно и нужно, но **односторонне**: команда пишет `EXPECTATIONS.md` (что хочу от соседей / что готова отдать) и строит **исполняемые стабы** соседей строго в своей зоне (`team-<block>/stubs/` или моки в тестах). Стабы в интеграционную ветку не мёржатся; стаб, доживший до прода, — дефект интеграции.
4. Нарушение изоляции ≠ дисквалификация (блок нельзя выбросить): блок объявляется **«скомпрометированным»** и разбирается первым на Interface Consilium; факт — в RETROSPECTIVE.

## Фазы

| Фаза | Название | Ориентир | Артефакт |
|------|----------|----------|----------|
| **0** | Brief + open | 0.5–1 день | `COWORK_SPRINT_BRIEF.md`, `COWORK_SPRINT_ACTIVE.md`, 3 ветки от одного BASE_SHA |
| **1** | Concept | 0.5–1 день | `team-<block>/CONCEPT.md` (свой блок изнутри) + первый `EXPECTATIONS.md` |
| **2** | Isolated build | 2–3 дня | Блок проходит **собственный DoD** на стабах; scoped CI зелёный; EXPECTATIONS дописывается по ходу |
| **3** | Interface Consilium | 2–4 часа | Одновременное вскрытие трёх EXPECTATIONS → `INTERFACE_CONTRACT.md` |
| **4** | Integration | 0.5–2 дня | Ветка `cowork/<id>/integration`: мёрж трёх блоков + адаптеры; smoke швов зелёный |
| **5** | Merge + archive | 0.5 дня | Один PR integration → main; `RETROSPECTIVE.md`; `yarn task:archive` |

### Точка интеграции (гейт 2→3) — событийная, не календарная

```
ready(block) = собственный DoD блока зелёный (тесты на стабах, scoped CI)
integration_gate = ready(A) ∧ ready(B) ∧ ready(C)  ∨  deadline из brief
```

Объявляет координатор по факту предиката. По дедлайну неготовый блок входит в интеграцию «как есть» с явной пометкой готовности. На каждую ветку — freeze-тег `cowork-<id>-<block>-interface-freeze`.

### Phase 3 — Interface Consilium

Протокол: `docs/discussions/cowork-sprint-<id>-interface-consilium.md`. Порядок:

1. **Вскрытие**: три `EXPECTATIONS.md` читаются всеми одновременно (до этого — никем).
2. Сведение в **`INTERFACE_CONTRACT.md`**, обязательные секции:
   - сигнатуры/схемы данных на каждом шве (A↔B, B↔C, A↔C — какие есть);
   - **инварианты потока данных**: единицы, частоты, порядок событий, владелец времени (два «зелёных» блока с разным пониманием timeline дают мусор на стыке);
   - **глоссарий стыка** — одинаковые слова для сущностей на границах (термин = граница);
   - **интеграционный smoke** — единый сценарий, проверяющий прохождение данных через все швы;
   - список адаптеров: что и где пишется на интеграции.
3. Скомпрометированные блоки (нарушение изоляции) разбираются первыми.

### Phase 4 — Integration

- Ветка `cowork/<id>/integration` от того же BASE_SHA; мёрж трёх блоков механически чист (зоны не пересекаются).
- Поверх — **адаптеры по контракту**. Блоки НЕ переписываются: несогласованность гасится адаптерами на швах; потребность переписать блок = дефект резки brief (фиксируется в RETROSPECTIVE, не лечится втихую).
- Ведёт координатор. DoD: интеграционный smoke зелёный **и** собственные тесты всех блоков зелёные в собранной ветке.

## Stop rules

| # | Условие | Действие |
|---|---------|----------|
| S-C1 | Блок 2 дня подряд красный по scoped CI | Пауза блока + `team-<block>/BLOCKER.md`; координатор **сужает блок** или двигает дедлайн интеграции (форфейта нет — блок обязателен) |
| S-C2 | EXPECTATIONS несводимы на Interface Consilium (конфликт моделей данных) | Эскалация владельцу, два варианта: пере-резка brief (дорого, признать честно) ИЛИ назначение **ведущей модели** одного блока с адаптацией двух других. Решение владельца, не команд |
| S-C3 | Нарушение изоляции | Блок «скомпрометирован» → первый на разборе Phase 3 + строка в RETROSPECTIVE |
| S-C4 | Brief change после Phase 1 | Только `BRIEF_AMENDMENT.md` + LGTM владельца и координатора |

## Артефакты и шаблоны

Каталог: `docs/cowork-sprint/<sprint-id>/`.

### Brief (`COWORK_SPRINT_BRIEF.md`)

```markdown
# Cowork Sprint Brief: <title>

| Поле | Значение |
|------|----------|
| sprintId | cowork-<slug> |
| GitHub Issue | #… |
| baseBranch | main |
| blocks | <slug-a>, <slug-b>, <slug-c> |
| integration deadline | <дата-fallback> |
| LGTM Product / координатор | … |

## Problem (общая разработка)
…

## Резка на блоки
| Блок | Суть | Файловая зона (пакеты/каталоги) | Собственный DoD (без соседей, на стабах) |
|------|------|--------------------------------|------------------------------------------|
<!-- зоны НЕ пересекаются; общие корневые файлы не входят ни в одну зону -->

## Constraints / Out of scope
…

## Интеграционный smoke (набросок — уточняется контрактом Phase 3)
…
```

### EXPECTATIONS.md (per block, односторонний)

```markdown
# Expectations — Block <slug>

## Что мне нужно от соседей
| От блока | Что | Форма (сигнатура/схема) | Инварианты (единицы, частоты, порядок, владелец времени) |

## Что я готов отдать
| Блоку | Что | Форма | Инварианты |

## Мои стабы (исполняемые, в моей зоне)
| Стаб | Замещает | Где живёт |
```

Пишется в Phase 1, **дописывается** по ходу Phase 2 (живой документ), запечатывается freeze-тегом в точке интеграции. Чужие EXPECTATIONS не читать до Phase 3.

### RETROSPECTIVE.md — обязательная строка

«Что пришлось адаптировать vs что пришлось переписать» — метрика качества резки brief; по ней калибруются следующие коворки.

## Реестр и команды

```json
{
  "id": "cowork-<slug>",
  "title": "Cowork Sprint: …",
  "promptPath": "docs/cowork-sprint/<sprint-id>/COWORK_SPRINT_BRIEF.md",
  "size": "L",
  "status": "active",
  "sprintKind": "cowork-sprint",
  "notes": "3 блока <slug-a>/<slug-b>/<slug-c>; изоляция до Interface Consilium; мёрж всех трёх"
}
```

| Команда | Действие |
|---------|----------|
| `yarn cowork:open --id <sprint-id> --blocks a,b,c` | Проверка brief, BASE_SHA, команды создания веток, ACTIVE-гард |
| `cowork:phase` / `cowork:close` | Follow-up; регламент валиден без них (прецедент comp) |

## Мастер-промпт агента блока

```markdown
# Cowork Sprint — agent instructions

Ты команда **Block <SLUG>** в Cowork Sprint `<SPRINT_ID>`.
Регламент: `docs/COWORK_SPRINT_REGULATION.md` (прочитай полностью).
Brief: `docs/cowork-sprint/<SPRINT_ID>/COWORK_SPRINT_BRIEF.md`.
Ветка: ТОЛЬКО `cowork/<SPRINT_ID>/<SLUG>`, свой worktree (membrana-worktree).

## Hard rules
- Твоя файловая зона — из brief; за её пределы не писать. Общие корневые файлы (package.json, registry.json) НЕ трогать.
- НЕ читать чужие ветки и чужие EXPECTATIONS.md; НЕ договариваться об интерфейсах до Phase 3.
- Об интерфейсах думать ОДНОСТОРОННЕ: веди team-<SLUG>/EXPECTATIONS.md и исполняемые стабы соседей в своей зоне.
- Собственный DoD блока обязан проходить на стабах, без кода соседей.

## Текущая фаза: <PHASE>   <!-- 1 | 2 | 3-consilium | 4-integration(координатор) -->

### PHASE = 1
CONCEPT.md (свой блок изнутри) + первый EXPECTATIONS.md. Commit: `cowork(<SPRINT_ID>/<SLUG>): phase1 concept`.

### PHASE = 2
Isolated build до собственного DoD; scoped CI зелёный; EXPECTATIONS дописывай по ходу.
Готовность объявляй координатору: «ready» = DoD зелёный. Tag: `cowork-<SPRINT_ID>-<SLUG>-interface-freeze` (ставит координатор).

### PHASE = 3-consilium
Кода не писать. Вскрытие EXPECTATIONS, выработка INTERFACE_CONTRACT.md.

### PHASE = 4-integration (только координатор)
Ветка integration: мёрж блоков + адаптеры по контракту; блоки не переписывать; стабы не мёржить.

## Формат ответа
[Block <SLUG> · Phase <PHASE>]: Сделано / Артефакты / CI / EXPECTATIONS-дельта / Следующий шаг / Blocker
```
