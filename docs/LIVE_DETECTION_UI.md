## 1. Сглаживание live-статуса (anti-flicker)

### Проблема

Сырой выход `HarmonicDetector.detect()` / `classifySpectrum()` на границе порога **мигает**: `isDrone` и `confidence` меняются каждый кадр (~10–15 Hz). Пользователь видит «дёрганье» без реального изменения сцены.

### Правило

**В UI никогда не показывать сырой `isDrone` напрямую.** Между классификатором и React-состоянием — слой сглаживания (хук или утилита).

### Нормативный алгоритм (три механизма)

| Механизм | Назначение | Рекомендуемые значения v0.1 |
|----------|------------|-----------------------------|
| **EMA confidence** | Плавная полоса / проценты | `emaAlpha = 0.22` (0..1; выше = быстрее) |
| **Гистерезис** | Разные пороги вкл/выкл | вкл: `threshold + 0.06`; выкл: `threshold − 0.10` |
| **Подтверждение по кадрам** | Debounce смены «дрон / не дрон» | вкл: **3** кадра подряд выше on-порога; выкл: **6** кадров ниже off-порога |

При пороге UI **55%**: включение «дрон» ≈ от **61%** smoothed/raw (с учётом streak), выключение ≈ ниже **45%**.

**Асимметрия:** выключать статус **медленнее**, чем включать (меньше ложных «потерял дрон» на шуме).

### API слоя сглаживания

```ts
interface SmoothedDetection {
  stableIsDrone: boolean;      // → главный индикатор, aria-live
  displayConfidence: number; // → progress / %
  displayReasoning?: string; // обновлять при смене stableIsDrone или реже
  rawConfidence: number;     // опционально, debug / tooltip
}
```

Классификатор в `packages/services/*` **не менять** ради UI; сглаживание — в `hooks/` demo или плагина клиента.

### Визуальные переходы

| Элемент | Длительность | Примечание |
|---------|--------------|------------|
| Смена статуса «дрон / не дрон» | **400–500 ms** | `transition-all` / `ease-in-out` |
| Полоса confidence | **400–500 ms** | `transition-[width,background-color]` |
| Reasoning | без collapse | см. §2 |

Не использовать `framer-motion` / `motion.*` в JSX.

### Клиент Membrana

- Плагин микрофона, header sensor (`droneDetectionHub`): те же пороги и streak, публикация в hub только после `stableIsDrone === true`.
- Слайдер порога в UI меняет `confidenceThreshold` детектора **и** on/off уровни гистерезиса относительно него.

### Demo-приложения

- Вынести логику в `demo/hooks/detection-smooth.ts` (или shared util при появлении второго demo).
- Unit-тесты на streak и EMA — в том же пакете.

## 2. Вёрстка demo и компактных live-панелей

### Проблема

- Выпадающие блоки (`<details>`, `collapse`) для reasoning/гармоник **излишни** в lab/demo.
- Контент то помещается на экран, то нет → **скроллбар появляется и исчезает**, сдвигая layout.

### Правило: одна высота viewport

```text
html.h-full → body.h-full.overflow-hidden → #root.h-full → flex-колонка на 100%
```

| Слой | Классы / CSS |
|------|----------------|
| `html` | `scrollbar-gutter: stable` (в `index.css` demo) |
| `body` | `h-full overflow-hidden` |
| Корень React | `h-full` + padding в одном месте |
| Карточка | `flex-1 min-h-0 flex flex-col` |

**Не** использовать `min-h-screen` на внутренних блоках, если цель — уместить всё без прокрутки.

### Правило: статические строки вместо collapse

Для вторичной информации детектора — **две фиксированные строки** (всегда в DOM):

```text
Гармоники: 120 Гц, 240 Гц   (или «—»)
Пояснение: <текст>           (или «—», line-clamp-2)
```

Запрещено в demo: `collapse`, `details`, выпадающие меню для reasoning/fundamentals.

### Стабильность высоты блоков

| Блок | Минимальная высота / поведение |
|------|--------------------------------|
| Главный статус (alert) | `min-h-[4.5rem]` — одинаково для «слушаем», «дрон», «не дрон» |
| Строка latency | всегда видна (`—` если нет данных) |
| Ошибка | показывать в зарезервированной зоне; не раздувать карточку без нужды |

### Компактность

- Заголовок demo: 1–2 строки, без длинного кода пакета в hero.
- Футер: **одна** короткая строка с техникой (FFT, сглаживание).
- Отступы карточки: `gap-4`, padding `p-5`.

## 3b. Живой перф-замер yamnet в combined (#416, Задача B 2026-07-14)

Замер DEV-лога `[drift-perf]` (`neuralTtlDetector`, только реальные инференсы)
на живом каденсе combined-плагина: реальный микрофон, dev-клиент (vite, Chrome,
Windows), вкладка на переднем плане, windowSec 2 с, опрос нейро раз в 6 с.

| Метрика | Значение | Комментарий |
|---------|----------|-------------|
| n | 44 | ~4.5 мин живой сессии |
| p50 | **150 мс** | steady-state инференс |
| p95 | **213 мс** | без учёта холодного старта не повторявшихся выбросов нет |
| min | 119 мс | |
| max (холодный старт) | 5020 мс | первый инференс: загрузка весов + компиляция шейдеров TF.js; за сессию не повторился |

**Вердикт (консилиум live-neural-combined-fusion, точка 3/6):** p95 213 мс
формально выше бюджета §2 «p95 < 100 мс», но **непопадания в живой каденс нет** —
инференс занимает 3.5% интервала опроса (6000 мс); TTL-субдискретизация уже
штатно изолирует нейро-стоимость от DSP-ветви (та отвечает каждое окно 2 с),
тревога не запаздывает. Follow-up на снижение `pollIntervalMs` **не требуется**.
Практическое следствие: холодный старт ~5 с — это время до первого нейро-вклада
в fusion после запуска; UI-метка модальностей честно показывает DSP-only,
пока нейро греется. Оффлайн-бенчмарк для сравнения: p50 97 / p95 132 мс
(node, `DETECTOR_BENCHMARK.md`) — браузерный dev-режим ~1.5× дороже, ожидаемо.
