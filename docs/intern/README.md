# docs/intern — онбординг второго разработчика в `background-office`

Каталог собирает документы одного направления: мягкое подключение
второго разработчика (парт-тайм) к сервису `background-office` под присмотром
куратора, с малым blast-radius по построению.

## Контекст

Стажёр пробует силы на реальном сервисе (`background-office`), но риск ограничен
конструкцией, а не доверием: скоупленный доступ к GitHub (write в репозиторий, но
`main` защищён ruleset'ом; работа только в фиче-ветках через PR), **свои**
капнутые ключи (это же и прозрачность по его запросам), ревью куратором на каждую
PR. Отложенные вещи остаются отложенными.

Стартовый набор — три задачи вертикалью, каждая сдаётся отдельной PR:

1. **Outbound self-check** — проверка внешних каналов.
2. **`/health` и `/ready`** — выставить готовность наружу.
3. **Ресёрч-дайджест** («утренняя пресса на стол») — первая реальная джоба по
   проверенному каналу, через Perplexity Sonar.

Порядок не случаен: задача 3 опирается на канал, проверенный задачей 1.

## С чего начать (по роли)

- **Куратор** → playbook, затем Phase 0 runbook.
- **Стажёр** → `INTERN_ONBOARDING_BACKGROUND_OFFICE.md`.
- **Агент** → сперва этот README (особенно «Границы» ниже), затем
  `AGENT_PROMPT_PHASE0_SCAFFOLD.md`.

## Границы (важно для агента)

Эти действия — **только за человеком**, агент их не выполняет автономно:

- настройки доступа и безопасности: branch protection, rulesets, права
  коллабораторов;
- ключи и аккаунты: Anthropic workspace и spend-лимит, Perplexity-ключ, любой
  реальный `.env` и любые секреты;
- проектные решения: выбор темы дайджеста, имя `yarn`-команды.

Агенту делегируется обратимая файловая часть (CODEOWNERS, `.env.example`,
каталог `docs/research/`, заведение issue, механическая проверка запуска без
секретов) — и всё это приходит как PR на ревью куратора, никогда напрямую в
`main`.

## Отложенное (не сейчас)

- Расписание и хостинг дайджеста — фаза два, когда джоба обкатана.
- Мультимодельность (OpenRouter) — отдельный апгрейд, когда понадобится
  сравнивать модели.
- Сборка Linear-агентов — после прохождения single-node шлюза детекции.
