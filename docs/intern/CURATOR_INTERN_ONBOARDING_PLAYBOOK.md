# Куратору: онбординг стажёра в `background-office` — playbook по фазам

**Дата:** 27 июня 2026 · **Версия:** 0.1 · **Статус:** рабочий
**Пара к:** `INTERN_ONBOARDING_BACKGROUND_OFFICE.md` (входящий документ стажёра)

> Твоя последовательность шагов как куратора. Фазы идут по порядку; каждая
> заканчивается **воротами** — условием, без которого не переходишь дальше.

## Принципы куратора

- **Малый blast radius по построению**, а не по доверию: доступ скоупленный,
  ключи у стажёра свои и капнутые, ревью на каждую PR.
- **Чёткий DoD вместо надзора.** Проверяешь результат против критерия
  готовности, а не «как он это делал».
- **Прозрачность держится на кредитале.** Смотришь usage его workspace, а не
  «честное слово». Доверие при этом остаётся доверием.
- **Отложенное остаётся отложенным.** Расписание, хостинг, OpenRouter,
  Linear-сборка — не пускаешь их в стартовый набор.

---

## Фаза 0. Подготовка (до прихода стажёра)

Цель: чтобы стажёр в первый день поднял сервис и начал, ничего не выпрашивая.

- [ ] **GitHub — защита.** Branch protection на `main` (PR обязателен, ревью
      обязательно); `CODEOWNERS` на чувствительные пути.
- [ ] **GitHub — доступ.** Дать стажёру fork либо outside-collaborator на
      ограниченный набор. **Write в `main` не давать.**
- [ ] **Anthropic Console.** Завести отдельный workspace под стажёра, выпустить
      workspace-ключ, поставить **месячный spend-лимит**.
- [ ] **Perplexity.** Обеспечить ему **свой** Sonar-ключ (не дольку твоего).
- [ ] **`.env.example`** актуален: `ANTHROPIC_API_KEY`, `PERPLEXITY_API_KEY`,
      `GITHUB_TOKEN`, порты. Сервис без обязательных env должен падать внятно.
- [ ] **Проверить инструкцию на себе.** Свежий клон → `yarn office:dev`
      поднимается по шагам входящего документа. Инструкция не должна врать.
- [ ] **Завести 3 issue** (по одной на задачу) с DoD из входящего документа.
- [ ] **Заполнить заглушки** входящего документа: тема дайджеста, имя
      `yarn`-команды джобы, путь к репозиторию и `.env`-примеру.

**Ворота:** всё выше отмечено → отдаёшь входящий документ.

## Фаза 1. Передача и первый контакт

Цель: убедиться, что петля «склонировал → поднял → PR» реально работает,
на безопасном материале.

- [ ] Отдал входящий документ + ссылку на репозиторий.
- [ ] Стажёр поднял `background-office` локально, dev стартует.
- [ ] **Первая микро-PR** (правка/дополнение onboarding-дока или раздела про
      локальный запуск) — проверка fork/branch → PR.
- [ ] Сверился: ключи у него **свои**, лежат в `.env`, не в репозитории.

**Ворота:** локальный запуск собрался + первая PR прошла ревью → к задачам.

## Фаза 2. Задача 1 — Outbound self-check

Ревью против DoD:

- [ ] Сводка по 4 эндпоинтам (`api.anthropic.com`, Linear, GitHub,
      `api.perplexity.ai`) читаема.
- [ ] Недостижимый эндпоинт **помечается**, не роняет проверку (таймаут).
- [ ] Не-цели соблюдены: нет ретраев-как-фичи, нет алертов, нет расписания.
- [ ] Запустил у себя — сходится ли с его выводом.

**Ворота:** PR смержена → задача 2.

## Фаза 3. Задача 2 — `/health` и `/ready`

- [ ] `/health` отвечает быстро, наружу не ходит.
- [ ] `/ready` отражает реальное состояние зависимостей; переиспользует логику
      из задачи 1.
- [ ] Не-цели: нет auth, нет стека метрик и дашбордов.
- [ ] Поведение кратко задокументировано.

**Ворота:** PR смержена → задача 3.

## Фаза 4. Задача 3 — Ресёрч-дайджест («утренняя пресса»)

- [ ] Команда создаёт файл по пути `docs/research/digest-YYYY-MM-DD.md`.
- [ ] ≤ 7 пунктов, по убыванию важности, 1–2 предложения на пункт.
- [ ] У каждого пункта **источник + дата**; неисточниковое исключено.
- [ ] Идемпотентность + `--dry-run` работают.
- [ ] **Регистр секретарский**, не инженерный; редактура сделана (не свалка
      из двадцати результатов).
- [ ] Тема — заданная в конфиге, не «новости вообще».

**Ворота:** PR смержена → первый прогон цикла завершён.

## Фаза 5. Стабильный режим и что дальше

- [ ] Раз в период — **Settings → Usage / Cost** по его workspace: расход в
      норме, spend-лимит не пробит. Это твоя постоянная прозрачность.
- [ ] Ревью-ритм налажен, обратная связь по каждой PR.
- [ ] **Не пускаешь отложенное:** расписание и хостинг дайджеста,
      мультимодельность (OpenRouter), Linear-agent-сборка — всё по своим
      триггерам, не сейчас.
- [ ] Если дайджест оказался полезен → фаза два: расписание + бокс, но только
      **после** outbound-теста (см. решение по хостингу).

### Быстрая карта ответственности

| Что                                     | Кто                  |
| --------------------------------------- | -------------------- |
| Доступ, ключи, лимиты, защита `main`    | куратор (Фаза 0)     |
| Поднять сервис локально, писать код, PR | стажёр               |
| Ревью против DoD, контроль не-целей     | куратор              |
| Мониторинг расхода (прозрачность)       | куратор (Фаза 5)     |
| Решение «пора на расписание/бокс»       | куратор, по триггеру |
