# Промпт для агента: скаффолдинг Фазы 0 (онбординг стажёра)

> Скопируй в агента с доступом к GitHub и файловой системе репозитория
> `officefish/Membrana`. Перед запуском подставь свой GitHub-username вместо
> `@OWNER`.

---

Ты работаешь в репозитории `officefish/Membrana` через файловую систему и
GitHub. Задача — подготовить скаффолдинг для онбординга нового разработчика.
**Работай в отдельной ветке и открой Pull Request. Не сливай его и не пушь в
`main`.**

## Что сделать

1. **`.github/CODEOWNERS`** — создай файл с содержимым:

   ```
   *                              @OWNER
   /.github/                      @OWNER
   /packages/background-office/   @OWNER
   ```

2. **`.env.example`** — создай/обнови в корне (только имена переменных, без
   значений):

   ```dotenv
   PORT=3000
   ANTHROPIC_API_KEY=
   PERPLEXITY_API_KEY=
   GITHUB_TOKEN=
   # LINEAR_API_KEY=
   ```

   Убедись, что реальный `.env` присутствует в `.gitignore`. Если нет — добавь.

3. **Каталог `docs/research/`** — создай, если его нет (для артефактов
   дайджеста). Положи `.gitkeep`.

4. **Три GitHub-issue** — заведи через `gh`, с этими заголовками и телами:
   - **Outbound self-check** — Диагностика: пинг `api.anthropic.com`, Linear,
     GitHub, `api.perplexity.ai`; сводка со статусом и латентностью.
     DoD: читаемая сводка по 4 эндпоинтам; недостижимый помечается, не роняет
     проверку (таймаут). Не-цели: не чинить связность, без ретраев-как-фичи,
     без алертов, без расписания.
   - **`/health` и `/ready`** — `/health` — живость (наружу не ходит);
     `/ready` — готовность зависимостей, переиспользует логику self-check.
     DoD: оба отвечают; `/ready` отражает реальное состояние; задокументировано.
     Не-цели: без auth, без стека метрик/дашбордов.
   - **Ресёрч-дайджест** — заметка по одной теме из конфига, Perplexity Sonar,
     артефакт `docs/research/digest-YYYY-MM-DD.md`. DoD: ≤ 7 пунктов по убыванию
     важности, у каждого источник+дата; идемпотентность; `--dry-run`;
     секретарский регистр. Не-цели: без расписания, мультимодельности, записи
     в Linear/GitHub, действий.

5. **Проверка локального запуска (без секретов).** В рабочей копии выполни:
   `corepack enable && corepack prepare yarn@4.5.0 --activate`, `yarn install`,
   затем `yarn office:dev` с **пустым** `.env`. Зафиксируй: падает ли сервис
   внятно при отсутствии обязательных переменных, или стартует. Результат
   опиши в PR. **Не подставляй никакие реальные ключи.**

6. **Открой PR** с изменениями из пунктов 1–3 и кратким описанием результата
   проверки (п. 5) и списком заведённых issue (п. 4).

## Чего НЕ делать (жёсткие границы)

- **Не** создавай, не меняй и не удаляй branch protection, rulesets и любые
  настройки репозитория или безопасности. Если считаешь, что правило нужно —
  подготовь `gh`-команду текстом в описании PR, но **не выполняй** её.
- **Не** добавляй и не меняй коллабораторов и права доступа.
- **Не** создавай, не запрашивай, не вставляй и не трогай никакие API-ключи,
  токены и секреты. **Не** создавай и не редактируй реальный `.env`.
- **Не** создавай аккаунты, workspace, ключи или лимиты в Anthropic/Perplexity.
- **Не** пушь в `main` и не сливай PR; не обходи защиту веток.
- **Не** придумывай тему дайджеста и имя `yarn`-команды джобы — оставь явный
  `TODO` и вынеси в список «требует решения человека».

## Что вернуть в конце

1. Ссылку на открытый PR.
2. Список выполненного (файлы, issue).
3. Список того, что ты намеренно **не** делал, потому что это за человеком
   (ruleset, права, ключи, выбор темы), — чтобы я закрыл это сам.
