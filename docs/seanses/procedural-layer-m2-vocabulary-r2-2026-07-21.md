Мой первый тезис: словарь должен быть **полуоткрытым** — закрытое ядро из четырёх категорий владельца плюс объявленная процедура расширения через ADR. Чисто закрытый застынет и заставит контрабандой протаскивать пятую категорию мимо замка.

[Музыкант]: По касательной — аудио тут не задет, но структуру я слышу как набор инструментов в партитуре. Закрытый словарь — это фиксированный состав ансамбля: четыре инструмента, пятый не заходит без переписывания партитуры. Полуоткрытый — это дирижёр, который может ввести новый голос, но только через объявленную ремарку в партитуре, не импровизацией на сцене. Поддержу Teamlead: чистая закрытость меня пугала бы, я на `ritual-inputs-echo` видел, как жёсткий список молча обрастает синонимами сбоку. Но процедура расширения обязана быть **дорогой** — иначе это открытый словарь под маской.

[Верстальщик]: С точки зрения потребителя словаря — а потребитель это агент, читающий процедуру, и человек в PR — словарь обязан быть **видимым в одном месте**, а не рассыпанным по коду слоя. Моя больная тема из `main-day-accuracy`: если категория используется, но нигде не объявлена как термин с определением, читатель видит мёртвую дверь — слово есть, статьи за ним нет. Требую: каждая категория — это статья с полями «имя, определение одним предложением, чем проверяется принадлежность». Без определения категория декоративна.

[Математик]: Формализую, иначе «полуоткрытый» останется вкусовым. Пусть словарь — конечное множество категорий `V = {команда, задача, ответственность, цель}`, `|V|=4`. Механизм проверки — это предикат `declared(term) → bool` над каждым термином, встречающимся в процедурном слое. Инвариант замка: `∀ term ∈ usedTerms(layer): declared(term) = true`. Нарушение — это `term ∈ usedTerms ∧ ¬declared(term)`, ловится грепом по носителю. «Полуоткрытость» строго определяется так: множество `V` расширяется только транзакцией с внешним гейтом (ADR + LGTM владельца), между транзакциями `V` константно. Это не вкус — это версионируемое множество с контролируемым переходом.

[Структурщик]: Начну с термина, иначе DAG поплывёт. «Словарь категорий» — это ровно **лемма-реестр**: один носитель, где каждая категория — словарная статья с однозначным смыслом в контексте слоя. Тут прямая аналогия с моим ремеслом: публичный API пакета — это `index.ts`, и категория, не экспортированная через него, для потребителя не существует. Носитель обязан быть **единственным** — два места объявления породят синонимы-слои, ровно то, против чего я всегда. Предлагаю: файл `docs/meeting/procedural-layer/VOCABULARY.md` (дом слоя из M1) как единственный `index.ts` словаря.

[Teamlead]: Принимаю носитель Структурщика, но уточню границу по «форме следует функции»: `VOCABULARY.md` — это человекочитаемая словарная статья, а гейт Математика требует **машиночитаемого** множества для предиката `declared`. Это две проекции одной леммы, и я не хочу два источника правды. Решение: канон — машиночитаемый (JSON или фронт-маттер), из него генерируется человекочитаемая часть, либо наоборот — один источник, вторая проекция производная. Кто источник — отдаю Структурщику и Математику согласовать, но правило жёсткое: **один источник, вторая форма выводится**.

[Музыкант]: Тут вклинюсь с флагом из опыта — на `dads-integration-m1-preset-delivery` мы ровно обожглись на двух носителях одного пресета: TS и JSON расходились молча. [Верстальщик]: Согласен про один источник, но добавлю про **видимость нарушения**. Предикат `declared` Математика ловит контрабанду грепом — хорошо, но где потребитель это увидит? Требую: отчёт проверки показывает не просто «нарушений N», а список `term → где встречен → почему не объявлен`. Молчаливое «нарушений: 0» с exit 0 меня уже обжигало — читается как чистота, которой нет. Отчёт с адресом каждой недекларированной категории — часть конструкции, не опция.

[Математик]: Поддержу Верстальщика и уточню численно. Проверка `declared` — чистая функция `check(layerText, vocabulary) → {violations: [{term, location, reason}]}`, детерминированная: один вход → один выход, без `Date.now()`, без сети. Но есть подводный камень — как выделить `usedTerms(layer)` из текста? Если тупой греп по подстроке, получим ложные срабатывания на «задача» в комментарии не по теме. Значит нужен явный синтаксис ссылки на категорию в процедурном слое — например маркер `@cat:задача`, тогда `usedTerms` — это множество маркеров, а не всех слов. Без этого предикат либо шумит, либо молчит.

[Структурщик]: Верно, и это ложится на мою норму «импорт — ссылка на статью, а не на случайный абзац». Маркер `@cat:<имя>` — это и есть явный импорт категории; слово «задача» в прозе — не импорт, оно термином не является, пока не помечено. Так граница словаря = граница явных ссылок, ровно как граница пакета = граница экспортов через `index.ts`. И тогда механизм расширения строгий: новая категория входит в `V` только парой (статья в источнике + первое употребление через `@cat:`), без статьи маркер — нарушение, без маркера статья — мёртвая (объявлена, не используется), тоже флаг.

Это не «открытый под маской», потому что процедура расширения дорогая и внешняя: без моего LGTM и записи в дом слоя множество не меняется. Возражения по статусу есть? [Математик]: Выдерживает. Полуоткрытость формально — это `V` как версионируемая константа: между ADR-транзакциями `V` неизменно, предикат `declared` работает над фиксированным множеством. Без рода читатель примет «ответственность» за такую же сущность, как «задача», и построит кривую процедуру. Это ровно моя иерархия через различие, не через украшение.

**Definition of Done:**
- В доме слоя (M1) создан единственный машиночитаемый источник словаря с четырьмя статьями, каждая с пятью полями (включая `род`).
- `VOCABULARY.md` генерируется из источника скриптом; ручная прав提ка производной запрещена (проверяется CI или пометкой «generated»).
- Реализован чистый предикат `check(layerText, vocabulary)` — детерминированный, без сети/времени; unit-тесты на ложное срабатывание (слово «задача» в прозе без маркера) и на пропуск (маркер `@cat:` без статьи).
- Синтаксис маркера `@cat:<имя>` задокументирован в доме слоя.
- Отчёт проверки выдаёт список `{term, location, reason}`; «0 нарушений» сопровождается пустым списком явно, exit-код честен.
- Процедура расширения `V` (ADR + LGTM) записана; diff множества `V` версионируется в git.
