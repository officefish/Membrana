[Математик]: Начинаю с живого дефекта журнала: 4 прогона подряд LLM роняет слоты, потому
что единый промпт просит модель выдать ВЕСЬ документ — структура и текст в одних руках.
По M2-K код владеет `id/order/title`. Формализую: `frame(): Slot[]` — 5 секций кодом;
LLM зовётся ТОЛЬКО per-slot: `fill(slot, ctx): {text, status}`; `assemble(slots, fills):
string` конкатенирует заголовки из frame + тела из fill. Модель не владеет ни множеством
секций, ни порядком, ни заголовками — ей физически нечего ронять.

[Структурщик]: Развожу леммы, чтобы сборка не слиплась: `frame` (структура) · `fill`
(наполнение) · `assemble` (склейка) · `sign` (подпись) · `gate` (страж) — пять разных
операций, пять чистых функций. Единственная точка недетерминизма — `fill`. Всё остальное
детерминировано и тестируемо без LLM. Единый промпт-скелет объявляем запрещённым.

[Teamlead]: Рамку держу: гейт owner-choice (топ-3) — предмет G, не наш; хозяйка — H.
Мы решаем только «как документ собирается и подписывается». Старое содержимое промпта
(Метаданные, Посылки probe, DoD, Ссылки) — раскладывается ВНУТРЬ пяти слотов как
контекст соответствующего `fill`, отдельных секций вне frame не остаётся.

[Музыкант]: Флаг тихой дыры: что при отказе одного fill? Требую тотальности: `assemble`
ВСЕГДА отдаёт 5 секций; `fill` возвращает `status ∈ {filled, empty, malformed}`;
пустой/битый слот рендерится видимым маркером «⚠» — документ честно показывает дыру,
а не прячет её и не роняет весь прогон. Ретраить можно один слот, не всю генерацию.

[Верстальщик]: По подаче: источник канона должен быть виден читателю. Бейдж в шапке:
`human` — акцентная «ручная чеканка», `llm` — нейтральный «сгенерирован»; иконка+текст,
не только цвет. В PR дифф канона читается по секциям — сборка per-slot делает дифф
атомарным по слотам, это подарок ревью.

[Математик]: Подпись чеканки (Д4). `sign(doc, author): {digest: sha256(canonicalize(doc)),
author ∈ {llm, human}, signedAt}` — author фиксируется на ВХОДЕ прогона (кто запускал),
не выводится задним числом из содержимого. Страж после этого — чистый предикат:
`structOk(doc) ∧ digestOk(doc)`. `author=human` — НЕ основание для блока: владелец
чеканит легитимно; страж проверяет структуру и целостность, не авторство.

[Структурщик]: Подпись — тем же helper'ом, что генераторы (`provenanceHeader`), одна
точка правды формата. Инструмент для владельца: `yarn canon:sign [--author human]` —
читает документ, канонизирует, подписывает; сегодняшняя ручная хирургия (node -e)
умирает. Место кода: `scripts/` (ops-контур), НЕ пакет `@membrana/*` — это обвязка
ритуала, не продукт.

[Teamlead]: Проверяю против журнала: дефект Д2 (смесь структур) закрывается запретом
единого скелета; Д4 (подпись руками) — командой canon:sign; вещдок-патч
(`MAIN_DAY_ISSUE.rejected.md`) сохраняется как штатная диагностика гейта. Из журнала
остаётся только Д1/Д3 — они в G, границу не нарушаем.

[Музыкант]: Про retry-политику: гейт скелета после этой сборки станет почти
неотказным (структуру собирает код), но оставляем его как страховку от регрессии —
и его отказ ТЕПЕРЬ означает баг сборки, не каприз модели. Красный = чинить код, не
перезапускать до зелёного.

[Верстальщик]: Слот «Список посылок» — прошу включить в frame постоянной секцией канона
дня (probe-посылки живут там) — тогда и наш собственный формат вердиктов, и канон дня
несут посылки конструктивно, а не по памяти.

[Математик]: Принимаю слот посылок в frame. Тесты: (1) `frame()` — ровно 5 слотов,
константные id/заголовки; (2) `output.sections.map(id) === frame().map(id)` — сборка
не теряет и не изобретает секций; (3) тотальность assemble на {filled, empty, malformed};
(4) sign: честный `author=human` с валидной структурой → pass, подмена дайджеста → fail.
Всё без сети и без реального LLM (fill мокается контрактом).

[Структурщик]: Итог по границам: пять функций — `scripts/lib/` рядом с day-plan-frame
(он уже несёт frame/fillInput — расширяем его, не плодим второй модуль). Существующие
`buildTop3`/`fillInput` переиспользуются. Ноль новых пакетов.

[Teamlead]: Возражений не слышу. Проверка эха рамки: «Сборка секций» — решена
(frame+fill+assemble, запрет единого скелета); «Подпись ручной чеканки» — решена
(sign + canon:sign, author=human легитимен). Собираю вердикт с посылками.

[Музыкант]: Принимаю. Маркер «⚠» в пустом слоте — прошу текстом, не только символом
(скринридер), — «слот пуст: наполнение не удалось».

[Верстальщик]: Принимаю; бейдж источника и ⚠-маркеры — в DoD отдельной строкой.

[Математик]: Финальная сверка инвариантов: единственный недетерминизм — fill; сборка,
подпись, страж — чистые; тотальность доказуема тестом. Возражений нет.

[Структурщик]: Термины закреплены, леммы не слипаются. Возражений нет.

## Definition of Done

1. `frame()` — ровно 5 слотов, константные id/заголовки (вкл. «Список посылок»); юнит-тест
   порядка и состава. **Владелец: Математик.**
2. Генератор зовёт `fill(slot)` per-slot; единый промпт-скелет удалён; тест
   `output.sections.map(id) === frame().map(id)`. **Владелец: Математик + Структурщик.**
3. `fill` возвращает `status ∈ {filled, empty, malformed}`; пустой/битый слот → «⚠» с
   текстом; документ всегда несёт 5 секций; ретрай одного слота. **Владелец: Музыкант (форма).**
4. `sign(doc, author)` + команда `yarn canon:sign --author human`; страж блокирует только
   `¬structOk ∨ ¬digestOk`; тесты: честный human → pass, подмена дайджеста → fail.
   **Владелец: Математик.**
5. Бейдж источника (human/llm) в шапке канона; иконка+текст. **Владелец: Верстальщик.**
6. Код — расширение `day-plan-frame.mjs` + `scripts/`; без новых пакетов; `test:scripts`
   зелёный. **Владелец: Структурщик.**
7. LGTM Teamlead с проверкой границы B↔G/H/GC. **Владелец: Teamlead.**
