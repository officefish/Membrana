## Итоговое решение консилиума

| Вопрос | Решение |
|--------|---------|
| Корень ошибки 16.07 | Генератор MAIN_DAY_ISSUE доверял вторичному сигналу (Issue-статус) как состоянию кода; посылки развилки не верифицировались против дерева |
| Первичный источник истины | **Код** (символ/файл/тест) — первичен; GitHub Issue / карточка реестра — вторичны |
| Расхождение код↔Issue | Не шум, а **находка**: `violated` → план обязан подсветить (реестр протух) |
| Механизм | Препроцессор-гейт `probeMainDayState()` — read-only чистая проверка посылок: `{claim, predicate, evidence, verdict}` |
| Где живёт | `scripts/*.mjs` (тонкая обвязка) + чистая функция вердикта в ядре рядом с `evaluateProdMainDivergence` (#413); **новый пакет не создаём** |
| Впайка | Хвост `ritual:day`; **падает** при нарушенной посылке (без `|| true`) |
| Контракт развилки | Ни одна ветка A/B не публикуется без {маркер в коде + связанная карточка/Issue + вердикт} |
| Обход | `--force` разрешён только с явной пометкой в теле MAIN_DAY_ISSUE + записью в резюме дня |
| Формат отчёта | Таблица «посылка → маркер → вердикт» в консоли и в MAIN_DAY_ISSUE; нарушенные сверху, цвет **и** текст; провенанс (SHA/дата) |
| Автоматическая починка | Запрещена — препроцессор только выносит вердикт, закрытие Issue = ручной человек-гейт |

**Definition of Done:**
- Чистая функция `probeAssertion(claim, evidence) → { verdict: holds|violated, evidence }` в проверяемом ядре, покрыта unit-тестами (holds / violated / расхождение код↔Issue) без мока DOM.
- Скрипт `scripts/main-day-probe.mjs` — read-only обвязка: читает маркеры (символ через `grep -l` по `packages/**`, наличие теста, `issue.state`), выводит таблицу вердиктов; ненулевой код выхода при `violated`.
- Впаян хвостом в `yarn ritual:day`; при `violated` генерация магистрали блокируется, кроме `--force` с пометкой в MAIN_DAY_ISSUE.
- Формат MAIN_DAY_ISSUE расширен блоком провенанса: каждая развилка несёт строку-доказательство (маркер + вердикт + SHA/дата снятия посылки); нарушенные развилки помечены текстом «ПОСЫЛКА НАРУШЕНА», не только цветом.
- Демонстрация на кейсе 16.07: препроцессор помечает обе посылки развилки `violated` (`fuseDetectorConfidences` существует; карточки #487 слиты).
- Границы пакетов не нарушены (проверка Структурщика); отчёт соответствует DESIGN.md (проверка Верстальщика); LGTM Teamlead.
