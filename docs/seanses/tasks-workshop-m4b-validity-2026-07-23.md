# Заседание M4B: контракт функции валидности

[Математик]: Начну с чистоты, потому что от неё зависит всё остальное. У нас в проекте есть
идиом, и он работает: `planPrShip`, `checkBaseFreshness`, `classifyHttpProbe` — чистая логика
отделена от исполнения и покрыта тестом. Валидность обязана лечь в тот же ряд, и вот почему это
не эстетика: **внешние системы недоступны чаще, чем кажется**. Anthropic исчерпан до августа,
Linear отвечает только через прокси, ключи живут в `.env` каждого worktree отдельно. Функция,
которая сама лезет в сеть, в такой среде даёт разный ответ на одних и тех же данных — то есть
не проверяется тестом вообще.

Значит два слоя. **Сборщик фактов** — грязный, ходит в GitHub и Linear, отдаёт слепок
`links = { issueState, linearState, promptExists, promptIsStub, insightExists }`. **Предикат** —
чистый: вход `(card, links)`, выход вердикт. Тест кормит предикат готовым слепком и не знает
про сеть.

Отдельно: слепок обязан различать «связь плохая» и «связь неизвестна». Если Linear не ответил,
это не `linearState: bad`, это `unknown`, и вердикт по такому полю обязан быть отдельного
уровня. Иначе ляжет прокси — и весь реестр покраснеет ложно.

[Структурщик]: Границы элемента и группы. Элемент — то, что проверяется по одной карточке и
её слепку: заполненность полей, согласованность статуса с датами, живость связей. Группа — то,
что видно только на всём реестре: уникальность `id`, две активные карточки на одну иссью,
ссылочная целостность `parentEpic`, расхождение производного `README.md` с реестром.

Проверка на дублирование: у группового слоя не должно быть ни одной проверки, которую можно
выполнить, глядя на одну карточку. Если такая появилась — она уехала не в тот слой.

[Teamlead]: Граница с `tasks:audit` — вопрос, ради которого его и задавали, иначе построим
второй такой же. `tasks:audit` отвечает на один вопрос: **что из `active` закрыто на самом
деле**, и он всегда read-only, только предлагает. Это вопрос про **соответствие внешнему миру**.
Валидность отвечает на другой: **цела ли карточка как запись** — заполнена ли, сцеплена ли,
не противоречит ли сама себе.

Отсюда разделение, а не поглощение: `audit` остаётся как есть и продолжает предлагать архивацию;
валидность — новый предикат, который `audit` может **использовать**, но не наоборот. Одна логика,
один носитель.

[Верстальщик]: Про шкалу уровней. Три — рабочий минимум, и каждый должен означать **действие**,
а не громкость. Blocker — работа по карточке не начинается, пока не починено. Warning — работать
можно, но след неполон, и это видно в отчёте. Note — заметка, не требующая действия сегодня.
Если уровень не меняет ничьего поведения, он лишний.

И требование к выводу: вердикт называет **что именно** нарушено и **где**, а не только «карточка
невалидна». Сегодняшний день это доказал: `linearId: null` заметили только потому, что смотрели
глазами.

[Музыкант]: Одна нота, которую легко потерять. Валидность не должна становиться гейтом,
роняющим ритуал. Она — зрение, а не забор. Пусть она даёт красный, но решение «стоп» принимает
тот, кто владеет процессом, а не предикат. Иначе первый же недоступный прокси остановит утро.

[Teamlead]: Согласен и фиксирую это отдельной строкой вердикта. LGTM.

## Итоговое решение консилиума

| Вопрос | Решение |
|--------|---------|
| **Две сигнатуры** | `validateTask(card, links) → { ok, level, findings[] }`; `validateRegistry(cards, links) → { ok, byCard, groupFindings[] }` |
| **Чистота** | Обе функции **чистые**, сети не знают. Внешние факты собирает отдельный грязный слой и передаёт слепком `links = { issueState, linearState, promptExists, promptIsStub, insightExists }` |
| **Граница элемент / группа** | Элемент — всё, что видно по одной карточке и её слепку. Группа — только то, что нельзя проверить, глядя на одну карточку: уникальность `id`, две активные карточки на одну иссью, целостность `parentEpic`, расхождение производного `README.md` с реестром. Проверка, выполнимая на одной карточке, в групповом слое запрещена |
| **Шкала уровней** | Три: **blocker** (работа не начинается), **warning** (работать можно, след неполон), **note** (заметка). Уровень, не меняющий ничьего поведения, не заводится |
| **Неизвестность ≠ нарушение** | Недоступность внешней системы даёт `unknown`, а не «плохо». Вердикт по `unknown` — не выше `warning`; ложно краснеть весь реестр при упавшем прокси запрещено |
| **Граница с `tasks:audit`** | `tasks:audit` отвечает «что из `active` закрыто на самом деле» — соответствие внешнему миру; валидность отвечает «цела ли карточка как запись». Поглощения нет: `audit` может использовать предикат валидности, обратное — нет. Одна логика, один носитель |
| **Форма находки** | Каждая находка называет **что** нарушено и **где** (карточка, поле); «карточка невалидна» без адреса — не находка |
| **Валидность не гейт** | Предикат даёт вердикт, но не останавливает ритуал сам: решение «стоп» принимает владелец процесса. Валидность — зрение, а не забор |

**Definition of Done:**

- [ ] `validateTask` и `validateRegistry` — чистые функции в `scripts/lib/`, без импортов сети и fs.
- [ ] Сборщик слепка `links` — отдельный модуль; недоступность внешней системы даёт `unknown`, не ошибку.
- [ ] Тест кормит предикат готовым слепком: покрыты `unknown`, blocker, warning, note.
- [ ] Ни одна групповая проверка не выполнима на одной карточке (тест-инвариант).
- [ ] `tasks:audit` переведён на общий предикат либо явно оставлен как есть с записанной причиной.
- [ ] Вывод находки содержит адрес (карточка + поле).
