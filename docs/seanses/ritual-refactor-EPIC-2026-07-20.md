# Эпик: Комплексный рефакторинг ритуала — свод заседания

**Заседание:** `ritual-refactor` · 2026-07-20 · формат `membrana-meeting` (M0 + M1..M5).
**Статус:** все вердикты ратифицированы владельцем. Заседание состоялось.

## Объединяющее задание

Ритуал **перепроверяет свежесть и структуру** и не рапортует зелёным по инерции.
Три несущих: **детерминизм каркаса**, **субъект действия** (у каждого документа явный
автор + явный страж), **анти-«молчун»** (ни один шаг не отчитывается успехом, не сделав дело).

Повод — два утра (19–20.07) ритуал тихо врал семейством одного класса: узел/канал не
перепроверял живость источника (стендап ел мёртвый файл, экстрактор доклада разошёлся,
зомби-акцент на отозванных кристаллах, `rag exit 0` при несделанном, план-структура
плавала). Сводный доклад: `ritual-refactoring-meeting-report-2026-07-20.md`.

## Ратифицированный порядок (M0)

**`A → K → {S → R → D}`** — DAG на контрактных рёбрах; `D→K` (сны питают блок) — рантайм-поток,
не цикл. Протокол: [`ritual-refactor-m0-order-2026-07-20.md`](ritual-refactor-m0-order-2026-07-20.md).

## Компоненты (ратифицированы)

### M1 · A — Ангелина-координатор
Детерминированный **оркестратор каскада + вызыватель страж-функций** на каждом ребре.
Свежесть = `version(d_i) ≤ readAt(d_j) ∧ digest`, версия по **`git log -1`** (не mtime),
три исхода `fresh/stale/unknown`; **`unknown ≠ fresh`**; `stale` → громкий стоп.
Провенанс `{author, guard, digest, readAt}` обязателен; **автор ∈ {5 персон, человек}**.
Не редактирует, не голосует. Протокол:
[`ritual-refactor-m1-angelina-2026-07-20.md`](ritual-refactor-m1-angelina-2026-07-20.md).

### M2 · K — 5-блочный каркас плана + топ-3
Стенка `Slot → Text`: детерминированный слой владеет `id/order/title` пяти слотов и
отбором топ-3; LLM — только текст. **5 слотов всегда** (пустой = легальное явное состояние).
Топ-3 — **детерминированный** ранжировщик (не модель), балансировка = ключ сортировки.
Гейт проверяет скелет, не текст. Протокол:
[`ritual-refactor-m2-karkas-2026-07-20.md`](ritual-refactor-m2-karkas-2026-07-20.md).

### M3 · S — стендап Тимлидом + движок задач
Чистая функция `standup(dayIssue, engineSnapshot) → Plan`; автор и единственный писатель —
`vesnin`. Три состояния пункта `назначено/пробел/осиротело`. Стендап **резолвит и
привязывает** существующие `taskRef`, **не создаёт** (Linear-контейнеры — в спринтах, не на
планёрке). `handoff` — ссылки, не копии, только для персон с назначениями. Активация памяти
по `memoryRef=(persona, revision=git log -1)`. `emptyPlan` при непустом Day Issue → warning.
Протокол: [`ritual-refactor-m3-standup-2026-07-20.md`](ritual-refactor-m3-standup-2026-07-20.md).

### M4 · R — доклад наружу
Линза = строгий **rephraser** (`text → text`) при **изоморфизме дерева**:
`structuralHash(report)=structuralHash(plan)`, НЕ redactor. Защита фактов:
`protectedTokens ⊆ tokens(report)` (числа/статусы/ссылки дословно). Чек ссылок:
`internalRef` (бинарно) + `externalRef` (тернарно `alive/dead/unverifiable`).
`canPublish = structuralIntact ∧ protectedTokensKept ∧ noInternalDead ∧ noExternalDead`;
`dead > 0` → не уходит. Ручной гейт «ок» владельца — сверху. Протокол:
[`ritual-refactor-m4-report-2026-07-20.md`](ritual-refactor-m4-report-2026-07-20.md).

### M5 · D — сны v2
Автономный сервис `@membrana/dreams`. **24 тика/сутки → соревнование четвёрками →
6 победителей** (`select: Dream[24] → Winner[6]`, детерминированно, `no-winner` явно).
Автор — субагент **«Мастер снов»** (версия = промпт; поправка к M1 для снов, LGTM владельца).
Failover-**кубик** `rollProvider(seed)` — воспроизводимая биекция 4 провайдеров; терминал
`synthesisFailed`; media-прокси. Стор — **append-only** на office, единственный писатель;
24 лежат все, 6 — derived; проигравшие в актив (контент-мейкер). Server-first office,
почасовой cron. Протокол (secretary, канон OK):
[`ritual-refactor-m5-dreams-2026-07-20.md`](ritual-refactor-m5-dreams-2026-07-20.md).

## Сквозной принцип (проходит все 6)

- **Детерминизм** там, где форма обязана быть стабильной (каркас, `select`, `structuralHash`,
  топ-3-отбор); **LLM** — только суждение (текст, сила темы в заезде).
- **Субъект действия** у каждого документа: автор (`vesnin` для стендапа, «Мастер снов» для
  снов) + страж (Ангелина по формальным признакам).
- **Анти-«молчун»**: `unknown ≠ fresh`, `stale`/`dead`/`no-winner`/`skipped`/`пробел` —
  явные метки, ни одной тихой двери.
- Граница держится: дрейф/граф правды детерминированы; «полезность/сила темы» — LLM.
