## Итоговое решение консилиума

| Вопрос | Решение |
|--------|---------|
| Что делаем | Рефакторинг **верхней зоны** `MicrophoneModule`: новый презентационный компонент `MicrophoneCapturePanel` (device + статус-герой + главная кнопка start/stop + ошибки) |
| Что не делаем | Отдельный пакет `@membrana/microphone-ui`; FFT/спектр в блоке; второй захват микрофона; изменение hub/coordinator API |
| Визуальная концепция (Boyarskiy) | Герой-статус «Слушаем» / «Выключено»; выразительный выбор устройства; одна главная CTA; явные empty/error для разрешений; имя активной дорожки у героя; мягкий live-пульс (CSS 150–200 ms) |
| Техдок в UI | Блок про `publishMicrophoneStream` — в `<details>` закрыт по умолчанию |
| Level-meter | Опционально: EMA + троттл 15 Hz, чистая функция + тест; **не блокер** первого PR |
| Ветка / роли | `boyarskiy`; реализация UI — Rodchenko + Boyarskiy (приёмка слуха/live); Ozhegov — структура файлов; Vesnin — LGTM по форме |
| Документация | Абзац в `MODULE_AND_PLUGIN_UI.md` про эталон capture-панели |

**Definition of Done (задача в коде):**

- [ ] `MicrophoneCapturePanel.tsx` вынесен; `MicrophoneModule` только оркестрирует engine, hub, coordinator, плагины.
- [ ] Статус live/off и ошибки `NotAllowedError` видны без опоры только на цвет (текст + иконка).
- [ ] `select` устройства + refresh; главная кнопка ≥40 px; `focus-visible` на интерактиве.
- [ ] Hub/coordinator тесты зелёные; smoke: старт → harmonic (или fft-threshold) видит тот же поток.
- [ ] Технический prose про hub — в `<details>`, не в лицо пользователю.
- [ ] (Опционально) level-meter с `smoothLevel` + unit-тест.
