# Консилиум: M4-F — форма вечернего аудит-отчёта как топливо агенту

**Повестка:** F1 — какую форму имеет вечерний аудит-отчёт и его генератор, чтобы он был топливом агенту (утренняя ассимиляция), а не отчётом для человека? Форма стоит в стоке DAG: содержание уже ратифицировано (граница B, объём M1′, конституция H1, воспроизводимость DA1, читатель T1).

**Участники:** Teamlead (Vesnin), Структурщик (Ozhegov), Математик (Dynin), Музыкант, Верстальщик (Rodchenko).

**Порядок реплик:** Математик → Музыкант → Структурщик → Teamlead → Верстальщик (циклически)

---

[Математик]: Формализую, иначе форма поплывёт в вёрстку. Отчёт — это множество элементов `report = {header, entries[], footer}`, где `entry = {claim, verdict ∈ {ok, drift, stale}, evidenceRef, scope ∈ {sverka, recheck}, originHash}`. Топливо агенту означает предикат `citable(entry) → bool`: элемент цитируем утром тогда и только тогда, когда у него непустой `originHash` и разрешимый `evidenceRef`. Если хоть один entry нецитируем — это не топливо, а декор. Начинаю с этого инварианта: форма обязана делать каждый элемент машинно-адресуемым, иначе T1 (ассимиляция по hash-цитате) невыполним.

[Музыкант]: Поддержу с аудио-стороны и повторю флаг из `evening-auditor-doc2`: не делайте генератор умным. Он тупой микшер — три фейдера сверху (сверка / перепроверка / вне-границы), вердикт снизу. Вся интеллектуальность у владельцев дорожек: probe, code-review, reaper. Формат должен звучать как трек-лист для машины, а не как рецензия для человека — то есть YAML/JSON-фронтматтер первичен, markdown-тело вторично. Если агент утром парсит прозу — это щелчок на ресемплинге: недетерминированно и хрупко.

[Структурщик]: Начну с термина, иначе носитель поплывёт. «Отчёт» — омоним: (1) **машинный артефакт** — структурированные данные, которые цитирует утренний агент по origin-hash; (2) **человекочитаемый снимок** — markdown для ретроспективы в PR. Это две леммы на одном носителе, и T1 уже сказал: первичный потребитель — агент. Значит несущее — машинный слой: фронтматтер `entries[]` с полями из DA1. Markdown-тело — производная проекция того же данного, генерируется из фронтматтера, а не пишется параллельно. Один источник правды, иначе получим два словаря для одного факта, как 17.07.

[Teamlead]: Рамку фиксирую сразу, чтобы форма не колонизировала уже решённое. Мы НЕ переоткрываем границу, объём, конституцию, читателя — они входы. Наш единственный вопрос: форма генератора и артефакта. По конструкции «форма следует функции»: функция задана T1 — топливо утреннему агенту. Значит несущая балка — машинный фронтматтер (Структурщик прав), markdown — производная проекция (Музыкант прав про тупой микшер). Принимаю оба тезиса как фундамент. Спорим дальше о полях и о том, как отчёт указывает на своё влияние на утро.

[Верстальщик]: С витрины закрою честность дверей, но зафиксирую отличие: даже «для агента» markdown-проекция читается человеком в PR — это вещдок. По моей норме нужен один главный акцент: вердикт дня (ok/drift/stale) — крупно в шапке, три блока ниже с чёткой границей `border-base-300`. И мёртвая дверь: если один из трёх входов отсутствует (probe не гонялся) — блок показывает «нет данных», вердикт = «неполный аудит», не ложное зелёное. Провенанс в шапке обязателен: `registry@sha` + дата построения + `audit@commit`.

[Математик]: Уточню поле влияния на утро, которое T1 передал в эту комнату. Нужен не свободный текст, а связь: `influence = {consumedOriginHash, morningArtifactRef}` — какой вчерашний факт процитирован сегодня утром. Но это поле заполняется НЕ вечером (вечер не знает будущего), а утренним ритуалом обратной записью. Формально: вечерний генератор оставляет слот `consumed: null`, утро его резолвит. Иначе hash-цитата выродится в штамп, о чём предупреждал T1 — согласен, и фиксирую как двухфазную запись.

[Музыкант]: Вот двухфазность — это ровно как overdub: вечер пишет дорожку, утро накладывает вокал. Согласен. Но риск-флаг с аудио-опыта: если утро не резолвит `consumed`, слот висит `null` вечно — это orphan из T1. Нужен честный таймаут: если через N дней `consumed` так и `null`, вердикт цикла = `orphan`, и это видно, не молчит. Не делаем «магическую кнопку», которая делает вид, что всё ассимилировано.

[Структурщик]: Принимаю двухфазность, но зафиксирую носитель обеих фаз, иначе будет два файла. Канон: `docs/DAILY_AUDIT.md` — живой, перезаписывается вечерним ритуалом; архивный снимок `docs/archive/daily-day/<date>/audit.md` — вещдок, неизменяемый. Обратную запись утра ведём в архивный снимок (он датирован), не в живой файл — живой уже перезаписан следующим вечером. Фронтматтер — единый: `{provenance, verdict, entries[], cycle: {consumed, state}}`. Генератор пишет фронтматтер, markdown-тело рендерит из него детерминированной проекцией `render(frontmatter) → md`.

[Teamlead]: Свожу к структуре, проверяю границы. Генератор `audit` производит `derivedReport` (граница B) → сериализует в фронтматтер (DA1-поля) → `render` в markdown. Это чистое ядро (Математик) + тонкая обвязка записи (fs, git-пин — снаружи ядра, как в DA1). Место: `docs/DAILY_AUDIT.md` + архив. Lifecycle: вечерняя цепочка после `truth.mjs cool` и `code-review`, `onBreak=report` — отчёт генерится даже при FAIL инварианта, роняет ритуал только при собственной ошибке сборки. Возражения по exit-семантике?

[Верстальщик]: Возражений нет, добавлю честность неполного входа на витрине. Три блока в фиксированном порядке: сверка (`scope: sverka`), перепроверка (`scope: recheck`), вне-границы (highlight на `registry-reaper`, `disabled`, некликабельная заглушка «27 карточек → другой процесс»). Каждый entry: `claim` крупно, `verdict`-badge (ok=`badge-success`, drift=`badge-warning`, stale=`badge-ghost badge-outline`), `evidenceRef` моноширинно, `originHash` усечённый с `title` на полный. Вердикт дня в шапке — `tabular-nums`, один акцент. Пустой блок = «нет данных», не пропуск.

[Математик]: Закрою exit-семантику формально, чтобы генератор не врал зелёным. `exit(generator) = 0` ⟺ отчёт сериализован и отрендерен (вердикт внутри может быть drift/stale); `exit ≠ 0` ⟺ генератор сам не смог собрать (нечитаемый вход, отсутствующий origin-hash — провал до вычисления по DA1). Вердикт дня — содержимое `frontmatter.verdict`, НЕ exit-код. Смешать = ошибка `issue.state`-прокси из `main-day-issue-accuracy`. Процесс успешен, если честно записал провал.

[Музыкант]: Принимаю exit-разведение всем нутром — это разные клоки, как аудитор (такт дня) и его сборка (мастеринг). Последнее с аудио-стороны: `render(frontmatter) → md` должен быть детерминированным бит-в-бит при фиксированном входе (DA1). Тестируется без сети, без fs — чистая функция от данных к строке. Это несущая балка, не эксперимент; курёхинский риск тут запрещён. Голосую за.

[Структурщик]: Термин чист, носитель един, DAG ацикличен: `audit → frontmatter → render`. Одно уточнение в DoD: тест на равенство `render(parse(render(fm))) === render(fm)` — идемпотентность проекции, гарантия что markdown не расходится с данными. И gate: пустой `originHash` в любом entry — FAIL сборки (exit≠0), а не тихий пропуск. Принимаю форму.

[Teamlead]: Свожу к вердикту. Форма ратифицирована: машинный фронтматтер первичен, markdown — детерминированная проекция, двухфазная запись `consumed` (вечер пишет null, утро резолвит в архивный снимок), тернарный вердикт в шапке, три блока + highlight на reaper, exit-код отделён от вердикта, `onBreak=report`. Всё в русле B/M1′/H1/DA1/T1 — форма следует функции, декора нет. Ставлю LGTM. Возражения?

[Верстальщик]: Принимаю. Витрина честна: один акцент, три блока с границами, мёртвые двери озвучены, провенанс в шапке, orphan-таймаут виден.

## Итоговое решение консилиума

| Вопрос | Решение |
|--------|---------|
| Первичный слой формы | **Машинный фронтматтер** (`{provenance, verdict, entries[], cycle}`); markdown — детерминированная производная проекция `render(frontmatter) → md` |
| Структура entry | `{claim, verdict ∈ {ok, drift, stale}, evidenceRef, scope ∈ {sverka, recheck}, originHash}`; непустой `originHash` обязателен |
| Влияние на утро (T1) | Поле `cycle: {consumed, state}`; двухфазная запись — вечер пишет `consumed: null`, утро резолвит обратной записью в архивный снимок; таймаут → `state: orphan` |
| Носитель | `docs/DAILY_AUDIT.md` (живой, перезаписывается вечером) + `docs/archive/daily-day/<date>/audit.md` (неизменяемый вещдок; сюда пишется обратная запись утра) |
| Вердикт дня | Тернарный (ok/drift/stale) в шапке, один акцент; зелёное только при ok + proven |
| Три блока витрины | сверка (`sverka`) → перепроверка (`recheck`) → вне-границы (highlight на `registry-reaper`, disabled). Пустой вход = «нет данных», вердикт «неполный аудит», не ложное зелёное |
| Exit-семантика | `exit=0` ⟺ отчёт собран (вердикт внутри любой); `exit≠0` ⟺ генератор сам сломался. Вердикт ≠ exit-код |
| Lifecycle | Вечерняя цепочка после `truth.mjs cool` и `code-review`; `onBreak=report` |
| Провенанс | `registry@sha` + дата построения + `audit@commit` в шапке |

**Definition of Done:**
- Чистое ядро: `audit(logs, dayPlan, truthCrystals[]) → derivedReport` без чтения git/fs (пин делает обвязка).
- `render(frontmatter) → md` — чистая функция, детерминизм бит-в-бит; unit-тест без сети/DOM/fs.
- Тест идемпотентности проекции: `render(parse(render(fm))) === render(fm)`.
- Gate: пустой `originHash` в любом entry → `exit ≠ 0` (FAIL сборки), не тихий пропуск.
- Дедупликация по `originHash`, вес согласия = число различных origin (DA1).
- Двухфазный `cycle.consumed`: вечерний генератор пишет `null`; утренний ритуал резолвит в архивный снимок; orphan-таймаут после N дней → `state: orphan`, видимо на витрине.
- Витрина (markdown-проекция): три блока с `border-base-300`, badge вердикта (ok=`badge-success`, drift=`badge-warning`, stale=`badge-ghost badge-outline`), `originHash` усечён с `title` на полный, `tabular-nums` для вердикта дня, `aria-live="polite"` на смену стабильного вердикта.
- Отсутствие прямых импортов между сервисами; генератор-обвязка в `scripts/*.mjs`, ядро — чистый TS.
- Соответствие `DESIGN.md` (палитра, 8px-сетка, WCAG AA) и `ARCHITECTURE.md` (границы пакетов).
