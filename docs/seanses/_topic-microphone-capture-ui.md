# Повестка: выразительный блок захвата микрофона

## Инициатор

**Boyarskiy (Музыкант)** поднимает вопрос: блок запуска микрофона и выбора устройства — не «служебная форма», а **главная сцена входа** в приложение Membrana. Пользователь здесь решает, слышит ли система мир; от этого зависят все плагины (детекторы, FFT, качество звука).

## Текущее состояние (код)

Файл: `apps/client/src/modules/microphone/MicrophoneModule.tsx`

- Карточка `card` с `badge` «Поток активен / остановлен» в правом верхнем углу.
- `select` «Источник звука» + подсказка про имена устройств после первого запуска.
- Кнопки: «Запустить поток» / «Остановить поток» (`btn-primary` / `btn-error`), «Обновить список устройств».
- Технический блок про `publishMicrophoneStream` / hub (для разработчика).
- Архитектура корректна: `audio-engine-service`, coordinator, hub — **не ломать**.

## Ограничения

- Не переносить DSP/FFT в UI блока микрофона.
- Не дублировать захват в плагинах — hub остаётся единственной точкой публикации потока.
