[Структурщик]: Начну с главного развода терминов: «затереть» ≠ «удалить». История git и
вердикты-провенанс неприкосновенны — они объясняют, почему мир такой. Затирается
**читаемость мёртвого пути живой сессией**. Предлагаю физический носитель: каталог
`void/` — отвергнутые документы и их производные переезжают туда с YAML-шапкой
(status, verdict, reason, дата). Один источник правды: без отдельного graveyard-реестра —
поля `rejectedReason`/`rejectedBy` живут в существующем registry.json.

[Teamlead]: Рамка ответственности, иначе GC станет опасен: **GC не решает сам, что
сценарий отвергнут**. Пометка rejected — решение с ответственным (закрытый вердикт
заседания/владельца). Машине остаётся детерминированный перенос уже приговорённого.
Граница: решение — человек/вердикт, исполнение — машина.

[Математик]: Формализую исполнение: `isDead(s) = status = rejected ∧ verdict_closed` —
чистый предикат; дата берётся из вердикта, не из `Date.now()`. Один детерминированный
проход `gc.mjs` в вечернем ритуале, после закрытия вердиктов дня. Производные (черновики,
промежуточные прогоны) следуют за родителем одной операцией — сирот не остаётся.

[Верстальщик]: По подаче: `void/` должен читаться как **кладбище с эпитафиями**, а не
свалка: каждая шапка — кто приговорил, за что, когда; свежие — `badge-error`, истёкшие —
`badge-ghost`. Человек-гейт — diff в PR: перенос виден построчно, автоудаления нет.

[Музыкант]: Флаг возрождения — главный риск класса: затёртое имеет привычку возвращаться
(шаг вернулся в цепочку, скрипт снова позвали). Требую **три независимых барьера**:
(1) YAML-шапка `status: rejected` в самом файле — читающий видит первой строкой;
(2) запись в индексе каталога — «путь ведёт в void»; (3) grep-инвариант CI: ни одной
активной ссылки на `void/*` извне. Один барьер обходится случайно, три — только намеренно.

[Структурщик]: Дополню анти-возрождение семантикой веса: `recent_void_penalty` — генераторы
и планировщики читают `void/` и штрафуют свежеотвергнутое, чтобы идея не переоткрылась
под новым именем через неделю. Свежесть — по `rejectedAt` из шапки.

[Teamlead]: По реестру следов: стартовый список — рукописный (известные экспонаты:
барьерный раннер, старый генератор плана и читатели, retired-sources, следы старого
ночного канала), но пополнение — добываемое: grep-инвентаризация по предмету перед
каждым новым контуром (тезис питомца из шторма — беру его сюда как норму GC-класса).
Инвентаризация до проектирования = профилактика новых альтернатив.

[Математик]: Про истечение: `isStale(s) = rejectedAt + 90 дней < today` — детерминированная
пометка «истёк» для веса штрафа. НЕ удаление: файлы из `void/` удалять запрещено, история —
актив (контент-мейкер, аудит, обучение). Стареет штраф, не память.

[Верстальщик]: Процессные следы — отвечаю на грань повестки: черновики, эхо в логах,
промежуточные прогоны = **производные родителя**, уезжают той же операцией с той же
шапкой. А вот нескоординированные процессы (install-война, чистка чужого дерева) — это
НЕ предмет GC-файлов: это дисциплина деревьев, у неё свой эпик (worktree-гигиена).
Граница проведена: GC затирает артефакты, не процессы; процессам — свой контур.

[Музыкант]: Поддержу границу, с одним мостом: GC-проход обязан быть **шумным** — отчёт
в вечернем ритуале («перенесено N, эпитафии: …»), не тихий фон. Молчаливый сборщик
мусора в нашем проекте — это самопародия.

[Структурщик]: Место в цепочке: `gc.mjs` — шаг `ritual:evening` ПОСЛЕ закрытия вердиктов
дня (раньше нельзя — приговор ещё не вынесен). Без фоновых демонов: раз в день, видимо,
с diff-гейтом. Каталог — `docs/void/` в корне docs (не в research: класс шире ресёрча).

[Teamlead]: Проверяю против прецедентов подбора мёртвого пути: холодная сессия 21.07
(старый сценарий) — барьер 1+2 остановили бы на первом чтении; зомби-акцент стратегии —
recent_void_penalty ловит переоткрытие; воскрешение шага в цепочке — grep-инвариант CI.
Все три класса закрыты. Принимаю.

[Математик]: Тестируемое: `isDead`/`isStale` — юнит без часов (дата из вердикта);
grep-инвариант — CI; перенос производных — тест «сирот нет». Возражений нет.

[Верстальщик]: Принимаю; прошу в DoD явную строку про отчёт-эпитафии в вечернем ритуале.

[Музыкант]: Принимаю. Шумность GC — в DoD.

[Структурщик]: Термины закреплены: void ≠ delete; приговор ≠ исполнение; артефакты ≠
процессы. Возражений нет.

[Верстальщик]: Сверка каталога: `docs/void/` в корне docs, README-шапка каталога с
правилами чтения («здесь лежит отвергнутое; живым не является»). Принято.

[Музыкант]: И проверка тишины наоборот: если GC-проход НЕ нашёл ничего к переносу —
отчёт всё равно печатает «перенесено 0», молчание не допускается. Принято.

[Математик]: Финальный инвариант: `void/` растёт монотонно (переносы), никогда не
уменьшается (удаления запрещены) — проверяется тестом на diff каталога. Принято.

Один источник правды, без отдельного graveyard. Стартовый список — рукописный (известные экспонаты), пополнение — grep-инвентаризация по предмету ДО проектирования нового контура (норма класса) |

## Список посылок

- **Слово владельца** (BRIEF_AMENDMENT, ратифицирован): «нужен garbage collector — не вижу,
  как мы затираем следы альтернативных сценариев»; тезис шторма «охотники за привидениями»
  (манифест следов до сноса, снос со ссылкой) — ратифицирован в #741.
- **Закрытые вердикты этого заседания:** C (статусная механика status/supersededBy,
  verify-canon — документы), H (манифест + атомарность + |entry|=1 — образец вычистки
  одного конкурента, GC обобщает в класс).
- **Прецеденты подбора мёртвого пути:** холодная сессия 21.07 (старый сценарий с боковой
  ветки — прецедент-постмортем); зомби-акцент стратегии на отозванных кристаллах 20.07;
  канон/скилл, детерминированно воспроизводящие старый мир (корень №4).
- **Живой инцидент 21.07:** чистка деревьев зацепила основное дерево (2019 файлов,
  восстановлены), node_modules-война — довод для явной границы «артефакты ≠ процессы».
- **Тезис питомца из шторма** (кандидат в конспекте #741, владельцем не подтверждён —
  вход как предложение): grep-инвентаризация пересечений перед проектированием.
- **Канон append-only** (граф правды, реестры): история — актив; затирается читаемость,
  не прошлое.

## Definition of Done

1. `scripts/gc.mjs` — чистый детерминированный проход (дата из вердикта, не Date.now);
   юнит-тесты `isDead`/`isStale`; перенос производных без сирот. **Владелец: Математик.**
2. Каталог `docs/void/` с обязательной YAML-шапкой; поля rejectedReason/rejectedBy в схеме
   registry.json + валидатор. **Владелец: Структурщик.**
3. Три барьера: шапка · индекс · grep-инвариант CI (нарушение — отчёт с указанием вердикта,
   не только exit-код). **Владелец: Математик + Структурщик.**
4. `recent_void_penalty` в генераторах-планировщиках (читают void, штрафуют свежее).
   **Владелец: Структурщик.**
5. Шаг в `ritual:evening` после закрытия вердиктов дня; отчёт-эпитафии (перенесено N,
   badge-error свежим, badge-ghost истёкшим). **Владелец: Верстальщик (подача) + Музыкант (шумность).**
6. Стартовый реестр следов заполнен известными экспонатами; норма grep-инвентаризации
   перед новым контуром — в канон. **Владелец: Teamlead.**
7. LGTM Teamlead: граница GC↔worktree-гигиена явная; провенанс не затронут. **Владелец: Teamlead.**
