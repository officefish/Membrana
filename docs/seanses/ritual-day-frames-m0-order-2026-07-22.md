По «форма следует функции»: функция фреймовой раскладки — сделать очередь утра читаемой без раскопок. Значит фундамент — тот вопрос, без вердикта которого варианты остальных теряют смысл. Проверяемый предикат фундамента: убери кандидат a — становится ли множество допустимых ответов остальных пустым или неопределённым? Если да — a фундамент. Проверю 1 этим предикатом: убрать инвентарь — у 2, 3, 4 исчезает область определения (holder чего? ранг какого кадра? граница с каким фреймом?). 1 — фундамент.

[Верстальщик]: Со стороны потребителя каскада — а потребитель это холодная сессия агента утром и человек в PR — граф обязан быть **виден в артефакте**, иначе владелец получит пять пунктов без стрелок. По существу поддержу: без описи кадров ни holder, ни ранг, ни граница не рисуются на холсте — им нечего подписывать.

[Структурщик]: Теперь рёбра из фундамента. **1 → 2**: карта holder'ов присваивает лицо каждому кадру; нет списка кадров — нет области значений функции holder. [Математик]: Проверю, есть ли ребро **2 ↔ 3**. holder wiring уже зафиксирован — ozhegov у `morning-wiring` (посылка-факт). [Музыкант]: Флаг тихой дыры на 5. ## Итоговое решение консилиума

| Вопрос | Решение |
|--------|---------|
| Фундамент | **Кандидат 1** — инвентарь полной очереди фреймов утра. Без него у 2, 3, 4 нет области определения. |
| Обязательный ранний вердикт | **Кандидат 1** — выносится первым; остальные обсуждать раньше бессмысленно. |
| Надстройка первого яруса | **Кандидаты 2 (holder'ы), 3 (ранг `morning-wiring`), 4 (граница кит↔фреймы)** — все зависят только от 1. |
| Независимые между собой | **2, 3, 4** — попарно нет рёбер (holder wiring уже зафиксирован → 3 не ждёт 2; граница про декомпозицию пина, ранг про позицию → 3 ⊥ 4). Общий предок — 1. |
| Терминал | **Кандидат 5** — приёмка/миграция; входящие рёбра 1→5, 2→5, 3→5, 4→5. |
| Фактурная блокировка | **Кандидат 5 не решается раньше #900 F1–F3 в main** (validateProcedure/auditPins). Это фактурная, а не контрактная зависимость. |
| Направленный граф | `1 → {2, 3, 4} → 5`; барьер `#900 F1–F3` перед 5. Циклов нет, топосортировка по слоям однозначна. |
| Риск обратного ранга | Приёмка пустоты: `frames[]` в MANIFEST без состава/holder'а/границы → дубль «файл+отрезок», слом `kitVersion` до F2/F3. |

**Definition of Done (только вопрос D1):**
- Артефакт несёт направленный граф `1 → {2,3,4} → 5` с обоснованием каждого ребра (стрелки видны, не линейный список).
- Явно назван фундамент (1) и явно названы независимые (2, 3, 4).
- Узел 5 помечен как заблокированный фактурой #900 F1–F3 (не в main) — блокировка видна в артефакте, а не подразумевается.
- Порядок ратифицирует владелец; председатель фиксирует, не решает.
