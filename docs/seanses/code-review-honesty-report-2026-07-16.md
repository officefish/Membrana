# Доклад: честен ли и свеж ли генератор code-review

> Расследование по образцу разбора дрейфа MAIN_DAY_ISSUE (16.07). Только evidence и код.
> Вопрос: производит ли `yarn code-review` честную оценку — или, как MAIN_DAY_ISSUE,
> документ, который выглядит авторитетно, но собран не из того, что заявляет.

## TL;DR

Два разных инструмента под одним именем.

- **Ревью по запросу** (`--pr` / `--branch` / `--uncommitted` / `--staged`) — **честное**: получает полный дифф. Это то, что запускается как `/code-review`.
- **Вечернее `daily`-ревью** (то, что пишет `docs/DAILY_CODE_REVIEW.md` и кормит утренний ритуал) — **структурно слепо к коду**. Оно не видит ни одной строки диффа: только имена изменённых файлов, сводку тестов/линта и список коммитов. И оценивает **рабочее дерево**, а не работу дня.

Итог 15.07: день с крупными слияниями кода получил вердикт **T0, docs-only, «Runtime не затронут»** — и этот вердикт ушёл в MAIN_DAY_ISSUE 16.07 как зелёный сигнал «✅ Вход в Фазу 0».

## Находка 1 — daily-ревью не видит кода (структурно)

`code-review.mjs` разводит режимы: [строки 70–76](../../scripts/code-review.mjs#L70-L76).

```js
if (cli.mode === 'daily') {
  contextBlock = collectRepositoryContext({ full: cli.full });  // git-снимок, БЕЗ диффа
} else {
  contextBlock = collectReviewContext(cli).text;                // полный дифф
}
```

`collectRepositoryContext` ([context-collector.mjs](../../scripts/context-collector.mjs)) собирает: ветку, последний коммит, `todaysCommits()` (темы коммитов), `git status --short`, `git diff --name-status` (имена, не содержимое), список каталога, сводку `turbo test`/`lint`. Самодокумент скрипта, строки 206–207:

> Скрипт **не обходит дерево исходников**: только git, yarn и ограниченный список верхнего уровня каталога.

Диффа коммитов дня нет нигде: `git diff --name-status` — это рабочее дерево против индекса. **Вечером, после того как всё закоммичено, дерево чисто → изменений ноль.** Проверено сейчас: `git status --short` и `git diff --name-status` на чистом дереве пусты.

При этом промпт задания ([code-review-ritual.mjs:280](../../scripts/lib/code-review-ritual.mjs#L280)) просит «Проведи вечернее daily code review… Укажи Tier». LLM просят вынести Tier-вердикт по коду, не показав кода.

## Находка 2 — ревью пересказывает день, а оценивает рабочее дерево

Живой случай — `docs/DAILY_CODE_REVIEW.md` от 15.07 (сгенерирован 15:54Z). Дословно:

> [Teamlead]: Сводка дня: за 15.07 в main влилась **большая серия закрытий** (PC-1/PC-2/PC-2b/PC-2c, hot-repo-flow, repo-clean, canon-tooling-debt, agent-tooling-friction-2, cowork FREE) — но всё это **уже смёржено**. Незакоммиченные правки в рабочем дереве — **только docs**… Runtime не затронут, тесты 57/57, lint 36/36 зелёные. Классифицирую как **T0** (docs/memory-снапшоты, без кода).

Несогласованность видна в одном абзаце: ревью **перечисляет** крупную кодовую работу дня и тут же оценивает её как **T0 docs-only**, потому что смотрит на остаток рабочего дерева (docs-снапшоты), а не на слитый код. Чем продуктивнее и чище закрыт день, тем пустее ревью — извращённый стимул: полностью закоммиченный день кода выглядит как «нечего ревьюить».

Что здесь честно, чтобы не преувеличивать: числа `57/57`, `36/36` — реальный вывод `turbo test`/`lint`, не выдумка. Слепота не в тестах, а в **классификации изменений**: «Runtime не затронут» ложно для дня, истинно только для остатка дерева.

## Находка 3 — daily слеп и к задаче дня

`appendTaskContext` (читает MAIN_DAY_ISSUE / CURRENT_TASK) вызывается только для `pr`/`branch`/`uncommitted`/`staged` — [code-review-ritual.mjs:364](../../scripts/lib/code-review-ritual.mjs#L364), в `daily`-ветке (строки 171–173) возврат `text: ''`. Вечернее ревью не знает, какая у дня была задача, — поэтому не может сказать «сделано не то, что планировалось». Оно слепо и к коду, и к намерению.

## Находка 5 — свежесть: по регенерации ОК, но без гварда

История `DAILY_CODE_REVIEW.md` — по коммиту в день, без пропусков (07-08 … 07-15). То есть регенерация работает, и заголовок честно проставляет дату. **Активного протухания нет.**

Но защиты нет. В `ritual:evening`: `… && (node scripts/code-review.mjs || true) && (node scripts/archive-daily-code-review.mjs || true) && …`. Сбой ревью **глотается** `|| true`. А в самом `code-review.mjs` (строки 143–148) зафиксировано: 15.07 фолбэк «Anthropic без кредита» отдавал 127. Если бы кредит не восстановили — вечер прошёл бы дальше, `DAILY_CODE_REVIEW.md` остался бы вчерашним, и ни standup, ни main-day-issue не проверяют его дату (гварда свежести нет — grep пуст). Латентная дыра того же класса `|| true`, что чинили сегодня в контуре планирования.

`code-review-ritual.test.mjs` тестирует парсинг CLI, но не сам факт «daily не получает диффа». Как и с `main-day-probe` (PG3: гейт врал при 12 зелёных тестах), поведение, которое ломает смысл инструмента, невидимо для набора.

## Чего в этом контуре НЕТ (проверено, чтобы не додумывать)

- **Замороженного хардкода** уровня `detection-planning-priorities.mjs` здесь нет. `CODE_REVIEW_REGULATION.md` статичен, но это **регламент** (правила ревью), а не снимок состояния — легитимно статичен, как «вердикты детекции».
- **Фабрикации чисел** нет: тест/линт-счётчики реальны.
- **Ревью по запросу честное**: `/code-review --pr N` тянет полный `gh pr diff`. Именно им прошёл сегодняшний тимлид-ревью #538 — и поймал мою ошибку идентичности. Инструмент работает, когда ему дают код.

## Вопросы (для решения / консилиума)

1. **Главный:** daily-ревью должно ревьюить **код дня** (дифф слитых за сутки коммитов: `git diff @{yesterday}..HEAD` или по `todaysCommits`), а не остаток рабочего дерева. Это меняет вход `collectRepositoryContext` для daily. Размер?
2. Пока daily слеп — честно ли он называется «code review»? Не переименовать ли артефакт в «сводку дня», чтобы downstream не читал T0 как «код проверен»?
3. Гвард свежести: должен ли standup/main-day-issue отвергать `DAILY_CODE_REVIEW.md` старше N часов (как гейт посылок отвергает протухший маркер)?
4. Слепота к задаче: подмешивать ли MAIN_DAY_ISSUE в daily, чтобы ревью мог сказать «сделано не то»?
5. `|| true` в вечернем ритуале: глотать сбой ревью или падать, как решили для `main-day-probe`?

## Приложение: воспроизведение

```bash
sed -n '70,76p' scripts/code-review.mjs                    # daily → collectRepositoryContext
sed -n '206,207p' scripts/context-collector.mjs            # «не обходит дерево исходников»
sed -n '171,173p' scripts/lib/code-review-ritual.mjs       # daily → text: ''
head -5 docs/DAILY_CODE_REVIEW.md                          # T0, «Runtime не затронут» на дне с кодом
grep -n 'DAILY_CODE_REVIEW' docs/MAIN_DAY_ISSUE.md         # потреблено как ✅ Вход в Фазу 0
git log --format='%ad' --date=short -- docs/DAILY_CODE_REVIEW.md | head   # регенерация без пропусков
```
