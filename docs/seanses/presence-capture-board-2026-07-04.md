## Итоговое решение консилиума

| Вопрос | Решение |
|--------|---------|
| **A1: Диагностика persistent-offline** | Логирование трёх событий WS-lifecycle в обе стороны: `node-ws-connect`, `node-ws-authenticate`, `node-ws-register` с timestamp. Математик реализует. Выявит H1 (auth-fail 4401) / H2 (рестарт сервера) / H3 (отсутствие mediaDeviceId) / H4 (Vite HMR). |
| **A2: Реальная живость на клиенте** | Расширить `nodeConnectionStore` полем `authFailedAt` (когда последняя ошибка auth). Верстальщик добавляет баннер в `NodeConnectionShell`: «Сопряжение истекло» с дебаунсом 2–3 сек + кнопка «Пересопрячь». Не спамит при HMR-разрывах. |
| **A3: Heartbeat (PL2b)** | **Отложить**. Фаза 1 обходится без heartbeat: полагаемся на `Device.lastSeenAt` (БД) и snapshot при подключении кабинета. |
| **A4: Кабинет — достоверный индикатор** | Расширить `getNodesByMembrane` полем `lastSeenAt` (ISO8601). Кабинет показывает: online (в `nodeSockets`), connecting (WS в процессе), seen Ns ago (из `lastSeenAt` > 5 мин назад), offline. Структурщик. |
| **B1: Capture → force view-only board** | На `board.capture`: `boardLeaseBridge` вызывает `useForceOpenDeviceBoard()` хук (из agenda) → `navigate('/device-board')` + флаг в сессию. Device-board модуль читает флаг → `readOnly=true` + `cabinet-view` сессия. На release → флаг снять. Верстальщик + Структурщик. |
| **B2: Инвариант жёсткого захвата** | Жёсткий захват блокирует редактирование; только просмотр + emergency stop (уже в `resolveDeviceCaptureFlags`). B1 не нарушает инвариант. |
| **D1: Health-эндпоинты API** | Структурщик добавляет: (1) `GET /v1/nodes/:id/link-state` → `{ paired, live, lastSeenAt }`; (2) `POST /v1/nodes/:id/health-ping` → `{ reachable, latencyMs }` с таймаутом 3с (echo по WS). Для диагностики оператором. |
| **D2: Мульти-устройство** | **Фаза 3 (отложить)**. Расширение `getPairStatus` на массив узлов требует переделки UI кабинета. Пока хватает V1 (один узел на membrane). |
