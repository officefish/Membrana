# Консилиум: борд detector-compare — trends DRONE_TIGHT vs yamnet в панели

**Участники:** Teamlead (Vesnin), Структурщик (Ozhegov), Математик (Dynin), Музыкант (Kuryokhin), Верстальщик (Rodchenko).

---

[Математик]: Начну с природы данных, она решает развилку 1. Сравнение детекторов — это **детерминированный офлайн-прогон** по фиксированному корпусу (бенчмарк-манифест free-v1 + drone, тот же, что `benchmark:detectors`). Live-вычисление в браузере или на office — недетерминированно и дорого: yamnet-инференс по 120+ сэмплам на каждый визит. Правильная форма — **экспортёр-скрипт**: один прогон обоих детекторов → версионируемый JSON-артефакт с per-sample вердиктами, скорами и деталями. Одинаковый корпус + одинаковые пороги → байт-в-байт одинаковый артефакт.

[Верстальщик]: Поддерживаю офлайн-артефакт — мне для таблицы нужен статичный JSON, а не крутящиеся спиннеры на 120 строк. По видению владельца зафиксирую UI-контракт: строка = трек (play + waveform, имя/класс/длительность/метаданные, две ячейки вердикта); в ячейке — вердикт словом + скор + значок «подробнее» с `aria-label`; попап — `dialog` с фокус-ловушкой, Esc, метрики + человеческое пояснение. Визуальный паритет с библиотекой сэмплов — да, но это паритет **вида**, не обязанность тащить её код.

[Структурщик]: Это и есть развилка 2. Эпик зафиксировал границу «panel зависит максимум от `@membrana/core`». `@membrana/sample-playback-service` — готовый waveform, но он тянет привычки клиентской библиотеки и создаёт прецедент размывания границы на второй же день жизни панели. Термины: панель — **витрина read-only над артефактами**, не вторая библиотека сэмплов. Предлагаю **локальный мини-waveform**: `decodeAudioData` + canvas, ~50 строк, нулевые зависимости. Границу эпика не трогаем.

[Музыкант]: Как автор звуковой части поддержу Ozhegov'а, но с условием к качеству: mini-waveform обязан честно показывать пики (downsample по max-|amplitude| на бакет, не среднее — иначе выстрел «gunshot» размажется в шум), и play должен идти через **один** AudioContext на страницу с остановкой предыдущего трека (анти-какофония — урок библиотеки). Формат аудио — те же wav из корпуса, без перекодирования: оператор должен слышать ровно то, что слышал детектор.

[Teamlead]: Фиксирую по 1 и 2: экспортёр-скрипт → артефакт; mini-waveform локально, граница эпика не ослабляется. Теперь где живёт артефакт и аудио (развилка 1б). Вариант «office-ручка» добавляет код в office ради раздачи статики — противоречит его роли тупого транспорта. Данные несекретны: те же числа публично лежат в `DETECTOR_BENCHMARK.md` открытого репо (прецедент решения по digest, ADR 0004 Р3), корпус free-v1 — открытый. Вердикт: **артефакт и аудио-бандл деплоятся статикой рядом с dist панели** (`/compare-data/latest.json`, `/compare-audio/<id>.wav`), Caddy раздаёт как файлы. Office не трогаем вовсе.

[Математик]: Тогда развилка 4 решается так: **данные публичны** (они уже публичны в репо), а **раздел в UI остаётся operator** — это UX-настройка рабочего места, не security-граница, и честно это задокументировать, чтобы никто не считал скрытие ссылки защитой. Q3-минимизация соблюдена: в артефакте только метрики детекторов и метаданные корпуса, никаких путей инфраструктуры.

[Структурщик]: Схема артефакта (развилка 3) — контракт между экспортёром и панелью, объявляю его в панели локальным типом (панель не импортирует детекторные пакеты!): `{ schemaVersion, generatedAt, corpus: {name, manifestSha}, thresholds: {trends, yamnet}, samples: [{ id, file, className, isDroneTruth, durationSec, meta: {sampleRate, source?}, detectors: { trends: Verdict, yamnet: Verdict } }] }`, где `Verdict = { isDrone, score, confidence, details }`. Экспортёр живёт в `scripts/` и знает детекторы; панель знает только JSON. Слабая связанность через файл — тот же паттерн, что journал persona-memory.

[Музыкант]: Наполнение `details` для попапа — моя зона с Dynin'ом. **trends (DRONE_TIGHT)**: имя совпавшего шаблона, счётчик совпадений/окон, пиковые частоты коридора, порог и фактическое значение — пояснение вида «совпал шаблон DRONE_TIGHT: N окон из M, пики ~180/360 Гц». **yamnet**: топ-5 классов с вероятностями, какие классы агрегированы в drone-скор, порог — пояснение «модель увидела „Helicopter“ 0.61 + „Propeller“ 0.22 → drone-скор 0.83 ≥ 0.5». Пояснение генерирует **экспортёр детерминированным шаблоном** — никакого LLM.

[Верстальщик]: По фильтрам и сортировке (развилка 5): фильтр — сегмент-контрол «все | дрон | не дрон» (по ground truth корпуса, словом в подписи «по разметке корпуса»), сортировка — «уверенные сверху/снизу» по выбранному детектору (селектор trends/yamnet у заголовка колонки). Добавляю настойчивую просьбу: чипы согласия — «детекторы расходятся» как отдельный фильтр: оператору интереснее всего именно расхождения. Всё client-side по загруженному JSON, без запросов.

[Математик]: Поддерживаю фильтр расхождений — это главный аналитический кейс борда: FP/FN каждого детектора видны сразу. И добавлю в шапку борда агрегаты из артефакта: P/R/F1/FPR обоих детекторов на корпусе (уже считаются бенчмарком) — таблица без сводки заставляет оператора считать в уме.

[Структурщик]: Форма задачи (развилка 6): **одна M-карточка** `panel-detector-compare-board` (не эпик — один связный набор артефактов): экспортёр + артефакт + раздел панели. Порядок внутри: экспортёр с тестами → артефакт в репо → UI. Из scope исключаю: изменения office, изменения детекторных пакетов, live-прогон, любой LLM. Деплой аудио-бандла — той же rsync-строкой из PANEL_DEPLOY.md (дополнить runbook одной строкой).

[Музыкант]: Ограничение на размер: аудио-бандл — только сэмплы корпуса сравнения (~120 × ≈5 с wav ≈ 50–100 МБ). В git бандл НЕ коммитим — только JSON-артефакт; wav копируются экспортёром из локального корпуса при деплое. Иначе раздуем репо.

[Teamlead]: Согласен: JSON — в git (версионируемый результат прогона), wav — деплой-тайм копия из корпуса. Итоговая сборка вердиктов. (1) Экспортёр `scripts/detector-compare-export.mjs` на инфраструктуре `benchmark:detectors` (те же детекторы, манифест, пороги) → `docs/reports/detector-compare/latest.json` в git. (2) Panel: mini-waveform локально (canvas, max-|amp| бакеты, один AudioContext), граница «≤ core» цела. (3) Схема артефакта — локальный тип панели, details + детерминированные пояснения из экспортёра. (4) Данные публичны (уже публичны в репо), раздел — operator как UX-гейт, задокументировать несекьюрность скрытия. (5) Фильтры все/дрон/не-дрон + «расхождения», сортировка по уверенности выбранного детектора, сводка P/R/F1/FPR в шапке. (6) Одна M-карточка, office и детекторные пакеты не трогаются. **LGTM на план.**

[Математик]: Принимаю. Экспортёр — чистые функции сборки записи и пояснений + снапшот-тест на фикстурных вердиктах; прогон детекторов — переиспользование бенчмарк-раннера, не копия.

[Верстальщик]: Принимаю. Таблица: `tabular-nums` для скоров, вердикт словом + иконка (не только цвет), попап — `<dialog>`, «подробнее» с aria-label, пустое/ошибочное состояние загрузки JSON — с первого коммита.

[Структурщик]: Принимаю. Контракт-файл, локальные типы, ноль новых зависимостей панели.

[Музыкант]: Принимаю. Пики честные, AudioContext один, wav без перекодирования, бандл вне git.

## Итоговое решение консилиума

| Развилка | Решение |
|----------|---------|
| 1. Данные | Офлайн-экспортёр `scripts/detector-compare-export.mjs` на инфраструктуре `benchmark:detectors` → **`docs/reports/detector-compare/latest.json` в git**; live-вычислений нет; office не трогается |
| 2. Аудио/waveform | Wav корпуса без перекодирования; бандл копируется при деплое (`/compare-audio/`), в git НЕ коммитится; **mini-waveform локально в панели** (canvas, downsample по max-амплитуде, один AudioContext) — граница «panel ≤ core» цела |
| 3. Схема/попап | Локальный тип панели: schemaVersion/corpus/thresholds/samples[{id,file,className,isDroneTruth,durationSec,meta,detectors:{trends,yamnet}}]; Verdict={isDrone,score,confidence,details}; пояснения — детерминированный шаблон в экспортёре (trends: шаблон/окна/пики; yamnet: топ-классы→агрегат), БЕЗ LLM |
| 4. Доступ | Данные публичны (те же числа уже в открытом репо — прецедент digest); раздел `detector-compare` в UI — operator как UX-гейт; несекьюрность скрытия задокументирована |
| 5. Фильтры/сортировка | Client-side: все / дрон / не дрон (по разметке корпуса) + фильтр «детекторы расходятся»; сортировка по уверенности выбранного детектора ↑/↓; сводка P/R/F1/FPR в шапке |
| 6. Форма | Одна **M-карточка `panel-detector-compare-board`** (магистраль дня); порядок: экспортёр+тесты → артефакт → UI; out of scope: office, детекторные пакеты, live-прогон, LLM |

**Definition of Done:**
- `yarn detector:compare:export` детерминированно строит `docs/reports/detector-compare/latest.json` (снапшот-тест чистых функций; повторный прогон не меняет файл при неизменном корпусе/порогах).
- Раздел `detector-compare` панели: таблица (трек: play+mini-waveform, метаданные; вердикты обоих детекторов словом+скор), попап «подробнее» (`<dialog>`, метрики + пояснение), фильтры все/дрон/не-дрон/расхождения, сортировка по уверенности, сводка P/R/F1/FPR.
- A11y: aria-label на play/подробнее, вердикт не только цветом, `tabular-nums`, состояния loading/error/empty.
- Аудио-бандл вне git; строка деплоя бандла добавлена в `PANEL_DEPLOY.md`.
- Границы: panel без новых зависимостей; office и `packages/services/detectors/*` не изменены.
- LGTM Teamlead (closure review) перед merge.
