## Итоговое решение консилиума

**ВЕРДИКТ O1**

Порядок разбора жизненного цикла инсайтов — это DAG с корнем C1 (объект закрытия) и единственным стоком C6 (workflow/enforcement).

**Легенда кандидатов:**

| ID | Кандидат |
|----|----------|
| C1 | Определить объект закрытия (INSIGHT / REVIEW-мандат / outcome) |
| C2 | Развести состояния decision / delivery-outcome / archive-visibility |
| C3 | Типизированный evidence contract + полнота task graph |
| C4 | Модель истории, производных views, обратимых событий |
| C5 | Forensic migration legacy + классификация четырёх кандидатов |
| C6 | Замкнуть workflow и enforcement (Cursor / Claude / Codex) |
| C7 | Operational safety: concurrency, live work, idempotency, rollback/reopen |

**DAG (прямые рёбра, транзитивные скрыты):**

| Ребро | Обоснование посылки |
|-------|---------------------|
| C1 → C2 | Состояния можно разводить только над определённым носителем закрытия |
| C2 → C3 | Evidence-contract доказывает переход между определёнными состояниями |
| C2 → C4 | История фиксирует переходы между определёнными состояниями |
| C2 → C5 | Миграция переносит legacy в целевую модель состояний |
| C3 → C5 | Классификация четырёх кандидатов = применение evidence-предиката |
| C4 → C5 | Миграция требует целевую модель истории/событий |
| C2 → C7 | Concurrency защищает определённые состояния |
| C3 → C7 | Idempotency — свойство evidence-операций |
| C4 → C7 | Rollback/reopen оперирует обратимыми событиями |
| C5 → C6 | Enforcement принуждает соблюдать согласованную (в т.ч. мигрированную) форму |
| C7 → C6 | Enforcement не должен принуждать к небезопасным операциям |

**Слои DAG:**

| Слой | Узлы | Природа |
|------|------|---------|
| 0 (корень) | C1 | Носитель — что закрывается |
| 1 | C2 | Разведение омонима состояний |
| 2 (параллельно) | C3, C4 | Доказательство ↔ история (ортогональные домены) |
| 3 (параллельно) | C5, C7 | Миграция ↔ безопасность рантайма |
| 4 (сток) | C6 | Замыкание workflow/enforcement |

**Инварианты:** I1 ацикличность — выполнена (топосорт существует); I2 «посылка раньше вопроса» — каждое ребро обосновано в таблице выше; содержательных решений по кандидатам не принято.

**Посылки, реально использованные для DAG:**

1. Ребро a→b означает «вердикт по b предполагает уже принятый вердикт по a» (определение Математика).
2. Транзитивные рёбра не рисуются; подразумевается транзитивное замыкание (норма Структурщика/Верстальщика).
3. C1 — единственный узел без входящих рёбер (носитель определяет адрес всех статей).
4. C3 и C4 ортогональны по домену (доказательство vs история переходов) → ребра между ними нет.
5. C5 и C7 независимы: миграция над результатом, safety над моделью → ребра между ними нет.
6. C6 — единственный сток (enforcement всегда финальное замыкание).
7. Safety (C7) предшествует enforcement (C6): нельзя принуждать к небезопасным операциям.

**Definition of Done:** неприменимо — прогон производит только DAG порядка вопросов; содержательные вердикты по C1–C7 выносятся отдельными сеансами в порядке слоёв 0→4.
