### Топ-3 для немедленного пилота

**1. `opencode:weekly:design-token-drift`**
- **Таймер:** Среда 07:00 UTC (GitHub Actions scheduled workflow).
- **Вход:** `DESIGN.md` (парсить Markdown таблицы токенов) + все файлы `src/components/**/*.tsx` (grep на `className`, `style`, переменные Tailwind).
- **Выход:** Markdown `docs/seanses/design-drift-YYYY-WW.md` с таблицей: файл → строка → используемый класс → предложение из DESIGN.md.
- **Почему proxy:** Требует интерпретации CSS-классов и соответствия DESIGN.md, LLM хорошо справляется.
- **Skip-последствия:** Консистентность дизайна ловится в ревью; пропуск не блокирует deploy.
- **Owner:** Верстальщик (Rodchenko) утверждает выход.

**2. `opencode:weekly:services-api-contract-drift`**
- **Таймер:** Понедельник 11:00 UTC.
- **Подготовка:** Одноразово — создать `packages/services/.api-contract.json` с версией и эталонными экспортами всех текущих сервисов. Структура: `{ "version": "1.0", "services": { "@membrana/audio-engine-service": { "exports": [...], "version": "0.x" } } }`.
- **Вход:** Текущие `packages/services/*/src/index.ts` + `.api-contract.json`.
- **Выход:** Markdown `docs/seanses/services-api-drift-YYYY-WW.md` с diff-таблицей (добавлены / удалены / изменены экспорты, версия).
- **Почему proxy:** Сравнение и рекомендации требуют контекста и суждения.
- **Skip-последствия:** API-изменения ловятся в ревью; пропуск не критичен.
- **Owner:** Структурщик (Ozhegov) утверждает и обновляет `.api-contract.json`.

**3. `opencode:weekly:monorepo-dependency-graph`**
- **Таймер:** Вторник 08:30 UTC.
- **Вход:** Вывод `turbo report` (граф зависимостей пакетов) + ARCHITECTURE.md (правила из §1).
- **Выход:** Markdown `docs/seanses/graph-drift-YYYY-WW.md` с перечислением новых циклов или нарушений ARCHITECTURE.md (прямые импорты между плагинами, зависимость agenda от device-board и т.п.).
- **Почему proxy:** Анализ графа требует контекста правил архитектуры.
- **Skip-последствия:** Циклы ловятся Турбо в CI; пропуск не критичен, но теряется человеческий контекст.
- **Owner:** Teamlead (Vesnin) + Структурщик (Ozhegov) утверждают.
