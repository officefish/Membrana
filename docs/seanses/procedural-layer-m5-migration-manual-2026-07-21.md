Посылки оглашены. Три пункта повестки.

[Структурщик]: Пункт 1 — форма. «Сразу все» ложна по посылке 9: девять доноров по цене дня каждый — это спринт-неделя, замаскированная под галочку; получим девять пустых каталогов — «Половину без провода». «Только новые» ложна по посылке 1: ритуалы — процедуры уже сейчас, оставить их вне слоя значит объявить слой музеем. Остаётся **«по касанию» с принудительным первым залпом**: два донора мигрируют немедленно (слой не пуст, образец существует), остальные — той работой, которая их коснётся. Форма включения наследует наш же дневной опыт: аудит по касанию, а не тотальный.

[Математик]: Пункт 2 — критерий, формализую. `migrated(p) := container(p) ∧ vocabulary(p) ∧ grammar(p)` — производный предикат, НЕ поле: хранить его нельзя, только вычислять, иначе разъедется с компонентами. Три компоненты раздельны: `container` — дом существует и проходит `validateProcedure` (посылка 4); `vocabulary` — словарная статья процедуры оперирует категориями M2; `grammar` — ветки работ процедуры соответствуют M4. Каждое `true` несёт провенанс `<persona>@<hash>` — кто и на какой ревизии подтвердил. Анти-критерий: каталог без резолвящегося манифеста миграцией не является ни при каком статусе.

[Верстальщик]: Носитель статуса — реестр `docs/procedures/registry.json`, отдельный от реестра задач: это разные предметы (задачи движутся, процедуры живут). Человекочитаемая проекция генерится из реестра — руками не правится (образец M2). Статус немигрированных — честное слово **`legacy`**, не `pending`: мы не обещаем скорый переезд, мы фиксируем факт. Шапка каждой процедуры показывает статус, держателя, ревизию — текстом.

[Музыкант]: Пункт 3 — доноры первого залпа. Первый — **`ritual-evening`**: он уже назначен жильцом в M1, менять донора значило бы пересмотреть M1 без права на то; и у ритуала самый богатый набор движков — стресс-тест манифеста. Второй — **`meeting`** (заседание): мы сами восемь раз за день упёрлись в его движок — миграция заседания в слой и запланированный фикс движка (отдельный блок работ) лягут в один контейнер, прецеденты уже готовы: восемь черновиков в rejected/ — это `precedents[]` будущего манифеста. Держатели по T6: ритуал — Ангелина (ведёт утро и вечер), заседание — Teamlead/Vesnin (председательствует).

[Структурщик]: Поправка к черновику движка, фиксирую явно: отдельного термина «держатель ≠ leadPersona» не заводим. Держатель процедуры — это и есть `leadPersona` её манифеста (посылка 4); он владелец адреса и словарной статьи и точка обращения при расхождении, и он не обязан быть автором кода — но это свойство роли, а не вторая сущность. Один термин, одна точка истины.
