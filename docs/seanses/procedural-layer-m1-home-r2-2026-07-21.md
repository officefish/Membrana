[Структурщик]: Начну с термина, иначе DAG поплывёт. «Процедурный слой» — это омоним двух лемм, и их нельзя слить. Первая: **контейнер процедуры** — markdown-дом (канон + манифест ссылок + прецеденты), без кода. Вторая: **движок процедуры** — плоский скрипт в `scripts/`. E1 спрашивает адрес **первой** леммы. Предлагаю корень `docs/procedures/<id>/` — по аналогии с `docs/audit/<id>/`, но семантически честнее: `audit` — это род процедуры, а слой шире. Минимальный контейнер: `README.md` (что это, одна лемма-определение), `MANIFEST.json` (ссылки на движки в `scripts/`, версия кита), `precedents/` (следы прогонов).

[Верстальщик]: С точки зрения потребителя — а потребитель здесь агент, читающий контейнер как вход следующего промпта, и человек в PR. Моя норма честности: дом должен быть **виден без раскопок**. Если процедурный слой размазан по `prompts/`, `storm/`, `meeting/`, `skills/` — это мёртвая дверь: читатель не знает, где канон процедуры. Прошу, чтобы `MANIFEST.json` нёс явную шапку: id процедуры, ведущая персона, список движков-ссылок, версия кита. Иначе «половина без провода» — контейнер есть, а куда он ссылается, невидимо.

[Математик]: Поддержу адрес Структурщика с оговоркой о проверяемости. `docs/procedures/<id>/` даёт предикат `resolvable(manifest) → bool`: каждая ссылка в `MANIFEST.json` обязана указывать на существующий путь в `scripts/`. Это тестируемо одной чистой функцией — читаем манифест, грепаем цель, возвращаем список битых ссылок. Без этого предиката «связь ссылкой» — декларация. И величина: минимальный контейнер должен иметь ненулевой `README` и непустой `MANIFEST`, иначе `settled = false` по определению.

[Музыкант]: Соглашусь, но добавлю флаг про версионирование — владелец развёл две скорости (T-посылки про `PINNED_SUBGRAPH_VERSIONING`). Кит версионируется, контейнер — нет. Это как альбом и трек: трек можно перетасовать, но релиз-версия альбома фиксирована. Значит версия живёт в `MANIFEST.json` на уровне **набора движков** (кита), а не на самом контейнере-папке. Если засунуть версию в имя папки — сломаем ссылки при каждом ремастере. Версия — поле манифеста, не часть адреса.

[Структурщик]: Соглашусь с «специализацией», но уточню границу термина, иначе разъедется. `docs/audit/*` — контейнеры, которые **несут свой анализ и кэш** (у них есть вычислительный вес на месте). Процедурный контейнер — **чистая ссылка**: он ничего не вычисляет, он адресует движок. Это разные подвиды одной леммы «групповой контейнер»: `audit` — контейнер-с-данными, procedure — контейнер-манифест. Критерий заселения виден на холсте: манифест резолвится, README непустой.

[Математик]: Здесь осторожно — «ритуал» большой, у него много движков, риск, что первый жилец окажется неподъёмным и заселение не завершится (анти-«половина без провода»). ## Итоговое решение консилиума

| Вопрос | Решение |
|--------|---------|
| Адрес процедурного слоя | Новый корень `docs/procedures/<id>/` — по одному каталогу на процедуру |
| Контракт минимального контейнера | `README.md` (лемма-определение + ведущая персона) + `MANIFEST.json` (`id`, `leadPersona`, `kitVersion`, `engines[]` — ссылки на `scripts/`, `precedents[]`); опционально `precedents/`. **Код и тесты запрещены** (посылка владельца) |
| Отношение к `GROUP_CONTAINERIZATION` | Не новый паттерн, а **подвид `manifest-only`**: расширить чеклист строкой «код/тесты/кэш запрещены, обязателен MANIFEST ссылок наружу» |
| Определение vs прогон | `docs/procedures/<id>/` — дом **определения** (класс); `docs/storm/`, `docs/meeting/` — дома **прогонов** (инстансы), не мигрируют, служат целями `precedents[]` |
| Версионирование | Версия кита — поле `kitVersion` в `MANIFEST.json`, **не** часть имени папки (`PINNED_SUBGRAPH_VERSIONING`: киты версионируются, контейнеры — нет) |
| Первый жилец | `docs/procedures/ritual-evening/` — манифест несёт **только резолвящиеся сегодня** движки `.mjs` |
| Критерий «жилец заселён» | `validateProcedure(dir)` истинна по всем трём: `resolvable` (все ссылки на `scripts/` существуют), `readmeNonEmpty`, `manifestSchemaOk` |
| Судьба разбросанных носителей | Не мигрируем (это M5); становятся целями ссылок из манифестов, остаются на месте |

**Definition of Done (E1):**
- Создан корень `docs/procedures/` с образцовым контейнером `ritual-evening/`.
- `MANIFEST.json` имеет фиксированную машиночитаемую схему (`id`, `leadPersona`, `kitVersion`, `engines[]`, `precedents[]`).
- Реализована чистая функция `validateProcedure(dir)` (детерминированная, без сети) + её вызов в CI; первый жилец проходит все три предиката.
- Чеклист `GROUP_CONTAINERIZATION` расширен подвидом `manifest-only`.
- README первого жильца фиксирует различие «определение (procedures) ↔ прогон (storm/meeting)» как образец.
- Не затронуты вопросы M2 (словарь категорий), M3 (граница слоёв), M4 (грамматика веток), M5 (миграция).
