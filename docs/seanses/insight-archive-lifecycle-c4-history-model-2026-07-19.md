[Музыкант]: Аудио тут не задет, голосую по касательной, но ритм я узнаю — это как лента записи мастер-сессии: дорожки только дописываются, старые дубли не стираются. Флаг из моего опыта `ritual-inputs-echo`: не делайте «магическую кнопку» current view, которая молча перезаписывает историю. Пусть current будет **сведением** (mixdown) поверх неизменной ленты событий, а не отдельным редактируемым мастером.

[Структурщик]: Начну с термина, иначе спор поплывёт. У нас две разные леммы под словом «view»: **event log** (append-only факты `assert/revoke/supersede/reopen`) — это source of truth; и **current/archive** — это `derived`-проекции, чистые функции от лога. Мы это ровно так решили в `truth-graph-unify-or-split` 17.07: узлы — токены, current — `derived-from` через replay. Лог хранится в `docs/truth/`-стиле реестра; проекции нигде не хранятся как самостоятельная истина, иначе получим два фасада на одно.

[Teamlead]: Рамку фиксирую сразу, чтобы C4 не колонизировал C3 и C6. Наш вопрос узкий: **минимальная** модель истории, которая делает audit/reopen/revoke/supersede воспроизводимыми — без полноценной event-sourcing платформы. По конструкции «форма следует функции»: единственный source of truth — append-only лог событий; всё остальное — `derived`. Декор без конструкции — это snapshot-хранилище current view как отдельная истина. Его не будет.

[Верстальщик]: С точки зрения потребителя — а потребитель это агент в audit и человек в PR — мне нужна **честность мёртвых дверей**. Archive view не должна выглядеть как живое решение: revoked/superseded токен обязан визуально читаться как «архив», не как current. Это projection одной и той же истории, а не второй список. Моя норма из `truth-graph-unify-or-split`: если проекция врёт про статус — это баг отображения, а не данных.

[Математик]: Формализую, иначе останется вкусовым. Source of truth — последовательность событий `E = [e₁, e₂, …, eₙ]`, каждое `eᵢ` immutable, с монотонной меткой `seq(eᵢ) = i`. Current view — чистая функция `fold: E → State`, `State = replay(E)`. Инварианты, проверяемые без побочных эффектов: (I1) `E` только растёт (append); (I2) `fold(E)` детерминирован — один и тот же `E` даёт один и тот же `State`; (I3) `fold(E[0:k])` восстанавливает историческое состояние на момент `k` — это и есть audit/replay.

[Музыкант]: Поддержу с живой стороны. `reopen` — это не «стереть трек и записать заново», это **новый трек поверх**: new ID, `D=proposed`, ссылка на старую ревизию. Как в моём контуре — новая версия сведения не перезаписывает оригинал. И `supersede` — event-only link, стрелка «этот дубль заменил тот», но оба дубля на ленте остаются. Оператор при желании отматывает и слушает исходник.

[Структурщик]: Соглашусь и уточню словарь событий, чтобы имена были однозначны. Четыре леммы: `assert(subject, axis, claim)` — новое утверждение; `revoke(assertionId)` — событие отмены, **не** delete; `supersede(oldDecisionAssertion, successorRevision)` — event-only link между уже существующими; `reopen(oldRevision) → new ID, D=proposed` — порождает новую ревизию **без transcription**. Жёсткий вопрос по хостингу: где живёт `successorRevision` — он должен существовать в логе **до** события supersede, иначе ссылка висит в воздухе. Ребро только между существующими узлами.

[Teamlead]: Принимаю словарь Структурщика как несущий. И фиксирую границу с C2: axes/enums/subjects **не пересматриваем** — они заморожены вердиктом C2. C4 добавляет только слой событий-переходов поверх immutable assertions. No-inference из C2 обязан пережить и replay, и обе проекции: ни current, ни archive не имеют права породить новое evidence/implication D/L/O. Проекция — это фильтр видимости, никогда не вывод.

[Верстальщик]: Со стороны читаемости добавлю про archive как projection. Archive view = `filter(fold(E), visible=false-or-superseded)` — тот же fold, другой фильтр видимости. По моей норме «один главный акцент»: current — крупно, archive — приглушённый слой того же холста, с явной меткой «supersededBy →» или «revoked at seq N». Никакого второго источника, только вторая линза. Потребитель всегда видит, ПОЧЕМУ токен в архиве.

[Математик]: Формализую archive точно, чтобы не расплылось. Пусть `State = fold(E)`. Тогда `current = π_cur(State) = { a ∈ State : ¬revoked(a) ∧ ¬superseded(a) }`, `archive = π_arc(State) = { a ∈ State : revoked(a) ∨ superseded(a) }`. Обе — чистые проекции одного `State`, `π_cur ∪ π_arc = State`, пересечение пусто. Это гарантирует: archive никогда не evidence — она подмножество тех же assertions, помеченных событием, а не новый вывод.

[Музыкант]: Тут вставлю флаг про декоррелированность — мы это решили в `ritual-inputs-echo`: три отражения одного снимка это `n=1`, не `n=3`. Для C4: если один `reopen` породил цепочку ревизий одного subject, replay не должен считать их независимыми свидетельствами. Но — это уже про C3 content, туда не лезем. Здесь достаточно, что цепочка `reopen→reopen` видна как линия версий, а не как параллельные факты.

[Структурщик]: Верно, и это ведёт к schema evolution — иначе через полгода лог не прочитается. Норма: каждое событие несёт `version`; чтение — через `upcast(eventₖ) : v_old → v_current`, чистая функция без побочных эффектов. Старые события **не переписываются** на диске — upcast применяется на чтении, в памяти. Это сохраняет hard no-delete: файл события immutable, эволюция схемы живёт в коде читателя, не в данных.

[Teamlead]: Принимаю upcast-on-read как единственную легитимную форму эволюции. Это ровно наш паттерн из `code-review-honesty-refactor`: имя должно держать обещание — «no-delete» значит байты события не меняются никогда. Upcast, который переписывает файлы, нарушил бы обещание. Фиксирую: schema evolution = `version` в каждом событии + `upcast` в reader, replay остаётся детерминированным относительно версии читателя.

[Верстальщик]: С точки зрения потребителя upcast должен быть **невидимым, но честным**. Если событие v1 показано через upcast до v3, в audit-view я хочу метку «прочитано через upcast v1→v3» — чтобы человек в PR не думал, что данные родились в v3. Мёртвая дверь наоборот: скрытая трансформация, о которой читатель не знает. Одна тихая строка `font-mono text-xs muted` под событием решает.

[Математик]: Согласен, и это формализуется предикатом. `upcast` обязан быть **идемпотентным на целевой версии**: `upcast_v3(e) = e` если `version(e) = 3`. И **тотальным**: для любой сохранённой версии определён путь до текущей. Без тотальности replay падает на старом событии — это регрессия no-delete через чёрный ход. Проверяется юнит-тестом: для каждой исторической версии схемы существует и однозначен `upcast → current`.

[Teamlead]: Теперь граница, где event sourcing становится избыточным — ключевой вопрос повестки. Моя рамка: мы берём **минимум** ES — append-only лог + fold + проекции + upcast. Мы **не** берём: снапшоты состояния, отдельный write/read-model bus, projection rebuild-инфраструктуру, event versioning framework. Граница проста: если проекция пересчитывается из полного лога за приемлемое время — снапшоты не нужны.

[Математик]: Подтвержу численно, чтобы «приемлемо» не было вкусовым. Стоимость replay = `O(|E|)`. Для C4 порядок величины: события — редкие акты решений/отмен, `|E|` растёт медленно (десятки-сотни в год на subject-space, не миллионы). При `|E| < 10⁴` `fold` укладывается в единицы миллисекунд — снапшот экономит незначимую величину, но добавляет второй source of truth (риск рассинхрона). **Вывод:** снапшоты запрещены до тех пор, пока `replay time` не станет измеримой болью. Это проверяемый порог, не мнение.

[Структурщик]: Тогда зафиксирую границу словарно, чтобы никто потом не притащил лишнее. Внутри C4: `EventLog` (append-only), `replay`, `π_cur`/`π_arc`, `upcast`. **Вне** C4 и запрещено как преждевременная абстракция: snapshot store, CQRS-разделение, event bus, saga. C3 evidence — `opaque`: событие хранит `evidenceRef` (ссылку/id), не content и не validity-criteria. C4 не читает внутрь evidence — только держит ссылку в payload.

[Музыкант]: Поддержу закрытие. Для меня как для того, кто дёргает крайние кейсы: `reopen` без transcription — это честно, старую ревизию не «перезаписываем в новую тональность», а оставляем как есть и стартуем `proposed`-дубль. Никаких магических кнопок, каждый параметр честный. С аудио-стороны блокеров нет, риск-флагов в production не вижу. Принимаю.

[Верстальщик]: Со стороны витрины всё сходится: одна лента, две линзы (current/archive), честная метка upcast, явный `supersededBy`/`revoked`-статус. Мёртвых дверей нет — каждый архивный токен объясняет свой статус. Принимаю.

[Математик]: Инварианты замкнуты: append-only, детерминированный `fold`, тотальный идемпотентный `upcast`, проекции без inference, порог снапшотов численно обоснован. Принимаю.

[Структурщик]: Словарь однозначен, граф без циклов (ребра supersede только между существующими узлами), C3 opaque не протекает. Принимаю.

[Teamlead]: Границы соблюдены, C2 не пересмотрен, C3/C5–C7 не тронуты. Форма следует функции — минимальный ES без декора. LGTM. Принимаю, свожу в вердикт.

## Итоговое решение консилиума

### ВЕРДИКТ C4

| Вопрос | Решение |
|--------|---------|
| **Source of truth** | Единственный — **append-only `EventLog`** immutable-событий с монотонным `seq`. Current и archive — производные (`derived`), не хранятся как самостоятельная истина. |
| **Форма событий (C2 заморожен)** | `assert(subject, axis, claim)`; `revoke(assertionId)` — событие, не delete; `supersede(oldDecisionAssertion, successorRevision)` — event-only link между **уже существующими** узлами; `reopen(oldRevision) → new ID, D=proposed`, **без transcription**. Axes/enums/subjects/scope не пересматриваются. |
| **Hard no-delete** | Correction/revoke/supersede — только append. Байты сохранённого события **никогда** не rewrite/delete. |
| **Deterministic replay** | `current State = fold(E)`, чистая функция; audit = `fold(E[0:k])`. `fold` детерминирован: один `E` → один `State`. Стоимость `O(|E|)`. |
| **Current / Archive views** | Проекции одного `State`: `π_cur = {¬revoked ∧ ¬superseded}`, `π_arc = {revoked ∨ superseded}`; `π_cur ⊍ π_arc = State`. Archive — **visibility projection**, никогда не evidence/implication D/L/O. Сохраняют C2 no-inference, immutable scope, reopen-без-transcription. |
| **Schema evolution** | Каждое событие несёт `version`; чтение через **тотальный, идемпотентный `upcast(e): v_old → v_current`** в reader (upcast-on-read). Файлы событий не переписываются. Юнит-тест: для каждой исторической версии — существует и однозначен путь до current. |
| **C3 evidence** | `opaque`: событие хранит только `evidenceRef` (ссылка/id). C4 не определяет content или validity-criteria. |
| **Граница избыточности event sourcing** | Берём минимум: `EventLog` + `replay` + `π_cur`/`π_arc` + `upcast`. **Запрещены как преждевременные:** snapshot store, CQRS write/read-модели, event bus, saga. Порог снапшотов — численный: только когда измеримое `replay time` станет болью (`|E|` порядка ≫10⁴); до тех пор второй source of truth недопустим. |
| **Границы C4** | C3 evidence content, C5 legacy, C6 workflow/UX/enforcement, C7 operational safety/concurrency — **не решены** (вне повестки). |

**Использованные premises / sources:**
- `C2_VERDICT.md` — заморожённые semantics: typed axes D/L/O/V, immutable subject/id/scope/claims, no-delete constraint, no-inference, reopen без transcription (неподвижный predecessor).
- `C4_TOPIC.md` — формулировка вопроса и требования к вердикту.
- `docs/seanses/truth-graph-unify-or-split-2026-07-17.md` — граф правды как DAG, `derived-from`, узлы/рёбра, декоррелированность origin-hash.
- `docs/seanses/code-review-honesty-refactor-2026-07-16.md` — «имя держит обещание» (обоснование запрета rewrite при заявленном no-delete).
- `docs/seanses/ritual-inputs-echo` (память Kuryokhin) — цепочка производных ≠ независимые свидетельства (граница с C3, не решаем).
- `docs/ARCHITECTURE.md` / `docs/SERVICES.md` — чистые функции, `derived` без побочных эффектов, единая точка истины.

**Definition of Done (если ведёт к задаче в коде):**
- `EventLog` реализован append-only; API не имеет метода удаления/перезаписи события.
- `fold: Event[] → State` — чистая функция, покрыта тестом детерминизма (`fold(E) === fold(E)`).
- `replay(E[0:k])` восстанавливает историческое состояние — тест audit-точки.
- `π_cur` и `π_arc` — чистые проекции; тест `π_cur ∪ π_arc = State`, `π_cur ∩ π_arc = ∅`; тест «archive не порождает нового D/L/O».
- `upcast` тотален и идемпотентен: тест для каждой исторической версии схемы → однозначный current.
- `supersede`/`reopen` не создают transcription; `supersede` ссылается только на существующие узлы (тест «нет висячих рёбер»).
- Событие хранит `evidenceRef` opaque; C4-код не читает внутрь evidence (тест границы).
- Отсутствие snapshot store / event bus в реализации (ревью Структурщика на преждевременную абстракцию).
- UI-проекция (когда появится): метка статуса archive (`revoked`/`supersededBy`) и метка `upcast v_old→v_current` видимы — по DESIGN.md, ревью Верстальщика.
