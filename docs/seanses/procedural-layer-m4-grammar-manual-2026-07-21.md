Посылки оглашены. Три пункта повестки, по очереди.

[Структурщик]: Пункт 1 — грамматика. Беру за основу сходимость отвергнутого черновика и упрощаю до кортежа: `[<персона>/]<kind>/<slug>` плюс ортогональные теги формата. `kind` — закрытый словарь в **одной точке истины** (тот же единый файл правил из M3, посылка 10): `feat, fix, docs, chore, tooling, meeting, storm, cowork, night…` — и этим словарём умирает дубль `feat`/`feature` (посылка 7): `feature` в словарь не входит. Персона-сегмент опционален и легален только из словаря персон виртуальной команды — не из словаря категорий M2, это разные словари (посылка 6, поправка к черновику).

[Математик]: Пункт 2 — каскад. Функция `resolveHolder(explicit, branchPersona) → persona | CONFLICT | MISSING`, тотальная и детерминированная. «Явное» — объединение двух источников с внутренним приоритетом: слово владельца при шипе, иначе `leadPersona` карточки реестра, когда `slug` равен id карточки (посылка 5 — сохраняем каскад шторма K2 внутри понятия explicit, не плодя пятое состояние). Ветви: оба источника заданы и равны → persona; заданы и различны → `CONFLICT`, красный гейт с показом обоих значений; ровно один → он; ни одного → `MISSING`. Оба красных исхода — это возврат на доработку по T8 (посылка 3), не мёртвый блок: гейт говорит, чего не хватает.

[Музыкант]: Держу тег `night` отдельной осью — он не про держателя, он про **жизненный цикл**: заморозка мерджа до флага (посылка 2). В каскад держателя не входит; проверка заморозки — отдельный предикат того же шип-гейта. Смешение осей в один сегмент — то самое, из-за чего в ретроспективе персона потерялась до 1%: каждая ось должна звучать своей дорожкой.

[Верстальщик]: Пункт 3 и видимость. Агент-префиксы (`cursor/`, `codex/`, `claude/`) в новый канон **не входят**: исполнитель — не держатель, его место в `Co-Authored-By` (ось K3 из ретроспективы); исторические ветки остаются как есть (посылка 9). Легаси-якоря (`techies68` и родня) — не входы для новых веток. Шапка шип-гейта рендерит три оси текстом, не цветом: `тип`, `держатель (+источник: слово/карточка/сегмент)`, `заморозка`; исход каскада — явный статус ✓ / CONFLICT / MISSING с подсказкой.

[Структурщик]: Про сирот: персона-сегмент вне словаря персон (исторический `boyarskiy`-случай из черновика) → `MISSING` с подсказкой валидатора, отдельного правила не заводим — словарь один, исключения плодить нельзя.

| Вопрос | Решение |

| Грамматика имени | `[<персона>/]<kind>/<slug>` + ортогональные теги формата; `kind` — закрытый словарь в едином файле правил (M3); `feature` в словарь не входит — дубль умирает |

| Тег `night` | Ортогональная ось жизненного цикла: заморозка мерджа до флага «ночь закончилась»; в каскад держателя не входит; отдельный предикат шип-гейта |

| Видимость | Шапка шип-гейта: `тип · держатель (+источник) · заморозка` текстом; исход каскада — явный статус с подсказкой |
