[Структурщик]: Сначала леммы «зависит», иначе DAG снова смешает два смысла. **Контрактная** зависимость: без вердикта a варианты b не имеют области определения. **Фактурная**: b ждёт артефакт в main/проде, которого ещё нет. Кандидат 9 (готовность к спринту) почти всегда терминал по контракту; кандидат 8 (как локальный агент докладывает usage) часто несёт фактуру (office endpoint). Разведу явно.

[Teamlead]: Рамку M0 фиксирую: только порядок, не «куда класть ключи» и не макет панели. Функция заседания — чтобы исчерпание Anthropic не валило code-review/consilium и чтобы владелец видел экономику дня. Фундамент — тот вопрос, без которого остальные теряют предмет. Мой первый кандидат в корень — **4 (контракт процедуры как единица маршрутизации)**: без стабильного id нечего ни маршрутизировать, ни считать в телеметрии, ни вешать на переключатель панели.

[Математик]: Обозначу V={1..9} как в повестке. Предикат фундамента: убери a — остаётся ли область ответов остальных определённой? Убери 4: у 1 (control plane) нет ключей реестра, у 2 (телеметрия) нет поля procedure, у 3 (панель) нечего переключать, у 5 (fallback) неясна единица отказа. 4 проходит предикат. Убери 7 (scope): остальные ещё формулируемы в общем виде — значит 7 не корень, хотя влияет на ширину API.

[Музыкант]: По тракту сигнала узнаю порядок: сначала имена шин (процедуры), потом куда идёт матрица маршрутизации, потом счётчики на шинах, потом пульт. Флаг из живого эпизода 22.07: code-review и consilium уже горят на одном Anthropic — если scope (7) откроем «все ритуалы» раньше контракта id, получим мёртвые двери в панели на процедуры, которых ещё нет в реестре. Сначала 4, потом узкий 7.

[Верстальщик]: С витрины потребителя (владелец на panel.mmbrn.tech и агент в worktree): без таблицы «procedure id → канал» переключатель — декорация. Рисую узлы: **4** в корне. Дальше жду рёбра, не линейный список из девяти пунктов — иначе в PR снова «всё важно».

[Структурщик]: Рёбра из корня. **4 → 7**: scope v1 («только CR+consilium» vs каркас на всех) оперирует множеством id; множество без схемы id — призрак. **4 → 1**: control plane хранит отображение procedure→channel; ключ отображения = id из 4. Оба ребра контрактные.

[Teamlead]: Принимаю 4→7 и 4→1. Между 7 и 1: можно ли параллелить? Scope сужает, *какие* ключи обязательны в v1, но форма control plane (git / office API / UI-only) ортогональна ширине. Предикат: убери 7 — определён ли ответ на 1? Да, «источник истины маршрутов» формулируется. Убери 1 — определён ли 7? Да, «в v1 жильцы CR и consilium». Ребра 7↔1 нет. Параллель под 4.

[Математик]: Кандидат 2 (телеметрия). Поля события требуют procedure id → **4 → 2**. Куда писать события: если SoT маршрутов на office, лог логично рядом → мягкое **1 → 2**. Жёстко ли 1→2? Если control plane = git-конфиг, а метрики всё равно в office — ребро слабее. Фиксирую обязательное 4→2; 1→2 как «предпочтительное», не блокер параллели 2 с уточнением хранилища внутри существа 2. Для DAG M0: **4 → 2** обязательно; **1 → 2** тоже ставлю — иначе 2 может выбрать хранилище, конфликтующее с SoT из 1.

[Музыкант]: Поддержу 1→2 жёстко. Иначе получим партитуру на двух пультах: маршруты в git, счётчики в никуда — тишина на слух. Кандидат 6 (шов `.env.llm-proxy`): это «куда деть уже лежащий кабель». Без ответа «где живёт политика» (1) шов не решить — **1 → 6**. Не наоборот: experimental не определяет control plane.

[Структурщик]: Кандидат 5 (fallback vs STOP). Кто меняет цепочку при отказе канала — админ панели или только деплой — это свойство control plane. **1 → 5**. Телеметрия (2) желательна для наблюдения failover, но не задаёт область определения политики отказа: политику можно сформулировать без счётчиков. Ребра 2→5 не ставлю как обязательное.

[Teamlead]: Кандидат 8 (как локальный агент докладывает usage). Это транспорт телеметрии с Windows/worktree. Без модели события (2) нечего постить; без адреса SoT/API (1) некуда. **1 → 8**, **2 → 8**. Фактурный привкус: нужен живой endpoint — но это существо 8, не повод поднимать 8 выше 1/2.

[Математик]: Кандидат 3 (панель). UI читает сводку дня ← 2; пишет переключение канала ← 1; показывает/редактирует fallback ← 5. [Музыкант]: Флаг на 3 раньше 2: нарисуем красивый дашборд без контракта события — снова декорация, как drift-борды без provenance. Запрет обратного ранга: 3 до 2 = панель без единиц учёта.

[Верстальщик]: Согласен. На холсте слои: L0: **4**. L1: **7** и **1** веером. L2: **6**, **2**, **5** из 1 (и 2 ещё из 4). L3: **8** из {1,2}, **3** из {1,2,5}. L4: **9**. Стрелки обязательны в артефакте ACTIVE/EPIC.

## Итоговое решение консилиума

| Вопрос | Решение |
|--------|---------|
| Фундамент | **Кандидат 4** — контракт процедуры как единица маршрутизации (стабильный id). |
| Обязательный ранний вердикт | **4** — первым; без него нет ключей маршрута и поля телеметрии. |
| Слой L1 (параллель) | **7 (scope v1)** и **1 (control plane / SoT)** — оба ←4; между собой рёбер нет. |
| Слой L2 | **6** (шов llm-proxy) ←1; **2** (телеметрия) ←4 и ←1; **5** (fallback/STOP) ←1. Между 6/2/5 обязательных рёбер нет. |
| Слой L3 | **8** (доклад usage с локального агента) ←{1,2}; **3** (панель mmbrn.tech) ←{1,2,5}. |
| Терминал L4 | **9** (критерий готовности к спринту) ← {1..8}. |
| Направленный граф | `4 → {7, 1}`; `1 → {6, 2, 5}`; `4 → 2`; `{1,2} → 8`; `{1,2,5} → 3`; `{1..8} → 9`. |
| Риск обратного ранга | Панель до телеметрии/id; готовность спринта до SoT; события без procedure id. |
| Вне заседания | Форма токенов правды; пополнение Anthropic; не решать существо кандидатов в M0. |

**Definition of Done (только вопрос O1):**
- В ACTIVE/EPIC после ратификации владельцем виден граф `4 → {7,1} → … → 9` со слоями и явными параллелями.
- Назван фундамент (4) и параллели L1/L2.
- Порядок комнат Mn следует DAG; внутри параллельного слоя — по слову владельца/ритму кредита.
- Список посылок присутствует в этом же протоколе.
