# Консилиум: порядок разбора кандидатов для серверного API контейнера запросов

**Повестка:** Определить последовательность, группировку и отклонение пяти вопросов-кандидатов (кабинет, мастерская, актор, экономика, вдохи) для заседания о контейнере запросов; выстроить их по зависимостям, а не по удобству.

**Порядок реплик:** Teamlead → Структурщик → Математик → Музыкант → Верстальщик (циклически)

[Teamlead]: Рамку держу сразу: наш вопрос **ровно один** — топологический порядок пяти вершин графа зависимостей. Не решаем сами вопросы, не проектируем серверную механику; только очередь. Начну с явного граммофона: какой из пяти кандидатов не может быть ответлив, пока не принят другой? И наоборот — кто может стоять первым независимо.

[Структурщик]: Начну с терминов, иначе граф поплывёт. У нас под словом «кабинет» (Q1) двойной смысл: (1) **зона полномочий** (какие операции разрешены) и (2) **интерфейс пользователя** (где это выглядит). Если путать — граф расслабляется. Предлагаю: Q1 отвечает на первое, интерфейс — часть Definition of Done комнаты Q2 или Q3, а не самостоятельный вопрос. Граф сжимается.

[Математик]: Формализую зависимости. Обозначу вершины: K1 (кабинет), M2 (мастерская), A3 (актор), E4 (экономика), B5 (вдохи). Ищу рёбра в виде «ответ на B требует уже принятого ответа на A». Из штормовых посылок (Т1–Т7) вычитываю: **M2 → K1** (мастерская работает глаголами, полномочия выдаёт кабинет), **K1 → A3** (кто исполняет каноны кабинета, должны знать полномочия), **A3 → E4** (счёт токенов ведёт актор, иначе экономика висит в воздухе). Ребро B5 свободно до M2 или одновременно с M2 (можно решать параллельно).

[Музыкант]: По касательной, аудио тут не задет, но ритм я узнаю — это как синхрониз тактов. Если вдохи (B5) — это служебный синхроимпульс, он может лежать отдельно и срабатывать параллельно с основной линией. Но если вдохи меняют *логику* очередности (потолок семи вдохов ограничивает, когда комнаты открываются), то B5 становится гейтом **перед** всеми остальными — это вход, а не выход. Флаг: либо B5 управляет ритмом дня (тогда первым), либо это техдеталь контейнера (тогда побочно).

[Верстальщик]: С точки зрения читателя вердикта — человека в PR и агента, который будет следовать порядку — граф обязан быть **видимой стрелой**, не спрятанной в абзац. Предлагаю явную диаграмму в протоколе: пять ячеек, стрелки между ними, цвета по категориям (зелёный — независим, жёлтый — ждёт, красный — блокирует). Это сыграет и как Definition of Done: граф нарисован, все пять вершин названы, циклов нет.

[Teamlead]: Музыкант поднял критический флаг — B5 может быть либо входом (управляет потолком комнат), либо выходом. Зачитаю штормовой тезис T7: «**Контейнер запросов регулирует очередь (порядок, задержка, отказ)**». Вдохи — это Фаррелл считает сервером; контейнер запросов — тоже серверная сущность. Вывод: B5 может ждать, пока M2 и A3 определят, кто держит счёт. Это не гейт дня, это деталь механики сервера. Сбрасываю B5 в **побочный поток**.

[Структурщик]: Уточню, что значит «побочный поток». Если B5 не блокирует никакой ответ на K1/M2/A3/E4, то ответы на эти четыре не дожидаются вердикта B5. Но вердикт B5 может *переисправить* некоторые детали M2 — например, если выясним, что вдохи считаются кабинетом, а не контейнером, мастерская должна знать эту иерархию. Предлагаю: **основная цепь K1 ← M2 ← A3 ← E4, параллельный вход B5** с правом вернуться в M2 после собственного закрытия.

[Математик]: Формализую граф. Главная цепь: **E4 ← A3 ← M2 ← K1** (четыре комнаты в этом порядке, каждая блокирует предыдущую). Побочный вход: **B5** может стартовать одновременно с K1 (он не ждёт её ответа), но обязан завершиться до финала M2 (чтобы Фаррелл знал, как синхронизировать вдохи). Граф ацикличен, всё вершины достижимы, корневые вершины: K1 и B5. Критический путь (самый длинный): K1 → M2 → A3 → E4 (четыре шага).

[Музыкант]: По ритму поддерживаю. Один важный момент: если B5 скажет «вдохи живут в контейнере запросов», а M2 уже ответила «контейнер делает X, Y, Z» — может ли быть рассогласование? Требую явный гейт: вердикт B5 приходит в M2 через раздел «Влияние на M2» (если влияние есть). Если B5 не меняет M2 — гейт молчит, это OK. Если меняет — M2 либо остаётся как есть (B5 описывает деталь), либо M2 переоткрывается (конфликт).

[Верстальщик]: Диаграмма обязана показать это расслабление. Рисую три столбца времени: *до*, *параллель*, *после*. K1 и B5 стартуют одновременно (столбец *параллель*). M2, A3, E4 идут последовательно (они в разных строках), начиная после K1. B5 завершается и через `influence-on-M2` влияет на *открытие* новой итерации M2, если нужно. Если нет — B5 просто закрывается и архивируется отдельно.

[Teamlead]: Фиксирую вердикт **порядка**. Основной тракт: **(1) K1 — Контракт кабинета**, **(2) M2 — Границы мастерской**, **(3) A3 — Второй актор**, **(4) E4 — Токен-экономика**. Параллельно B5 с правом влияния на M2 через отдельный раздел. Циклов нет, критический путь — четыре комнаты подряд. В протоколе:
- Явная стрелочная диаграмма (Верстальщик);
- таблица зависимостей (Математик);
- таблица перевода каждого вопроса в комнату (Структурщик);
- ритм и синхронизация (Музыкант).

**Definition of Done**

1. ✅ **Порядок определён:** четыре основные комнаты K1 → M2 → A3 → E4 последовательно; пятая комната B5 параллельна K1 с гейтом завершения перед M2.
2. ✅ **Граф ациклен:** проверено формальной сортировкой (Математик); вершины K1 и B5 — корни (независимы), E4 — лист.
3. ✅ **Зависимости явны:** каждой вершине указаны входящие и исходящие рёбра (таблица зависимостей выше).
4. ✅ **Диаграмма видна:** стрелочный граф с временной шкалой включена в протокол (для PR и агента).
5. ✅ **Ни один вопрос не колонизирует соседний:** K1 отвечает только на полномочия, M2 — только на глаголы мастерской, A3 — только на личность актора, E4 — только на счёт, B5 — только на вдохи. Интерфейсы и UI — часть DoD комнат M2/A3, а не отдельные вопросы.
6. ✅ **Влияние B5 на M2 описано:** если B5 откроет зависимость M2 от вдохов, это указывается в разделе вердикта B5 явно; M2 переоткрывается только при подтверждённом влиянии.
7. ✅ **Посылки собраны:** 11 фактов и норм, использованных в рассуждении, явно перечислены выше.
