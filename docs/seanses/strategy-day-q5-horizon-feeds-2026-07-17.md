## Итоговое решение консилиума

| Вопрос | Решение |
|--------|---------|
| Как горизонт наполняется внешним знанием при оборванных каналах? | Один читатель `collectHorizonInputs(horizon, {insights, research}) → {highlights[], provenance[]}` — чистая функция в `scripts/lib/`; флаги `includeInsights`/`includeAnalyzersResearch` уходят из замороженной недельной в параметры этой функции (один источник правды) |
| Два канала — как не смешать? | Две леммы с разными доменами: `insights: memory[] → highlight[]` (внутреннее свидетельство), `research: horizon → question[3] → finding[]` (внешнее знание). Оба вливаются **только как highlight**, никогда как assign/DoD (посылка Q1) |
| Эхо одного снимка (06.07 в трёх местах)? | Не третий канал, а устаревшее свидетельство. Дедуп по `origin-hash = sha(source@commit)` схлопывает три строки в `n=1`; снимок растворяется в общем механизме, спецкода нет |
| Свежесть / stale | TTL-предикат привязан к **вехе горизонта** (не к календарным дням, согласовано с Q2 `d/dt=0`): `age > веха → highlight помечен stale`, вес падает, Верстальщик рисует приглушённо |
| Гейт формулировки ресёрча | `externalizeQuery(q) → q'` перед отправкой в Perplexity: словарь запрещённого жаргона (`ADR`, имена пакетов, `MAIN_DAY_ISSUE`, `night-hunt`) → замена или явный FAIL с читаемым сообщением. Устраняет урок 17.07 (ЕГРЮЛ/ADR) |
| Цепочка ресёрча | `pickTopic` (Q4b, 3 вопроса из горизонта) → `externalizeQuery` → Perplexity → `nightYield` (finding→архив с origin) → утром `collectHorizonInputs` подхватывает как highlight. Ночь наполняет, утро осмысляет |
| Хостинг / границы | Организационный слой (`scripts/lib/` + чистое ядро дедупа), пакетный граф не трогается; модуль читает и дедуплицирует, но НЕ решает, что станет задачей |
| UI горизонта | Шапка «gate+phase» → «критерии (веха)» → «входы»: группа `подсветки` (бейджи источника `инсайт`/`ресёрч`/`снимок` + свежести, stale приглушены) отдельно от `task_refs`. Highlight никогда не выглядит как кнопка действия |

**Definition of Done (ведёт к эпику):**
- `collectHorizonInputs` — чистая функция, детерминированная (один вход → один выход), тест без сети.
- Дедуп: вход с тремя строками одного `origin-hash` → выход `n=1` (тест на эхо 06.07).
- `externalizeQuery` ловит словарь жаргона на фикстуре (`ADR`, имя пакета) → FAIL с читаемым сообщением; отклонённый вопрос — видимое состояние, не тихий сбой.
- stale-предикат срабатывает при `age > веха` текущего горизонта; помеченный highlight рисуется приглушённо с бейджем даты.
- Флаги `includeInsights`/`includeAnalyzersResearch` удалены из тела недельной, стали параметрами `collectHorizonInputs`; один читатель.
- Все входы вливаются как `highlight`, ни один канал не имеет права на `assign` (проверка границы в ревью).
- UI горизонта: `highlight` и `task_ref` — визуально разные леммы; бейджи источника и свежести присутствуют.
- Пакетный граф `ARCHITECTURE.md` не изменён (только `scripts/lib/` + ядро дедупа).
