[Математик]: Формализую B1, иначе «собирает кодом» останется вкусовым. Есть две функции, разделённые стенкой M2-K: `frame: () → Slot[]` — тотальная, детерминированная, `|Slot[]| = 5`, порядок константный; и `fill: (Slot, Context) → Text` — недетерминированная (LLM), но **не влияющая на множество и порядок слотов**. Дефект Д2 — это ровно нарушение инварианта: сейчас LLM владеет и каркасом, и наполнением, поэтому роняет слоты. Требование численно: `assert(output.sections.map(s => s.id) === frame().map(s => s.id))` — порядковое равенство id, а не «похоже».

[Структурщик]: Начну с термина, иначе «канон» поплывёт. Под «сборкой канона» я понимаю **конкатенацию кодом**: `assemble: (Slot[], Map<SlotId, Text>) → Document`, где заголовки берутся из `frame()`, а не из выхлопа LLM. Это разводит две леммы, которые Д2 слил: **скелет** (владение кодом) и **наполнение** (владение LLM). Модель вообще не видит и не пишет заголовки — она получает `fill(slot)` по одному слоту и возвращает только тело. Заголовок «Список посылок» — это `Slot` с фиксированным id, не текст, который модель может уронить.

[Teamlead]: Рамку фиксирую сразу, чтобы B не сполз в G (гейты) и H (хозяйка). Мы решаем **ровно** B1: код собирает 5 секций, LLM наполняет по одному слоту; и как подписывается ручная чеканка. По конструктивизму «форма следует функции»: заголовок оправдан тем, что он несущая стенка секции, а не декор — значит его владелец код, не модель. Вердикт M2-K не переоткрываем, мы его **применяем**: `frame()` уже в main, надо перестать давать LLM единый промпт-скелет. Принимаю направление Математика и Структурщика.

[Музыкант]: По касательной, аудио тут не задет, но структуру сет-листа я узнаю — это как жёсткий порядок номеров концерта, где импровизация живёт **внутри** номера, не в порядке номеров. Мой живой флаг из `ritual-inputs-echo`: не давать «магическую кнопку», которая молча роняет номер из сет-листа. Сейчас именно она — LLM молча теряет 3 из 5 секций, и никто не слышит провал. Требую: если `fill(slot)` вернул пусто/мусор — это **слышимая** ошибка (слот помечен `fillFailed`), не тихая дыра.

[Верстальщик]: С точки зрения потребителя плана — а потребитель это агент-исполнитель утром и человек в PR — пять секций должны читаться как пять устойчивых заголовков с фиксированным порядком, без «плавающей» нумерации и без хвостов. Моя норма из `main-day-accuracy`: смешанный поток = мёртвая дверь. Если заголовок «Список посылок» приделан хвостом-подражанием чеканке владельца — читатель не понимает, где кончается канон и начинается импровизация. Заголовки — из `frame()`, дословно, без права модели их трогать.

[Математик]: Уточню контракт `fill`, чтобы флаг Музыканта был проверяемым, а не декларативным. `fill(slot, ctx) → {text, status}`, где `status ∈ {filled, empty, malformed}`. Пустой слот — не «ошибка сборки»: `assemble` всё равно выдаёт документ с 5 секциями, но секция несёт видимый маркер `⚠ слот не наполнен`. Инвариант завершимости: `assemble` тотальна — при любом наборе `fill`-результатов возвращает документ с ровно 5 заголовками. Модель не может уронить структуру, только качество тела одного слота.

[Структурщик]: Согласен, и зафиксирую границу пакета, иначе «генератор» расплывётся по скриптам. `frame()` и `assemble()` — чистые функции, место им в `scripts/` рядом с генератором main-day-issue (это ops-контур Математика, не пакет `@membrana/*`). Публичный контракт: `frame(): Slot[]`, `fill(slot, ctx): FillResult`, `assemble(slots, fills): string`. LLM вызывается **только** внутри `fill`, по одному слоту — это единственная точка недетерминизма. Никаких «слить весь скелет в один промпт» — это и есть корень Д2.

[Teamlead]: Теперь ручная чеканка (Д4) — это второй вопрос эха, не забываем. Форма: ручная чеканка владельца — это **та же структура из 5 секций**, но `fill` для слотов выполнил человек, не LLM. Значит подпись не про «обойти стража», а про **атрибуцию источника**: `author ∈ {llm, human}` на уровне документа + след в провенансе. Страж не должен бить по `author=human`, если структура (5 секций из `frame()`) соблюдена. Принимаю: подпись = метаданные источника + гейт стража проверяет **структуру**, а не авторство.

[Математик]: Поддержу и формализую подпись, иначе «подписал руками» останется вкусовым. Подпись — детерминированная функция над документом: `sign(doc, author) → {digest: sha256(canonicalize(doc)), author, signedAt}`. Страж проверяет два независимых предиката: `structOk(doc) = (doc.sections.map(id) === frame().map(id))` и `digestOk(doc) = (recordedDigest === sha256(canonicalize(doc)))`. `author=human` **не** является основанием для блока — блокирует только `¬structOk ∨ ¬digestOk`. Это снимает Д4 без дыры: подделать нельзя (дайджест), а честную ручную чеканку не бьёт.

[Музыкант]: Флаг риска на failover: если LLM-провайдер `fill` упал, это не должно молча превращаться в `human`-чеканку и наоборот. `author` фиксируется **явно** на входе прогона (кто наполнял слоты), не выводится задним числом. Как в записи: если дорожку писал живой музыкант, а не семплер — это в метаданных сессии с самого начала, не дорисовывается на мастеринге. Все попытки `fill`, включая упавшие, ложатся в провенанс слота.

[Верстальщик]: Со стороны витрины и PR: `author=llm` и `author=human` не должны выглядеть одинаково серым — это разные двери. Бейдж источника у документа: `human` — акцентная метка «ручная чеканка владельца», `llm` — нейтральная «сгенерировано». Слот со `status: empty/malformed` — видимый `⚠`, не пустая строка (иначе читатель думает, что так и задумано). `aria-live` не требуется — это статичный md-артефакт, но иконка+текст обязательны, не только цвет. Мёртвая дверь чинится именно честностью маркеров.

[Структурщик]: Один термин уточню, чтобы страж и подпись не слиплись. **Страж** — предикат `gate: Document → pass ⊕ fail(reasons)`, чистый, без сети. **Подпись** — артефакт метаданных, вход для стража, не сам страж. Их нельзя слить: страж читает и `structOk`, и `digestOk`, но `sign` их только производит. И вещдок-патч из журнала 21.07 (при отказе гейта выхлоп → `MAIN_DAY_ISSUE.rejected.md`) — оставляем: это диагностика, не обход.

[Teamlead]: Свожу к решению. Сборка секций: `frame()` собирает 5 заголовков кодом, LLM вызывается только через `fill(slot)` по одному слоту, `assemble()` конкатенирует — модель не владеет ни множеством, ни порядком, ни текстом заголовков. Подпись ручной чеканки: `sign(doc, author)` пишет `{digest, author, signedAt}`; страж проверяет `structOk ∧ digestOk`, `author=human` не блокирует. LGTM на форму. Секция «Список посылок» — слот с фиксированным id, не доверяется LLM.

[Математик]: Принимаю. Инвариант завершимости и два предиката стража численно проверяемы, дыры Д4 нет.

[Музыкант]: Принимаю. Тихих дыр нет: упавший `fill` — слышимый маркер, `author` фиксируется на входе.

[Верстальщик]: Принимаю. Заголовки дословно из `frame()`, маркеры источника и пустых слотов честны — мёртвых дверей нет.

[Структурщик]: Принимаю. Границы чисты: `frame`/`fill`/`assemble`/`sign`/`gate` — разные леммы, LLM только в `fill`.

## Итоговое решение консилиума

| Вопрос | Решение |
|--------|---------|
| **Сборка секций** | `frame(): Slot[]` собирает **5 секций кодом** (id + заголовки, константный порядок). LLM вызывается **только** через `fill(slot, ctx): {text, status}` по одному слоту. `assemble(slots, fills): string` конкатенирует заголовки из `frame()` + тела из `fill`. Модель не владеет множеством, порядком и текстом заголовков. Дефект Д2 закрыт: единый промпт-скелет запрещён. |
| **Подпись ручной чеканки** | `sign(doc, author): {digest: sha256(canonicalize(doc)), author ∈ {llm,human}, signedAt}`. `author` фиксируется на входе прогона, не выводится задним числом. Страж = чистый предикат `structOk(doc) ∧ digestOk(doc)`; `author=human` **не** основание для блока. Дефект Д4 закрыт без дыры. |
| Пустой/битый слот | `assemble` тотальна: всегда 5 секций; `status: empty/malformed` → видимый маркер `⚠`, не тихая дыра. |
| Витрина/PR | Бейдж источника: `human` — акцентная «ручная чеканка», `llm` — нейтральная; иконка+текст, не только цвет. |
| Место кода | `frame`/`fill`/`assemble`/`sign`/`gate` — чистые функции в `scripts/` (ops-контур), не пакет `@membrana/*`. Вещдок-патч `MAIN_DAY_ISSUE.rejected.md` сохраняется как диагностика. |
| Границы (Структурщик) | `frame`/`fill`/`assemble`/`sign`/`gate` — разные леммы; единственная точка недетерминизма — `fill`. |

**Definition of Done:**
- `frame()` возвращает ровно 5 `Slot` с константными id и заголовками (включая слот «Список посылок»); юнит-тест на порядок и состав.
- Генератор вызывает `fill(slot)` пер-слот; единый промпт-скелет удалён; тест: `output.sections.map(id) === frame().map(id)`.
- `fill` возвращает `status ∈ {filled, empty, malformed}`; пустой/битый слот рендерится маркером `⚠`, документ всё равно несёт 5 секций.
- `sign(doc, author)` пишет `{digest, author, signedAt}`; страж блокирует только `¬structOk ∨ ¬digestOk`; `author=human` при валидной структуре проходит.
- `MAIN_DAY_ISSUE.rejected.md` пишется при отказе гейта.
- Тест ручной чеканки: honest `author=human` с валидной структурой → pass; подмена дайджеста → fail.
