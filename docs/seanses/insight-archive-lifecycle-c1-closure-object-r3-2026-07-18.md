[Математик]: Формализую, иначе «closable» останется вкусовым. Границей закрытия я называю предикат `boundary(id) → sliceSet` — конечное множество срезов, которые этот id обязуется покрыть. Ключевое: `id` и `boundary` неизменяемы, а факты о покрытии — переменные во времени. Мы дважды путали эти две категории: неизменяемую identity и изменяемую оценку. Non-implication — это утверждение о независимости предикатов, не о переходах: переходы — C2.

## Итоговое решение консилиума

### ВЕРДИКТ C1

**1. Closable-сущности с immutable identity и границами закрытия**

| Сущность | Immutable identity | Граница закрытия |
|----------|--------------------|-------------------|
| **INSIGHT** | `id` инсайта, сохраняется через любые ревизии — терять запрещено | содержательный узел; закрывается как объект, не как факт |
| **MANDATE** (мандат-ревизия) | собственный `id`; каждая новая фаза = новый id | `boundary(MANDATE) = {slice…}` — конечное множество срезов-обязательств |
| **SLICE** | идентификатор среза внутри границы мандата | единица покрытия; закрывается как объект по своей границе |

Состояние сущности — монотонный предикат `open → closed` (необратимый для данного id). Переходы и prerequisites — **не** здесь (C2).

**2. Четыре разных факта и их носители (без технической схемы)**

| Факт | Носитель (домен определения) |
|------|------------------------------|
| `accepted` | пара (актор, **MANDATE**) — мандат принят к исполнению |
| `transcribed` | **INSIGHT** — содержание зафиксировано/расшифровано |
| `delivered` | **SLICE** — срез передан |
| `outcome-proven` | **SLICE** — по срезу доказан результат |

Разные домены ⇒ четыре независимых предиката. Факт **не** является closable object.

**3. Два типа покрытия и допустимые non-implications**

- `delivery-covered(slice)` — срез сдан/передан.
- `outcome-proven(slice)` — по срезу доказан результат.
- Ни один не является универсальным `covered`; ни один не выводится из другого.
- Non-implications (только независимость, без переходов/prerequisites):
  `accepted ⇏ transcribed`, `transcribed ⇏ delivered`, `delivered ⇏ outcome-proven`.

**4. Partial delivery, partial outcome, новая фаза**

- Partial delivery = часть срезов границы имеет `delivered`; partial outcome = (иное) подмножество имеет `outcome-proven`. Подмножества независимы: partial delivery ⇏ partial outcome.
- Ни то, ни другое не есть закрытие MANDATE как объекта.
- **Новая фаза = новый immutable mandate/revision с новым ID.** Старый MANDATE и его оценки **не** пересчитываются задним числом; identity INSIGHT сохраняется.

**5. Исчерпывающий список реально использованных посылок C1**

1. Отдельный заголовок `ВЕРДИКТ C1`.
2. Сначала сущности с immutable identity и границами, затем факты; факт ≠ closable object.
3. Identity INSIGHT нельзя терять.
4. Только три non-implications, без prerequisites и переходов.
5. Различать `delivery-covered(slice)` и `outcome-proven(slice)`; ни один не универсальный `covered`; не выводить один из другого.
6. Новая фаза = новый immutable mandate/revision без мутации прошлого.
7. Запрет на hash / обязательные поля / git-команду / storage / evidence schema (C3+).
8. Запрет классификации Hermes / Comms / Telegram / Persona (C5).
9. State machine / evidence contract / history/storage / CLI — C2–C7.

(Аналогии Музыканта и метафоры Верстальщика — иллюстрации, **не** логическое основание.)

**6. Явно оставлено открытым**

| Узел | Оставлено |
|------|-----------|
| **C2** | prerequisites, переходы, state machine между состояниями/фазами |
| **C3** | hash, обязательные поля, evidence schema, evidence contract |
| **C4** | (git-команда / storage-механика, если отнесена сюда) |
| **C5** | классификация Hermes, Comms, Telegram, Persona |
| **C6** | CLI |
| **C7** | history/storage lifecycle |

| Вопрос | Решение |
|--------|---------|
| Отделены ли immutable closable-сущности от фактов? | **Да** — INSIGHT/MANDATE/SLICE как объекты; accepted/transcribed/delivered/outcome-proven как факты о них |
| Разведены ли delivery- и outcome-coverage? | **Да** — два предиката, ни один не выводится из другого, нет универсального `covered` |
| Новая фаза мутирует прошлое? | **Нет** — новый immutable mandate/revision с новым id; identity INSIGHT сохранена |
| Затронуты ли C2+? | **Нет** — переходы, schema, storage, CLI, классификация персон явно оставлены открытыми |
