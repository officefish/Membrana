# Заседание M3: оси декомпозиции реестра задач

[Структурщик]: Начну с того, что уже стоит и работает. `yarn tasks:decompose` раскладывает
активный реестр по категориям из `scripts/tasks-decompose.config.json`, и там действует норма
Р5: менять конфиг, не код. Обязателен markdown-вывод, карточки без категории показываются
строкой «ВНЕ КАТЕГОРИЙ», а не прячутся. Это не черновик, это живая механика — значит вопрос не
«какие оси придумать», а «как добавить оси, не сломав то, что есть». Категория остаётся осью
по умолчанию; всё новое встаёт рядом, а не вместо.

[Математик]: Разделю оси по природе входа, иначе получим кашу из разных типов.

Первая природа — **поля карточки**: `size`, `sprintKind`, `leadPersona`, `status`. Это готовые
дискретные значения, раскладка по ним детерминирована и не требует ничего, кроме реестра.
Вторая — **производные от полей**: возраст считается из `createdAt` и текущей даты, значит
нужны корзины, а границы корзин — конфиг, не код. Третья — **производные от связей**: состояние
сцепки вычисляется из полей, но осмысленно только вместе с вердиктом валидности из M4B.

Отсюда прямой ответ на подвопрос повестки: ось «здоровье» **вводит зависимость от M4B** и в
первый заход не идёт. Ось «связи» идёт, потому что смотрит на **наличие** полей (`githubIssue`,
`linearId`, `promptPath`), а не на их истинность. Наличие — факт реестра, истинность — работа
валидности. Эту границу размывать нельзя.

[Верстальщик]: Со стороны читателя вывода — одна ось за прогон. Комбинация двух осей даёт
матрицу, а матрица в markdown читается плохо и не помещается в терминал. Нужна матрица — это
два прогона и сверка глазами, но не `decompose`. И обязательно сохранить «ВНЕ КАТЕГОРИЙ» как
общее правило для **любой** оси: карточка без значения по оси не исчезает, а показывается
отдельной строкой. Иначе раскладка врёт молчанием — самый дорогой вид вранья в отчёте.

[Музыкант]: По касательной, но по делу: ось — это то, по чему вещь звучит по-разному. Если
две оси дают одинаковую раскладку на живом реестре, вторая — не ось, а синоним. Проверять это
надо не рассуждением, а на реальных карточках: если `sprintKind` и `category` совпадают на 90%,
держать обе — самообман. Правило: новая ось входит в набор, только если на текущем реестре даёт
раскладку, отличную от существующих.

[Teamlead]: Свожу. Оси задаются флагом `--by`, значения и параметры — из конфига, не из
кода: это продолжение нормы Р5 и оно же позволяет добавить ось без правки скрипта. Ось по
умолчанию — `category`, чтобы нынешние вызовы не сломались. Одна ось за прогон. «Здоровье»
откладываем до вердикта M4B и заводим follow-up, а не пункт этого вердикта. LGTM.

## Итоговое решение консилиума

| Вопрос | Решение |
|--------|---------|
| **Канонический набор осей** | `category` (по умолчанию, нынешний конфиг), `size` (`XS/S/M/L`), `age` (корзины из конфига), `lead` (`leadPersona`), `kind` (`sprintKind`), `links` (наличие `githubIssue` / `linearId` / `promptPath`) |
| **Механизм задания** | Флаг `--by <ось>`; допустимые значения и их параметры (границы корзин возраста, порядок групп) — в `scripts/tasks-decompose.config.json`. Норма Р5 сохраняется: новая ось — правка конфига, не кода |
| **Ось по умолчанию** | `category` — поведение `yarn tasks:decompose` без флага не меняется |
| **Комбинации осей** | **Запрещены.** Одна ось за прогон: матрица не читается в markdown и не помещается в терминал |
| **Ось «здоровье»** | **Отложена** до вердикта M4B: требует суждения валидности, а не факта реестра. Заводится follow-up, в этот вердикт не входит |
| **Граница «наличие против истинности»** | `links` смотрит **наличие** полей (факт реестра) и остаётся в `decompose`; **истинность** связей (жива ли иссью, соответствует ли Linear) — работа валидности, в `decompose` не тянется |
| **Инвариант вывода** | Правило «ВНЕ КАТЕГОРИЙ» распространяется на **любую** ось: карточка без значения показывается отдельной строкой, а не исчезает |
| **Критерий новой оси** | Ось входит в набор, только если на текущем реестре даёт раскладку, отличную от существующих; совпадающая — синоним, а не ось |

**Definition of Done:**

- [x] `--by` реализован; без флага поведение прежнее (`category`).
- [x] Оси `size`, `age`, `lead`, `kind`, `links` работают на живом реестре.
- [x] Параметры осей вынесены в конфиг; код осей не знает.
- [x] Строка «ВНЕ КАТЕГОРИЙ» выводится для каждой оси; тест на карточку без значения.
- [x] Комбинация осей отклоняется с внятным сообщением, а не молча берёт первую.
- [x] Follow-up на ось `health` заведён и ссылается на вердикт M4B ([#1104](https://github.com/officefish/Membrana/issues/1104) → после #1061).
