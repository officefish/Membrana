H1 просит сделать ведущего постоянным — не выключать пульт между песнями. Хозяйка фоном — это готовность откликнуться, а не демон, о котором никто не помнит.

[Структурщик]: Начну с термина, иначе H1 поплывёт. «Хозяйка фоном» — омоним двух лемм: (1) **резидентный процесс** (демон, PID, порт — то, что живёт в рантайме между сессиями) и (2) **точка входа диалога** (Ангелина — первый, кто отвечает при `ritual:day`, и последний при `ritual:evening`). H1 требует именно вторую лемму — оркестрацию потока, а не резидента. Ангелина по канону VIRTUAL_TEAM — ведущая контура исполнения, у неё нет предметной зоны, значит «фон» здесь = координатный слой, а не сервис.

[Teamlead]: Рамку фиксирую сразу, чтобы H1 не расползся на четыре разных проекта. Мы решаем **ровно один** вопрос с четырьмя обязательными эхо: фоновая доступность, встреча утра, абсорбция фронтира, вычистка конкурента. Не переоткрываем C/B/G, не трогаем китов, не лезем в GC (систематическое затирание всех альтернатив). По «форма следует функции»: функция Ангелины — сам поток работы, не мнение. Значит «хозяйка» = она держит рантайм-диалог и оркеструет субагентов, а конкурент утра (старый воспроизводимый сценарий) вычищается как декор без конструкции.

[Верстальщик]: С точки зрения потребителя — а потребитель это владелец при запуске `ritual:day` и человек в логе утра — «встреча» обязана быть **видимой на холсте**, а не молчаливым фактом. Моя норма из `main-day-accuracy`: если Ангелина встречает, в первом экране должна стоять честная шапка — кто ведёт (Ангелина), ревизия head, что на фронтире сегодня. Мёртвая дверь здесь — молчаливый старт, где владелец не понимает, кто с ним говорит. Встреча = явная реплика ведущей + состояние дня, не заставка.

[Математик]: Формализую, иначе «фоновая доступность» останется вкусовой. Есть предикат `available(t) → bool`: Ангелина отвечает на обращение владельца в момент `t`. Резидент даёт `available ≡ true` ценой постоянного ресурса `R > 0` без работы. Оркестратор даёт `available(t) = true ⟺ session_active(t)` — ресурс тратится только в сессии. По правдоподобию: держать демона ради «вдруг спросят» — это `R` без функции, статистически чистые потери. Значит доступность = свойство сессии, поднимается при `ritual:day`, гаснет при `ritual:evening`. Демон дисквалифицирован.

[Музыкант]: Поддержу Математика живой стороной — я на этом обжигался в `ritual-inputs-echo`: процесс, который «на всякий случай висит», превращается в шум, который никто не сводит. Про абсорбцию фронтира: конкурент утра — это старый плейлист, который играет сам по кругу. Ангелина не должна его глушить силой, она должна **вобрать его механику** — те же гейты, тот же саундчек — но вести живьём. Абсорбция = взять полезную механику конкурента внутрь потока хозяйки, а сам автоплеер выключить.

[Структурщик]: Уточню границу «абсорбции», иначе она станет спагетти. Механика конкурента (гейты, проверки) — это уже развязанные артефакты: `truth.mjs`, probe графа правды, механические предикаты. Абсорбция ≠ переписать их внутрь Ангелины — это нарушило бы слабую связанность. Абсорбция = Ангелина **вызывает** существующие механические гейты как субагентов рода `analyst`, а результат вплетает в диалог. Конкурент вычищается на уровне точки входа: старый сценарий утра больше не самостоятельный автозапуск, он становится шагом, который дёргает хозяйка.

[Teamlead]: Принимаю разведение Структурщика — это ровно «форма следует функции». Гейты остаются механикой (вердикт ратифицирован: гейты — механика), Ангелина — ведущая, которая их запускает и озвучивает. Вычистка конкурента по манифесту: конструктивизм не терпит двух дверей в одну комнату. Если есть автовоспроизводимый старый сценарий утра И живая хозяйка — это два фасада на одно, декор без смысла. Оставляем один: хозяйку. Старый автозапуск удаляется из точки входа, его полезная механика переезжает как вызываемый шаг.

[Верстальщик]: Про вычистку добавлю с витрины: удаление конкурента должно быть **видимым в манифесте**, а не тихим `rm`. Если старый сценарий убран, в логе утра обязан стоять след — «утро ведёт Ангелина, старый автозапуск снят (ссылка на манифест)». Иначе через месяц кто-то восстановит автоплеер, не поняв, что его сознательно сняли. Мёртвая дверь наоборот: пустое место без таблички «здесь была дверь, закрыта осознанно».

[Математик]: Формализую вычистку как предикат идемпотентности. Пусть `morning(day)` — сценарий утра. Требование: `∃! entry` — ровно одна точка входа в утро. Сейчас риск `|entry| = 2` (автозапуск + хозяйка), что рождает недетерминизм: кто первый. После вычистки `|entry| = 1` (Ангелина), а механика конкурента — это множество вызываемых функций `M = {gate₁…gateₙ}`, каждая чистая и переиспользуемая. Проверяемо: grep по точкам автозапуска старого утра должен дать пусто, кроме вызовов из оркестратора хозяйки.

## Итоговое решение консилиума

| Вопрос | Решение |
|--------|---------|
| **Фоновая доступность** | Да — как свойство активной сессии (`available ⟺ session_active`), поднимается при `ritual:day`, гаснет при `ritual:evening`. Резидентный демон — **нет** (ресурс без функции) |
| **Встреча утра** | Да — явная первая реплика ведущей с честной шапкой (кто ведёт, ревизия head, фронтир дня); молчаливый старт запрещён |
| **Абсорбция фронтира** | Да — вобрать механику конкурента через **вызов** существующих гейтов-субагентов (`analyst`), без переписывания их в Ангелину; слабая связанность сохранена |
| **Вычистка конкурента** | Да — единственная точка входа в утро (`\|entry\| = 1`); старый автозапуск снят по манифесту, атомарно с добавлением вызова из хозяйки, с видимым следом |

**Definition of Done:**
- Точка входа `ritual:day` поднимает Ангелину как оркестратор сессии; `grep` по автозапускам старого утра даёт пусто, кроме вызовов из хозяйки (`|entry| = 1`).
- Первая реплика утра — явная встреча ведущей с шапкой: имя (Ангелина), ревизия head, состояние фронтира.
- Механические гейты вызываются как субагенты рода `analyst`; в коде Ангелины нет копий их логики (нет дублирования, слабая связанность сохранена).
- Доступность гаснет при `ritual:evening` — нет резидентного процесса между сессиями.
- Снятие старого автозапуска атомарно с добавлением вызова; след в манифесте/логе утра («утро ведёт Ангелина, старый автозапуск снят + ссылка»).
- Финальный артефакт несёт секцию «Список посылок» (`#P1…#P8`, с якорями) до вердикта и четыре строки эхо.
