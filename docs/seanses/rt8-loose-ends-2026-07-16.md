**Definition of Done (A1, сейчас):**
- `isFresh(artifact)` и `dateOf(artifact)` — чистые функции, `today` инъектируется параметром (без мока системной даты).
- Тест на оба пути: свежий артефакт (сегодня) проходит; вчерашний падает с внятным exit-кодом и человекочитаемым сообщением («устарел на N дн.»).
- `dateOf` читает дату из фронтматтера `DAILY_CODE_REVIEW.md`, не из mtime.
- Явный массив `NON_CRITICAL_STEPS` в скрипте ритуала; все шаги вне списка теряют `|| true` и падают громко.
- Downstream (standup, main-day-issue) проверяет свежесть на входе, падает явно при несвежести.
- Сообщение об ошибке читаемо: что устарело + на сколько дней (UI-принцип Rodchenko, применим и к CLI).

**Definition of Done (A2, карточка, следующая волна):**
- Поле `precision` принимает два значения: `exact` | `working-tree`, детерминированно выводимое из `diffAvailable`.
- Любой рендер метки: badge не только цветом + строка-объяснение «дифф недоступен, показано текущее дерево».
- Карточка зарегистрирована в реестре сейчас (регистрация до кода — канон lifecycle).
