Техническое задание: Внедрение удалённого выполнения AI-задач с использованием мощного ПК и пробуждения по расписанию

1. Цель
   Обеспечить выполнение вечернего code-review и ночных спринтов (рефакторинг, создание Pull Request'ов) с использованием локальных LLM-моделей (Ollama) без затрат на токены, при этом основная вычислительная нагрузка переносится на удалённый мощный ПК с 32 ГБ ОЗУ, который может находиться в спящем режиме для экономии электроэнергии. Запросы от локального ноутбука разработчика направляются через промежуточный VDS-сервер (1 ГБ ОЗУ) по безопасному SSH-туннелю.

2. Исходные данные
   Компонент Характеристики Роль
   Локальный ноутбук разработчика Windows, Intel Iris Xe (0,12 ГБ VRAM), используется для разработки Отправляет запросы (yarn local-code-review, ночные спринты)
   VDS (промежуточный сервер) 1 vCPU, 1 ГБ ОЗУ, Ubuntu 22.04, публичный IP Прокси-сервер для маршрутизации запросов через SSH-туннель
   Мощный удалённый ПК 32 ГБ ОЗУ, Windows / Linux, поддерживает Wake-on-LAN или пробуждение по расписанию, находится в локальной сети Выполняет Ollama с моделями 7B–8B (например, mistral:7b, qwen2.5-coder:7b)
   Репозиторий проекта Git, существует регламент DEVELOPER_RHYTHM.md, скрипты local-code-review.mjs Уже есть заготовка для локального ревью, необходимо адаптировать под удалённый Ollama
3. Требования к системе
   3.1. Софт на мощном ПК
   Установленный Ollama (последняя версия) с загруженными моделями (mistral:7b, qwen2.5-coder:7b, опционально mixtral:8x7b).

OpenSSH Client (встроен в Windows 10/11, в Linux предустановлен).

SSH-ключи для подключения к VDS (без пароля).

Скрипт автоматического запуска при пробуждении (запуск Ollama, установка Reverse SSH-туннеля).

Настроенное расписание пробуждения (через Планировщик задач Windows или cron в Linux).

3.2. Софт на VDS
SSH-сервер с включённым GatewayPorts (опционально, если требуется внешний доступ к порту 11434).

Открытый порт 11434 только для локального ноутбука (рекомендуется использовать второй туннель, см. раздел безопасности).

(Опционально) установленный autossh для поддержания туннеля.

3.3. Софт на локальном ноутбуке
Модифицированный скрипт local-code-review.mjs, поддерживающий переменную окружения OLLAMA_HOST.

Возможность запуска вечернего ритуала с удалённым Ollama (команда yarn ritual:evening:remote).

(Опционально) скрипт проверки доступности туннеля.

4. Архитектура взаимодействия

Два варианта реализации:

Вариант 1 (рекомендуемый для простоты): На локальном ноутбуке поднимается прямой -L туннель на VDS:
ssh -L 11434:localhost:11434 root@<VDS_IP> -N -f
При этом на VDS уже должен быть активен Reverse-туннель с мощного ПК:
ssh -R 11434:localhost:11434 root@<VDS_IP> -N -f
Тогда локальный ноутбук обращается к localhost:11434 и через VDS попадает на мощный ПК.

Вариант 2 (если открыт порт на VDS): Задать OLLAMA_HOST=http://<VDS_IP>:11434 и не использовать -L туннель, но тогда необходимо защитить порт паролем или IP-ограничениями.

Далее по тексту будем использовать Вариант 1 как более безопасный и простой.

5. Подготовка мощного ПК
   5.1. Установка Ollama и моделей
   Скачать и установить Ollama для Windows или для Linux.

Загрузить модели:

powershell
ollama pull mistral:7b
ollama pull qwen2.5-coder:7b
5.2. Настройка SSH-ключа для доступа к VDS
На мощном ПК (в PowerShell с правами администратора или в WSL):

powershell

# Генерация ключа

ssh-keygen -t rsa -b 4096 -f "$env:USERPROFILE\.ssh\id_rsa"

# Копирование публичного ключа на VDS

type $env:USERPROFILE\.ssh\id_rsa.pub | ssh root@<VDS_IP> "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"
Проверить подключение: ssh root@<VDS_IP> (пароль не должен запрашиваться).

5.3. Скрипт запуска Ollama и туннеля при пробуждении
Создать файл C:\Scripts\start-ollama-tunnel.ps1:

powershell

# start-ollama-tunnel.ps1

Write-Host "Starting Ollama server..."
Start-Process "C:\Program Files\Ollama\ollama.exe" -WindowStyle Hidden
Start-Sleep -Seconds 10
Write-Host "Establishing reverse SSH tunnel..."
ssh -R 11434:localhost:11434 -N -f root@<VDS_IP>
Write-Host "Tunnel established."
Сделать этот скрипт исполняемым (разрешить выполнение PowerShell-скриптов).

5.4. Настройка автоматического пробуждения по расписанию
В Планировщике задач Windows создать задачу:

Имя: WakeUpForMembrana

Триггер: Ежедневно в 22:00 (или за 15 минут до вашего вечернего ритуала).

Действие: Запуск программы powershell.exe -ExecutionPolicy Bypass -File "C:\Scripts\start-ollama-tunnel.ps1"

Условия: Включить «Разбудить компьютер для выполнения задачи».

Также можно настроить автоматический переход в сон после бездействия (в параметрах электропитания).

5.5. Настройка автоматического закрытия туннеля и сна (опционально)
Если вы хотите, чтобы ПК засыпал после завершения работы, можно добавить в скрипт мониторинг: по истечении времени бездействия (например, 30 минут) выполнить shutdown /h (гибернация) или rundll32.exe powrprof.dll,SetSuspendState 0,1,0 (сон). Для упрощения можно довериться системным настройкам электропитания «Переходить в сон через 30 минут».

6. Настройка VDS (промежуточного сервера)
   6.1. Проверка SSH-конфигурации
   Убедиться, что в /etc/ssh/sshd_config есть (или добавить) строки:

text
GatewayPorts no # оставляем no, если используем только Reverse туннель, без внешнего доступа
ClientAliveInterval 60
ClientAliveCountMax 3
Перезапустить SSH: systemctl restart sshd.

6.2. Проверка, что порт 11434 не занят другими службами
netstat -tlnp | grep 11434 – должно быть чисто.

7. Настройка локального ноутбука
   7.1. Адаптация скрипта local-code-review.mjs
   Отредактировать scripts/local-code-review.mjs, добавив поддержку OLLAMA_HOST:

javascript
// В начале после импортов
const OLLAMA_HOST = process.env.OLLAMA_HOST || 'http://localhost:11434';

// В функции callOllama заменить url на:
const url = `${OLLAMA_HOST}/api/generate`;
Если у вас уже есть скрипт без этой переменной — внесите изменения. Если нет — создайте по аналогии с тем, что мы разрабатывали ранее.

7.2. Создание команды для установки локального туннеля
На локальном ноутбуке (также должен быть SSH-ключ для доступа к VDS) можно создать скрипт scripts/tunnel-up.ps1 (или .sh для WSL):

powershell

# tunnel-up.ps1

ssh -L 11434:localhost:11434 -N -f root@<VDS_IP>
Write-Host "Local tunnel to VDS established."
И для отключения: ssh -O exit root@<VDS_IP>.

7.3. Создание пакетной команды для вечернего ритуала
В package.json добавить:

json
"ritual:evening:remote": "yarn archive:daily-day && yarn local-code-review && yarn save-code-review && yarn task:close-github"
При этом перед запуском ритуала нужно убедиться, что туннель активен. Можно объединить в один скрипт:

json
"ritual:evening:remote": "node scripts/ensure-tunnel.mjs && yarn archive:daily-day && yarn local-code-review && yarn save-code-review && yarn task:close-github"
Где ensure-tunnel.mjs проверяет доступность localhost:11434 через curl http://localhost:11434/api/tags и, если недоступно, пытается запустить туннель (через exec). Но это уже опционально.

7.4. Проверка работы
Запустить yarn local-code-review на локальном ноутбуке, убедиться, что запрос проходит через VDS на мощный ПК и возвращается ответ.

8. Безопасность
   SSH-ключи должны быть защищены парольной фразой (но для автоматического запуска парольная фраза не подойдёт — можно использовать ssh-agent или оставить без пароля, если ключ хранится в безопасном месте).

На VDS рекомендуется ограничить доступ к порту 11434 только для локального хоста (по умолчанию Reverse туннель слушает только localhost, что безопасно). Внешний доступ не открывать.

Использовать GatewayPorts no на VDS, чтобы порт 11434 был доступен только с localhost.

На мощном ПК не открывать порты наружу, все соединения идут через SSH.

Регулярно обновлять ПО (Ollama, SSH).

9. Управление питанием мощного ПК
   9.1. Пробуждение по расписанию (основной метод)
   Установить задачу в Планировщике задач (как описано в п.5.4) на время, когда обычно проводится вечерний ритуал (например, 22:00).

После завершения ритуала (или через N минут бездействия) ПК автоматически уходит в сон благодаря стандартным настройкам электропитания.

9.2. Ручное внеплановое пробуждение (опционально)
Если требуется запустить ночной спринт в другое время, можно:

Подключиться к ПК по RDP (если он не в глубоком сне) и разбудить движением мыши.

Использовать Wake-on-LAN (WoL) через VPN или через роутер (требует дополнительной настройки). Это выходит за рамки ТЗ, но может быть реализовано при необходимости.

10. План тестирования
    Тест туннеля: С мощного ПК запустить ssh -R 11434:localhost:11434 -N -f root@<VDS_IP>. На локальном ноутбуке выполнить ssh -L 11434:localhost:11434 -N -f root@<VDS_IP>. Затем выполнить curl http://localhost:11434/api/tags и убедиться, что возвращается список моделей (с мощного ПК).

Тест локального ревью: В репозитории запустить yarn local-code-review и дождаться ответа. Проверить, что docs/DAILY_CODE_REVIEW.md создан.

Тест пробуждения: Настроить расписание на ближайшее время, перевести ПК в сон, дождаться срабатывания задачи, проверить, что туннель поднят и работает.

Тест вечернего ритуала: Выполнить yarn ritual:evening:remote и убедиться, что все шаги проходят успешно.

Тест ночного спринта (если реализован): Запустить скрипт ночного спринта, который также использует удалённый Ollama, и проверить создание PR (при наличии соответствующих скриптов).

11. Возможные проблемы и пути решения
    Проблема Решение
    Туннель обрывается Использовать autossh на мощном ПК для автоматического переподключения.
    ПК не просыпается по расписанию Проверить настройки BIOS (включить Wake-on-LAN, разрешить пробуждение по таймеру). В Windows проверить, что задача имеет права на пробуждение.
    Медленная работа на CPU Использовать менее тяжёлую модель (phi3:3.8b) или увеличить число потоков в Ollama (переменная OLLAMA_NUM_PARALLEL).
    Не хватает ОЗУ для mixtral Использовать qwen2.5-coder:7b или mistral:7b — они легче.
    Не удаётся установить SSH-туннель Проверить сетевые настройки, открытые порты на VDS (22), ключи. Использовать -v для диагностики.
12. Документация для разработчиков
    Описать в DEVELOPER_RHYTHM.md новый режим: использование удалённого Ollama через туннель. Добавить команду ritual:evening:remote.

Создать памятку по запуску туннеля перед вечерним ритуалом.

Включить проверку доступности Ollama в скрипты, чтобы при отсутствии туннеля выдавалось понятное сообщение.

13. Сроки и ответственные
    Настройка мощного ПК (установка Ollama, SSH-ключи, скрипты, расписание) – 2 часа.

Настройка VDS (проверка SSH, возможно, установка autossh) – 1 час.

Адаптация скриптов локального ноутбука – 1 час.

Тестирование и отладка – 2 часа.
Общее время: ~6 часов.

Ответственный: разработчик (вы).

Приложение A: Пример скрипта для автоматического запуска туннеля на мощном ПК (Linux)
Если мощный ПК работает под Linux, аналогичный скрипт:

bash
#!/bin/bash

# start-ollama-tunnel.sh

ollama serve &
sleep 10
ssh -R 11434:localhost:11434 -N -f root@<VDS_IP>
Добавить в crontab: @reboot или через systemd.

Приложение B: Пример скрипта проверки туннеля (ensure-tunnel.mjs)
javascript
import { execSync } from 'child_process';
try {
execSync('curl -s http://localhost:11434/api/tags', { stdio: 'ignore' });
console.log('Ollama tunnel is active.');
} catch {
console.log('Tunnel not active, attempting to start...');
execSync('ssh -L 11434:localhost:11434 -N -f root@<VDS_IP>', { stdio: 'inherit' });
// повторная проверка
execSync('curl -s http://localhost:11434/api/tags', { stdio: 'ignore' });
console.log('Tunnel started.');
}
ТЗ утверждено и готово к реализации.
