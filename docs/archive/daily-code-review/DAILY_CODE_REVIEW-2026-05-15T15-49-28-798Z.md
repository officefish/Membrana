## [Teamlead — Vesnin]

**Итоговая оценка:** Состояние **жёлтое** (amber). Проведена массированная работа по инфраструктуре и документации, но есть нарушения границ и несогласованность в выкладке.

### Что сделано корректно

1. **Архитектурная чистота инструмента `ask`:** Новый CLI правильно разделяет ответственность:
   - Подкладывает контекст из `WHITE_PAPER.md` + `ARCHITECTURE.md`.

   - Не смешивает фронтовый код с background-office логикой.

2. **Новая инфраструктура для задач (consilium, main-day-issue):** Скрипты разумно раскладывают по `scripts/lib/` — это снижает энтропию root.

3. **Доля обновлений PROMPT'ов:** Расширение `PROMPT_LAYOUT_DEVELOPER.md` и `PROMPT_STRUCTURER.md` показывает, что команда вызревает в самосознании.

### Риски и нарушения

#### 🔴 **Критичное: миксинг документации**

- **`docs/VIRTUAL_TEAM_PROMPT.md`** переписан и содержит ~90% содержимого текущего файла (таблица персонажей, правила, формат ответа).
- **`docs/prompts/TASK_PROMPT_WORKFLOW.md`** тоже обновлён.
- **Вопрос:** Эти файлы должны были быть **одной** системной инструкцией или **двумя**?
  - `VIRTUAL_TEAM_PROMPT.md` — координация и регламент ролей.
  - `TASK_PROMPT_WORKFLOW.md` — как именно оформлять issue → prompt → work → archive.
  - Сейчас есть **дублирование** (ссылки на task prompts, описание Registry) и **рассеивание** контекста.
  - **Действие:** На следующем ревью провести **deduplicate** — выделить одну master-инструкцию, остальное → подразделы или ссылки.

#### 🟡 **Среднее: untracked файлы в scripts**

- `consilium.mjs`, `main-day-issue.mjs` и тесты **не закоммичены**.
- Это означает, что `yarn consilium` и `yarn main-day-issue` **не работают** для кого-то, кто скло́нил репо.
- **Действие:** `git add scripts/consilium.mjs scripts/main-day-issue.mjs scripts/lib/` и коммит **сейчас же**.

#### 🟡 **Среднее: AGENTS.md — перекрытие с VIRTUAL_TEAM_PROMPT.md**

- `AGENTS.md` содержит таблицу роли + персонажи и цели.
- `VIRTUAL_TEAM_PROMPT.md` (он же текущий файл) содержит ту же таблицу.
- **Вопрос:** AGENTS.md был задумкой для внешних (для пользователей, CI/CD). Но теперь он теряет смысл как отдельный документ.
- **Действие:** Уточнить: AGENTS.md → краткая справка для запуска `yarn ask`, VIRTUAL_TEAM_PROMPT.md → **единая служебная инструкция** для агента.

#### 🟡 **Среднее: README.md — расширение, но без навигации**

- README.md обновлён, но нет ясной таблицы содержания для новых разделов (consilium, main-day-issue, ежедневный ритм).
- Читатель не видит, что внутри `docs/DEVELOPER_RHYTHM.md`.

### Что проверить при слиянии

2. **Скрипты:** `yarn consilium --dry-run "test"` — есть ли ошибки в промптировании?

## [Структурщик — Ozhegov]

**Вердикт:** Слабая связанность **сохранена**, но появились новые точки соприкосновения.

### Анализ импортов и модульной границы

1. **Новые сервисы и стоки:**
   - `@membrana/background-office` — остаётся неизменным (background-worker + telemetry журнал).
   - **Вопрос:** Где живут функции для парсинга обсуждений из `docs/discussions/*.md`?
     - Если в `background-office` → всё хорошо (инкапсуляция).
     - Если в root-скриптах → миксинг ответственности (scripts должны только **вызывать** сервисы, не содержать логику).

2. **Новые скрипты (`consilium.mjs`, `main-day-issue.mjs`):**
   - Живут в `scripts/lib/` (хороший уровень изоляции).
   - Импортируют ли они из `packages/` напрямую или через `background-office`?
   - **Действие:** Проверить, нет ли в этих скриптах импортов типа `import from "../../packages/..."`. Они должны работать только с файловой системой и API (OpenAI, Linear).

3. **Плагины (FFT, Sound Quality, Drone Detector):**
   - Согласно commit'ам, три плагина добавлены/обновлены.
   - Каждый из них импортирует ли соседей или только сервисы?
   - **Действие:** Проверить `packages/plugins/` на прямые импорты между плагинами (антипаттерн).

### Критичная проверка

```bash
# Для каждого пакета:
grep -r "from.*plugins/" packages/plugins/*/src/
# Должно быть пусто или только на сервис-слой.

# Для скриптов:
grep -r "from.*packages/" scripts/consilium.mjs scripts/main-day-issue.mjs scripts/lib/
# Должны быть **только** типы, интерфейсы и конфиги, не бизнес-логика.
```
