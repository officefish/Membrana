## [Математик — Dynin]

**Состояние ядра:**

Видно изменение `harmonics.ts` и `classifier.ts`. Без доступа к diff не могу вынести финальный вердикт, но вот **чеклист вопросов**:

1. **Входы/выходы гармонического анализа:**
   - `harmonics(samples: Float32Array, sampleRate: number): HarmonicResult` ← формат результата соответствует `HarmonicResult` в `@membrana/core` (типы)?
   - Побочные эффекты? (должны быть исключены; только чистые вычисления FFT → пики → гармоники)

2. **Классификатор:**
   - Зависит ли от состояния (напр., истории фреймов)? Если да — это часть **плагина**, а не чистой функции.
   - Нужна ли нормировка результатов (дБ, вероятности)? Это должно быть задокументировано в заголовке функции.

3. **Бенчмарк**: `data/detectors-benchmark/v0.1/reports/latest.json` обновлён. Если метрики:
   - **Точность < 85%** →需要 доп. оптимизация алгоритма.
   - **Время > 50 мс на фрейм** → профилировать FFT, кэширование окон.
   - **Нестабильность** (σ > 5% от µ) → проверить граничные случаи (тишина, шум, экстремальные частоты).

**Фокус на PR:**
- Нужна ревью: есть ли регрессия в точности по сравнению с прошлым бенчмарком?
- Есть ли unit-тесты в `__tests__/` (проверка edge cases: нулевой сигнал, постоянная, высокие гармоники)?
