⚠️ **Потенциальные нарушения:**

1. **Циклические импорты?** В `packages/background-office/.env.docker` виден контекст Docker (порты, хосты). Нужно проверить:
   - `background-office` → не импортирует ли прямо code из `background-media`? Только HTTP-запросы.
   - Cabinet Web → не импортирует ли `background-office` типы/утилиты напрямую?

   ```bash
   # Быстрая проверка
   grep -r "from.*background-office" packages/background-media/src/
   grep -r "from.*background-media" packages/background-office/src/
   ```

2. **Импорты в `scripts/`:** SSH-скрипты в корне могут знать о структуре сервисов (это OK, если это утилиты CI), но нужно документировать в `CONTRIBUTING.md`.

3. **Docker Compose сеть:** В коммитах упоминается `host.docker.internal` и `membrana-platform` — это детали развёртывания, которые должны быть в `docker-compose.yml` (не в код-базе). Проверить, что нет хардкода хостов в переменных окружения сервисов (должны быть ENV var'ы).

**Рекомендация:** Запустить `/refactor background-office background-media` для проверки циклических зависимостей и уточнения контрактов.
