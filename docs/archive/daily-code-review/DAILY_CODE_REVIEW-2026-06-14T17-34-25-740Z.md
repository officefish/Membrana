   - Фронт: `cabinet` (UI лимита)
   
   ✅ **Зелено:** архитектура не нарушена (слои чётко разделены), но нужно проверить:
   - Нет ли циклических импортов `tariff-dataset ← background-cabinet`?
   - `MembraneRegistry lifecycle` в коммите `2fd328d` — может означать, что регистр не был готов к парам. Проверить: инициализируется ли регистр ДО получения квоты?

2. **Cabinet перешла на SPA + удалённые операции (`f3483c2`):**
   - новый `background-media` сервис (каталог образцов)
   - CORS патчи для `cabinet.membrana.space`
   - Встроенный плеер в строку таблицы (`bd170ad`)
   
   ⚠️ **Риск слабой связанности:**
   - Фронт `cabinet` теперь зависит от `@membrana/core` (для плеера), от `@membrana/media-library` (для каталога)
   - Нужна проверка: нет ли прямых импортов между `cabinet` и каталогом? (через фасад `background-media`?)

3. **Docker build context расширился:**
   - `e0c5b16`, `64e9a37` добавляют `core` и `media-library` в контекст
   - Если рост образа > 50 МБ, это может замедлить CI/CD
