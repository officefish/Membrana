# docs/void/ — кладбище альтернативных сценариев

**Здесь лежит отвергнутое. Живым не является.** (Вердикт M5-GC заседания
`angelina-hostess`, 21.07, ратифицирован владельцем.)

Правила чтения:
- Каждый файл несёт YAML-эпитафию: кто приговорил (`verdict`/`rejectedBy`), за что
  (`rejectedReason`), когда (`rejectedAt`). Эпитафия — первое, что видит читатель.
- **Не восстанавливать без нового вердикта.** Приговор выносит только закрытый вердикт
  с ответственным; GC исполняет, не судит.
- **Удаление из void запрещено** — история актив (аудит, контент-мейкер, обучение);
  каталог растёт монотонно. Стареет штраф свежести (90 дней), не память.
- Барьеры против возрождения: эпитафия в файле · этот индекс · grep-инвариант CI
  (активные ссылки на void/* извне запрещены) · `recent_void_penalty` в генераторах.

Механика: `scripts/lib/gc-void.mjs` (isDead/isStale/epitaph) — проход в вечернем ритуале
после закрытия вердиктов дня; человек-гейт = diff в PR.
