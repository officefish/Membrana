# DATASET — датасет для бенчмарка детекторов

> **Статус:** скелет. Записи и разметка — отдельный поток (`DATASET_BOOTSTRAP_PROMPT`).
> Минимум для stage-gate уточняется после первого сбора.

## Классы (целевая структура)

| Класс | Описание |
|-------|----------|
| `drone-multirotor` | Мультиротор, типичные обороты |
| `drone-fixed-wing` | Крыло / планер |
| `bird` | Птица (ложноположительный контроль) |
| `wind` | Ветер, порывы |
| `traffic` | Дорожный транспорт |
| `human-speech` | Речь (фильтр ложных срабатываний) |
| `silence` | Тишина / фоновый шум без события |

## Минимум для stage-gate

- **≥ 30 сэмплов на класс** на test-split (число может быть пересмотрено в
  `DATASET_BOOTSTRAP_PROMPT`).
- Train / val / test: **60 / 20 / 20** (заглушка до утверждения Teamlead).

## Протокол расширения

1. Новые записи — отдельный PR с метаданными: источник, условия, расстояние до
   источника (м), SNR (если известен).
2. Формат файлов и каталоги — зафиксируются в bootstrap-промпте.
3. Не коммитить сырые записи с персональными данными без политики хранения
   (см. `WHITE_PAPER.md` §10).
