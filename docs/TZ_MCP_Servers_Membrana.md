# Техническое задание: установка и настройка MCP-серверов

**Проект:** Мембрана
**Дата:** 20 мая 2026
**Версия:** 1.0

---

## Контекст

Необходимо настроить локальную среду MCP (Model Context Protocol) для работы над проектом **«Мембрана»**. Целевые MCP-клиенты — **Claude Desktop** (основной) и **Cursor / VS Code** (вторичный). Конфигурация должна быть переносимой между клиентами за счёт идентичной структуры `mcpServers`.

## Цель

Развернуть стабильный набор MCP-серверов, обеспечивающий следующие возможности агента:

- веб-поиск с цитированием источников и фильтрацией по свежести
- автоматизация браузера (навигация, заполнение форм, скриншоты, скрапинг)
- структурное извлечение символов из кодовой базы для эффективной навигации
- (опционально) работа с живым браузером пользователя через расширение

## Состав серверов

| Сервер | Назначение | Приоритет | API-ключ |
|---|---|---|---|
| Perplexity MCP | research, актуальный поиск | обязательный | требуется |
| Playwright MCP | автоматизация изолированного браузера | обязательный | нет |
| Glyph MCP | карта символов кодовой базы | желательный | нет |
| Chrome MCP | работа с живым Chrome пользователя | опциональный | нет |
| mcp-firewall | политики и аудит вызовов | отложено до production | нет |

## Требования к окружению

- **Node.js** ≥ 18 (для Perplexity, Playwright, Chrome MCP)
- **uv** (Python package manager) — для Glyph
- **Git** — для клонирования Glyph
- **Chrome** последней версии — только если ставится Chrome MCP
- ОС: macOS или Windows (указать в начале выполнения)

## Этап 1. Подготовка runtime

### macOS

```bash
brew install node git
curl -LsSf https://astral.sh/uv/install.sh | sh
node --version && uv --version && git --version
```

### Windows (PowerShell от администратора)

```powershell
winget install OpenJS.NodeJS.LTS
winget install Git.Git
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
# перезапустить PowerShell, затем:
node --version; uv --version; git --version
```

**Критерий приёмки этапа:** все три команды возвращают версии без ошибок.

---

## Этап 2. Получение API-ключа Perplexity

1. Зарегистрироваться/залогиниться на `perplexity.ai`
2. Settings → API → Generate Key
3. Привязать платёжный метод (требование Perplexity)
4. Сохранить ключ формата `pplx-...` в безопасном месте (менеджер паролей)

**Критерий приёмки:** ключ получен, не закоммичен в git, не вставлен в чаты.

---

## Этап 3. Установка Glyph из исходников

### macOS

```bash
cd ~
git clone https://github.com/benmyles/glyph
cd glyph
uv sync
pwd  # зафиксировать абсолютный путь
```

### Windows

```powershell
cd $env:USERPROFILE
git clone https://github.com/benmyles/glyph
cd glyph
uv sync
(Get-Location).Path  # зафиксировать абсолютный путь
```

**Критерий приёмки:** команда `uv run --directory <путь_к_glyph> glyph-mcp --help` отрабатывает без ошибок импорта.

## Этап 4. Конфигурация Claude Desktop

### 4.1. Расположение конфига

- **macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Windows:** `%APPDATA%\Claude\claude_desktop_config.json`

Если файл отсутствует — создать. Альтернативно: в Claude Desktop **Settings → Developer → Edit Config**.

### 4.2. Содержимое конфига (macOS / Linux)

```json
{
  "mcpServers": {
    "perplexity": {
      "command": "npx",
      "args": ["-y", "@perplexity-ai/mcp-server"],
      "env": {
        "PERPLEXITY_API_KEY": "pplx-ЗАМЕНИТЬ_НА_РЕАЛЬНЫЙ_КЛЮЧ"
      }
    },
    "playwright": {
      "command": "npx",
      "args": ["@playwright/mcp@latest"]
    },
    "glyph": {
      "command": "uv",
      "args": ["run", "--directory", "/АБСОЛЮТНЫЙ/ПУТЬ/К/glyph", "glyph-mcp"]
    }
  }
}
```

### 4.3. Содержимое конфига (Windows)

```json
{
  "mcpServers": {
    "perplexity": {
      "command": "cmd",
      "args": ["/c", "npx", "-y", "@perplexity-ai/mcp-server"],
      "env": {
        "PERPLEXITY_API_KEY": "pplx-ЗАМЕНИТЬ_НА_РЕАЛЬНЫЙ_КЛЮЧ"
      }
    },
    "playwright": {
      "command": "cmd",
      "args": ["/c", "npx", "@playwright/mcp@latest"]
    },
    "glyph": {
      "command": "uv",
      "args": ["run", "--directory", "C:\\АБСОЛЮТНЫЙ\\ПУТЬ\\К\\glyph", "glyph-mcp"]
    }
  }
}
```

**Замены перед сохранением:**

- `pplx-ЗАМЕНИТЬ_НА_РЕАЛЬНЫЙ_КЛЮЧ` → ключ из Этапа 2
- путь к Glyph → значение из Этапа 3 (на Windows — двойные обратные слеши)

### 4.4. Перезапуск

- **macOS:** `Cmd+Q` в Claude Desktop (не просто закрыть окно), затем заново открыть
- **Windows:** правый клик по иконке в трее → Quit, затем перезапуск

### 4.5. Проверка

В окне ввода должна появиться иконка инструментов. Развернуть список — увидеть инструменты `perplexity_*`, `browser_*`, `glyph_*`.

В случае ошибок: **Settings → Developer → MCP Log** покажет логи запуска каждого сервера.

**Критерий приёмки этапа:** все три сервера видны в списке инструментов, логи не содержат красных ошибок.

## Этап 5. Конфигурация Cursor (опционально)

### 5.1. Расположение

- **macOS:** `~/.cursor/mcp.json`
- **Windows:** `%USERPROFILE%\.cursor\mcp.json`

### 5.2. Содержимое

Скопировать без изменений содержимое `claude_desktop_config.json` из Этапа 4. Формат идентичен.

### 5.3. Проверка

Cursor → Settings (`Cmd/Ctrl + ,`) → MCP → серверы должны быть со статусом «active» (зелёная индикация).

**Критерий приёмки:** инструменты доступны из Cursor Composer.

## Этап 7. Chrome MCP (опционально, по решению)

Выполнять **только если** есть подтверждённая необходимость работать с залогиненными сессиями браузера, не покрываемая Playwright.

1. `npm install -g mcp-chrome-bridge`
2. Скачать релиз расширения с `github.com/hangwin/mcp-chrome/releases`, распаковать
3. Chrome → `chrome://extensions` → включить **Developer mode** → **Load unpacked** → выбрать распакованную папку
4. Кликнуть иконку расширения → запустить bridge → скопировать путь к `mcp-server-stdio.js`
5. Добавить в `mcpServers`:

   ```json
   "chrome": {
     "command": "node",
     "args": ["/скопированный/путь/mcp-server-stdio.js"]
   }
   ```

6. Перезапустить Claude Desktop

> ⚠️ **Предупреждение безопасности:** Chrome MCP получает доступ ко всем твоим залогиненным сессиям. Не использовать на машинах с чувствительными корпоративными аккаунтами без отдельного профиля Chrome.

После успешной приёмки:

1. Сохранить итоговый `claude_desktop_config.json` в защищённое место (без API-ключа — заменить плейсхолдером)

   - назначение каждого
   - сценарии использования для команды
3. Добавить `claude_desktop_config.json` в `.gitignore` (если он попадает в репозиторий)

---

## Чего НЕ нужно делать

- **Не писать собственные `.mjs` или `.sh` скрипты для запуска серверов.** MCP-клиент сам управляет жизненным циклом процессов через `command` + `args` в конфиге.
- **Не запускать серверы вручную в терминале.** Они работают через stdio и ожидают подключения клиента.
- **Не коммитить конфиг с реальным API-ключом.**
- **Не ставить mcp-firewall на этом этапе** — он обоснован только перед выходом в production или при предоставлении доступа третьим лицам.

---

## Откат и удаление

В случае проблем:

1. Закрыть Claude Desktop
2. Переименовать `claude_desktop_config.json` → `claude_desktop_config.json.bak`
3. Запустить Claude Desktop (запустится без MCP-серверов)
4. Диагностировать причину по логам, починить конфиг, переименовать обратно
