# Процедура: bridge — комната «мостик» капитана

**Держатель:** Ангелина (`leadPersona` манифеста) — ведущая комнаты (конспект и
честный отчёт; кода не пишет, кодекс #922).

Особый формат диалога капитана с командой мостика: **Ангелина** (конспект),
**Фаррелл** (`origin: pet`, свободный голос, не гейт), **попугай** (память
техдолгов, немногословен). Спринт-источник: #936.

## Жизненный цикл (канон — слово капитана, две поправки 22.07)

- **Открытие — ЯВНОЕ** (`yarn bridge open`): по слову капитана; не обязано быть с утра.
- **Закрытие — НЕ ЯВНОЕ**: шаг `bridge-close` вечернего ритуала закрывает сам.
- **Конспект при закрытии уезжает ФРЕЙМОМ** в репозиторий — не прямым пушем
  (сходится с инсайтом `change-routing-office`: конспект едет заявкой).

## Очередь фреймов комнаты (до контракта frames[] #900)

1. `open` — явное открытие + попугай зачитывает живые долги (Б3).
2. диалог — Ангелина ведёт конспект в `docs/bridge/<день>/CONSPECTUS.md`.
3. `debt add/settle` — попугай пополняет/гасит реестр (append-only, с вещдоком).
4. `bridge-close` — неявное закрытие вечером + постановка конспекта на отправку фреймом.

## Границы (Ожегов)

- Ядро состояния (`bridge-room.mjs`) и реестр долгов (`bridge-debts.mjs`) — чистые,
  без fs/сети; персистентность и дата — в адаптере `bridge.mjs`.
- Один источник истины состояния — `docs/bridge/state.json`.
- Реестр долгов `docs/bridge/DEBTS.md` — **append-only**: settled помечается, не стирается.
- Комната **не пушит сама** — готовит фрейм-заявку (двухпоточный роутинг).

Код и тесты — в плоском `scripts/` (Т12 шторма фреймов: контейнер процедуры без
собственного кода). Провод в цепочку — `docs/tasks/evening-ritual-steps.json`.
