# Процедура: one-shot — первый фрейм (подобранная S-правка за один проход)

**Определение.** One-shot — процедура **одного прохода** по подобранной (не
заказанной) мелкой находке: агент нашёл сам или всплыло из прецедента. Допуск —
двойной гейт (размер **S** и не сервер); исполнение кончается на «весь код
дописан». Ревью и мердж в формат **не входят**.

**Держатель:** Vesnin (`leadPersona` манифеста). Support предиката — Dynin.

**Источники:** шторм [`storm-one-shot-format-2026-07-23`](../../storm/storm-one-shot-format-2026-07-23/REPORT.md);
инсайт [`insight-one-shot-format`](../../insights/insight-one-shot-format/REVIEW.md)
(7.2, adopted); промпт [`ONE_SHOT_FIRST_FRAME_PROMPT.md`](../../prompts/ONE_SHOT_FIRST_FRAME_PROMPT.md);
Issue [#1022](https://github.com/officefish/Membrana/issues/1022).

## 1. Развод с `procedure-frames` (#900) — чьё понятие «фрейм»

**Используется понятие фрейма из спринта `procedure-frames` (#900):** элемент
очереди манифеста `{ id, holder, pins? }` в полосах `preflight` | `frames` |
`post` (схема в `validateProcedure` / ADR-0015). Первый фрейм этой процедуры
объявлен в [`MANIFEST.json`](./MANIFEST.json) как `frames[0].id = first-frame`.

**Второго понятия «фрейм» не заводим.** Текст ниже описывает *содержание*
первого кадра (шаги допуска и исполнения), а не альтернативную модель кадров.
Механика `frames[]`, пин отрезков (`segmentHash`) — чужой ACTIVE-скоуп #900:
пользуемся, не проектируем.

## 2. Первый фрейм — контракт шагов

```
находка (агент нашёл сам / всплыла из прецедента)
   → запрос штампа S у тимлида
   → чистая функция по критериям подтверждает «да, это действительно S»
   → инлайн чек-лист плана
   → согласование и ратификация владельцем
   → исполнение до «весь код дописан»
```

| Шаг | Кто | Правило |
|-----|-----|---------|
| Находка | агент | Подобранная работа (T2), не заказ из бэклога |
| Штамп S | тимлид по запросу | Самоаттестация размера запрещена (T4) |
| Предикат | `evaluateOneShotS` | Детерминированный вердикт с названными основаниями (T5) |
| Чек-лист | агент → владелец | Пункты в повелительном наклонении, каждый проверяемый (T6/T10) |
| Ратификация | владелец | Без «да» владельца исполнение не начинается (T10) |
| Выход фрейма | агент | «Весь код дописан» — ревью/мердж снаружи (T11) |

**Вход (оба обязательны):** размер **S** И работа **не** затрагивает сервер
(office / media / cabinet / deploy / prisma migrations).

**Отменяется ровно одно:** регистрация карточки в реестре задач. Ветка, коммит,
ревью, мердж, Linear — как обычно, **снаружи** формата.

---

## 3. Осознанное отступление: тикет снимаем

Research Q1: индустрия оставляет тикет даже для мелкой правки (малый PR + тикет +
проверяемость + обратимость). Мы регистрацию карточки **снимаем** (хвост Х1
шторма). Это **осознанное отступление от ориентира**, не «так принято».

**Основание:** роль следа несут прецеденты согласия/отказа (T8/T9), а не карточка
реестра. GitHub Issue / Linear для one-shot **не требуются**; внешний PR-цикл
остаётся.

## 4. Предикат S-ности

Движок: [`scripts/lib/one-shot-s-predicate.mjs`](../../../scripts/lib/one-shot-s-predicate.mjs)
(`evaluateOneShotS`). Тесты границ:
[`scripts/one-shot-s-predicate.test.mjs`](../../../scripts/one-shot-s-predicate.test.mjs).

| Вход | Порог по умолчанию |
|------|-------------------|
| Число файлов | ≤ 8 |
| Строки диффа (`linesChanged`) | ≤ 200 |
| Запретные префиксы | `packages/background-office/`, `packages/background-media/`, `packages/background-cabinet/`, `deploy/` |
| Запретные пути | `**/prisma/migrations/**` |
| Цепочка | смежные one shot'ы за 7 суток: ≤ 3 выстрела, сумма файлов ≤ 8, сумма строк ≤ 200 |

Смежность путей — по `pathFamily` (первые 2–3 сегмента). Цепочка — прямая
контрмера **capability chaining** (research Q3): каждый шаг формально S, сумма — нет.

Вердикт несёт **названные основания**: `empty_diff`, `exceeds_file_count`,
`exceeds_line_count`, `touches_server`, `capability_chaining` либо позитивные
`within_*` / `no_server_paths` / `chain_clear`.

Проверка (без LLM):

```bash
node --test scripts/one-shot-s-predicate.test.mjs
```

## 5. Инлайн чек-лист (контракт UI плана)

Читается за пять секунд. Пункты — **повелительное наклонение**, каждый
проверяемый. Шаблон:

```text
One-shot чек-лист
- [ ] Назови находку одной фразой
- [ ] Запроси штамп S у тимлида (не ставь сам)
- [ ] Прогони evaluateOneShotS на дифф + recentShots; приложи reasons
- [ ] Перечисли файлы, которые тронешь (не больше порога)
- [ ] Подтверди: серверных путей нет
- [ ] Дождись ратификации владельца
- [ ] Допиши весь код до «готово к ревью»; ревью/мердж — снаружи
```

### Судьба чек-листа при потере сессии

**Решение:** чек-лист **сразу дублируется** в прецедент (черновик согласия или
отказа) в момент показа владельцу. Формат **не переживает** обрыв сессии без
этого следа: чат-единственный план = импровизация, это запрещено молча.

## 6. Оба исхода оставляют след

| Исход | След |
|-------|------|
| Промах (отказ тимлида / предиката / владельца) | Прецедент отказа + четыре маршрута: шторм · инсайт · консилиум · спринт (T8) |
| Успех (ратификация + код дописан) | Согласие **с обоснованием** (T9); согласие без обоснования следом не считается |

**Финал без категоричности:** мотивированная рекомендация «как лучше поступить»,
не жёсткий вердикт за владельца.

## 7. Что вне этого захода

- Второй и последующие фреймы — не проектируем (шторм их не назвал).
- Строка в [`docs/FORMATS.md`](../../FORMATS.md) — **запрещена** до готовности
  процедуры; канон форматов позже.
- `yarn pr:ship` / `membrana-ship` — транспорт снаружи формата, не путать с one-shot.
