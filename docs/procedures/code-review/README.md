# Процедура: code-review

Ревью изменений виртуальной командой с **ведущим из пяти** (T3 шторма
branch-taxonomy 21.07): ведущий назначается каскадом «явное слово владельца →
`leadPersona` карточки, чей id виден в ветке/диффе → скоуп путей диффа →
Teamlead + пометка "вне конвенции"»; на ревью работает его персональная память
(T4), дифф сверяется с [бестиарием антипаттернов](../../bestiary/BESTIARY.md)
(T5), вердикт ведущего — **пропуск или блок**.

Категории слоя: виртуальная команда (ведущий), задача (карточка-основание),
ответственность (кто пропустил/заблокировал — след в артефакте ревью).

- **Держатель процедуры:** vesnin (Teamlead — владелец LGTM-контура).
- **Ведущий прогона:** определяется per-diff движком `review-lead` — держатель
  и ведущий сознательно разведены (как секретарь ≠ докладчик).
- Режимы: `daily` (вечер) · `--pr N` · `--branch` · `--uncommitted` / `--staged`;
  явное слово: `--lead <persona>`.
- Граница: грамматика имён веток и `resolveHolder` — процедура-сосед
  (`attribution`, Р3/Р4); здесь только подстрочное совпадение id карточки.

Инстансы (прогоны) живут в `docs/DAILY_CODE_REVIEW.md` /
`docs/discussions/uncommitted-code-review.md` / артефактах PR-ревью — сюда не
мигрируют.
