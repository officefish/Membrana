# Процедура: ritual-day — утренний ритуал

**Определение.** Утренний ритуал — процедура открытия рабочего дня: гигиена
репо, план/стратегия, стендап, центральная задача дня и секретарский контур
Ангелины. Оперирует категориями слоя: командой (роли в стендапе и фидбеке),
задачами (MAIN_DAY / реестр), ответственностью (гейты утра) и стратегической
целью (фокус дня).

**Держатель:** Ангелина (`leadPersona` манифеста).

## Кит (K4 / #819)

Набор шагов утра пинится китом [`kits/angelina-morning/`](../../../kits/angelina-morning/)
через поле **`kitVersion`**: `kits/angelina-morning`. Полный subgraph (roots +
транзитивные `scripts/lib/*`) — в манифесте кита, **не** дублируется в
`engines[]`.

| Режим | Правило |
|-------|---------|
| **latest** | интерактив; `yarn kits:audit --mode latest` |
| **pinned** | autonomous (night/cron/office); только от пина кита |

## Движки

В контейнере кода нет (Т12). `engines[]` несёт **точку входа цепочки**
`yarn ritual:day` — `scripts/morning-care.mjs`. Остальные корни утра живут в
ките; ласточка / hermes — корни кита, вызываются по ритуалу вручную после
артефакта (см. `.claude/CLAUDE.md`).

## Манифест

[`MANIFEST.json`](./MANIFEST.json) — `id`, `leadPersona`, `kitVersion`,
`engines[]`, `precedents[]`. Optional очередь кадров (F1 / #927): `preflight` /
`frames` / `post` — элемент `{id, holder, pins?}` (ADR-0015); пока не объявлена
(`D_live` после #928–#929). Зуб: `validateProcedure` (в т.ч. резолв
`kitVersion` → дом кита с `MANIFEST.json`).
