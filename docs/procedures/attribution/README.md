# Процедура: attribution — распределение ответственности

**Определение.** Attribution — процедура вычисления и распределения
ответственности за изменения: от вывода держателя ветки на шип-гейте до
распределения по файлам сложных процедурных коммитов парсером на code-review
(T6). Оперирует категорией «ответственность» (род: отношение — устанавливает и
разрывает связь «работа ↔ персона», не мутируя её концы).

**Держатель:** dynin (`leadPersona`; держатель фазы Р3, где домен получил адрес).

**Статус: адрес, не механизм.** По T9 сейчас спроектировано место жительства,
не сам парсер: этот контейнер — точка старта любого расследования прецедента.
Механизм парсера — будущая работа; его реализация — кандидат в **первый кит**
(#761, режим pinned по `PINNED_SUBGRAPH_VERSIONING`).

## Маршрут расследования прецедента (вердикт m3-boundary-manual)

1. Вердикт attribution несёт `rulesSha` — SHA файла правил
   [`../layer-rules.json`](../layer-rules.json), зафиксированный **в момент
   вердикта** (не HEAD).
2. `rulesSha` → git-история файла правил (кто и когда менял отношение легальности).
3. Прецеденты — `precedents[]` манифеста; история git не переписывается (посылка 9).

приедет в Р4 и впишется сюда движком.
