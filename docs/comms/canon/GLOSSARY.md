# GLOSSARY — заблокированный глоссарий Membrana

Канонический словарь терминов и формы для контура внешних коммуникаций (CC6). Обеспечивает
**терминологическую согласованность** между слайдами, лендингом, white paper и one-pager.
Если термин зафиксирован здесь — он используется во всех документах без вариаций.

Источники формы: `docs/foundation/CLAUDE_PROJECT_SYSTEM_PROMPT.md`,
`docs/foundation/MEMBRANA_ONE_PAGER.md`. Термины по существу — `FACTS_SHEET.md` (Слой 3).
При расхождении — источник выигрывает, глоссарий правится.

---

## 1. Заблокированные термины

Левая колонка — **канонический** термин. Правая — чем его **нельзя** заменять.

| Говорим | НЕ говорим | Что значит |
|---------|-----------|------------|
| **нижнее небо** | «воздушное пространство», «зона», «low altitude» | Слой от земли до 1500 м — среда работы системы. |
| **сенсорный узел** / **узел** | «нода», «датчик», «устройство», «сенсор» (в одиночку), «микрофон» (как весь узел) | Микрофон + плата уровня Raspberry Pi + точное время. |
| **сеть узлов** / **распределённая сеть** | «рой», «массив», «кластер» | Несколько узлов с известной геометрией + центральный слой слияния. |
| **дешёвый узел, умная сеть** | «умный сенсор», «интеллектуальное устройство» | Архитектурный принцип: сложность — в слиянии данных, не в узле. |
| **семейство детекторов** | «наша нейросеть», «AI-модель», «движок» | Три класса независимых детекторов под единым контрактом. |
| **эшелон** | «уровень», «tier» (в рус. тексте) | Эшелон A — DSP; эшелон B — нейросетевые и агентные. |
| **stage-gate** / **шлюз качества** | «KPI», «планка», «цель по точности» | Блокирующий порог P ≥ 85% / R ≥ 90% на одном узле перед переходом к сети. |
| **наблюдательная система** | «система ПВО», «средство защиты», «комплекс противодействия» | Фиксирует факт полёта; не управляет, не глушит, не перехватывает. |
| **локализация по разнице времён прихода (TDOA)** | «триангуляция», «радар» | Вычисление координат по разнице времён прихода звука на узлы. |
| **акустический отпечаток** / **портрет** | «сигнатура», «след» | Несущая 80–250 Гц, гармоники до 2–5 кГц. |
| **малый БПЛА** / **малый дрон** | «беспилотник» (как жаргон), «аппарат» без уточнения | Целевой класс: мультироторные и фиксированно-крыльевые малые БПЛА. |
| **контроль воздушного фона** | «слежка», «мониторинг угроз» (см. §3) | Нейтральная формулировка постоянного наблюдения. |
| **пост-инцидентный анализ** | «расследование», «форензика» | Восстановление траектории/характеристик из локальных буферов узлов. |

## 2. Что это — одной строкой (эталон формулировки)

> **Membrana — распределённая акустическая разведка нижнего неба: обнаружение, классификация и
> трекинг малых БПЛА по звуку, в реальном времени, без радара.**

Это камертон из one-pager. Любая более короткая формулировка сверяется с ним.

## 3. Запрещённые слова и фразы

Из `CLAUDE_PROJECT_SYSTEM_PROMPT.md` §«Запрещённые слова». Избегать вместе с синонимами и кальками.

**Хайп / маркетинг:**
«революционный», «прорывной», «передовой», «беспрецедентный», «уникальный», «не имеет аналогов»,
«единственное решение», «изменит мир», «новая эра», «следующее поколение», «трансформирует индустрию»,
«синергия», «экосистема», «парадигма», «дисрапт», «инновация» (в корпоративно-маркетинговом значении),
«AI-powered».

**«Искусственный интеллект» без уточнения** — вместо: «нейросетевые модели», «модели машинного обучения»,
«алгоритмы обработки сигналов» с конкретикой.

**Военная риторика:** «поле боя», «противник», «вооружённые силы», «военный», «удар», «поражение»,
«уничтожение».

## 4. Dual-use переформулировки (военное → нейтральное)

| Вместо | Пишем |
|--------|-------|
| поле боя / зона боевых действий | охрана стационарных объектов |
| противник / враг | (не персонифицируем; «неизвестный аппарат», «нерегламентированный полёт») |
| военный объект | режимный объект, критическая инфраструктура |
| перехват / подавление / поражение дрона | обнаружение, классификация, фиксация факта полёта |
| угроза (как враг) | угроза как абстрактный риск воздушного фона |

«Нейтрализация» — допустимо только в отношении угрозы как абстрактного риска, **не** в отношении самих дронов.

## 5. Правила формы

- `[✅]` Тон: сдержанный, инженерный, конкретный, не назидательный (как технический отчёт). Никакого хайпа.
- `[✅]` Любое утверждение — либо факт из `FACTS_SHEET.md`, либо явная пометка «гипотеза/планируется».
- `[✅]` Терминологическая согласованность: выбранный термин (нижнее небо, сенсорный узел) — во всех документах.
- `[❌]` **Никаких эмодзи в выходных документах** (слайды, лендинг, one-pager) — сигнал серьёзности.
- `[❌]` Не цитировать внутренние источники прямо («согласно WHITE_PAPER §…») — переформулировать своими словами.
- Аудитория: pre-seed/seed VC РФ/СНГ, госфонды, корпоративные VC; вторично — CTO/юротдел/пилотные заказчики.
  Анти-аудитория: B2C, civil drone-community, privacy-активисты.

> Примечание: маркеры `[✅]`/`[❌]` здесь — служебные для канон-документа; в **выходных** материалах
> контура эмодзи и такие маркеры не появляются (§«Правила формы»).
