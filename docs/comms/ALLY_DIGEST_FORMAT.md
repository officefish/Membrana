# Формат партнёрского дайджеста (ласточка)

> Канон структуры утреннего и вечернего дайджеста союзникам. Детерминированно
> извлекается из артефактов ритма (`MAIN_DAY_ISSUE.md` / `team-evening-feedback-*.md`),
> форматируется в office и уходит в приватную telegram-группу. Правило: **тезисно,
> без простыней; детали — во вложенном md-файле.**
>
> Реализация: `scripts/lib/ritual-digest-extract.mjs` (извлечение) →
> `packages/background-office/.../telegram-format.ts` (вёрстка) →
> `telegram.client.ts sendDocument` (вложение).

## Принципы

1. **Тезисно.** Каждый пункт — одна строка-суть, не абзац. Партнёр читает за 20 секунд.
2. **Структура фиксирована** (см. ниже) — не свободный поток.
3. **Детали — файлом.** Утром вложен `MAIN_DAY_ISSUE.md`, вечером — `DAILY_CODE_REVIEW`.
4. **Свёрнутая шапка-пояснение** (`ALLY_DIGEST_HEADER.md`) остаётся первым блоком.

---

## Утренний дайджест (`--kind day`)

```
Membrana — план на DD.MM.YYYY

[свёрнутая шапка-пояснение]

🎯 Центральная задача дня:
<одна фраза — что главное и зачем>

⬆️ Высокий приоритет (2–3):
• <тезис>
• <тезис>

🔭 Перспективные направления (2–3):
• <тезис>
• <тезис>

🌙 Критерий вечера:
<одна фраза — по чему поймём, что день удался>

📎 MAIN_DAY_ISSUE.md
```

**Источники (детерминированно из `MAIN_DAY_ISSUE.md`):**

| Секция дайджеста | Откуда берётся |
|------------------|----------------|
| Центральная задача | ⚡-заголовок бокса + «Одна фраза дня» |
| Высокий приоритет | таблица «Issues в скоупе» → строки «Магистраль/критпуть» + «Долг (side-слот)» |
| Перспективные направления | таблица «Issues в скоупе» → строка «Отложено (пост-FREE)» |
| Критерий вечера | строка «🎯 Критерий вечера» из бокса |

---

## Вечерний дайджест (`--kind evening`)

```
Membrana — итоги дня DD.MM.YYYY

[свёрнутая шапка-пояснение]

✅ Сошлось (2–3):
• <тезис>

⚠️ Не сошлось / перенесено (1–3):
• <тезис>

🎲 Неожиданно всплыло (0–3):
• <тезис>

Команда оценивает день: X/10.

📎 DAILY_CODE_REVIEW-DD.MM.md
```

**Источники (детерминированно из `team-evening-feedback-<date>.md`):**

| Секция дайджеста | Откуда берётся |
|------------------|----------------|
| Сошлось | DoD-чекбоксы MAIN_DAY_ISSUE со статусом ✅ + вердикт дня |
| Не сошлось / перенесено | DoD-чекбоксы без отметки + явная секция «Перенесено» |
| Неожиданно всплыло | новая секция «Неожиданные задачи» в team-evening-feedback |
| Оценка | «Средний балл команды» |

> Для «сошлось/не сошлось/неожиданно» вечерний артефакт
> (`team-evening-feedback`) должен нести эти три подраздела — иначе секции уходят
> пустыми (graceful). Мелкое расширение генератора `team-evening-feedback`.

---

## Вложения (файлом, не текстом)

- Утро: `docs/MAIN_DAY_ISSUE.md` — полный фокус дня.
- Вечер: свежий `docs/archive/daily/**/DAILY_CODE_REVIEW-*.md` — код-ревью дня.
- Транспорт: office `TelegramClient.sendDocument` (Bot API `sendDocument`,
  proxy-aware как `sendMessage`). Скрипт кладёт содержимое файла в payload
  (`documentMd` + `documentName`); office шлёт как документ после текста.
- Graceful: файла нет / слишком велик (>Telegram-лимита) → уходит только текст.

---

## Инварианты

- Дайджест **детерминирован** (кредит LLM не нужен) — только извлечение + вёрстка.
- Клэмп под лимит Telegram 4096 сохраняется; вложение снимает нагрузку с текста.
- Изменение формата = правка office → **редеплой office** (`_ssh-office-prod-up`).
- Никогда не слать устаревший артефакт (гейт свежести по дате остаётся).
