# docs/tooling-atlas — атлас контейнеров (общая документация по туллингу)

**Контейнер контейнеров** (`plane=meta`). Его группа — сами контейнеры проекта вместе
с их мастерскими. Он не хранит собственных описаний, а **агрегирует** документацию
из каждого контейнера в единый производный индекс. Реализация паттерна
[`GROUP_CONTAINERIZATION`](../patterns/GROUP_CONTAINERIZATION.md); оснастка — паттерн
[`HOME_WORKSHOP`](../patterns/HOME_WORKSHOP.md).

## Плоскости индекса (domain / report / meta)

Атлас **не** сплющивает предмет и отчёт в один ряд без метки. В индексе различают:

| `plane` | Смысл | Примеры `home` |
|---------|--------|----------------|
| **domain** | предметный дом группы | `docs/tasks`, `docs/procedures`, `docs/precedents` |
| **report** | слот плоскости отчётов [`docs/audit/`](../audit/README.md) | `docs/audit/git`, `docs/audit/tasks`, `docs/audit/llm-calls`, … |
| **meta** | этот атлас | `docs/tooling-atlas` |

Важно: [`docs/tasks`](../tasks/) (задания) ≠ [`docs/audit/tasks`](../audit/tasks/)
(отчёты **про** задания, `role: derivative`). Ссылки в `registry/ATLAS.md` и Mintlify
ведут на **`home`** (каталог контейнера), не на хвост `worksOn` (`registry.json` /
`registry/`). Движок — спринт [`atlas-report-plane`](../prompts/ATLAS_REPORT_PLANE_PROMPT.md)
(#1097); колонки `role`/`plane` — фаза W2.

## Что это и чем НЕ является

- **Это** — «одно окно» по туллингу-контейнерам: какие дома есть, на какой плоскости,
  primary/derivative, глаголы мастерской, кит.
- **Это НЕ** копия документации контейнеров. **Источник истины остаётся в каждом
  контейнере** (его `README.md` + `workshop.manifest.json`); атлас лишь собирает индекс.
  Копий нет — иначе эхо-дрейф (тот самый, против которого паттерн).

## Таксономия — контейнер контейнеров (мета-уровень)

Как `docs/procedures/` — контейнер контейнеров (его элементы сами контейнеры), так и
атлас: его **элемент = целый контейнер**. Разница: процедурный дом группирует процедуры,
атлас группирует **все дома вообще** (включая процедурный и, рекурсивно, себя).

## Контракт (что писать / что коммитить)

| Орган | Что | Форма |
|-------|-----|-------|
| **Элемент** | один контейнер = его `README.md` + `workshop.manifest.json` (живут В контейнере, не здесь) | внешний источник |
| **Реестр** | производный индекс всех контейнеров (overwrite, Meta) | `registry/ATLAS.md` — пересобирается |
| **Витрина** | mintlify-страница общей документации по туллингу | `apps/docs/tooling/containers.mdx` — генерится |
| **Инструмент** | осмотр/декомпозиция/рассмотрение + пересбор индекса и витрины | `yarn tooling:atlas` |

**Источник истины — README + манифест каждого контейнера.** `registry/ATLAS.md` и
mintlify-страница руками не правятся (дрейф ловит `--check`).

## Мастерская (предоставляет информацию о контейнерах)

[`workshop.manifest.json`](./workshop.manifest.json) — три глагола:

- **`audit`** (`--audit`): инвентарь контейнеров + здоровье их мастерских (валиден ли
  манифест, есть ли README, каких глаголов не хватает).
- **`decompose`** (`--decompose --by family|plane|holder|kit`): раскладка (в т.ч.
  `--by plane` → report / domain / meta). Список = `home`. Цель — не смешивать
  слот `docs/audit/tasks` с домом `docs/tasks`.
- **`inspectElement`** (`--inspect <home>`): один контейнер вглубь — адрес по **`home`**,
  манифест, глаголы, выжимка README.

## Витрина mintlify

`yarn tooling:atlas --render` пересобирает и `registry/ATLAS.md`, и mintlify-страницу
`apps/docs/tooling/containers.mdx` (секция «Tooling» в `docs.json`). Оба — производные.

## Границы

- Атлас индексирует контейнеры, несущие `workshop.manifest.json` (дома с мастерскими).
  Контейнер без мастерской (напр. плоский `scripts/`) — кандидат на включение позже.
- Только агрегация и чтение; собственных описаний контейнеров атлас не держит.
- Каталог-UI сверх mintlify-страницы — вне области.
