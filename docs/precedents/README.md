# docs/precedents — контейнер прецедентов

Дом-контейнер прецедентов: единичных задокументированных случаев (симптом → корень →
фикс → профилактика) с классом и счётчиком рецидива. Реализация паттерна
[`GROUP_CONTAINERIZATION`](../patterns/GROUP_CONTAINERIZATION.md); оснастка — паттерн
[`HOME_WORKSHOP`](../patterns/HOME_WORKSHOP.md) (третий живущий жилец). Родом из инсайта
[`insight-precedent-container`](../insights/insight-precedent-container/INSIGHT.md).

## Таксономия — прецедент ≠ антипаттерн ≠ урок (читать ПЕРВЫМ)

Три каталога легко путаются и расползаются в дубли. Граница жёсткая:

| Что | Где живёт | Единица |
|-----|-----------|---------|
| **Прецедент** | `docs/precedents/` (здесь) | **единичный случай** с root cause и счётчиком рецидива класса |
| **Антипаттерн** | [`docs/audit/bestiary/`](../audit/bestiary/README.md) | **обобщённый класс** дефекта + specimens |
| **Урок** | [`USERCASE_COMPETITION_LESSONS.md`](../actions/device-board/USERCASE_COMPETITION_LESSONS.md) | **вывод для процесса** (симптом→корень→фикс→профилактика UserCase) |

Правило: прецедент — «это случилось вот тогда-то, вот корень»; когда класс повторяется и
обобщается — он поднимается в **антипаттерн** (бестиарий), а не размножается прецедентами.
Прецедент **ссылается** на антипаттерн/урок, но не дублирует их.

## Контракт (что писать / что коммитить)

| Орган | Что | Форма |
|-------|-----|-------|
| **Запись** | один прецедент = один markdown-файл `<id>.md` с мета-блоком | `<!-- precedent-meta { … } -->` + проза |
| **Схема** | типизированные поля мета (см. ниже); класс — из [`classes.json`](./classes.json) | валидатор `precedent:register` |
| **Реестр** | канонический снимок `registry/PRECEDENTS.md` (overwrite, Meta) — **производный**, не источник | пересобирается инструментом |
| **Кеш** | сырые дампы | `cache/` под gitignore |
| **Инструмент** | единственная точка записи + валидатор + пересбор снимка | `yarn precedent:register` |

**Источник истины — сами файлы прецедентов**, реестр агрегируется из них. Реестр руками
не править (дрейф ловит `--validate`).

## Схема записи (мета-блок)

```
<!-- precedent-meta
{
  "id": "2026-07-22-<slug>",        // обяз., = имя файла без .md
  "date": "2026-07-22",             // обяз., YYYY-MM-DD
  "class": "cold-start",            // обяз., из classes.json (enum)
  "symptom": "…",                   // обяз., короткий симптом
  "rootCause": "…",                 // обяз., системная причина
  "fix": "…",                       // обяз., что сделали/делаем
  "canonicalCause": "…",            // реком., нормализованная причина (для счётчика)
  "prevention": "…",                // опц., профилактика
  "actionItems": [{"text":"…","owner":"ozhegov","status":"open"}],  // опц.
  "related": ["<precedent-id>"]     // опц., родственные прецеденты
}
-->
```

Ниже мета — человеческая проза (хронология, вещдоки, разбор). Мета машиночитаема, проза — нет.

## Инструмент

```bash
yarn precedent:register --new <slug> --class <c> --title "…"   # каркас записи по схеме + пересбор
yarn precedent:register --validate                             # проверить все записи (зуб)
yarn precedent:register --rebuild                              # пересобрать снимок-реестр
```

Осмотр/декомпозиция/рассмотрение — мастерская дома
([`workshop.manifest.json`](./workshop.manifest.json), паттерн HOME_WORKSHOP).

## Границы

- Прецедент — единичный случай, не обобщённый класс (это бестиарий) и не вывод процесса (это урок).
- Счётчик рецидива — детерминированная агрегация по `class`/`canonicalCause`, **не ML**.
- Каталог-UI — вне области (deferred).
