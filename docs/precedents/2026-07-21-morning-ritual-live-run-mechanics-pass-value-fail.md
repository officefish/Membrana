## Часть 3. Журнал находок прогона

1. **№1 · Вечер не доносит артефакты в main** (×2 проявления): team-evening-feedback
   20.07 и DAILY_CODE_REVIEW 19–20.07 остались в ветке `fix/adr-0013-accepted` → утро
   с чистого main слепо ко вчерашнему вечеру; RT-9 отбил стендап по «трёхдневному»
   ревью, которое на самом деле было свежим — но в другой ветке. Лечение: вечерний
   ритуал обязан завершаться доносом артефактов в main (PR вечера).
2. **№2 · Встреча в конце вместо начала:** greet реализован внутри `angelina.mjs`,
   который стоит последним шагом цепочки; вердикт M4-H требует первую реплику. Разнести:
   `--greet` первым шагом, страж — последним.
3. **№3 · Нет замка перегенерации после чеканки:** повторный `ritual:day` после
   owner-choice/подписи human молча перепишет канон дня выбором LLM. Нужен предикат
   «канон зачеканен → перегенерация только по слову владельца».
4. **№4 · Порядок стендап↔канон не перевёрнут:** живая цепочка = старый порядок
   (стендап-обзор → канон); ратифицированный M3-S = канон → стендап Тимлида по нему
   (handoff). S-ядро в main, но не вшито.
5. **№5 · Режим процедуры не объявлен промптом:** процедурный агент посреди испытания
   ушёл в ремонт (≈20 permission-запросов). Принцип владельца: «он внутри процедуры —
   должен просто идти по шагам; когда хочу импровизацию — не замыкаю в процедуру».
   Спам прав = симптом смены режима; always-yes глушит сигнал, а не чинит.
6. **№6 · Корень отказов гейта скелета — обрезка хвоста промпта** (диагноз испытуемой
   сессии, поправивший диагноз архитектора): задание со скелетом лежало в хвосте сборки;
   при переполнении 95К хвост срезался → модель не видела задание и писала по образцу
   вчерашнего файла. Все 6 отказов гейта — одна причина. Фикс агента: переиспользован
   `assembleStandupPrompt` (защищённая сборка, чинилась 16.07), 22/22 теста; НЕ
   закоммичен — судьба через дифф дня → ревью → PR.
   **№6б:** контекст раздут реестром — 176 активных задач; реестр-обжора физически
   ломает генерацию. Довод в переезд реестра и GC-архивацию.
7. **№7 · Правило без шаблона не исполняется:** «структура доклада = зеркало плана»
   есть в скилле строкой — агент проскочил. В скилл нужен мини-шаблон блоков дословно.

## Часть 4. Процессные уроки (обе сессии)

- **Коллизия наблюдателя:** архитектор дважды ошибся о топологии испытуемой сессии —
  проверял не то дерево («агент ничего не сделал» — а тот всё сделал в worktree) и
  увёл ветку у неё из-под ног (`checkout --detach` для «освобождения»), что она
  зафиксировала вещдоком (`Already on` → `Switched to`). Правило «одно дерево — один
  агент» нарушил сам его автор.
- **Испытание ≠ ремонт:** боевой прогон чинят только минимальной разблокировкой
  жёсткого стопа; остальное — в журнал. Промпт прогона обязан это объявлять.
- Гейт, который «мешает» (6 отказов скелета), может быть героем: он 6 раз не дал
  записать канон, собранный вслепую, и довёл до настоящего корня.
