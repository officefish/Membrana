## Исход — согласие с обоснованием

Владелец ратифицировал мягкий вариант (предупреждение, а не FAIL): ручной протокол — другой
канал записи, а не нарушение регламента; нарушением было бы выдать его за произведённый
инструментом.

**Обоснование согласия:** ложный зелёный подрывает то, ради чего аудитор существует —
правду о процедуре; правка читает значение уже существующей пометки, новых требований к
авторам протоколов не вводит; код возврата не меняется, значит ни один сегодняшний зелёный
прогон не станет красным задним числом.

**Что сделано:** `TOOL_CHANNEL` + `protocolChannel()` (значение канала, регистр не значим);
проверка [6] разведена на три ветки — `llm` → PASS, иной канал → `ПРЕДУПР` с прямой
формулировкой «гейт посылок при записи НЕ вызывался», пометки нет → FAIL как прежде;
печать получила знак `⚠`; три теста.

**Живая проверка:** `yarn meeting:audit --id tasks-workshop` — ровно три ручных протокола
(m3, m4b, m4d) стали ⚠, семь инструментальных остались ✓, «Нарушений: 0» не изменилось.

Фактический дифф: 3 файла, 57 строк. Предикат на цепочке: `ok=true`, `chain_clear`
(2 выстрела, 5 файлов, 104 строки).

**Выход фрейма:** «весь код дописан». Ветка `fix/oneshot-1047-channel-hand` (от `origin/main`).

## По существу: как шёл выстрел (отчёт по требованию владельца)

**1. Вход.** Прочитал #1047 и код проверки [6] (`scripts/lib/meeting-audit.mjs:198-221`)
с её комментарием — он оказался важнее иссью. Комментарий объясняет, зачем правило вообще
введено: 18.07 нашли, что девять протоколов `evening-auditor` писались прямо в `docs/seanses`,
минуя `consilium.mjs`, то есть гейт «не сработал» неверно — он НЕ ВЫЗЫВАЛСЯ. Отсюда видно,
что проверка [6] по замыслу отвечает на вопрос «вызывался ли гейт при записи», а
реализация отвечала на другой — «есть ли комментарий в шапке». Иссью описывала симптом;
разрыв замысла и реализации я взял из кода.

**2. Развилка строгости.** Правка допускала две трактовки: ручной протокол — нарушение
(FAIL, валит код возврата) или другой канал записи (предупреждение). Иссью просила
предупреждение, но решение имело последствие: FAIL сегодня же покрасил бы три протокола
`tasks-workshop`, на которых стоят живые вердикты. Вынес развилку владельцу отдельным
вариантом ответа. Владелец выбрал предупреждение.

**3. Штамп.** Тем же обходом, что и в выстреле 1 (`yarn ask` мёртв до 01.08): лог цепочки
`anthropic … rate_limit` → `openrouter … ok`. Вердикт S ✓ **с условием на цепочку**: третий
выстрел остаётся в формате, только если он правда 1–2 файла и меньше 100 строк. Это условие
сработало через час — на выстреле 3 тимлид дал BLOCK.

**4. Предикат — здесь он работал не формально.** Впервые считалась ЦЕПОЧКА: вход
`recentShots = [выстрел 1: 2 файла / 44 строки]`. Соседство определяется по `pathFamily`,
и оба выстрела попадают в семейство `scripts/lib` — то есть контрмера capability chaining
включилась по-настоящему, а не вхолостую: `chainShots 2 · chainFiles 5 · chainLines 104`
против порогов 3 / 8 / 200. Именно этот счёт потом и сделал третий выстрел последним
возможным за день.

**5. Исполнение.** Не стал прятать разбор канала в условие внутри проверки, а вынес
`protocolChannel()` — чистую функцию, читающую ЗНАЧЕНИЕ пометки (регистр не значим,
кириллица в значении допустима), и константу `TOOL_CHANNEL = 'llm'` — единственное значение,
при котором гейт реально отработал при записи. Проверка [6] разведена на три ветки.
Новый статус `ПРЕДУПР` намеренно НЕ попадает в `violations`: обоснование взял из соседнего
кода — статус `НЕЧЕМ` там уже не валит код возврата по принципу «отсутствие зубов ≠
нарушение». Формулировку заметки написал прямой: «канал «hand»: протокол вёлся вручную,
гейт посылок при записи НЕ вызывался — секции проверены только постфактум», без слова
«произведён инструментом», из-за которого весь сыр-бор.

**6. Проверка — живая, на настоящем заседании.** `yarn meeting:audit --id tasks-workshop`:

```
✓ [6] m0 · ✓ m1 · ✓ m2 · ⚠ m3 · ✓ m4a · ⚠ m4b · ✓ m4c · ⚠ m4d · ✓ m5a · ✓ m5b
Нарушений: 0.
```

Три ⚠ — ровно те три ручных протокола, что названы в эпизоде иссью; семь инструментальных
остались ✓; строка «Нарушений: 0» не изменилась, то есть ни один сегодняшний зелёный прогон
не покраснел задним числом. Тесты: `node --test scripts/meeting-audit.test.mjs
scripts/protocol-validator.test.mjs` — 47/47 (16 + 31), три новых.

**Чего выстрел НЕ закрыл:** аудитор теперь честно говорит, что гейт при записи не звали, —
но он по-прежнему не может сказать, ЧТО именно ручной протокол потерял. Проверки 1–4 судят
структуру постфактум, семантику (спрятанная посылка, колонизация без упоминания ID) машина
не судит вовсе — это записано в самом коде как честный предел и моей правкой не сдвинуто.
Хвост S-M5 заседания `tasks-workshop` (не было независимого аудитора-человека) этим
выстрелом тоже не снят.
