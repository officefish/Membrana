## Находка

Первый прогон M0 заседания `tasks-workshop` отработал 28 реплик комнаты и умер при
сохранении: `ReferenceError: model is not defined` (`scripts/consilium.mjs:707`) — регрессия
перехода на procedure channels (#1021). Починено PR #1032 (`model: answeredBy ?? chainLabel`),
теста на путь сохранения нет.

## Предложенный объём (не исполнен)

2 файла / ~61 строка: гард запуска + экспорт `wrapSeanseFile` + чистая `seanseModelLabel`

## Отказ тимлида — BLOCK

Vesnin (через openrouter, `anthropic/claude-haiku-4.5`) штамп **не поставил**. Основание
дословно по смыслу: тест закрывает класс функции, но не класс регрессии. ReferenceError
жил в пути `main()` → цепочка → `wrapSeanseFile`; предлагаемый тест этот путь не исполняет,
точка 707 ему не видна. Штамп S дал бы ложное ощущение, что регресс закрыт, — это хуже,
чем незакрытый регресс. Формулировка тимлида: «выстрел без полного покрытия пути — это не
выстрел, это галочка».

Предел покрытия назвал сам агент в запросе штампа, до вердикта; тимлид на нём и отказал.
Предикат `evaluateOneShotS` на этой находке не запускался — двойной гейт остановился раньше,
на штампе.

## Мотивированная рекомендация (не вердикт за владельца)

Перевести #1051 в обычную задачу с явным DoD: **путь сохранения исполняется тестом с
мокнутой цепочкой** — оба исхода (ответило первое звено / ответило второе) и обе ветки
записи (`docs/seanses/` и `docs/seanses/rejected/`). Это же снимет вторую половину эпизода:
дорогой прогон умирает ПОСЛЕ генерации, когда деньги уже потрачены — сегодня такой сбой
ловится только живым прогоном.

Пока задачи нет, дешёвая страховка вне процедуры: гард запуска в `consilium.mjs` (сейчас
`main()` зовётся безусловно, любой импорт прогоняет консилиум) — но это отдельная находка,
и подавать её как «тот самый тест» было бы ровно той галочкой, за которую тимлид отказал.

**Итог цепочки дня:** три запроса, два штампа S, один отказ; исполнено два выстрела
(5 файлов, 104 строки), лимит выстрелов (≤3) не исчерпан.
