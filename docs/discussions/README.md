# Обсуждения и журналы

Папка с архивом **хронологических тредов по одной теме / задаче**.
Заполняется двумя способами:

1. **Автоматически** — скриптом `scripts/ask-persona.mjs` при обращении
   к виртуальной команде через `yarn ask <persona>`. Это исторически
   первичный сценарий — отсюда название «обсуждения».
2. **Вручную** — журналом разработки крупного пакета / эпопеи (например
   [`background-office-v0.1.md`](./background-office-v0.1.md);
   для `background-media` — консилиум в [`../seanses/`](../seanses/)). Такой файл
   ведётся постановщиком или Teamlead-ом параллельно с работой и служит:
   - подсказкой будущим агентам, которые подхватят тему,
   - заготовкой описания/комментариев для Linear-ticket'а,
   - архивной памятью «как мы к этому пришли», когда Linear-доступа нет.

## Соглашение по именам

| Источник | Имя файла |
|----------|-----------|
| `yarn ask ... --gh-issue N` | `gh-issue-<N>.md` |
| `yarn ask ... --ticket-file foo.md` | `foo.md` |
| `yarn ask ... --save-as TEC-42` | `TEC-42.md` |
| Журнал разработки пакета / эпопеи | `<пакет-или-тема>-vX.Y.md` |
| свободный вопрос без контекста (через `yarn ask`) | не сохраняется по умолчанию |

Linear-ID (например `TEC-42`) — самый удобный способ именовать обсуждение,
потому что один и тот же файл копит весь тред по одной задаче в Linear.
Для журналов разработки удобнее имя по пакету (`background-office-v0.1.md`),
потому что один пакет проходит несколько Linear-ticket'ов.

## Жизненный цикл

1. **Создание:** первый вызов `yarn ask` с подходящим контекстом порождает файл
   с заголовком и первым блоком вопрос-ответ.
2. **Накопление:** последующие вызовы **дописывают** новые блоки в конец.
3. **Архивация:** при закрытии задачи в Linear / GitHub файл остаётся в `docs/discussions/`
   — это публичный архив, ищется через git history.

## Что НЕ хранится здесь

- Финальный отчёт о выполнении задачи — в GitHub Issue (см. `docs/TASKS_MANAGEMENT.md`
  раздел 6).
- Технические решения «как реализовать» — в коде и в PR-описании.
- Обсуждения с не-virtual членами команды — в Linear (Шаг 2) или в чате.
- Task-промпты (постановки больших задач) — в [`../prompts/`](../prompts/).
  Журнал может ссылаться на task-промпт, но не дублирует его.
