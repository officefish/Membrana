**Вопрос:** Ты знаком с таким подходом в разработке UI/UX интерфейса?

**Ответ:**

Да, знаком. Это **архитектурный паттерн слабой связанности с фасадами и сервис-ориентированным подходом** — построение UI как набора независимых модулей через **чистые слои** (pure TS → тонкие React-обёртки) и явные контракты.

В контексте Membrana это выглядит как:

1. **Разделение logic ↔ UI:** сервис живёт отдельно (чистая логика в `.ts`), React-обёртка в хуках — только синхронизация состояния, никаких вычислений.
2. **Фасады вместо прямых импортов:** `MembranaRegistry` — единая точка регистрации модулей и плагинов вместо того, чтобы каждый плагин мусолил друг друга напрямую.
3. **Lifecycle явный:** плагины имеют `install()` / teardown, живут в реестре, активируются через store, а не через глобальное состояние в React-компонентах.
4. **Hub-паттерн для обмена:** `microphoneStreamHub` — публичная точка подписки вместо проброса props или Context-адской цепочки.

Это похоже на Redux + sagaи, но строже: **нет горизонтальных импортов между плагинами**, вся коммуникация идёт через hub или replaySubject, а сам движок (audio-engine) забаррикадирован в одном месте.
