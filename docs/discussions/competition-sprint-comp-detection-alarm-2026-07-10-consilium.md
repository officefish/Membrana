## Team Alpha — «Партитура из шести актов» (lead Ozhegov)

**Strengths:** максимально полная цепочка (report-build + track-upload); плоский граф
①–⑥ с одной exec-магистралью; изящный mirror-узел fusion в alarm (score-гейт proximity
виден на канвасе, dataflow-only, прецедент fn-1-block); отдельный
`detection-alarm-competition-user-case-entries.ts` — чистый каталожный паттерн;
идемпотентность отчёта «по построению» (отчёт один раз в main).

**Weaknesses:** hand-authored статический документ 840 строк — самый тяжёлый в
сопровождении (не переиспользует канон MVP bootstrap); report-build путь на живом
клиенте реджектится (находка Gamma) — smoke прошёл на stub-host, live упал бы;
14 новых тестов — меньше всех.

**Open questions:** миграция 840-строчного документа при следующем изменении канона MVP.

## Team Beta — «Измеренный сценарий» (lead Dynin)

**Strengths:** деривация bundled MVP v2.0-async (клон → замена report-секции) —
минимальный дрейф канона, bootstrap/журнал/teardown наследуются; каждый порог выведен
из DETECTOR_BENCHMARK (тревога 0.55 = minConfidence trends, инвариант «combined не мягче
одиночного гейта»); fusion ровно 2 входа с обоснованием (члены ансамбля уже слиты в
EnsembleProducer — сырые 3–4 входа удвоили бы коррелированные ошибки); 24 новых теста —
больше всех, включая грань 0.545/0.55 и находку presence-гейта (нетипизированный, иначе
collect-store до flush бросает type-mismatch); зафиксировала главный платформенный gap
(exec-subgraph не пробрасывает basn-store'ы).

**Weaknesses:** operator clarity наследует компоновку MVP — граф читается хуже
«плаката»/«партитуры»; report-build путь имеет тот же live-gap, что у Alpha (не отловлен
самой командой — отловила Gamma).

**Open questions:** нет.

## Team Gamma — «Прозрачный сценарий / плакат ①–⑥» (lead Rodchenko)

**Strengths:** лучшая наблюдаемость — print только в поворотных точках, тренд
«ближе/дальше» перезаписью одного слота (без спама), группы совпадают с шагами
demo-скрипта; **единственная команда, чей async-путь работает на живом host**:
обнаружила, что клиент принимает только `track-upload` (report-build → reject), честно
перевела отчёт на detached `on-async-resolved` промиса трека, ADR G5 задокументирован;
тест идемпотентности повторного combined-report.

**Weaknesses:** отход от буквы брифа (report-build канал заменён обходным путём) —
оправданный, но канонизировать его без решения жюри нельзя; сборка через новый builder —
средний дрейф от канона MVP.

**Open questions:** судьба detached-паттерна после починки report-build на host.

## Cross-cutting themes

| Theme | Alpha | Beta | Gamma |
|-------|-------|------|-------|
| Конструкция документа | hand-authored 840 LOC | деривация MVP-builder | новый builder |
| Fusion входы | 2 | 2 (ADR-обоснование) | 2 |
| Async отчёта | report-build (live-reject) | report-build (live-reject) | detached через track-промис (live-ok) |
| Новые тесты | 14 | 24 | 16 |
| Каталог | отд. файл + splice | правка bundled напрямую | отд. файл + барреллы |
| Читаемость канваса | партитура ①–⑥ | наследует MVP | плакат ①–⑥ + print-история |

**Платформенные находки спринта (gap main, не команд):**
1. `exec-subgraph.ts` не пробрасывает fusion/ensemble/proximity stores в
   `executeScenarioBlock` → живой браузерный run basn-узлов невозможен (~6 строк wiring). — Beta
2. Клиентский host (`scenarioMicJournalBridge.startAsyncJob`) принимает только
   `track-upload`; `report-build` реджектится. — Gamma
3. commit-msg хук не знает `comp(...)`-тип из регламента — все команды документированно
   использовали `SKIP_COMMIT_MSG=1`. — все

## Журнал реплик

[Teamlead — Vesnin]: Все три уложились в DoD и не тронули запрещённое (нет новых узлов,
границы чисты, изоляция веток соблюдена — DQ-оснований нет). Спринт окупился до
голосования: две платформенные находки, которые в одиночном треке мы бы поймали только
на живом smoke. По архитектуре берегу деривацию Beta — это ответ на главный риск
каталога: дрейф форков от канона (боль прошлого async-v2 спринта).

[Структурщик — Ozhegov]: C2/C5 факты. Beta — единственная, чьё решение ДЕШЕВЕЕТ со
временем: канон MVP меняется → деривация подхватывает. Alpha красива на канвасе, но
840 hand-authored строк — это второй канон, который разъедется. Gamma посередине.
Каталожно чище всех Alpha (отдельный файл — паттерн community-competition); правка
bundled напрямую у Beta — минус балл C2, на 5b исправим консолидацией.

[Математик — Dynin]: C3. Матрица Beta образцовая: грань порога с двух сторон
(0.545/0.55), пустой fusion, идемпотентность, alarm frames-гейт, plus находка
presence-гейта с фиксирующим тестом. 24 теста против 14/16 — и не количеством, а
краями. Пороги выведены, а не угаданы — «объяснимый сценарий» дословно. У Alpha
контрпример ниже порога тоже хорош, but mirror-узел не покрыт негативным кейсом.

[Музыкант — Kuryokhin]: аудио-каденс. Beta: окно трендов 4 c ≤ окно записи 5 c, alarm
0.5 c под каденс 400 мс — согласованная временная сетка, единственная команда, где
это выведено явно. Gamma — единственный live-исполнимый async-путь: на реальном
микрофоне ноль rejected-jobs. Это не мелочь: demo-скрипт п.5 у Alpha/Beta сегодня бы
покраснел. Балл C6 Gamma — максимальный.

[Верстальщик — Rodchenko]: C4. Плакат Gamma — лучший операторский опыт: история
детекции читается из журнала без документации, слот-перезапись тренда — находка против
спама. Партитура Alpha рядом. Beta функциональна, но оператору нужен CONCEPT, чтобы
понять цифры на узлах — жирный минус не ставлю (числа хотя бы подписаны policy-узлами).

[Teamlead — Vesnin]: Rebuttal-окно предлагалось всем трём в отчётах — существенных
возражений нет (примечания команд про commit-msg хук учтены в Cross-cutting п.3).
Переходим к бюллетеням.

   Задачи A (живой smoke), S-задача сразу после merge победителя.
2. **Host: поддержать `report-build`** в scenarioMicJournalBridge.startAsyncJob (или
   официально канонизировать detached-паттерн Gamma) — решение при wiring-фиксе.
3. commit-msg хук: допустить `comp(<sprint-id>/<team>)`.
4. Cherry-pick идей (не diff): print-слот наблюдаемости Gamma и нумерованные группы
   Alpha — рекомендованы победителю при 5b-полировке.
