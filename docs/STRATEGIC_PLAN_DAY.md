## 3. Магистраль дня (главная задача)

### 🎯 S2 — Combined UserCase: fusion спектр + нейро + alarm-loop «ближе/дальше»

Это прямая цель форсайта и главный незакрытый шаг перед FREE. Основания из контекста:

1. Форсайт явно ставит S2 первым в цепочке до 17.07.
2. Fusion должен брать **сырой confidence yamnet, не бинарный вердикт** (профили ошибок DSP/нейро слабо коррелированы — заметка ND3).
3. Граф спектра под это уже развязан (PC-2c, `#511`): поток+FFT без рекордера.

**Конкретные шаги (в порядке приоритета):**

1. **Разведка перед реализацией** — картина по S2 из коммитов **неполная**. Нужно прочитать перед стартом:
   - task-промпт/карточку S2 combined UC (в git его в явном виде не видно — **см. §5 разведка**);
   - `docs/seanses/foresight-2026-07-06.md` — точная декомпозиция S2;
   - существующий граф `usercase-free-spectrum-live.ts` (обновлён вчера) — как база для добавления нейро-ветки.
2. **Проектирование fusion-узла**: как в графе device-board соединить FFT-наблюдение (trends/спектр) с сырым confidence yamnet-детектора. Ключ — не бинарный OR, а объединение confidence.
3. **Alarm-loop «ближе/дальше»** по громкости (RMS) — механика есть в `fft-analyzer` (RMS уже считается).
4. Консилиум → impl → adversarial verify → Teamlead LGTM (канонный цикл, как в PC-2).

**Соответствие архитектуре:** fusion-логика графа живёт в `device-board`; детекторы остаются сервисами в `packages/services/detectors/*`; горизонтальных зависимостей между сервисами не вводить (§ARCHITECTURE 1a/1e).
