# Device Board MVP (v0.4)

**Device board** — visual scripting редактор сценария поведения прибора: два слоя (**Signal** и **Scenario**), scenario runtime, переменные document-scope, palette-узлы и host-bridge к Web Audio через client.

## Статус

После PR [#96–#123](https://github.com/officefish/Membrana/pulls?q=is%3Apr+device-board) пакет `@membrana/device-board` **приближается к MVP**:

| Возможность | Статус |
|-------------|--------|
| Fullscreen shell, ветки обработчиков (On start, onMainTick, …) | ✅ |
| Переменные + variable-get/set | ✅ |
| Системный Event-узел, exec + data рёбра | ✅ |
| `resolveInput`, `Ref.valid`, isValid | ✅ |
| Palette: Print, GetMicrophone, streaming chain | ✅ |
| Scenario runtime (initial → main/alarm loops) | ✅ |
| Manual save / draft, branch export JSON | ✅ |
| End-to-end mic → stream → sample → FFT (через host) | ✅ smoke |
| Mintlify docs (этот сайт) | 🚧 Phase 0+ |
| Scheduled jobs, cloud-only persist | ❌ out of MVP |

## Операторский сценарий MVP

1. Открыть device-board в client или cabinet.
2. На **On start**: Event → set переменных → GetMicrophone → StartStreaming.
3. На **onMainTick**: isValid(stream) → GetSample → GetFFTFrame → Print.
4. **Run** — Print с metadata в консоли.
5. **Export** ветки или полного JSON; **Save** по кнопке.

## Что не входит в MVP

- Полный alarm-cookbook «из коробки»
- `scheduled` ветка сценария
- Membrana Device как отдельный desktop SKU
- Pre-run warning «GetSample без StartStreaming» (планируется)
