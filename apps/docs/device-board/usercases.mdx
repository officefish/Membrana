# UserCases (catalog U9)

**UserCase** — полный сценарий device-board (`DeviceScenarioDocument` v2): все обработчики,
пользовательские функции и визуальные группы, собранные редакторами Membrana в один шаблон
с LR-layout.

Канон: [`DEVICE_BOARD_CONCEPT.md`](https://github.com/officefish/Membrana/blob/main/packages/device-board/DEVICE_BOARD_CONCEPT.md) §20.

<Info>
С **U10** системные UserCases на доске открываются в режиме **только просмотр**. Редактирование —
через **клон** в [user workspace](/device-board/user-workspace). Выбор контекста — в **модуле**
«Доска устройства», не в шапке доски.
</Info>

## Включить каталог

1. Откройте модуль **«Доска устройства»** на дашборде.
2. В секции **«Системные UserCases»** включите toggle **«Каталог»**.
3. Карточки показывают entitlement: **Bundled** (бесплатно) или **Тариф** (stub до cabinet).

## Просмотр и клон

1. Выберите карточку системного шаблона.
2. **Открыть доску** — preview/run без Save (badge «Только просмотр»).
3. **Клонировать в мой сценарий** — копия в user-слот (считается в квоту N/max); доска откроется в режиме редактирования.

Подробно: [User Workspace](/device-board/user-workspace).

Sign-off: [`USERCASE_MVP_MICROPHONE_LGTM.md`](https://github.com/officefish/Membrana/blob/main/docs/device-board-scripts/USERCASE_MVP_MICROPHONE_LGTM.md).

## Tariff (stub)

Карточки с tier `tariff` без активного SKU показываются как **locked** («Доступно в тарифе»).
Интеграция cabinet SKU — отдельная задача; entitlement читается в client, не в `@membrana/core`.

## Для разработчиков

| Script | Назначение |
|--------|------------|
| `yarn usercase:build <id>` | Сборка embedded document + layout canon |
| `yarn usercase:verify-kinds <id>` | Проверка kinds ⊆ palette |
| `yarn usercase:verify-layout <id>` | CI layout metrics |

Источники: `docs/device-board-scripts/usercase-*/manifest.json`.
