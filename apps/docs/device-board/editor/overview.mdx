## Save и draft

- **Manual save** — сценарий не уходит на сервер автоматически; `isDirty` сравнивает fingerprint с последним сохранённым снимком.
- Persist-адаптер (cabinet / local) — только по явному Save.
- Переключение handler-ветки без Save — см. [branch snapshot](/device-board/editor/edit-and-navigation#branch-snapshot-f7).

## Export JSON

| Контекст | Файл |
|----------|------|
| Вкладка **Signal** | полный `DeviceScenarioDocument` |
| Вкладка **Scenario** (активная ветка) | `branch-scenario` JSON только этой ветки |

Имя файла ветки: `device-scenario-{deviceKind}-{branch}.json`.

## Clear branch

Очищает узлы **текущей** ветки; системные Event-entry и loop-repeat (∞) сохраняются.
Действие попадает в **undo depth-1** (один шаг отката).

## Typed ports и edges

- Data-рёбра окрашены по `SocketType` (DeviceRef, MicrophoneRef, AudioStreamRef, …).
- Passthrough labels на exec/data для variable-set и streaming-узлов.

## Variable inspector

При **Run** — снимок входов/выходов выбранного узла (`inspectRuntimeNode`). Канвас read-only.

## UserCases

Готовые сценарии из каталога — [UserCases](/device-board/usercases). Apply-all заменяет scenario-слой;
signal graph не трогается. При `isDirty` — подтверждение перед заменой.
