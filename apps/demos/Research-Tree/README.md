## Марафон: формат работы с графом

Граф развивается **марафонами** — серией коротких рабочих сессий с чётким фокусом
на одном шаге, а не единым долгим штурмом.

### Принципы марафона

1. **Одна сессия — один шаг.** Не пытаться сверить все 14 узлов за раз.
   Типичная сессия: 3–5 узлов одной ветки.
2. **Верифицировать, а не придумывать.** Каждый снятый `verify` — это факт из
   репозитория, не предположение. Если данных нет — оставить `note` с обоснованием.
3. **Валидатор зелёный после каждой сессии.** Частичная сверка — ок; сломанный граф — нет.
4. **PR после блока сессий**, не после каждой (чтобы не дробить историю).

### Как запустить сессию

```bash
# 1. Убедиться, что JSON валиден
python -m json.tool apps/demos/Research-Tree/membrana-knowledge-graph.json > /dev/null

# 2. Прогнать валидатор инвариантов (из AGENT_TASK.md)
python scripts/validate-knowledge-graph.py   # или inline-скрипт из AGENT_TASK.md

# 3. Выбрать ветку узлов для сверки (например, branch=infra-ops)
# 4. Сверить каждый verify-узел с репозиторием, снять флаг или добавить note
# 5. Прогнать валидатор снова — должен быть зелёным
# 6. Зафиксировать в коммите с пометкой сессии
```

### Очерёдность сверки (рекомендуется)

| Приоритет | Ветка / группа | Узлы |

| 1 | infra-ops | `stack.docker`, `stack.vps-timeweb`, `stack.vps-deploy` |

| 4 | frontend | `stack.electron-studio` |
