# Граф знаний Membrana — конвенции

Карта областей, которые проект вскрыл, и возможностей, которые они открыли.
Покрывает и математику детекции, и весь производственный стек, и процесс, и
внешние коммуникации. Не дорожная карта с датами — та в white paper.

Источник истины: `membrana-knowledge-graph.json`. Документы и рендеры ссылаются
на него, а не дублируют.

## Узел

```jsonc
{
  "id": "stack.docker",
  "title": "...",
  "type": "knowledge | capability | service | process",
  "branch": "infra-ops", // топика, для layout
  "epoch": "E0", // null, если узел вне лестницы детекции
  "facets": { "skill": "hard|soft", "process": "internal|external" },
  "state": "fog | available | exploring | established",
  "requires": ["..."],
  "cost": { "tokens_budget": null, "tokens_spent": null }, // опц.
  "gate": { "criterion": "...", "status": "open|passed" }, // опц.
  "verify": true, // опц.: состояние по памяти, сверить
  "note": "...",
  "evidence": "...", // опц.
}
```

## Три ортогональные оси

- **branch** — топический кластер (для раскладки).
- **epoch** — лестница зрелости продукта детекции (tier). `null` вне лестницы.
- **facets** — стратегическая классификация (2×2 → позже 4×4).

Это разные оси. Один узел несёт все три.

## Типы и состояния

Типы: `knowledge` (математика/физика), `capability` (инженерный слой),
`service` (детектор/сервис), `process` (дисциплина/коммуникация, не код).

| Состояние     | Значение                                      |
| ------------- | --------------------------------------------- |
| `fog`         | хотя бы один `require` не `established`       |
| `available`   | все `require` `established`, работа не начата |
| `exploring`   | работа идёт, шлюз не пройден                  |
| `established` | шлюз пройден / узел без шлюза завершён        |

`fog`/`available` выводятся из графа. Узел в `exploring`/`established` обязан
иметь все `requires` `established` (инвариант, проверяется скриптом).

## Грань facets (2×2)

|                    | internal (для себя)       | external (наружу)                     |
| ------------------ | ------------------------- | ------------------------------------- |
| **hard** (техника) | машинерия разработки      | развёрнутый продукт, интеграции, поле |
| **soft** (процесс) | дисциплина и ритм команды | инвесторам, фондам, рынку             |

Квадрант = `facets.skill + "." + facets.process`. Рендер может сечь холст на
четыре секции по квадранту; ветки — кластеры внутри.

**4 → 16 — третья грань `materiality`.** 16 = 2(skill) × 2(process) × 4(уровня).
Уровни — вес артефакта от физического к информационному:

- **M1 Материя** — физический капитал; медленно/дорого менять (capex).
- **M2 Субстрат** — внешняя основа, на которой строим, не владея.
- **M3 Структура** — наши композиции: собрали и держим.
- **M4 Действие** — гранулярное, эфемерное, разовое (opex).

|        | hard·внутр                | hard·внешн                | soft·внутр                       | soft·внешн                       |
| ------ | ------------------------- | ------------------------- | -------------------------------- | -------------------------------- |
| **M1** | dev-железо                | полевой узел, GPS, VPS    | конституция: stage-gate, границы | базовая позиция: dual-use        |
| **M2** | Node, TS, Nest, Turbo     | внешние прод-API, хостинг | заимствованные практики          | нормы среды: фонды, регуляторика |
| **M3** | core, контракт, benchmark | развёрнутые контуры       | наши процессы и роли             | наши материалы: WP, pitch        |
| **M4** | .mjs, ритуал, CI-шаги     | деплой, полевой прогон    | разовые акты                     | встреча, заявка                  |

Каденция: M1=capex (редко/крупно), M4=opex (часто/мелко) — ложится на дневной
поток токенов. Цена разворота: M1 необратимо, M4 сбрасывается.

Узлы `type=knowledge` **вне шкалы** (`materiality: null`) — чистое понимание
есть информационный предел, а не артефакт.

**Баланс по квадрантам** — второй измеритель зрелости рядом с эпохами:
`established / total` в каждом квадранте. Прорыв в одном при пустоте в другом —
видимый перекос портфеля (для pre-seed важнее, чем баланс по веткам).

## Эпохи и tier продукта

Эпоха — полоса узлов, соответствующая `product_tier`. Два порога:

- **entered** — закреплён ≥1 узел эпохи (работа в фазе началась).
- **tier_reached** — закреплён `tier_node` (продукт перешёл в фазу зрелости).

Эпохи касаются в основном hard-узлов. soft-узлы (process/comm) обычно
`epoch: null`. `process.stage-gate` — исключение: он управляет переходами, но
сам вне лестницы.

## Двухфазное освоение и экономика токенов

`gate` = механика «синапс → закрепление». Узел с `gate` остаётся `exploring`,
пока `gate.status != passed`.

Токены — дневной **поток**, не запас: ограничивают скорость и параллелизм, не
достижимость. `cost.tokens_budget` — оценка; `cost.tokens_spent` — факт.
**Трата токенов узел не закрывает; закрывает `gate`.** Числа — из реального
учёта, не выдумывать; `null` = не измерено.

## Стратегия

- **balanced** — широкий фронт перед переходом.
- **breakthrough** — одна цепочка до `tier_node`, остальное в долг.
- **debt** — `established/total < 1` при переходе = технический долг.

## Контур: эпистемическая ось пути

Четвёртая ось — насколько ясен **путь** к узлу. Отдельно от `state` (готовности
предпосылок): узел бывает `fog` и при этом «план».

| Контур        | Значение                                 |
| ------------- | ---------------------------------------- |
| `реальность`  | узел `established`                       |
| `план`        | путь прочерчен до ядра; энергия известна |
| `горизонт`    | направление видно, путь с пробелами      |
| `возможность` | несвязный узел во внешнем кольце         |

Ручной вход — поле `path: complete (умолч.) | partial | floating`. Контур
выводится и **транзитивен**: узел не яснее своей туманнейшей предпосылки.

```jsonc
// припаркованная идея во внешнем кольце:
{
  "id": "research.swarm-behaviour",
  "title": "...",
  "type": "knowledge",
  "branch": "spatial",
  "epoch": null,
  "state": "fog",
  "facets": { "skill": "hard", "process": "internal" },
  "requires": [],
  "path": "floating",
}
```

**Энергия считается только для плана** — сумма `cost.tokens_budget` по
непройденным узлам замыкания. Для горизонта и возможности не определена: путь с
пробелами не тарифицируется.

**Две ортогональные жизни узла:**

- `state` (исполнение): `fog → available → exploring → established`
- контур (знание пути): `возможность → горизонт → план → реальность`

Встречаются в `established` = `реальность`.

**Раскладка.** Контур = концентрические кольца (радиус): `реальность` в центре,
`возможность` на ободе. Радиус занят контуром — значит эпоху кодируем цветом, а
квадрант — сектором холста. Каналов восприятия немного; лишние оси уводим в
фильтры-переключатели, а не рисуем все разом.

## Артефакты: спутники узла

Артефакт — собираемый деливерабл, орбитирующий узел (не узел и не ребро).
Освоение узла = построение связи **плюс** сбор его спутников.

Отдельный массив `artifacts` (many-to-many через `node`/`also`), не вложен:

```jsonc
{
  "id": "art.cfg-1tight",
  "title": "...",
  "node": "layer.trends-fft",
  "also": ["..."],
  "kind": "config|dataset|module|report|doc|script|command|hardware",
  "status": "collected|in_progress|missing",
  "materiality": "M3",
  "cost": {},
  "path": "...",
  "verify": true,
}
```

- **Прогресс узла** = `collected / total`. Текстура `exploring`: «собрано 2 из 4».
- **Энергия** живёт на артефактах; `node.cost` = свёртка их `cost`.
- **Материальность** опускается с узла на спутники, где ей место.
- **Инвариант**: `established` ⇒ нет `missing` требуемых спутников.
- Сбор ⟂ `state`: `fog`-узел может иметь собранные доки; `exploring` — ноль.
- Поглощает прежнее поле `evidence` (вырожденный артефакт-одиночка).

## Ресурсы: материя, на которой держится граф

Три ресурса разной природы. **Биндят не одинаково** — в этом вся суть.

| Ресурс      | Природа                                | Роль                                       |
| ----------- | -------------------------------------- | ------------------------------------------ |
| **Инфра**   | потолок ёмкости (запас, capex, M1)     | бюджет = предел параллелизма и скорости    |
| **Токены**  | возобновляемый поток (opex, M4)        | объём машинной работы в день; редко биндит |
| **Стамина** | дефицитный поток + WIP-лимит (человек) | число активных фронтов; **истинное горло** |

Лежат в `resources` (инфра со спеками; `spend`; стамина). Узлы тянут ресурс
через `node.draws` (tokens + stamina); инфра — потолок, который узел
`constrains`-ит.

**Денежная ось `spend` — три формы биллинга:**

- **metered** (Claude API, Perplexity, OpenRouter, GLM) — per-token → `draws.tokens`.
- **subscription** (Cursor Pro+, Codex, Perplexity Pro) — flat месяц + потолок;
  предоплаченная ёмкость, ближе к инфре.
- **overhead** (интернет, хостинг, сотовая) — flat, не атрибутируется.

**Базовый burn** = Σ(subscriptions) + Σ(overhead): фиксированный пол, горит и в
ноль-прогресс день. Метеред горит только в работе. Аренда/электричество — в
`out_of_scope` (реальны, вне фокуса).

**Tool sprawl.** Подписки и метеред множат машинную пропускную способность, не
стамину. Оплаченная ёмкость сверх `wip_limit` простаивает.

**Диагноз биндинга (калибровка).** Сейчас не насыщены ни стамина, ни токены —
есть запас. Текущий лимитер — **серийный критический путь**: основной продукт
движется малыми шагами, человек в контуре, 3–4 спринта/день. Это Амдаль: фон
параллелит необязательное, серийную долю не ускоряет.

**Полосы дня** (`day_structure`) — не плоский бюджет, а чередование:

- **Пик** (~2ч из 5) — Claude Plus + Codex на критическом пути; биндит **квота
  подписки** (выжиг ~2ч, затем регенерация), не стамина.
- **Поддержка** (~3ч) — Cursor + офлайн после премиум-квоты.
- **Фон** (ночь, async) — OpenRouter/Ollama/z-code на необязательных
  накопительных операциях (исследования, дайджесты, эрудиция агентов); нулевая
  стамина; структурно **не касается критического пути**, кормит внешнее кольцо.

**Спринт** = единица критического пути; `sprint ≈ один артефакт-шаг` у
`exploring`-узла. Каденция 3–4/день стыкуется со слоем артефактов.

**WIP-лимит.** `count(exploring) ≤ stamina.wip_limit`. Это формализует «не могу
во все направления сразу»: фронтов не больше ёмкости внимания. Виртуальная
команда множит токен-пропускную способность, **не стамину** — интеграция и
решения остаются на одном человеке. Хронический перерасход роняет `wip_limit`
(выгорание = человеческий долг прорыва).

Тарифы — публичные ставки, сверяются со своим планом. Спеки железа, дневной
баланс и стамина — приватный ввод (`null`/`verify`).

## Жизненный цикл узла: capital vs sustained

До сих пор `established` был монотонным. Операционные узлы это ломают:

- **capital** (умолч.) — открыт один раз, держится без затрат. Знание, конфиг,
  скрипт. 53 из 60 узлов.
- **sustained** — `upkeep = {energy, kind:[tokens|stamina|money|infra]}`: дренит
  ресурс, пока `established`. При неоплате **деградирует** `established →
exploring`. Кластеризуются в hard/внешн и поздних эпохах (полевая сеть,
  data-plane, работающие агенты).

В модель входит распад: держать узел — это работа, а не галочка.

## Capacity-gate эпохи

Переход в эпоху — конъюнкция: `tier_node закрыт` **И** `стоячая ёмкость ≥ порога`
(`epoch.capacity_gate`: люди / токен-поток / инфра-бюджет). Порог приблизительно
равен **сумме upkeep sustained-узлов эпохи**: нельзя войти в развёртывание, пока
нет ёмкости _удерживать_ развёрнутое. E2–E4 несут такой gate.

## Команда: провайдеры ёмкости

Венчурный слой расширяем. Участники поставляют ёмкость:

- **Стамина — вектор по квадрантам**, не скаляр. Найм заполняет компоненту там,
  где движется путь: маркетолог → `soft.external`, сильный dev → `hard.internal`.
- Команда даёт параллелизм **между** квадрантами; серийное ядро критического
  пути людьми не ускоряется (Брукс поверх Амдаля).
- Виртуальная AI-команда множит токен-поток, **не** стамину.

`resources.team.members` со `status: active|onboarding|planned`. Привлечение
людей может быть и условием, и следствием `capacity_gate`.

## Расширение

Новый узел — добавить в `nodes`. Новая категория — расширить `meta.node_types`
или дробное значение `facets`. Новая эпоха — в `epochs` с `tier_node`.
Избегать рёбер назад во времени (граф ацикличен) и дублей знания в разных
ветках (одно знание — один id).
