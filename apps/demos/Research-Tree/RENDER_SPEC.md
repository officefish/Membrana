# RENDER_SPEC — интерактивная карта Research-Tree

Блюпринт рендера графа знаний. Одновременно — **бриф для Replit** (прототипер с
коротким контекстом): собрать ровно этот изолированный элемент, затем импорт в
`apps/demos/Research-Tree` и доработка репо-агентами.

## Назначение

Один экран, который показывает: что вскрыто, что фронтир, что за туманом, и на
чём всё держится. Радиальная карта + две вспомогательные панели. Не редактор —
визуализатор поверх `membrana-knowledge-graph.json`.

## Вход (контракт данных)

Рендер читает `membrana-knowledge-graph.json`. Использует поля:

- `nodes[]`: `id, title, type, branch, epoch, facets{skill,process,materiality},
  state, requires[], path?, lifecycle?, upkeep?, verify?`
- `epochs[]`: `id, title, product_tier, tier_node`
- `artifacts[]`: `id, title, node, also[], kind, status`
- `resources{}`: `infra, spend, stamina, team, day_structure, sprint` (для нижней панели)

`null`-значения (приватные числа) выводятся как «TBD», не ломают рендер.

## Кодирование (главная карта)

Позиция несёт структуру, внешний вид — статус:

| Канал | Свойство |
|-------|----------|
| **радиус** (кольца) | контур: `реальность` в центре → `возможность` на ободе |
| **угловой сектор** (4 четверти) | квадрант `skill×process` (h/i, h/e, s/i, s/e) |
| **угол внутри сектора** | ветка (кластеры) |
| **цвет/тон** | эпоха E0…E4 |
| **регистр узла** (заливка/обводка/свечение/пунктир) | состояние fog/available/exploring/established |
| **размер узла** | материальность: M1 крупный (тяжёлый) → M4 мелкий |
| **спутники на орбите** | артефакты; дуга вокруг узла = collected/total |
| **пульс/кольцо дрена** | `lifecycle: sustained` (узел подъедает энергию) |

Узлы `knowledge` (materiality=null) — нейтральный мелкий размер.

## Производные (считать на клиенте)

```js
const est = new Set(nodes.filter(n=>n.state==='established').map(n=>n.id));
const base = n => n.requires.every(r=>est.has(r)) ? 'available' : 'fog'; // для проверки fog/available
function contour(n, by){                       // by: id->node
  if (n.state==='established') return 'реальность';
  if (n.path==='floating') return 'возможность';
  if (n.path==='partial')  return 'горизонт';
  const up = n.requires.map(r=>contour(by[r], by));
  return up.some(u=>u==='горизонт'||u==='возможность') ? 'горизонт' : 'план';
}
const ratio = id => {                            // прогресс коллекции
  const a = artifacts.filter(x=>x.node===id);
  return a.length ? a.filter(x=>x.status==='collected').length / a.length : null;
};
```

## Панели

- **16-клеточная тепловая карта** (quadrant × materiality, `established/total`) —
  диагностика портфеля; клик по ячейке подсвечивает её узлы на карте.
- **Ресурсная панель снизу**: три шкалы (инфра-потолок, токен-поток, стамина +
  индикатор WIP `count(exploring) vs wip_limit`), полосы дня (пик/поддержка/фон),
  каденция спринтов, ростер команды по статусам. `null` → «TBD».

## Интеракции

- **Hover** — тултип узла: тип, эпоха, квадрант, состояние, контур, upkeep,
  артефакты (collected/total).
- **Клик** — раскрыть спутники и предпосылки (подсветить рёбра `requires`).
- **Фильтры-переключатели** (не постоянные каналы): по эпохе, по квадранту, по
  контуру, показать только `verify`, показать только `sustained`.
- **Туман** — `fog`-узлы притушены; ободные `возможность` едва видны.

## Тех-ограничения (для прототипа)

- **Один самодостаточный файл** (HTML + inline JS; SVG или d3 c CDN). Без
  build-шага, без зависимостей от остального репозитория.
- Данные: `fetch('./membrana-knowledge-graph.json')` с **встроенным снапшотом как
  фолбэк**, чтобы прототип работал в Replit без файла.
- **Без** localStorage/sessionStorage и любого браузерного хранилища.
- Масштаб — ~60 узлов; раскладка должна читаться без наложений.
- Выход — файл, готовый к импорту в `apps/demos/Research-Tree/`.

## Не-цели

- Редактирование графа, персистентность, авторизация, сервер.
- Заполнение приватных чисел (стамина/баланс/спеки/пороги).
- Перенос на React/репо-стек — это уже доработка агентами после импорта.

## Критерии готовности

1. Все 60 узлов размещены по кодированию; рёбра `requires` доступны по клику.
2. Производные (контур, fog/available, collection ratio) считаются на клиенте и
   совпадают с валидатором графа.
3. Работают hover, клик, фильтры; туман различает контуры.
4. Обе панели рендерятся; `null` показан как «TBD».
5. Один файл, без build и без браузерного хранилища, данные грузятся с фолбэком.
