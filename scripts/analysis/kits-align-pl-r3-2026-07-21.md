# Kits align с pl-r3 — 2026-07-21 (S3 / #795)

**Scope (Scenario C HARD GATE):** выравнивание контейнера `scripts/` с контрактом
кит-манифеста из Р3 — без имплементации слоя `kits/` и без второй JSON-схемы.

## Термины (ozhegov)

| Лемма | Не путать с |
|-------|-------------|
| **Дом скриптов** (`scripts/`) | плоский SoT кода движков |
| **Слой кит** (`kits/` в layer-rules) | группировка по ссылке; дом слоя — [`kits/`](../../kits/README.md) (K1+) |
| **Кит-манифест** | контракт Р3 + схема [`kits/MANIFEST.schema.json`](../../kits/MANIFEST.schema.json); в `scripts/` схемы **нет** |
| **kitVersion** | поле манифеста *процедуры* → пин кита |
| **PINNED_SUBGRAPH_VERSIONING** | как версионировать кит (путь→SHA), когда #761 оживёт |
| **GitHub Releases** | ядро *раздачи* пина (core карты контейнеров), не замена манифеста |

## Канон (потребить)

Контейнер `scripts/` **потребляет** контракт Р3 и паттерн #761; собственного
схемного острова кита не заводит. Следующий код китов — только по #761, с манифестом
по правилам Р3.
